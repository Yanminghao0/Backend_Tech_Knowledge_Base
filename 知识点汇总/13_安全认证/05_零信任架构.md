## 1. 零信任概述

### 传统安全模型的问题

```
传统模型（城堡护城河）：
  - 内网可信，外网不可信
  - 一旦突破边界，内网畅通无阻
  - VPN接入后即"可信"

问题：
  - 内网横向移动攻击
  - VPN凭据泄露 → 全网沦陷
  - 云原生无明确边界
```

### 零信任核心思想

```
"永不信任，始终验证"（Never Trust, Always Verify）

1. 不区分内外网 → 所有访问都需要验证
2. 不基于IP信任 → 基于身份+上下文
3. 最小权限 → 只授予完成任务所需权限
4. 持续验证 → 不是一次性认证
```

---

## 2. 核心原则

```
1. 身份为基石：用户身份+设备身份+服务身份
2. 最小权限：按需授权，定期回收
3. 微隔离：服务间默认拒绝，显式允许
4. 持续评估：实时风险评估，异常即降权
5. 加密一切：传输加密+存储加密
```

### 身份认证（SPIFFE/SPIRE）

```yaml
# SPIFFE ID标识服务身份
# spiffe://example.com/ns/prod/sa/payment-service

# 服务间mTLS认证
# 每个服务有SVID证书，自动轮转
# Istio/Linkerd自动实现mTLS
```

### 微隔离

```yaml
# K8s NetworkPolicy + 服务网格
apiVersion: networking.k8s.io/v1
kind: NetworkPolicy
spec:
  podSelector:
    matchLabels:
      app: payment
  policyTypes:
    - Ingress
  ingress:
    - from:
        - podSelector:
            matchLabels:
              app: order    # 只允许order访问payment
      ports:
        - port: 8080
```

### 持续验证

```java
// 每次API调用都验证
@PreAuthorize("hasPermission(#resource, 'read')")
@GetMapping("/orders/{id}")
public Order getOrder(@PathVariable Long id) {
    // JWT验证 + 权限检查 + 设备健康检查 + 风险评分
    return orderService.getOrder(id);
}
```

## 5. 面试要点

### Q1: 零信任和传统安全模型的区别？

```
传统：内网可信，边界防护（防火墙/VPN）
零信任：永不信任，每次访问都验证

区别：
1. 信任边界：内网vs外网 → 无信任边界
2. 认证方式：一次性登录 → 持续验证
3. 授权：基于IP/网段 → 基于身份+上下文
4. 隔离：大区隔离 → 微隔离（服务级）
```

### Q2: 微服务中如何实现零信任？

```
1. mTLS：服务间双向TLS认证（Istio自动实现）
2. NetworkPolicy：K8s网络策略，默认拒绝
3. JWT + RBAC：每次API调用验证身份和权限
4. 设备健康：检查设备是否合规（补丁/安全软件）
5. 审计日志：记录所有访问行为
```
