## 1. 密码学概述

```
密码学三大核心目标：
  机密性 — 只有授权方能读懂数据（加密）
  完整性 — 数据未被篡改（Hash/MAC）
  真实性 — 数据来源可信（数字签名）

加密分类：
  ┌──────────────────────────────────────────┐
  │  对称加密   │  非对称加密  │   Hash   │
  │  AES/DES   │  RSA/ECC   │  SHA/MD5 │
  │  加解密同密钥│  公钥加密   │  不可逆   │
  │  速度快     │  私钥解密   │  定长输出 │
  │  密钥分发难  │  速度慢     │  完整性校验│
  └──────────────────────────────────────────┘
```

## 2. 对称加密

```java
// AES加密（推荐）
// 注意: IV应使用独立随机值，不能从密钥派生（密码学安全要求）
public class AESUtil {
    
    private static final String ALGORITHM = "AES";
    private static final String TRANSFORMATION = "AES/CBC/PKCS5Padding";
    
    // 加密(IV随机生成，与密文一起返回)
    public static String encrypt(String plaintext, String key) throws Exception {
        SecretKeySpec keySpec = new SecretKeySpec(key.getBytes(), ALGORITHM);
        // IV必须随机生成，不能复用密钥
        byte[] ivBytes = new byte[16];
        SecureRandom.getInstanceStrong().nextBytes(ivBytes);
        IvParameterSpec iv = new IvParameterSpec(ivBytes);
        
        Cipher cipher = Cipher.getInstance(TRANSFORMATION);
        cipher.init(Cipher.ENCRYPT_MODE, keySpec, iv);
        byte[] encrypted = cipher.doFinal(plaintext.getBytes());
        // IV + 密文拼接返回(解密时需要IV)
        byte[] result = new byte[ivBytes.length + encrypted.length];
        System.arraycopy(ivBytes, 0, result, 0, ivBytes.length);
        System.arraycopy(encrypted, 0, result, ivBytes.length, encrypted.length);
        return Base64.getEncoder().encodeToString(result);
    }
    
    // 解密(从密文头部提取IV)
    public static String decrypt(String ciphertext, String key) throws Exception {
        SecretKeySpec keySpec = new SecretKeySpec(key.getBytes(), ALGORITHM);
        byte[] decoded = Base64.getDecoder().decode(ciphertext);
        // 前16字节是IV
        IvParameterSpec iv = new IvParameterSpec(decoded, 0, 16);
        byte[] encrypted = new byte[decoded.length - 16];
        System.arraycopy(decoded, 16, encrypted, 0, encrypted.length);
        
        Cipher cipher = Cipher.getInstance(TRANSFORMATION);
        cipher.init(Cipher.DECRYPT_MODE, keySpec, iv);
        byte[] decrypted = cipher.doFinal(encrypted);
        return new String(decrypted);
    }
}

// AES密钥长度：128/192/256位，推荐256
// DES已不安全（56位密钥），3DES也逐步淘汰
```

## 3. 非对称加密

```java
// RSA加密
public class RSAUtil {
    
    // 生成密钥对
    public static KeyPair generateKeyPair() throws Exception {
        KeyPairGenerator generator = KeyPairGenerator.getInstance("RSA");
        generator.initialize(2048);  // 推荐2048位
        return generator.generateKeyPair();
    }
    
    // 公钥加密
    public static String encryptByPublicKey(String plaintext, PublicKey publicKey) throws Exception {
        Cipher cipher = Cipher.getInstance("RSA");
        cipher.init(Cipher.ENCRYPT_MODE, publicKey);
        byte[] encrypted = cipher.doFinal(plaintext.getBytes());
        return Base64.getEncoder().encodeToString(encrypted);
    }
    
    // 私钥解密
    public static String decryptByPrivateKey(String ciphertext, PrivateKey privateKey) throws Exception {
        Cipher cipher = Cipher.getInstance("RSA");
        cipher.init(Cipher.DECRYPT_MODE, privateKey);
        byte[] decrypted = cipher.doFinal(Base64.getDecoder().decode(ciphertext));
        return new String(decrypted);
    }
}

// RSA特点：
// - 公钥加密，私钥解密 → 机密性
// - 私钥签名，公钥验证 → 真实性
// - 速度慢，不适合大数据加密
// - 通常用于交换对称密钥（混合加密）
```

## 4. Hash算法

```java
// Hash：将任意长度数据映射为定长摘要
public class HashUtil {
    
    // SHA-256（推荐）
    public static String sha256(String input) {
        try {
            MessageDigest digest = MessageDigest.getInstance("SHA-256");
            byte[] hash = digest.digest(input.getBytes(StandardCharsets.UTF_8));
            return bytesToHex(hash);
        } catch (NoSuchAlgorithmException e) {
            throw new RuntimeException(e);
        }
    }
    
    // MD5（不推荐用于安全场景，但常用于文件校验）
    public static String md5(String input) {
        try {
            MessageDigest digest = MessageDigest.getInstance("MD5");
            byte[] hash = digest.digest(input.getBytes(StandardCharsets.UTF_8));
            return bytesToHex(hash);
        } catch (NoSuchAlgorithmException e) {
            throw new RuntimeException(e);
        }
    }
    
    // HMAC-SHA256（带密钥的Hash，用于消息认证）
    public static String hmacSha256(String data, String key) throws Exception {
        Mac mac = Mac.getInstance("HmacSHA256");
        SecretKeySpec keySpec = new SecretKeySpec(key.getBytes(), "HmacSHA256");
        mac.init(keySpec);
        byte[] hmac = mac.doFinal(data.getBytes());
        return Base64.getEncoder().encodeToString(hmac);
    }
}

// Hash vs 加密：
// Hash：不可逆（单向），用于完整性校验
// 加密：可逆（双向），用于机密性
// MD5：128位，已被碰撞攻击，不推荐
// SHA-1：160位，已被攻破
// SHA-256：256位，推荐
// SHA-3：新一代Hash标准
```

## 5. 数字签名与证书

```
数字签名流程：

  发送方：
  ┌────────────────────────────────────┐
  │ 1. 对消息计算Hash摘要               │
  │ 2. 用私钥加密Hash → 数字签名         │
  │ 3. 发送：消息 + 数字签名              │
  └────────────────────────────────────┘
  
  接收方：
  ┌────────────────────────────────────┐
  │ 1. 对消息计算Hash摘要               │
  │ 2. 用公钥解密数字签名 → 原Hash       │
  │ 3. 对比两个Hash → 一致则验证通过     │
  └────────────────────────────────────┘

  签名 = 用私钥加密Hash值
  作用：防篡改 + 防否认
```

```java
// Java数字签名
public class SignatureUtil {
    
    // 签名
    public static byte[] sign(String data, PrivateKey privateKey) throws Exception {
        Signature signature = Signature.getInstance("SHA256withRSA");
        signature.initSign(privateKey);
        signature.update(data.getBytes());
        return signature.sign();
    }
    
    // 验签
    public static boolean verify(String data, byte[] signatureBytes, PublicKey publicKey) throws Exception {
        Signature signature = Signature.getInstance("SHA256withRSA");
        signature.initVerify(publicKey);
        signature.update(data.getBytes());
        return signature.verify(signatureBytes);
    }
}
```

```
数字证书（X.509）：

  证书 = 公钥 + 持有者信息 + CA签名
  
  ┌──────────────────────────┐
  │  证书内容                  │
  │  ├── 持有者：example.com   │
  │  ├── 公钥：RSA 2048        │
  │  ├── 有效期：2024-2026     │
  │  ├── 颁发者：CA机构         │
  │  └── CA签名：SHA256withRSA │
  └──────────────────────────┘

  CA（Certificate Authority）：受信任的第三方
  证书链：根CA → 中间CA → 终端证书
  HTTPS = HTTP + TLS（TLS使用证书验证服务端身份）
```

## 6. Java加密实战

```java
// 密码存储：BCrypt（不要用MD5/SHA存密码！）
@Service
public class PasswordService {
    
    @Bean
    public PasswordEncoder passwordEncoder() {
        return new BCryptPasswordEncoder(10);  // strength=10
    }
    
    // 加密密码
    public String encode(String rawPassword) {
        return passwordEncoder.encode(rawPassword);
        // $2a$10$N9qo8uLOickgx2ZMRZoMy...
    }
    
    // 验证密码
    public boolean matches(String rawPassword, String encodedPassword) {
        return passwordEncoder.matches(rawPassword, encodedPassword);
    }
}

// BCrypt特点：
// - 每次加密结果不同（内含随机盐）
// - strength参数控制计算耗时（防暴力破解）
// - 不可逆，无法从密文推出原文

// JWT签名
public class JwtUtil {
    private static final String SECRET = "my-secret-key-at-least-256-bits";
    
    public String generateToken(Long userId) {
        return Jwts.builder()
            .subject(userId.toString())
            .issuedAt(new Date())
            .expiration(new Date(System.currentTimeMillis() + 86400000))
            .signWith(Keys.hmacShaKeyFor(SECRET.getBytes()))
            .compact();
    }
}
```

## 7. 面试题速查

**Q1: 对称加密和非对称加密的区别？**
```
对称：加解密同一密钥，速度快，密钥分发难（AES）
非对称：公钥加密私钥解密，速度慢，解决密钥分发（RSA）
实际用混合加密：非对称交换对称密钥，对称加密数据
```

**Q2: 为什么密码用BCrypt而不是MD5？**
```
MD5：可被彩虹表破解，无盐则相同密码Hash相同
BCrypt：自带随机盐（每次不同），可调计算强度，抗暴力破解
密码存储三原则：不可逆、加盐、慢算法
```

**Q3: HTTPS的加密过程？**
```
1. 客户端请求HTTPS连接
2. 服务端返回证书（含公钥）
3. 客户端验证证书（CA签名+有效期+域名）
4. 客户端生成对称密钥，用公钥加密发给服务端
5. 服务端用私钥解密获得对称密钥
6. 双方用对称密钥加密通信
```

**Q4: 数字签名的作用？**
```
防篡改：签名与内容绑定，改内容则签名失效
防否认：私钥只有持有者有，签名不可否认
完整性：Hash摘要验证数据完整
```
