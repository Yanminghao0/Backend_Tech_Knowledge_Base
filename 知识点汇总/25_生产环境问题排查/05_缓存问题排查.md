**1. 连接数达到上限**

## 缓存穿透

### 问题描述

**缓存穿透**：查询不存在的数据，缓存未命中，请求直接打到数据库

**影响**：
```
- 大量请求打到数据库
- 数据库压力大
- 可能影响正常业务
```

### 排查方法

**1. 查看缓存命中率**

```bash
# 查看命中率
redis-cli INFO stats | grep keyspace_hits
redis-cli INFO stats | grep keyspace_misses

# 计算命中率
命中率 = keyspace_hits / (keyspace_hits + keyspace_misses)
```

**2. 分析请求日志**

```bash
# 查看请求日志
grep "cache miss" application.log
grep "database query" application.log
```

### 解决方案

**1. 布隆过滤器**

```java
// 使用布隆过滤器快速判断
if (!bloomFilter.exists(key)) {
    return null;  // 不存在，直接返回
}

// 查询缓存
String value = redis.get(key);
if (value != null) {
    return value;
}

// 查询数据库
value = db.query(key);
if (value != null) {
    redis.set(key, value);
} else {
    // 缓存空值（短期）
    redis.setex(key, 300, "");  // 5分钟
}
```

**2. 缓存空值**

```java
// 查询不存在的数据，缓存空值
String value = redis.get(key);
if (value != null) {
    if (value.isEmpty()) {
        return null;  // 空值，直接返回
    }
    return value;
}

value = db.query(key);
if (value != null) {
    redis.set(key, value);
} else {
    // 缓存空值（短期，避免穿透）
    redis.setex(key, 300, "");  // 5分钟
}
```

**3. 参数校验**

```java
// 参数校验，过滤无效请求
if (id == null || id <= 0) {
    return null;
}
```

## 缓存击穿

### 问题描述

**缓存击穿**：热点key过期，大量请求同时打到数据库

**影响**：
```
- 数据库压力突增
- 可能影响系统性能
```

### 排查方法

**1. 查看热点key**

```bash
# 查看热key
redis-cli --hotkeys

# 分析key访问频率
redis-cli MONITOR | grep "GET"
```

**2. 分析请求模式**

```bash
# 查看请求日志
grep "cache miss" application.log
grep "database query" application.log
```

### 解决方案

**1. 热点key永不过期**

```java
// 热点key设置为永不过期
redis.set("hot:key", value);
redis.persist("hot:key");  // 取消过期时间
```

**2. 互斥锁更新缓存**

```java
// 使用分布式锁，只允许一个线程更新缓存
String lockKey = "lock:" + key;
if (redis.setnx(lockKey, "1", 60)) {  // 获取锁
    try {
        // 查询数据库
        String value = db.query(key);
        if (value != null) {
            redis.set(key, value);
        }
    } finally {
        redis.del(lockKey);  // 释放锁
    }
} else {
    // 未获取到锁，等待一段时间后重试
    Thread.sleep(100);
    return redis.get(key);
}
```

**3. 逻辑过期（异步更新）**

```java
// 逻辑过期：key永不过期，但存储过期时间
String value = redis.get(key);
if (value != null) {
    long expireTime = getExpireTime(key);
    if (expireTime > System.currentTimeMillis()) {
        return value;  // 未过期
    }
    // 过期，异步更新
    asyncUpdateCache(key);
    return value;  // 返回旧值
}
```

## 缓存雪崩

### 问题描述

**缓存雪崩**：大量key同时过期，大量请求同时打到数据库

**影响**：
```
- 数据库压力突增
- 系统性能下降
- 可能影响系统稳定性
```

### 排查方法

**1. 查看key过期情况**

```bash
# 查看key过期时间
redis-cli TTL key

# 分析过期时间分布
redis-cli --scan | while read key; do
    echo "$key: $(redis-cli TTL $key)"
done
```

**2. 分析请求模式**

```bash
# 查看请求日志
grep "cache miss" application.log | tail -100
```

### 解决方案

**1. 过期时间加随机值**

```java
// 过期时间加随机值，避免同时过期
int expireTime = 3600 + (int)(Math.random() * 600);  // 3600-4200秒
redis.setex(key, expireTime, value);
```

**2. 多级缓存**

```java
// 多级缓存：本地缓存 + Redis
// 本地缓存：Caffeine（更短过期时间）
// Redis缓存：较长过期时间
```

**3. 限流降级**

```java
// 限流保护：限制数据库请求数
if (rateLimiter.tryAcquire()) {
    // 查询数据库
} else {
    // 返回默认值或降级
    return getDefaultValue();
}
```

**4. 缓存预热**

```java
// 系统启动时，预热热点数据
@PostConstruct
public void warmUpCache() {
    List<String> hotKeys = getHotKeys();
    for (String key : hotKeys) {
        String value = db.query(key);
        redis.set(key, value);
    }
}
```

## 大key和热key

### 大key问题

**症状**：
```
✅ Redis内存占用高
✅ 操作慢（GET/SET）
✅ 网络传输慢
✅ 可能阻塞其他操作
```

**排查方法**：

```bash
# 查找大key
redis-cli --bigkeys

# 分析key大小
redis-cli MEMORY USAGE key
```

**解决方案**：

```java
// 1. 拆分大key
// ❌ 大key
redis.set("user:10001:orders", largeJson);

// ✅ 拆分
redis.set("user:10001:orders:count", count);
redis.lpush("user:10001:orders", order1, order2, ...);

// 2. 压缩数据
String compressed = compress(largeJson);
redis.set("key", compressed);

// 3. 使用Hash分片
for (int i = 0; i < 100; i++) {
    redis.hset("key:hash", "field" + i, value);
}
```

### 热key问题

**症状**：
```
✅ 单个key访问频率高
✅ Redis CPU使用率高
✅ 可能成为瓶颈
```

**排查方法**：

```bash
# 查找热key
redis-cli --hotkeys

# 监控key访问
redis-cli MONITOR | grep "GET"
```

**解决方案**：

```java
// 1. 本地缓存
// 使用Caffeine本地缓存热点数据
Cache<String, String> localCache = Caffeine.newBuilder()
    .maximumSize(1000)
    .expireAfterWrite(5, TimeUnit.MINUTES)
    .build();

String value = localCache.get(key, k -> redis.get(k));

// 2. 主从读写分离
// 读请求路由到从库
Jedis jedis = slaveJedisPool.getResource();
String value = jedis.get(key);

// 3. key分片
// 将热点key分片到多个key
String shardKey = "key:" + (hash(key) % 10);
String value = redis.get(shardKey);
```

### 布隆过滤器实现缓存穿透防护

**1. 引入Guava布隆过滤器**

**2. 初始化布隆过滤器**

```java
@Configuration
public class BloomFilterConfig {
    
    @Bean
    public BloomFilter<Long> idBloomFilter() {
        // 预计数据量100万，误判率0.01
        int expectedInsertions = 1000000;
        double fpp = 0.01;
        
        return BloomFilter.create(
            Funnels.longFunnel(),
            expectedInsertions,
            fpp
        );
    }
}
```

**3. 预热布隆过滤器**

```java
@Component
public class BloomFilterInitializer implements CommandLineRunner {
    
    @Autowired
    private BloomFilter<Long> idBloomFilter;
    
    @Autowired
    private ProductMapper productMapper;
    
    @Override
    public void run(String... args) throws Exception {
        // 从数据库加载所有有效ID到布隆过滤器
        List<Long> allIds = productMapper.selectAllIds();
        for (Long id : allIds) {
            idBloomFilter.put(id);
        }
    }
}
```

**4. 接口层防护实现**

```java
@Service
public class ProductService {
    
    @Autowired
    private BloomFilter<Long> idBloomFilter;
    
    @Autowired
    private RedisTemplate<String, Object> redisTemplate;
    
    @Autowired
    private ProductMapper productMapper;
    
    public Product getProductById(Long id) {
        // 1. 参数校验
        if (id == null || id <= 0) {
            return null;
        }
        
        // 2. 布隆过滤器过滤无效ID
        if (!idBloomFilter.mightContain(id)) {
            log.warn("无效ID请求: {}", id);
            return null;
        }
        
        // 3. 查询缓存
        String key = "product:" + id;
        Product product = (Product) redisTemplate.opsForValue().get(key);
        if (product != null) {
            return product;
        }
        
        // 4. 查询数据库
        product = productMapper.selectById(id);
        if (product != null) {
            // 5. 写入缓存
            redisTemplate.opsForValue().set(key, product, 30, TimeUnit.MINUTES);
        } else {
            // 6. 缓存空值
            redisTemplate.opsForValue().set(key, NULL_VALUE, 5, TimeUnit.MINUTES);
        }
        
        return product;
    }
}
```

**5. Redis布隆过滤器实现（适用于分布式系统）**

```java
@Component
public class RedisBloomFilter {
    
    @Autowired
    private StringRedisTemplate redisTemplate;
    
    // 添加元素到布隆过滤器
    public <T> void add(String filterName, Funnel<T> funnel, T value) {
        int numBits = 1 << 29; // 536870912 bits
        int numHashFunctions = 7;
        
        long[] indices = Hashing.murmur3_128().hashObjectHash(value, funnel);
        for (int i = 0; i < numHashFunctions; i++) {
            long index = indices[i] % numBits;
            redisTemplate.opsForValue().setBit(filterName, index, true);
        }
    }
    
    // 判断元素是否在布隆过滤器中
    public <T> boolean contains(String filterName, Funnel<T> funnel, T value) {
        int numBits = 1 << 29;
        int numHashFunctions = 7;
        
        long[] indices = Hashing.murmur3_128().hashObject(value, funnel).asLongs();
        for (int i = 0; i < numHashFunctions; i++) {
            long index = indices[i] % numBits;
            if (!redisTemplate.opsForValue().getBit(filterName, index)) {
                return false;
            }
        }
        return true;
    }
}
```

### 多级缓存架构实现

**1. 多级缓存设计**

```
本地缓存 (Caffeine) → Redis 分布式缓存 → 数据库
```

**2. Caffeine本地缓存配置**

```java
@Configuration
public class CaffeineConfig {
    
    @Bean
    public Cache<String, Object> localCache() {
        return Caffeine.newBuilder()
            .maximumSize(10_000)  // 最大缓存数量
            .expireAfterWrite(5, TimeUnit.MINUTES)  // 写入后过期时间
            .recordStats()  // 开启统计
            .build();
    }
}
```

**3. 多级缓存实现**

```java
@Service
public class ProductService {
    
    @Autowired
    private Cache<String, Object> localCache;
    
    @Autowired
    private RedisTemplate<String, Object> redisTemplate;
    
    @Autowired
    private ProductMapper productMapper;
    
    public Product getProductById(Long id) {
        // 1. 检查本地缓存
        String key = "product:" + id;
        Product product = (Product) localCache.getIfPresent(key);
        if (product != null) {
            return product;
        }
        
        // 2. 检查Redis缓存
        product = (Product) redisTemplate.opsForValue().get(key);
        if (product != null) {
            // 3. 写入本地缓存
            localCache.put(key, product);
            return product;
        }
        
        // 4. 查询数据库
        product = productMapper.selectById(id);
        if (product != null) {
            // 5. 写入Redis
            redisTemplate.opsForValue().set(key, product, 30, TimeUnit.MINUTES);
            // 6. 写入本地缓存
            localCache.put(key, product);
        }
        
        return product;
    }
}
```

**5. 热点数据自动预热**

```java
@Component
public class HotDataPreloader {
    
    @Autowired
    private ProductMapper productMapper;
    
    @Autowired
    private RedisTemplate<String, Object> redisTemplate;
    
    @Autowired
    private Cache<String, Object> localCache;
    
    @Scheduled(cron = "0 0 1 * * ?")  // 每天凌晨1点执行
    public void preloadHotData() {
        // 1. 查询热点数据
        List<Product> hotProducts = productMapper.selectHotProducts(100);
        
        // 2. 预热Redis缓存
        for (Product product : hotProducts) {
            String key = "product:" + product.getId();
            redisTemplate.opsForValue().set(key, product, 1, TimeUnit.DAYS);
            
            // 3. 预热本地缓存
            localCache.put(key, product);
        }
    }
}
```
