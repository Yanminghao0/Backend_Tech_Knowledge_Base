**症状**：

## 重复消费

### 排查方法

**1. 查看消息ID**

```java
// 记录消息ID，检查是否重复
String messageId = message.getMessageId();
if (processedIds.contains(messageId)) {
    log.warn("重复消费: {}", messageId);
    return;  // 跳过
}
processedIds.add(messageId);
```

**2. 查看消费日志**

```bash
# 查看消费日志
grep "重复消费" application.log
grep "messageId" application.log | sort | uniq -d
```

### 常见原因

**1. 消费失败重试**

```
- 消费失败后重试
- 消息被重复投递
- 消费者重启后重复消费
```

**2. 消息队列机制**

```
- Kafka: 消费者偏移量未提交
- RocketMQ: 消息重试机制
```

### 解决方案

**1. 幂等性设计**

```java
// 使用唯一标识保证幂等性
String messageId = message.getMessageId();
String key = "processed:" + messageId;

// 使用Redis检查是否已处理
if (redis.exists(key)) {
    return;  // 已处理，跳过
}

// 处理消息
processMessage(message);

// 标记已处理
redis.setex(key, 3600, "1");  // 1小时过期
```

**2. 数据库唯一约束**

```sql
-- 使用唯一约束防止重复
CREATE TABLE message_processed (
    message_id VARCHAR(64) PRIMARY KEY,
    processed_time TIMESTAMP
);
```

**3. 分布式锁**

```java
// 使用分布式锁保证幂等性
String lockKey = "lock:" + messageId;
if (redis.setnx(lockKey, "1", 60)) {  // 获取锁
    try {
        processMessage(message);
    } finally {
        redis.del(lockKey);  // 释放锁
    }
} else {
    log.warn("消息正在处理中: {}", messageId);
}
```

## 消息丢失

### 排查方法

**1. 查看消息确认**

```java
// Kafka: 手动提交偏移量
@KafkaListener(topics = "topic")
public void consume(ConsumerRecord<String, String> record) {
    try {
        processMessage(record);
        // 处理成功后才提交偏移量
        // 注意：如果这里抛出异常，消息会丢失
    } catch (Exception e) {
        log.error("处理失败", e);
        // 处理失败，不提交偏移量，消息会重新消费
    }
}
```

**2. 查看消息持久化**

```bash
# Kafka查看消息持久化
kafka-run-class kafka.tools.DumpLogSegments \
  --files /tmp/kafka-logs/topic-0/00000000000000000000.log \
  --print-data-log
```

### 常见原因

**1. 消息未持久化**

```
- 消息队列宕机
- 消息未落盘
- 消息在内存中丢失
```

**2. 消费失败未处理**

```
- 消费失败后未重试
- 异常被吞掉
- 消息被丢弃
```

**3. 偏移量提交错误**

```java
// ❌ 处理失败但提交了偏移量
try {
    processMessage(message);
} catch (Exception e) {
    // 异常被吞掉
}
// 偏移量被提交，消息丢失

// ✅ 处理失败不提交偏移量
try {
    processMessage(message);
} catch (Exception e) {
    log.error("处理失败", e);
    throw e;  // 抛出异常，不提交偏移量
}
```

### 解决方案

**1. 消息持久化**

```java
// Kafka: 设置acks=all，保证消息持久化
properties.put("acks", "all");
properties.put("retries", 3);
properties.put("max.in.flight.requests.per.connection", 1);
```

**2. 手动提交偏移量**

```java
// 手动提交偏移量，保证消息处理成功
@KafkaListener(topics = "topic", 
               containerFactory = "kafkaListenerContainerFactory")
public void consume(ConsumerRecord<String, String> record,
                    Acknowledgment ack) {
    try {
        processMessage(record);
        ack.acknowledge();  // 处理成功后提交
    } catch (Exception e) {
        log.error("处理失败", e);
        // 不提交偏移量，消息会重新消费
    }
}
```

**3. 死信队列**

```java
// 处理失败的消息发送到死信队列
try {
    processMessage(message);
} catch (Exception e) {
    log.error("处理失败", e);
    // 发送到死信队列
    deadLetterQueue.send(message);
}
```

```java
@Service
public class EmergencyMessageConsumer {
    @Autowired
    private KafkaTemplate<String, String> kafkaTemplate;
    
    // 快速跳过无效消息
    @KafkaListener(topics = "order_topic", groupId = "emergency_consumer")
    public void consume(ConsumerRecord<String, String> record, Acknowledgment ack) {
        try {
            // 1. 检查消息时间戳，跳过过期消息
            long messageTimestamp = record.timestamp();
            if (System.currentTimeMillis() - messageTimestamp > 3600_000) { // 1小时过期
                log.warn("跳过过期消息: {}", record.offset());
                ack.acknowledge();
                return;
            }
            
            // 2. 快速处理或转发到临时队列
            processMessage(record.value());
            ack.acknowledge();
        } catch (Exception e) {
            // 3. 异常消息转发到死信队列
            kafkaTemplate.send("order_topic_dlq", record.value());
            ack.acknowledge();
        }
    }
}
```

### 消息幂等性解决方案

**1. 基于数据库唯一索引实现**

```java
@Service
public class OrderService {
    @Autowired
    private OrderMapper orderMapper;
    
    @Transactional
    public void processOrder(String messageId, OrderDTO orderDTO) {
        // 1. 插入消息日志，利用唯一索引防止重复处理
        MessageLog log = new MessageLog();
        log.setMessageId(messageId);
        log.setStatus(1); // 处理中
        
        try {
            messageLogMapper.insert(log);
        } catch (DuplicateKeyException e) {
            // 唯一索引冲突，说明已处理过
            log.warn("消息已处理: {}", messageId);
            return;
        }
        
        try {
            // 2. 业务处理
            orderMapper.insert(orderDTO);
            // 3. 更新消息状态为成功
            log.setStatus(2);
            messageLogMapper.updateById(log);
        } catch (Exception e) {
            // 4. 更新消息状态为失败
            log.setStatus(3);
            log.setErrorMsg(e.getMessage());
            messageLogMapper.updateById(log);
            throw e;
        }
    }
}
```

**2. 基于Redis实现分布式锁**

```java
@Component
public class IdempotentProcessor {
    @Autowired
    private StringRedisTemplate redisTemplate;
    
    // 处理幂等性任务
    public <T, R> R process(String key, T data, Function<T, R> processor) {
        // 1. 获取分布式锁
        Boolean locked = redisTemplate.opsForValue().setIfAbsent(
            "idempotent:" + key,
            "processing",
            30, TimeUnit.SECONDS
        );
        
        if (Boolean.TRUE.equals(locked)) {
            try {
                // 2. 检查是否已处理
                String result = redisTemplate.opsForValue().get("result:" + key);
                if (result != null) {
                    return JSON.parseObject(result, (TypeReference<R>) new TypeReference<R>() {});
                }
                
                // 3. 处理业务
                R processResult = processor.apply(data);
                
                // 4. 存储处理结果
                redisTemplate.opsForValue().set(
                    "result:" + key,
                    JSON.toJSONString(processResult),
                    24, TimeUnit.HOURS
                );
                
                return processResult;
            } finally {
                // 5. 释放锁
                redisTemplate.delete("idempotent:" + key);
            }
        } else {
            // 6. 其他线程处理中，返回等待结果
            throw new ServiceException("请求处理中，请稍后再试");
        }
    }
}
```

**3. 消息去重设计**

```java
// 生产者端生成消息ID
@Service
public class OrderProducer {
    @Autowired
    private KafkaTemplate<String, String> kafkaTemplate;
    
    public void sendOrder(OrderDTO order) {
        // 1. 生成唯一消息ID (UUID或雪花算法)
        String messageId = UUID.randomUUID().toString();
        
        // 2. 设置消息头
        ProducerRecord<String, String> record = new ProducerRecord<>("order_topic", JSON.toJSONString(order));
        record.headers().add(new RecordHeader("MESSAGE_ID", messageId.getBytes()));
        
        // 3. 发送消息
        kafkaTemplate.send(record);
    }
}

// 消费者端获取消息ID
@KafkaListener(topics = "order_topic")
public void consume(ConsumerRecord<String, String> record) {
    // 获取消息ID
    String messageId = "";
    for (Header header : record.headers()) {
        if ("MESSAGE_ID".equals(header.key())) {
            messageId = new String(header.value());
            break;
        }
    }
    
    // 使用消息ID进行幂等处理
    orderService.processOrder(messageId, JSON.parseObject(record.value(), OrderDTO.class));
}
```
