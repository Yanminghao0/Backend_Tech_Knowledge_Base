## 📖 Dubbo架构概览

```
┌─────────────────────────────────────────────────────────────┐
│                    Dubbo架构                                 │
├─────────────────────────────────────────────────────────────┤
│                                                             │
│  ┌─────────────┐         ┌─────────────┐                   │
│  │  Consumer   │         │  Provider   │                   │
│  │  (服务消费者)│         │  (服务提供者)│                   │
│  └──────┬──────┘         └──────┬──────┘                   │
│         │                       │                           │
│         │ ③invoke              │ ①register                │
│         │                       │                           │
│         ▼                       ▼                           │
│  ┌─────────────────────────────────────────┐               │
│  │              Registry                    │               │
│  │              (注册中心)                   │               │
│  └─────────────────────────────────────────┘               │
│         │                       │                           │
│         │ ②subscribe           │                           │
│         │                       │                           │
│         ▼                       ▼                           │
│  ┌─────────────────────────────────────────┐               │
│  │              Monitor                     │               │
│  │              (监控中心)                   │               │
│  └─────────────────────────────────────────┘               │
│                                                             │
│  调用流程：                                                  │
│  1. Provider启动，向Registry注册服务                        │
│  2. Consumer启动，向Registry订阅服务                        │
│  3. Registry返回Provider地址列表                            │
│  4. Consumer根据负载均衡选择Provider调用                    │
│  5. Consumer和Provider定时向Monitor上报统计                 │
│                                                             │
└─────────────────────────────────────────────────────────────┘
```
