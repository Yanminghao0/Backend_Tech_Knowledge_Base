## 1. 覆盖率基础

### 1.1 什么是测试覆盖率

```
测试覆盖率 = 被测试代码执行的代码量 / 总代码量 × 100%

覆盖率衡量维度:
┌─────────────────────────────────────────┐
│  行覆盖率 (Line Coverage)               │
│  → 被执行的代码行数 / 总行数             │
│  → 最直观的覆盖率指标                    │
├─────────────────────────────────────────┤
│  分支覆盖率 (Branch Coverage)            │
│  → 被执行的分支数 / 总分支数             │
│  → if/else/switch的分支覆盖              │
├─────────────────────────────────────────┤
│  方法覆盖率 (Method Coverage)            │
│  → 被调用的方法数 / 总方法数             │
│  → 检查是否有方法完全没被测试            │
├─────────────────────────────────────────┤
│  类覆盖率 (Class Coverage)              │
│  → 被测试的类数 / 总类数                │
│  → 最粗粒度的覆盖率                     │
├─────────────────────────────────────────┤
│  指令覆盖率 (Instruction Coverage)       │
│  → 被执行的Java字节码指令数 / 总指令数   │
│  → JaCoCo最底层的指标                    │
├─────────────────────────────────────────┤
│  圈复杂度覆盖率 (Cyclomatic Complexity)  │
│  → 被覆盖的路径数 / 总路径数             │
│  → 最严格的覆盖率指标                    │
└─────────────────────────────────────────┘
```

### 1.2 覆盖率不是目的

```java
// ❌ 追求100%覆盖率的陷阱

// 这段代码有100%的行覆盖率，但测试质量为零
public class Calculator {
    public int divide(int a, int b) {
        return a / b;  // 这行被执行了
    }
}

@Test
void test() {
    Calculator calc = new Calculator();
    calc.divide(10, 2);  // 行覆盖率100%，但没断言！
    // 没有测试b=0的情况
    // 没有断言结果是否正确
}

// ✅ 高质量的测试应该关注边界条件和断言
@Test
void testDivide() {
    assertEquals(5, calc.divide(10, 2));
    assertEquals(-5, calc.divide(-10, 2));
    assertThrows(ArithmeticException.class, () -> calc.divide(1, 0));
    assertEquals(0, calc.divide(0, 5));
    assertEquals(Integer.MAX_VALUE, calc.divide(Integer.MAX_VALUE, 1));
}
```

## 3. 覆盖率类型详解

### 3.1 行覆盖率

```java
public class OrderService {
    public String processOrder(Order order) {
        if (order == null) {                          // Line 1: ✓ 被执行
            throw new IllegalArgumentException();      // Line 2: ✗ 未执行
        }
        order.validate();                              // Line 3: ✓ 被执行
        order.setStatus("PROCESSED");                  // Line 4: ✓ 被执行
        return "SUCCESS";                              // Line 5: ✓ 被执行
    }
}

// 测试1: 覆盖了行3,4,5，行覆盖率3/5 = 60%
@Test
void testProcessOrder() {
    orderService.processOrder(validOrder());
}

// 测试2: 增加null检查，覆盖行1,2，行覆盖率5/5 = 100%
@Test
void testNullOrder() {
    assertThrows(IllegalArgumentException.class,
        () -> orderService.processOrder(null));
}
```

### 3.2 分支覆盖率

```java
public class DiscountCalculator {
    public BigDecimal calculate(Order order) {
        BigDecimal discount = BigDecimal.ZERO;
        
        if (order.getAmount().compareTo(new BigDecimal("1000")) > 0) {  // 分支1: true / false
            discount = order.getAmount().multiply(new BigDecimal("0.1"));
        } else if (order.getCustomer().isVip()) {                       // 分支2: true / false
            discount = order.getAmount().multiply(new BigDecimal("0.05"));
        }
        
        if (order.hasCoupon()) {                                         // 分支3: true / false
            discount = discount.add(new BigDecimal("50"));
        }
        
        return discount;
    }
}

// 分支覆盖率计算：
// 总分支数 = 3个if × 2 = 6个分支

// 测试1: amount=500, 非VIP, 无优惠券
// 覆盖: 分支1(false), 分支2(false), 分支3(false)
// 分支覆盖率 = 3/6 = 50%

// 测试2: amount=1500, VIP, 有优惠券
// 覆盖: 分支1(true), 分支3(true)
// 分支覆盖率 = (3+2)/6 = 83%

// 测试3: amount=500, VIP, 有优惠券
// 覆盖: 分支1(false), 分支2(true), 分支3(true)
// 合计: 6/6 = 100% 分支覆盖率
```

## 4. 覆盖率目标

### 4.1 分层覆盖率目标

```yaml
# 推荐的分层覆盖率目标

# 领域模型/核心逻辑: 90%+
domain:
  line: 90%
  branch: 85%
  method: 95%
  reason: "核心业务逻辑，必须高覆盖"

# 服务层: 80%+
service:
  line: 80%
  branch: 75%
  method: 85%
  reason: "业务编排逻辑，重要但依赖较多"

# 控制器层: 70%+
controller:
  line: 70%
  branch: 65%
  method: 80%
  reason: "HTTP处理+参数校验，集成测试补充"

# 持久层: 60%+
repository:
  line: 60%
  branch: 50%
  method: 70%
  reason: "简单CRUD，集成测试覆盖更有效"

# 配置/工具类: 50%+ 或排除
config:
  exclude: true
  reason: "配置类通常不需要单元测试"

# DTO/Entity: 排除
dto:
  exclude: true
  reason: "数据载体，逻辑简单"
```

## 5. CI门禁

### 5.1 Maven门禁配置

```xml
<plugin>
    <groupId>org.jacoco</groupId>
    <artifactId>jacoco-maven-plugin</artifactId>
    <executions>
        <execution>
            <id>coverage-check</id>
            <phase>verify</phase>
            <goals>
                <goal>check</goal>
            </goals>
            <configuration>
                <haltOnFailure>true</haltOnFailure>  <!-- 覆盖率不达标时构建失败 -->
                <rules>
                    <!-- 规则1: 整体行覆盖率 -->
                    <rule>
                        <element>BUNDLE</element>
                        <limits>
                            <limit>
                                <counter>LINE</counter>
                                <value>COVEREDRATIO</value>
                                <minimum>0.80</minimum>
                            </limit>
                        </limits>
                    </rule>
                    
                    <!-- 规则2: 每个类的分支覆盖率 -->
                    <rule>
                        <element>CLASS</element>
                        <limits>
                            <limit>
                                <counter>BRANCH</counter>
                                <value>COVEREDRATIO</value>
                                <minimum>0.60</minimum>
                            </limit>
                        </limits>
                        <excludes>
                            <exclude>*Test</exclude>
                            <exclude>*Configuration</exclude>
                        </excludes>
                    </rule>
                    
                    <!-- 规则3: 每个方法至少被调用一次 -->
                    <rule>
                        <element>METHOD</element>
                        <limits>
                            <limit>
                                <counter>METHOD</counter>
                                <value>MISSEDCOUNT</value>
                                <maximum>0</maximum>
                            </limit>
                        </limits>
                    </rule>
                </rules>
            </configuration>
        </execution>
    </executions>
</plugin>
```

```yaml
  needs: [test-coverage]
  script:
    - |
      # 解析覆盖率报告

      import xml.etree.ElementTree as ET
      tree = ET.parse('target/site/jacoco/jacoco.xml')
      root = tree.getroot()
      for counter in root.findall('counter'):
          if counter.get('type') == 'LINE':
              missed = int(counter.get('missed'))
              covered = int(counter.get('covered'))

              print(f'{covered/total*100:.2f}')

      echo "Line Coverage: ${LINE_COV}%"
      
      # 门禁检查

      if (( $(echo "$LINE_COV < $THRESHOLD" | bc -l) )); then
        echo "❌ 覆盖率 ${LINE_COV}% 低于阈值 ${THRESHOLD}%"

      fi
      echo "✅ 覆盖率达标"
```

### 6.2 XML报告解析

```java
// 解析JaCoCo XML报告
public class CoverageReportParser {
    
    public CoverageSummary parse(String xmlPath) throws Exception {
        DocumentBuilderFactory factory = DocumentBuilderFactory.newInstance();
        DocumentBuilder builder = factory.newDocumentBuilder();
        Document doc = builder.parse(new File(xmlPath));
        
        Element root = doc.getDocumentElement();
        
        CoverageSummary summary = new CoverageSummary();
        
        // 解析包级别覆盖率
        NodeList packages = root.getElementsByTagName("package");
        for (int i = 0; i < packages.getLength(); i++) {
            Element pkg = (Element) packages.item(i);
            String packageName = pkg.getAttribute("name");
            
            PackageCoverage pkgCoverage = new PackageCoverage(packageName);
            
            // 解析计数器
            NodeList counters = pkg.getElementsByTagName("counter");
            for (int j = 0; j < counters.getLength(); j++) {
                Element counter = (Element) counters.item(j);
                String type = counter.getAttribute("type");
                int missed = Integer.parseInt(counter.getAttribute("missed"));
                int covered = Integer.parseInt(counter.getAttribute("covered"));
                pkgCoverage.addCoverage(type, missed, covered);
            }
            
            summary.addPackage(pkgCoverage);
        }
        
        return summary;
    }
    
    public static class CoverageSummary {
        private final List<PackageCoverage> packages = new ArrayList<>();
        
        public double getOverallLineCoverage() {
            int totalMissed = 0, totalCovered = 0;
            for (PackageCoverage pkg : packages) {
                int[] line = pkg.getCoverage("LINE");
                totalMissed += line[0];
                totalCovered += line[1];
            }
            return (double) totalCovered / (totalMissed + totalCovered) * 100;
        }
    }
}
```

## 7. 常见误区

### 7.1 覆盖率不等于测试质量

```java
// ❌ 误区1: 高覆盖率 = 高质量

// 100%行覆盖率但测试无意义
public class UserService {
    public User createUser(String name, int age) {
        if (name == null || name.isEmpty()) {
            throw new IllegalArgumentException("名称不能为空");
        }
        if (age < 0 || age > 150) {
            throw new IllegalArgumentException("年龄不合法");
        }
        User user = new User();
        user.setName(name);
        user.setAge(age);
        return user;
    }
}

@Test
void testCreateUser() {
    userService.createUser("张三", 25);  // 覆盖了所有行
    userService.createUser("", 25);       // 覆盖了异常分支
    userService.createUser("李四", -1);   // 覆盖了异常分支
    // 但是！没有断言返回值！
    // 没有测试边界值：age=0, age=150
    // 没有测试特殊字符的name
}

// ✅ 高质量的测试
@Test
void testCreateUser_Valid() {
    User user = userService.createUser("张三", 25);
    assertEquals("张三", user.getName());
    assertEquals(25, user.getAge());
}

@Test
void testCreateUser_BoundaryAge() {
    assertDoesNotThrow(() -> userService.createUser("test", 0));
    assertDoesNotThrow(() -> userService.createUser("test", 150));
    assertThrows(IllegalArgumentException.class, 
        () -> userService.createUser("test", -1));
    assertThrows(IllegalArgumentException.class, 
        () -> userService.createUser("test", 151));
}
```

### 7.2 只追求行覆盖率

```java
// ❌ 误区2: 只看行覆盖率，忽略分支覆盖率

public class OrderProcessor {
    public String getStatus(Order order) {
        String status = "DEFAULT";
        if (order.isPaid() && order.isShipped()) {  // 4个分支组合
            status = "COMPLETED";                    // 只需要false分支就覆盖此行
        }
        return status;
    }
}

// 这个测试有100%行覆盖率但只有50%分支覆盖率
@Test
void testGetStatus() {
    Order order = new Order();
    order.setPaid(false);
    order.setShipped(false);
    assertEquals("DEFAULT", processor.getStatus(order));
```

## 9. 面试题速查

**Q1: 行覆盖率和分支覆盖率有什么区别？哪个更重要？**
> 行覆盖率统计被执行的代码行比例，分支覆盖率统计if/else/switch等分支的覆盖比例。分支覆盖率更重要——100%行覆盖率可能只覆盖了50%的分支（如if的true分支被执行但false分支没有）。理想情况下两个指标都应达标，但分支覆盖率更能反映测试的完整性。

**Q2: 测试覆盖率达到100%意味着什么？**
> 意味着所有代码行/分支都被执行过，但不意味着代码没有Bug。覆盖率只衡量"代码是否被执行"，不衡量"执行结果是否正确"。100%覆盖率可能包含无断言的测试、只覆盖正常路径忽略边界条件、没有验证异常场景。覆盖率是必要条件但非充分条件。

**Q3: JaCoCo的check目标如何配置质量门禁？**
> 通过`<rules>`配置规则，每条规则包含`<element>`（检查级别：BUNDLE/PACKAGE/CLASS/METHOD）、`<limits>`（计数器和阈值）。`<counter>`指定覆盖率类型（LINE/BRANCH/METHOD等），`<value>`指定比较方式（COVEREDRATIO/MISSEDCOUNT等），`<minimum>`/`<maximum>`设定阈值。`haltOnFailure=true`时覆盖率不达标会导致构建失败。

**Q4: 如何处理DTO/Entity等不需要测试的类的覆盖率？**
> 1）在JaCoCo中配置`<excludes>`排除这些包/类。2）使用`@Generated`注解标记生成代码，JaCoCo会自动忽略。3）分层设定覆盖率目标——DTO/Entity可以设置更低阈值或完全排除。4）合理选择需要测试的代码——有业务逻辑的方法才需要测试。

**Q5: 什么是增量覆盖率？为什么比全量覆盖率更有意义？**
> 增量覆盖率只统计新增/修改代码的覆盖率，而非全量代码。更有意义的原因：1）历史代码可能没有测试但难以补充——全量覆盖率会永远偏低。2）新代码应该有测试——增量覆盖率直接反映当前PR的质量。3）鼓励持续改进——不会因历史债而沮丧。SonarQube的"New Code Coverage"就是增量覆盖率。

**Q6: 覆盖率应该如何融入CI/CD流水线？**
> 1）测试阶段执行`jacoco:report`生成报告。2）质量阶段执行`jacoco:check`进行门禁检查——不达标则构建失败。3）将报告上传到SonarQube/Codecov进行可视化。4）在PR上展示覆盖率变化。5）can-i-deploy检查包含覆盖率验证。6）定期监控覆盖率趋势，持续改进。

**Q7: 如何提升测试覆盖率而不降低测试质量？**
> 1）先写测试再写代码（TDD）——自然达到高覆盖。2）关注业务逻辑而非getter/setter。3）使用参数化测试覆盖多场景。4）优先覆盖圈复杂度高的方法。5）补充边界条件和异常路径测试。6）Code Review中要求新代码有测试。7）不要为凑覆盖率写无断言的测试。
