## 1. AI应用架构概述

### AI应用的挑战

```
1. 成本高：LLM调用按token收费
2. 延迟高：生成响应需要几秒
3. 不确定性：LLM输出不完全可控
4. 依赖外部：LLM服务可能不可用
5. 安全风险：提示注入、数据泄露
```

### 设计目标

```
1. 低成本：模型路由 + 语义缓存
2. 低延迟：流式响应 + 预计算
3. 高可用：降级 + 重试 + 多模型备选
4. 安全可控：内容过滤 + 审计日志
5. 可观测：Token监控 + 质量评估
```

## 3. 模型路由

```java
@Service
public class ModelRouter {
    
    public ModelChoice route(AiRequest request) {
        // 1. 简单问答 → 便宜模型
        if (request.getEstimatedTokens() < 500 
                && !request.isComplex()) {
            return ModelChoice.of("deepseek-v4", 0.3);
        }
        
        // 2. 代码相关 → Claude
        if (request.containsCode()) {
            return ModelChoice.of("claude-4-sonnet", 3.0);
        }
        
        // 3. 中文知识库 → Qwen
        if (request.isChinese() && request.isKnowledgeQuery()) {
            return ModelChoice.of("qwen3-plus", 0.5);
        }
        
        // 4. 复杂推理 → GPT-5
        return ModelChoice.of("gpt-5", 5.0);
    }
}

// 带降级的模型路由
public ModelChoice routeWithFallback(AiRequest request) {
    try {
        ModelChoice choice = route(request);
        // 检查模型是否健康
        if (modelHealthChecker.isHealthy(choice.getModel())) {
            return choice;
        }
    } catch (Exception e) {
        log.warn("模型路由失败，使用默认模型", e);
    }
    // 降级到默认模型
    return ModelChoice.of("qwen3-plus", 0.5);
}
```

## 4. 缓存策略

### 语义缓存

```java
@Service
public class SemanticCacheService {
    
    @Autowired
    private VectorStore vectorStore;
    
    @Autowired
    private RedisTemplate<String, String> redis;
    
    private static final double SIMILARITY_THRESHOLD = 0.95;
    
    public Optional<String> get(String query) {
        // 1. 精确缓存（Redis）
        String exactKey = "ai:cache:" + DigestUtils.md5Hex(query);
        String cached = redis.opsForValue().get(exactKey);
        if (cached != null) {
            metrics.recordCacheHit("exact");
            return Optional.of(cached);
        }
        
        // 2. 语义缓存（向量检索）
        List<Document> similar = vectorStore.similaritySearch(
            SearchRequest.builder()
                .query(query)
                .topK(1)
                .similarityThreshold(SIMILARITY_THRESHOLD)
                .build());
        
        if (!similar.isEmpty()) {
            String cacheKey = similar.get(0).getMetadata().get("cacheKey");
            cached = redis.opsForValue().get(cacheKey);
            if (cached != null) {
                metrics.recordCacheHit("semantic");
                return Optional.of(cached);
            }
        }
        
        metrics.recordCacheMiss();
        return Optional.empty();
    }
    
    public void put(String query, String response, Duration ttl) {
        String cacheKey = "ai:cache:" + DigestUtils.md5Hex(query);
        
        // 存Redis
        redis.opsForValue().set(cacheKey, response, ttl);
        
        // 存向量库（用于语义检索）
        vectorStore.add(List.of(
            new Document(query, Map.of(
                "cacheKey", cacheKey,
                "time", System.currentTimeMillis()
            ))
        ));
    }
}
```

## 5. 降级与容错

```java
@Service
public class AiServiceWithFallback {
    
    @Autowired
    private List<LlmProvider> providers;  // 按优先级排序
    
    @Autowired
    private SemanticCacheService cache;
    
    public String ask(String prompt) {
        // 1. 缓存
        Optional<String> cached = cache.get(prompt);
        if (cached.isPresent()) return cached.get();
        
        // 2. 依次尝试多个Provider
        for (LlmProvider provider : providers) {
            try {
                String response = provider.chat(prompt);
                cache.put(prompt, response, Duration.ofMinutes(30));
                return response;
            } catch (Exception e) {
                log.warn("Provider {} 失败: {}", provider.getName(), e.getMessage());
                metrics.recordProviderError(provider.getName());
            }
        }
        
        // 3. 所有Provider失败 → 模板降级
        log.error("所有AI Provider不可用，降级到模板回复");
        return templateFallback(prompt);
    }
    
    private String templateFallback(String prompt) {
        // 返回预设的模板回复
        if (prompt.contains("订单")) {
            return "您的订单正在处理中，请稍后查看详情。";
        }
        return "抱歉，AI服务暂时不可用，请稍后重试。";
    }
}
```

## 6. 流式响应

```java
@RestController
@RequestMapping("/api/ai")
public class AiStreamController {
    
    @Autowired
    private ChatClient chatClient;
    
    @GetMapping(value = "/chat/stream", produces = MediaType.TEXT_EVENT_STREAM_VALUE)
    public Flux<String> streamChat(@RequestParam String prompt) {
        return chatClient.prompt()
            .user(prompt)
            .stream()
            .content()
            // 首token延迟监控
            .doOnNext(token -> metrics.recordFirstToken())
            // 超时控制
            .timeout(Duration.ofSeconds(30))
            // 错误处理
            .onErrorResume(e -> Flux.just("[AI服务异常，请重试]"));
    }
}
```

## 7. 面试要点

### Q1: AI应用的架构设计要点？

```
1. 模型路由：简单→便宜模型，复杂→强模型
2. 语义缓存：相似问题直接返回缓存，节省90%+成本
3. 降级策略：LLM不可用时降级到模板回复
4. 流式响应：SSE/WebSocket，减少首token延迟
5. 多Provider：GPT/Claude/Qwen互为备份
6. 安全过滤：提示注入防护、内容审核
7. 可观测：Token用量、延迟、成本、质量监控
```

### Q2: 语义缓存和普通缓存的区别？

```
普通缓存：Key完全匹配才命中
  "今天天气怎么样" → 命中
  "今天天气如何" → 未命中（字面不同）

语义缓存：语义相似就命中
  "今天天气怎么样" → 命中
  "今天天气如何" → 也命中（语义相同）

实现：查询向量化 → 向量检索相似问题 → 相似度>0.95则返回缓存
效果：缓存命中率从20%提升到60%+
```

### Q3: AI应用如何降级？

```
1. 模型降级：GPT-5 → Qwen3 → DeepSeek（依次尝试）
2. 功能降级：AI回复 → 模板回复 → 默认回复
3. 流式降级：流式生成 → 一次性返回
4. 缓存降级：实时生成 → 返回缓存的历史回复
5. 异步降级：实时响应 → 异步处理+通知
```

### Q4: 如何监控AI应用？

```
1. 成本监控：Token用量、API费用、模型分布
2. 性能监控：首token延迟、总延迟、吞吐量
3. 质量监控：用户反馈、回答质量评分
4. 异常监控：Provider故障率、超时率、错误率
5. 缓存监控：命中率、缓存大小、过期率

工具：Prometheus + Grafana + 自定义AI指标
```
