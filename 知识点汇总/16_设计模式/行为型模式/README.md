# 行为型模式

行为型模式关注对象之间的交互和职责分配，通过定义对象间的通信方式来完成复杂功能。

## 模式列表
- [01_命令模式](01_命令模式.md)：将请求封装为对象，支持参数化和可撤销操作
- [02_责任链模式](02_责任链模式.md)：为请求创建处理链，避免请求发送者与接收者耦合
- [03_观察者模式](03_观察者模式.md)：对象间一对多依赖，当一个对象改变状态时通知所有依赖者
- [04_迭代器模式](04_迭代器模式.md)：提供遍历聚合对象元素的方法，无需暴露内部实现
- [05_解释器模式](05_解释器模式.md)：给定语言，定义其文法的表示，并定义解释器
- [06_中介者模式](06_中介者模式.md)：用一个中介对象封装一系列对象交互
- [07_备忘录模式](07_备忘录模式.md)：在不破坏封装的前提下捕获对象内部状态并保存
- [08_状态模式](08_状态模式.md)：允许对象在内部状态改变时改变其行为
- [09_策略模式](09_策略模式.md)：定义算法族，使它们可互换
- [10_模板方法模式](10_模板方法模式.md)：定义算法骨架，允许子类重写特定步骤
- [11_访问者模式](11_访问者模式.md)：表示一个作用于某对象结构中的各元素的操作

## 核心思想
行为型模式通过对象间的交互来完成复杂功能，强调对象的职责分配和通信方式，提高系统的灵活性和可扩展性。

## 应用场景
- 需要优化对象间通信时
- 需要定义对象间复杂交互规则时
- 需要实现可复用的算法或行为时
- 需要封装对象状态变化或行为变化时
