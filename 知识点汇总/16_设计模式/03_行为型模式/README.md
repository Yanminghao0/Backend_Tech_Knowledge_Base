## 🎯 核心思想

行为型模式通过**对象间的交互**来完成复杂功能，强调对象的职责分配和通信方式，提高系统的灵活性和可扩展性。

```
行为型模式解决的问题：
├── 请求的处理方式 → 命令、责任链
├── 对象间通知机制 → 观察者
├── 遍历方式封装 → 迭代器
├── 复杂交互解耦 → 中介者
├── 状态与行为耦合 → 状态模式
├── 算法选择与切换 → 策略模式
├── 算法骨架固定 → 模板方法
├── 操作与结构分离 → 访问者
├── 状态保存恢复 → 备忘录
└── 语言解释执行 → 解释器
```

## 📊 模式对比

| 维度 | 策略 | 状态 | 责任链 | 观察者 |
|------|------|------|--------|--------|
| 意图 | 算法切换 | 状态驱动 | 请求传递 | 事件通知 |
| 关系 | 平等可互换 | 状态间转换 | 链式传递 | 一对多 |
| 运行时切换 | 是 | 是 | 否(链固定) | 否 |
| 典型场景 | 支付方式 | 订单状态流 | 审批流程 | 事件总线 |

## 📈 学习路径

```
1. 模板方法模式 — 理解算法骨架与步骤扩展（最基础）
      ↓
2. 策略模式 — 理解算法族封装与切换（面试高频）
      ↓
3. 观察者模式 — 理解事件驱动机制（Spring事件基础）
      ↓
4. 责任链模式 — 理解请求处理链（Filter/Interceptor基础）
      ↓
5. 状态模式 — 理解状态驱动的行为变化
      ↓
6. 命令模式 — 理解请求封装与撤销
      ↓
7. 迭代器模式 — 理解统一遍历接口
      ↓
8. 备忘录模式 — 理解状态保存与恢复
      ↓
9. 中介者模式 — 理解多对象交互解耦
      ↓
10. 访问者模式 — 理解操作与结构分离（最复杂）
      ↓
11. 解释器模式 — 理解文法解析
```

---

## 💡 应用场景

- 需要优化对象间通信时（观察者、中介者）
- 需要定义对象间复杂交互规则时（责任链、命令）
- 需要实现可复用的算法或行为时（策略、模板方法）
- 需要封装对象状态变化或行为变化时（状态、备忘录）
- 需要对对象结构执行多种操作时（访问者）
