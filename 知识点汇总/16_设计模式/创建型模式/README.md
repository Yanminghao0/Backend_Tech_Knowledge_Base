# 创建型模式

创建型模式关注对象的创建过程，将对象的创建与使用分离，提供了灵活的对象创建机制。

## 模式列表
- [01_单例模式](01_单例模式.md)：确保一个类只有一个实例，并提供全局访问点
- [02_工厂方法模式](02_工厂方法模式.md)：定义创建对象的接口，让子类决定实例化哪个类
- [03_抽象工厂模式](03_抽象工厂模式.md)：提供一个创建一系列相关或相互依赖对象的接口
- [04_建造者模式](04_建造者模式.md)：将复杂对象的构建过程与表示分离
- [05_原型模式](05_原型模式.md)：通过复制现有对象来创建新对象

## 核心思想
创建型模式的核心在于封装对象创建的细节，通过抽象工厂、建造者等方式，提供灵活的对象创建策略，同时隐藏具体实现。

## 应用场景
- 需要控制对象创建过程时
- 系统需要独立于其产品的创建、组合和表示时
- 希望通过配置或参数决定创建哪种对象时
