# 创建型模式

> 创建型模式关注对象的创建过程，将对象的创建与使用分离，提供灵活的对象创建机制

## 🎯 核心思想

创建型模式的核心在于**封装对象创建的细节**，通过抽象工厂、建造者等方式，提供灵活的对象创建策略，同时隐藏具体实现，使客户端代码不依赖于具体类。

```
创建型模式解决的问题：
├── new的直接依赖 → 解耦创建与使用
├── 对象创建逻辑复杂 → 封装创建过程
├── 需要控制实例数量 → 单例控制
├── 需要创建复杂对象 → 建造者分步构建
└── 需要创建对象族 → 抽象工厂统一创建
```

## 📊 模式对比

| 维度 | 单例 | 工厂方法 | 抽象工厂 | 建造者 | 原型 |
|------|------|----------|----------|--------|------|
| 关注点 | 唯一性 | 单产品创建 | 产品族创建 | 复杂对象构建 | 对象复制 |
| 扩展性 | 低 | 高(加子类) | 高(加工厂) | 高(加步骤) | 中 |
| 复杂度 | 低 | 中 | 高 | 中 | 低 |
| 典型场景 | 配置管理、连接池 | 日志工厂 | UI跨平台组件 | SQL构建器 | 缓存对象复制 |

```

      ↓

      ↓
3. 抽象工厂模式 — 理解产品族的概念
      ↓

      ↓

```

---

## 💡 应用场景

- 需要控制对象创建过程时（单例、工厂）
- 系统需要独立于其产品的创建、组合和表示时（抽象工厂）
- 希望通过配置或参数决定创建哪种对象时（工厂方法）
- 构建复杂对象时需要分步骤组装时（建造者）
- 需要避免重复初始化开销，通过复制创建对象时（原型）
