## 📚 目录

### 一、设计模式概述

#### 设计模式六大原则
1. **单一职责原则（SRP）** - Single Responsibility Principle
2. **开闭原则（OCP）** - Open Closed Principle
3. **里氏替换原则（LSP）** - Liskov Substitution Principle
4. **依赖倒置原则（DIP）** - Dependence Inversion Principle
5. **接口隔离原则（ISP）** - Interface Segregation Principle
6. **迪米特法则（LoD）** - Law of Demeter

> 关注对象的创建过程，将对象的创建和使用分离

| 模式 | 描述 | 使用场景 | 文档链接 |
|------|------|----------|----------|
| 单例模式 | 确保一个类只有一个实例 | 配置管理、线程池、日志对象 | [查看详情](./创建型模式/01_单例模式.md) |
| 工厂方法模式 | 定义创建对象的接口，让子类决定实例化 | JDBC驱动加载、日志框架 | [查看详情](./创建型模式/02_工厂方法模式.md) |
| 抽象工厂模式 | 创建相关或依赖对象的家族 | 跨平台UI组件、数据库访问层 | [查看详情](./创建型模式/03_抽象工厂模式.md) |

## 📊 设计模式关系图

```
创建型模式 ──┐
           ├──> 对象创建与使用分离
结构型模式 ──┤
           ├──> 类和对象的组合
行为型模式 ──┘
           └──> 对象间的通信

常用组合：
- 工厂 + 单例
- 抽象工厂 + 单例
- 建造者 + 组合
- 策略 + 工厂
- 模板方法 + 工厂
- 观察者 + 单例
- 责任链 + 组合
```

## 💡 设计模式在Java/Spring中的应用

### Java核心库
- **单例**: `Runtime`
- **工厂**: `Calendar.getInstance()`
- **建造者**: `StringBuilder`
- **适配器**: `InputStreamReader`
- **装饰器**: `BufferedReader`
- **代理**: JDK动态代理
- **迭代器**: `Iterator`
- **观察者**: `java.util.Observable`
- **模板方法**: `AbstractList`

### Spring框架
- **单例**: Spring Bean默认作用域
- **工厂**: `BeanFactory`
- **代理**: Spring AOP
- **模板方法**: `JdbcTemplate`, `RestTemplate`
- **观察者**: `ApplicationListener`
- **策略**: `Resource`
- **适配器**: `HandlerAdapter`
- **责任链**: `FilterChain`, `Interceptor`

### MyBatis
- **建造者**: `SqlSessionFactoryBuilder`
- **工厂**: `SqlSessionFactory`
- **代理**: Mapper接口动态代理
- **装饰器**: `CachingExecutor`
- **模板方法**: `BaseExecutor`

## 🔍 快速检索

**按使用频率**:
- 高频: 单例、工厂、代理、策略、观察者
- 中频: 建造者、适配器、装饰器、模板方法、责任链
- 低频: 原型、桥接、享元、中介者、备忘录、访问者

**按应用场景**:
- 对象创建: 单例、工厂、建造者、原型
- 接口适配: 适配器、桥接、外观
- 功能增强: 装饰器、代理
- 算法选择: 策略、模板方法
- 状态管理: 状态、备忘录
- 对象通信: 观察者、中介者、责任链
