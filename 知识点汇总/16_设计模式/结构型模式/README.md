# 结构型模式

结构型模式关注类和对象的组合方式，通过继承或组合来实现新的功能或更高效的结构。

## 模式列表
- [01_适配器模式](01_适配器模式.md)：将一个类的接口转换成客户期望的另一个接口
- [02_桥接模式](02_桥接模式.md)：将抽象部分与实现部分分离，使它们可以独立变化
- [03_组合模式](03_组合模式.md)：将对象组合成树形结构以表示“部分-整体”的层次结构
- [04_装饰器模式](04_装饰器模式.md)：动态地给对象添加一些额外的职责
- [05_外观模式](05_外观模式.md)：为子系统中的一组接口提供一个一致的界面
- [06_享元模式](06_享元模式.md)：运用共享技术有效地支持大量细粒度的对象
- [07_代理模式](07_代理模式.md)：为其他对象提供一种代理以控制对这个对象的访问
- [08_过滤器模式](08_过滤器模式.md)：使用不同的标准过滤一组对象

## 核心思想
结构型模式通过组合类或对象来形成更大的结构，解决系统中类和对象的组合关系，提高代码的灵活性和可复用性。

## 应用场景
- 需要优化类结构以减少复杂度时
- 需要在不修改原有代码的情况下扩展功能时
- 需要简化客户端与复杂子系统的交互时
