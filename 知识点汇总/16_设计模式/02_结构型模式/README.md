# 结构型模式

> 结构型模式关注类和对象的组合方式，通过继承或组合来实现新的功能或更高效的结构

## 🎯 核心思想

结构型模式通过**组合类或对象**来形成更大的结构，解决系统中类和对象的组合关系，提高代码的灵活性和可复用性。

```
结构型模式解决的问题：
├── 接口不兼容 → 适配器模式
├── 抽象与实现耦合 → 桥接模式
├── 树形结构处理 → 组合模式
├── 功能动态扩展 → 装饰器模式
├── 子系统复杂度高 → 外观模式
├── 大量相似对象 → 享元模式
├── 访问控制/延迟加载 → 代理模式
└── 多条件过滤 → 过滤器模式
```

| 维度 | 适配器 | 装饰器 | 代理 | 外观 |

| 意图 | 接口转换 | 增加职责 | 控制访问 | 简化接口 |

| 对象数量 | 1对1 | 可叠加 | 1对1 | 1对多 |

## 📈 学习路径

```
1. 适配器模式 — 理解接口转换
      ↓
2. 代理模式 — 理解间接访问控制（Spring AOP基础）
      ↓
3. 装饰器模式 — 理解动态功能扩展
      ↓
4. 外观模式 — 理解子系统封装
      ↓
5. 组合模式 — 理解树形结构统一处理
      ↓
6. 桥接模式 — 理解多维度变化分离
      ↓
7. 享元模式 — 理解对象共享与池化
      ↓
8. 过滤器模式 — 理解多标准筛选
```

---

## 💡 应用场景

- 需要优化类结构以减少复杂度时（外观、适配器）
- 需要在不修改原有代码的情况下扩展功能时（装饰器、代理）
- 需要简化客户端与复杂子系统的交互时（外观）
- 需要处理树形层次结构时（组合）
- 需要减少大量相似对象的内存开销时（享元）
