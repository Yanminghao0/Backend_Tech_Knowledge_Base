# 堆

堆（Heap）是一种特殊的完全二叉树数据结构，除了基本的堆操作外，在实际应用中还有许多高级用法和优化策略。本文档将重点介绍堆的高级应用场景、优化技术以及在复杂问题中的解决方案。

---

## 📋 基本信息

| 项目 | 说明 |
|------|------|
| 数据结构类型 | 完全二叉树的一种特殊形式 |
| 存储结构 | 数组（为主） |
| 核心操作 | 构建堆、插入、删除、堆化、合并 |
| 扩展操作 | Top-K查询、中位数维护、区间最值查询 |
| 时间复杂度 | 构建堆：O(n)，插入/删除：O(log n)，Top-K：O(n log k) |
| 空间复杂度 | O(n) |
| 特点 | 高效的优先级管理，适合处理动态数据集合的最值问题 |

## 🎯 核心特性

1. **优先级管理**：能够快速访问和删除最大/最小值，适合实现优先级队列

2. **部分排序**：堆排序只需O(n log n)时间，可用于需要部分有序结果的场景

3. **动态集合**：支持高效插入和删除操作，适合处理数据流

4. **内存效率**：数组实现无需额外指针，缓存友好

5. **合并能力**：某些高级堆结构（如斐波那契堆）支持高效合并操作

6. **稳定性**：基本堆结构不稳定，但可通过扩展实现稳定排序

7. **并行性**：堆操作可并行化，适合多核处理环境

## 💻 代码实现

### 1. Top-K问题的多种解法

#### 解法一：小顶堆法（适合大数据量）
```java
import java.util.PriorityQueue;
import java.util.Comparator;

public class TopKUsingMinHeap {
    public int[] findTopK(int[] nums, int k) {
        if (k <= 0 || nums == null || nums.length == 0) {
            return new int[0];
        }

        // 创建小顶堆
        PriorityQueue<Integer> minHeap = new PriorityQueue<>(Comparator.naturalOrder());

        // 遍历数组，维护堆大小为k
        for (int num : nums) {
            if (minHeap.size() < k) {
                minHeap.offer(num);
            } else if (num > minHeap.peek()) {
                minHeap.poll();
                minHeap.offer(num);
            }
        }

        // 提取结果
        int[] result = new int[minHeap.size()];
        int index = 0;
        while (!minHeap.isEmpty()) {
            result[index++] = minHeap.poll();
        }
        return result;
    }
}
```

#### 解法二：大顶堆法（适合小数据量）
```java
import java.util.PriorityQueue;
import java.util.Comparator;
import java.util.Arrays;

public class TopKUsingMaxHeap {
    public int[] findTopK(int[] nums, int k) {
        if (k <= 0 || nums == null || nums.length == 0) {
            return new int[0];
        }

        // 创建大顶堆
        PriorityQueue<Integer> maxHeap = new PriorityQueue<>(Comparator.reverseOrder());

        // 所有元素入堆
        for (int num : nums) {
            maxHeap.offer(num);
        }

        // 提取前k个元素
        int size = Math.min(k, maxHeap.size());
        int[] result = new int[size];
        for (int i = 0; i < size; i++) {
            result[i] = maxHeap.poll();
        }
        return result;
    }
}
```

### 2. 滑动窗口最大值
```java
import java.util.PriorityQueue;
import java.util.Comparator;
import java.util.ArrayList;
import java.util.List;

public class SlidingWindowMaximum {
    static class Element {
        int value;
        int index;

        Element(int value, int index) {
            this.value = value;
            this.index = index;
        }
    }

    public int[] maxSlidingWindow(int[] nums, int k) {
        if (nums == null || nums.length == 0 || k <= 0) {
            return new int[0];
        }

        // 大顶堆，按值排序，值相等则按索引降序
        PriorityQueue<Element> maxHeap = new PriorityQueue<>(
            (a, b) -> a.value != b.value ? b.value - a.value : b.index - a.index
        );

        List<Integer> result = new ArrayList>();

        for (int i = 0; i < nums.length; i++) {
            // 添加当前元素
            maxHeap.offer(new Element(nums[i], i));

            // 移除窗口外的元素
            while (maxHeap.peek().index <= i - k) {
                maxHeap.poll();
            }

            // 窗口形成后开始记录结果
            if (i >= k - 1) {
                result.add(maxHeap.peek().value);
            }
        }

        // 转换为数组
        return result.stream().mapToInt(Integer::intValue).toArray();
    }
}
```

### 3. 数据流中的中位数
```java
import java.util.PriorityQueue;
import java.util.Comparator;

public class MedianFinder {
    private PriorityQueue<Integer> maxHeap; // 存储较小一半元素
    private PriorityQueue<Integer> minHeap; // 存储较大一半元素

    public MedianFinder() {
        maxHeap = new PriorityQueue<>(Comparator.reverseOrder());
        minHeap = new PriorityQueue<>(Comparator.naturalOrder());
    }

    public void addNum(int num) {
        // 保证maxHeap元素数量大于等于minHeap
        if (maxHeap.isEmpty() || num <= maxHeap.peek()) {
            maxHeap.offer(num);
        } else {
            minHeap.offer(num);
        }

        // 平衡两个堆的大小
        if (maxHeap.size() > minHeap.size() + 1) {
            minHeap.offer(maxHeap.poll());
        } else if (minHeap.size() > maxHeap.size()) {
            maxHeap.offer(minHeap.poll());
        }
    }

    public double findMedian() {
        if (maxHeap.size() == minHeap.size()) {
            return (maxHeap.peek() + minHeap.peek()) / 2.0;
        } else {
            return maxHeap.peek();
        }
    }
}
```

## 📊 复杂度分析

| 操作 | 时间复杂度 | 空间复杂度 | 适用场景 |
|------|------------|------------|----------|
| Top-K（小顶堆） | O(n log k) | O(k) | 大数据集，k较小 |
| Top-K（大顶堆） | O(n + k log n) | O(n) | 小数据集，k较大 |
| 滑动窗口最大值 | O(n log k) | O(k) | 窗口大小为k的数据流 |
| 中位数维护 | O(log n) | O(n) | 动态数据流 |
| 堆排序 | O(n log n) | O(1) | 原地排序，不稳定 |
| 优先队列 | O(log n) | O(n) | 任务调度，事件驱动 |

> 注：
> - 小顶堆法求解Top-K问题在n很大时优势明显
> - 双堆法维护中位数实现了O(log n)的插入和O(1)的查询
> - 滑动窗口问题中，堆解法并非最优，可使用单调队列优化至O(n)

## ⚠️ 注意事项

1. **堆的选择**：
   - 大顶堆适合获取最大值，小顶堆适合获取最小值
   - 双堆组合适合中位数等中间值查询
   - 高级堆结构（如斐波那契堆）适合频繁合并操作

2. **边界处理**：
   - 空堆操作会导致异常，需提前检查
   - 数据流为空时的中位数返回策略
   - Top-K问题中k=0或k大于数据量的情况

3. **性能优化**：
   - 原始数组构建堆可通过heapify操作优化至O(n)
   - 避免频繁的堆大小调整
   - 大数据量时考虑并行堆实现

4. **内存考量**：
   - 堆排序是原地排序，适合内存受限场景
   - 优先队列实现可能需要额外内存
   - 滑动窗口问题中注意清理过期元素

5. **稳定性问题**：
   - 堆排序是不稳定排序
   - 处理对象排序时需注意相等元素的顺序

## 🎓 最佳实践

1. **选择合适的实现**：
   - Java中优先使用`PriorityQueue`，而非手动实现
   - 大数据量Top-K问题优先使用小顶堆法
   - 滑动窗口最大值考虑使用单调队列替代堆

2. **性能调优**：
   - 预分配堆容量减少扩容开销
   - 自定义比较器时优化比较逻辑
   - 批量操作时考虑先排序后处理

3. **高级应用技巧**：
   - 使用延迟删除策略处理堆中的无效元素
   - 结合哈希表快速查找堆中元素
   - 多级堆结构处理复杂优先级场景

4. **线程安全**：
   - 多线程环境使用`PriorityBlockingQueue`
   - 或使用锁机制保护普通堆操作

5. **算法选择指南**：
   - 静态数据排序：优先考虑快速排序
   - 动态数据最值：必须使用堆或优先队列
   - 海量数据处理：结合外部排序和堆
