# 大整数乘法

> 大整数乘法（Large Integer Multiplication）是处理超出标准数据类型表示范围的整数相乘问题，分治策略可显著优化其时间复杂度。

---

## 📋 基本信息
- **类别**：分治算法、数值计算
- **难度**：困难
- **核心思想**：将大整数分解为较小部分递归计算
- **时间复杂度**：传统分治O(n²)，Karatsuba算法O(n^log₂3)≈O(n¹.⁵⁸⁵)
- **空间复杂度**：O(n)
- **关键词**：分治、大数运算、Karatsuba算法、高精度计算

## 🎯 分治策略
大整数乘法的分治思想体现在：
1. **分解**：将两个n位大整数X和Y各分为两部分，X=A×10ⁿ/²+B，Y=C×10ⁿ/²+D
2. **解决**：递归计算AC、AD、BC、BD四个乘积
3. **合并**：通过公式X×Y=AC×10ⁿ+(AD+BC)×10ⁿ/²+BD合并结果

**Karatsuba优化**：通过计算(A+B)(C+D)-AC-BD得到AD+BC，将4次乘法减少为3次

## 💻 代码实现
### Java实现（Karatsuba算法）
```java
public class LargeIntegerMultiplication {
    // 主函数：大整数乘法（字符串表示）
    public String multiply(String num1, String num2) {
        // 处理负数
        boolean negative = false;
        if (num1.charAt(0) == '-' && num2.charAt(0) == '-') {
            num1 = num1.substring(1);
            num2 = num2.substring(1);
        } else if (num1.charAt(0) == '-') {
            negative = true;
            num1 = num1.substring(1);
        } else if (num2.charAt(0) == '-') {
            negative = true;
            num2 = num2.substring(1);
        }

        String result = karatsuba(num1, num2);
        return negative ? "-" + result : result;
    }

    // Karatsuba算法实现
    private String karatsuba(String x, String y) {
        // 基础情况：若数字较短，直接相乘
        if (x.length() <= 3 || y.length() <= 3) {
            return multiplySmall(x, y);
        }

        int n = Math.max(x.length(), y.length());
        int half = (n + 1) / 2;

        // 分割数字
        String a = x.length() > half ? x.substring(0, x.length() - half) : "0";
        String b = x.length() > half ? x.substring(x.length() - half) : x;
        String c = y.length() > half ? y.substring(0, y.length() - half) : "0";
        String d = y.length() > half ? y.substring(y.length() - half) : y;

        // 递归计算三个乘积
        String ac = karatsuba(a, c);
        String bd = karatsuba(b, d);
        String abcd = karatsuba(add(a, b), add(c, d));
        String adbc = subtract(subtract(abcd, ac), bd);

        // 合并结果：ac*10^(2*half) + adbc*10^half + bd
        String result = add(add(pow10(ac, 2 * half), pow10(adbc, half)), bd);

        return result;
    }

    // 辅助函数：较小数字相乘
    private String multiplySmall(String a, String b) {
        int numA = Integer.parseInt(a);
        int numB = Integer.parseInt(b);
        return String.valueOf(numA * numB);
    }

    // 辅助函数：字符串数字加法
    private String add(String a, String b) {
        // 实现字符串表示的大整数加法
        int i = a.length() - 1;
        int j = b.length() - 1;
        int carry = 0;
        StringBuilder sb = new StringBuilder();

        while (i >= 0 || j >= 0 || carry > 0) {
            int sum = carry;
            if (i >= 0) sum += a.charAt(i--) - '0';
            if (j >= 0) sum += b.charAt(j--) - '0';
            carry = sum / 10;
            sb.append(sum % 10);
        }

        return sb.reverse().toString().replaceFirst("^0+", "");;
    }

    // 辅助函数：字符串数字减法（假设a >= b）
    private String subtract(String a, String b) {
        // 实现字符串表示的大整数减法
        int i = a.length() - 1;
        int j = b.length() - 1;
        int borrow = 0;
        StringBuilder sb = new StringBuilder();

        while (i >= 0 || j >= 0) {
            int digitA = i >= 0 ? a.charAt(i--) - '0' : 0;
            int digitB = j >= 0 ? b.charAt(j--) - '0' : 0;

            digitA -= borrow;
            if (digitA < digitB) {
                digitA += 10;
                borrow = 1;
            } else {
                borrow = 0;
            }

            sb.append(digitA - digitB);
        }

        String result = sb.reverse().toString().replaceFirst("^0+", "");
        return result.isEmpty() ? "0" : result;
    }

    // 辅助函数：数字乘以10^n（字符串表示）
    private String pow10(String num, int n) {
        if (num.equals("0")) return "0";
        StringBuilder sb = new StringBuilder(num);
        for (int i = 0; i < n; i++) {
            sb.append("0");
        }
        return sb.toString();
    }
}
```

## 📊 复杂度分析
- **时间复杂度**：
  - 传统分治算法：O(n²)（需4次递归乘法）
  - Karatsuba算法：O(n^log₂3)≈O(n¹.⁵⁸⁵)（仅需3次递归乘法）
  - Toom-Cook算法：更高阶优化可达到O(n¹.⁴⁶⁵)或更低
- **空间复杂度**：O(n)
  - 存储递归计算的中间结果
  - 字符串操作所需的临时空间

## ⚠️ 注意事项
1. **符号处理**：需单独处理正负数乘法
2. **前导零**：运算结果需去除前导零
3. **边界情况**：处理其中一个数为0的情况
4. **递归深度**：过深递归可能导致栈溢出
5. **基础情况阈值**：过小阈值会增加函数调用开销，过大则失去分治优势

## 💡 最佳实践
- 选择合适的基础情况阈值（通常3-6位数字）
- 结合FFT（快速傅里叶变换）实现更高效的大整数乘法
- 使用字符数组而非字符串提升性能
- 实现缓存机制避免重复计算
- 对极端大整数考虑使用并行分治策略
- 结合硬件加速指令优化底层运算
