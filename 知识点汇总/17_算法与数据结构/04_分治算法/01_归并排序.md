## 基本信息
- **类别**：分治算法
- **难度**：中等
- **稳定性**：稳定
- **时间复杂度**：O(n log n)（所有情况）
- **空间复杂度**：O(n)
- **关键词**：分治、合并、递归、稳定排序、外部排序
- **发明者**：John von Neumann（1945年）
- **特点**：时间复杂度恒定，适合需要稳定排序的场景

## 🎯 分治策略
归并排序的分治思想体现在三个步骤：

1. **分解（Divide）**：将数组递归拆分为两个 halves，直到每个子数组只包含一个元素
2. **解决（Conquer）**：递归地对两个子数组进行归并排序
3. **合并（Combine）**：将已排序的子数组合并为一个有序数组

### 执行流程图解

```
原始数组: [38, 27, 43, 3, 9, 82, 10]

分解阶段:
                    [38, 27, 43, 3, 9, 82, 10]
                   /                          \
           [38, 27, 43, 3]              [9, 82, 10]
           /            \              /           \
       [38, 27]      [43, 3]       [9, 82]       [10]
       /     \       /     \       /     \         |
    [38]   [27]   [43]   [3]    [9]   [82]     [10]

合并阶段:
       [27, 38]      [3, 43]       [9, 82]       [10]
           \            /              \           /
           [3, 27, 38, 43]              [9, 10, 82]
                   \                          /
                   [3, 9, 10, 27, 38, 43, 82]

最终结果: [3, 9, 10, 27, 38, 43, 82]
```

### 递归树深度分析
```
第0层: [38, 27, 43, 3, 9, 82, 10]        → 1个区间，长度7
第1层: [38,27,43,3] [9,82,10]           → 2个区间
第2层: [38,27] [43,3] [9,82] [10]       → 4个区间
第3层: [38] [27] [43] [3] [9] [82] [10] → 7个区间，长度1（基准情况）

总层数: ⌈log₂(7)⌉ + 1 = 4层
每层合并操作总工作量: O(n)
总时间复杂度: O(n log n)
```

## 核心概念详解

### Merge操作
Merge是归并排序的核心操作，将两个已排序的子数组合并为一个有序数组：
- 使用双指针分别遍历两个子数组
- 每次比较两个指针所指元素，取较小者放入结果数组
- 当一个子数组遍历完后，将另一个子数组的剩余元素直接复制

### 自顶向下 vs 自底向上
- **自顶向下（Top-Down）**：递归实现，从整个数组开始不断二分
- **自底向上（Bottom-Up）**：迭代实现，从长度1的子数组开始，逐步合并成更大的有序数组

## 💻 代码实现

### Java实现（递归版 - 自顶向下）
```java
public class MergeSort {
    
    /**
     * 归并排序入口
     */
    public void mergeSort(int[] arr) {
        if (arr == null || arr.length <= 1) return;
        int[] temp = new int[arr.length];  // 预分配辅助数组，避免重复创建
        sort(arr, 0, arr.length - 1, temp);
    }

    /**
     * 递归排序
     */
    private void sort(int[] arr, int left, int right, int[] temp) {
        if (left < right) {
            int mid = left + (right - left) / 2;
            // 递归排序左半部分
            sort(arr, left, mid, temp);
            // 递归排序右半部分
            sort(arr, mid + 1, right, temp);
            // 合并两个有序子数组
            merge(arr, left, mid, right, temp);
        }
    }

    /**
     * 合并两个有序子数组
     * 左子数组: arr[left..mid]
     * 右子数组: arr[mid+1..right]
     */
    private void merge(int[] arr, int left, int mid, int right, int[] temp) {
        int i = left;       // 左子数组起始索引
        int j = mid + 1;    // 右子数组起始索引
        int t = 0;          // 临时数组指针

        // 合并两个子数组：每次取较小者
        while (i <= mid && j <= right) {
            if (arr[i] <= arr[j]) {
                temp[t++] = arr[i++];
            } else {
                temp[t++] = arr[j++];
            }
        }

        // 将左子数组剩余元素复制到临时数组
        while (i <= mid) {
            temp[t++] = arr[i++];
        }

        // 将右子数组剩余元素复制到临时数组
        while (j <= right) {
            temp[t++] = arr[j++];
        }

        // 将临时数组元素复制回原数组
        t = 0;
        while (left <= right) {
            arr[left++] = temp[t++];
        }
    }
}
```

### Java实现（迭代版 - 自底向上）
```java
/**
 * 自底向上归并排序：无递归，避免栈溢出
 * 从长度1的子数组开始，逐步合并
 */
public void mergeSortIterative(int[] arr) {
    if (arr == null || arr.length <= 1) return;
    int n = arr.length;
    int[] temp = new int[n];
    
    // 子数组大小从1开始，每次翻倍: 1 → 2 → 4 → 8 → ...
    for (int size = 1; size < n; size *= 2) {
        // 合并相邻的两个长度为size的子数组
        for (int left = 0; left < n - size; left += 2 * size) {
            int mid = left + size - 1;
            int right = Math.min(left + 2 * size - 1, n - 1);
            merge(arr, left, mid, right, temp);
        }
    }
}
```

### Python实现
```python
def merge_sort(arr):
    """归并排序 - Python递归版"""
    if len(arr) <= 1:
        return arr
    
    mid = len(arr) // 2
    left = merge_sort(arr[:mid])
    right = merge_sort(arr[mid:])
    
    return merge(left, right)

def merge(left, right):
    """合并两个有序数组"""
    result = []
    i = j = 0
    
    while i < len(left) and j < len(right):
        if left[i] <= right[j]:
            result.append(left[i])
            i += 1
        else:
            result.append(right[j])
            j += 1
    
    result.extend(left[i:])
    result.extend(right[j:])
    return result

# 原地归并排序
def merge_sort_inplace(arr):
    _merge_sort(arr, 0, len(arr) - 1)
    return arr

def _merge_sort(arr, left, right):
    if left < right:
        mid = (left + right) // 2
        _merge_sort(arr, left, mid)
        _merge_sort(arr, mid + 1, right)
        _merge(arr, left, mid, right)

def _merge(arr, left, mid, right):
    temp = arr[left:right + 1]
    i, j, k = 0, mid - left + 1, left
    
    while i <= mid - left and j <= right - left:
        if temp[i] <= temp[j]:
            arr[k] = temp[i]; i += 1
        else:
            arr[k] = temp[j]; j += 1
        k += 1
    
    while i <= mid - left:
        arr[k] = temp[i]; i += 1; k += 1
    while j <= right - left:
        arr[k] = temp[j]; j += 1; k += 1
```

### 链表归并排序
```java
/**
 * 链表归并排序：空间复杂度O(1)，不需要额外数组
 */
public ListNode mergeSortList(ListNode head) {
    if (head == null || head.next == null) return head;
    
    // 快慢指针找中点
    ListNode slow = head, fast = head.next;
    while (fast != null && fast.next != null) {
        slow = slow.next;
        fast = fast.next.next;
    }
    
    ListNode mid = slow.next;
    slow.next = null;  // 断开
    
    // 递归排序
    ListNode left = mergeSortList(head);
    ListNode right = mergeSortList(mid);
    
    // 合并
    return mergeLists(left, right);
}

private ListNode mergeLists(ListNode l1, ListNode l2) {
    ListNode dummy = new ListNode(0);
    ListNode curr = dummy;
    
    while (l1 != null && l2 != null) {
        if (l1.val <= l2.val) {
            curr.next = l1; l1 = l1.next;
        } else {
            curr.next = l2; l2 = l2.next;
        }
        curr = curr.next;
    }
    curr.next = (l1 != null) ? l1 : l2;
    return dummy.next;
}
```

## Merge操作详解

### 双指针合并过程示例
```
左子数组: [3, 27, 38, 43]    右子数组: [9, 10, 82]
i=0 → 3                      j=0 → 9

步骤1: 3 ≤ 9  → temp=[3],  i=1
步骤2: 27 > 9  → temp=[3,9],  j=1
步骤3: 27 > 10 → temp=[3,9,10], j=2
步骤4: 27 ≤ 82 → temp=[3,9,10,27], i=2
步骤5: 38 ≤ 82 → temp=[3,9,10,27,38], i=3
步骤6: 43 ≤ 82 → temp=[3,9,10,27,38,43], i=4 (左子数组耗尽)
步骤7: 复制右子数组剩余 → temp=[3,9,10,27,38,43,82]

最终: [3, 9, 10, 27, 38, 43, 82]
```

### 稳定性分析
```
归并排序保持稳定性的关键: 当 arr[i] == arr[j] 时，优先取左子数组的元素

不稳定的情况（如果使用 < 而非 <=）:
  左: [3a, 5, 7]    右: [3b, 6]
  若 arr[i] < arr[j]: 3a < 3b → 先取3b，破坏稳定性
  若 arr[i] <= arr[j]: 3a <= 3b → 先取3a，保持稳定性 ✓
```

### 时间复杂度
```
递推关系: T(n) = 2T(n/2) + O(n)

展开:
  T(n) = 2T(n/2) + cn
       = 4T(n/4) + 2cn
       = 8T(n/8) + 3cn

无论输入数据如何（最好、最坏、平均），时间复杂度恒为O(n log n)
```

## 优化策略

### 1. 小数组切换插入排序
```java
private static final int INSERTION_THRESHOLD = 15;

private void sort(int[] arr, int left, int right, int[] temp) {
    if (right - left < INSERTION_THRESHOLD) {
        insertionSort(arr, left, right);
        return;
    }
    int mid = left + (right - left) / 2;
    sort(arr, left, mid, temp);
    sort(arr, mid + 1, right, temp);
    merge(arr, left, mid, right, temp);
}

private void insertionSort(int[] arr, int left, int right) {
    for (int i = left + 1; i <= right; i++) {
        int key = arr[i];
        int j = i - 1;
        while (j >= left && arr[j] > key) {
            arr[j + 1] = arr[j];
            j--;
        }
        arr[j + 1] = key;
    }
}
```

### 2. 跳过已有序的合并
```java
private void sort(int[] arr, int left, int right, int[] temp) {
    if (left < right) {
        int mid = left + (right - left) / 2;
        sort(arr, left, mid, temp);
        sort(arr, mid + 1, right, temp);
        
        // 优化：如果 arr[mid] <= arr[mid+1]，说明已经有序，跳过合并
        if (arr[mid] <= arr[mid + 1]) {
            return;
        }
        
        merge(arr, left, mid, right, temp);
    }
}
```

### 3. 避免频繁复制（交替使用原数组和辅助数组）
```java
/**
 * 优化：交替使用arr和temp，减少复制操作
 * src和dst在每一层递归中交替
 */
private void sort(int[] src, int[] dst, int left, int right) {
    if (left >= right) return;
    
    if (right - left < INSERTION_THRESHOLD) {
        insertionSort(dst, left, right);
        return;
    }
    
    int mid = left + (right - left) / 2;
    sort(dst, src, left, mid);      // 注意: src和dst交换
    sort(dst, src, mid + 1, right);
    merge(src, dst, left, mid, right);
}

private void merge(int[] src, int[] dst, int left, int mid, int right) {
    int i = left, j = mid + 1, k = left;
    while (i <= mid && j <= right) {
        if (src[i] <= src[j]) dst[k++] = src[i++];
        else dst[k++] = src[j++];
    }
    while (i <= mid) dst[k++] = src[i++];
    while (j <= right) dst[k++] = src[j++];
}
```

### 4. 并行归并排序
```java
/**
 * 并行归并排序：利用ForkJoin框架并行处理
 */
public class ParallelMergeSort extends RecursiveAction {
    private final int[] arr;
    private final int[] temp;
    private final int left, right;
    private static final int THRESHOLD = 10000;
    
    public ParallelMergeSort(int[] arr, int left, int right, int[] temp) {
        this.arr = arr;
        this.temp = temp;
        this.left = left;
        this.right = right;
    }
    
    @Override
    protected void compute() {
        if (right - left < THRESHOLD) {
            // 小任务直接排序
            Arrays.sort(arr, left, right + 1);
            return;
        }
        int mid = left + (right - left) / 2;
        // 并行执行两个子任务
        invokeAll(
            new ParallelMergeSort(arr, left, mid, temp),
            new ParallelMergeSort(arr, mid + 1, right, temp)
        );
        merge(arr, left, mid, right, temp);
    }
}
```

## 经典应用

### 1. 逆序对计数
```java
/**
 * 利用归并排序计算逆序对数量
 * 逆序对: i < j 但 arr[i] > arr[j] 的数对
 * 时间复杂度: O(n log n)  vs 暴力法 O(n²)
 */
private int count = 0;

public int countInversions(int[] arr) {
    if (arr == null || arr.length <= 1) return 0;
    int[] temp = new int[arr.length];
    mergeSortAndCount(arr, 0, arr.length - 1, temp);
    return count;
}

private void mergeSortAndCount(int[] arr, int left, int right, int[] temp) {
    if (left < right) {
        int mid = left + (right - left) / 2;
        mergeSortAndCount(arr, left, mid, temp);
        mergeSortAndCount(arr, mid + 1, right, temp);
        mergeAndCount(arr, left, mid, right, temp);
    }
}

private void mergeAndCount(int[] arr, int left, int mid, int right, int[] temp) {
    int i = left, j = mid + 1, t = 0;
    while (i <= mid && j <= right) {
        if (arr[i] <= arr[j]) {
            temp[t++] = arr[i++];
        } else {
            // arr[i] > arr[j]，说明 arr[i..mid] 都与 arr[j] 构成逆序对
            count += (mid - i + 1);
            temp[t++] = arr[j++];
        }
    }
    while (i <= mid) temp[t++] = arr[i++];
    while (j <= right) temp[t++] = arr[j++];
    t = 0;
    while (left <= right) arr[left++] = temp[t++];
}
```

### 3. 合并K个有序链表/数组
```java
/**
 * 合并K个有序数组：使用最小堆 + 归并思想
 * 时间复杂度: O(N log K)，N为总元素数，K为数组数
 */
public int[] mergeKSortedArrays(int[][] arrays) {
    // 最小堆，存储 {值, 数组索引, 元素索引}
    PriorityQueue<int[]> minHeap = new PriorityQueue<>((a, b) -> a[0] - b[0]);
    
    // 初始化：每个数组的第一个元素入堆
    for (int i = 0; i < arrays.length; i++) {
        if (arrays[i].length > 0) {
            minHeap.offer(new int[]{arrays[i][0], i, 0});
        }
    }
    
    List<Integer> result = new ArrayList<>();
    while (!minHeap.isEmpty()) {
        int[] curr = minHeap.poll();
        result.add(curr[0]);
        int arrIdx = curr[1], elemIdx = curr[2];
        if (elemIdx + 1 < arrays[arrIdx].length) {
            minHeap.offer(new int[]{
                arrays[arrIdx][elemIdx + 1], arrIdx, elemIdx + 1
            });
        }
    }
    
    return result.stream().mapToInt(Integer::intValue).toArray();
}
```

## ⚠️ 注意事项

1. **额外空间**：需要O(n)的辅助空间，不适合内存受限场景
2. **小规模数据**：对小规模数据效率不如插入排序（递归开销占比较大）
3. **递归深度**：递归实现可能导致栈溢出，大规模数据建议用迭代实现（自底向上）
4. **缓存效率**：数组实现的缓存局部性不如快速排序（合并时跳跃式访问）
5. **稳定性关键**：合并时必须使用 `<=` 而非 `<`，否则会失去稳定性
6. **辅助数组复用**：应在排序入口预分配一次辅助数组，避免每次merge都创建新数组

## 💡 最佳实践
- 对小规模子数组（长度<15）使用插入排序优化
- 采用自底向上的迭代实现避免栈溢出
- 合并操作中减少元素复制次数（交替使用原数组和辅助数组）
- 对于链表实现，可实现O(1)空间复杂度
- 与快速排序结合使用（如Timsort算法）
- 检查 `arr[mid] <= arr[mid+1]` 跳过已有序的合并

## 面试题速查

**Q1: 归并排序为什么是稳定的？**
> 归并排序在合并两个子数组时，当左右两边元素相等时（`arr[i] == arr[j]`），使用 `<=` 优先取左子数组的元素，保证了相等元素的相对顺序不变，因此是稳定排序。

**Q2: 归并排序和快速排序哪个更好？**
> 各有优势。归并排序：稳定、最坏O(n log n)、适合外部排序和链表排序，但需O(n)额外空间。快速排序：原地排序、缓存局部性好、实际更快，但最坏O(n²)、不稳定。工程实践中原始类型用快排（Arrays.sort），对象类型用Timsort（归并变体）。

**Q3: 如何用归并排序计算逆序对？**
> 在Merge阶段，当左子数组的元素arr[i] > 右子数组的arr[j]时，arr[i..mid]都与arr[j]构成逆序对，计数器加上 `mid - i + 1`。时间复杂度O(n log n)，比暴力法O(n²)快很多。

**Q4: 归并排序适合链表排序吗？为什么？**
> 非常适合。链表归并排序不需要随机访问，且合并操作只需修改指针，空间复杂度为O(1)（不需要辅助数组）。而快速排序依赖随机访问，不适合链表。

**Q5: 什么是外部排序？归并排序如何用于外部排序？**
> 外部排序用于排序无法全部装入内存的大文件。步骤：(1)将大文件分块读入内存，每块排序后写回临时文件；(2)多路归并合并所有临时文件。归并排序的顺序访问特性使其特别适合外部排序。

**Q6: Timsort是什么？和归并排序有什么关系？**
> Timsort是Tim Peters于2002年发明的混合排序算法，结合了归并排序和插入排序。它检测数据中已有的有序段（run），利用已有序性减少合并次数。最好情况O(n)，平均和最坏O(n log n)，稳定。Python和Java（对象排序）都使用Timsort。

**Q7: 归并排序的空间复杂度能优化到O(1)吗？**
> 数组实现难以做到O(1)（有原地归并算法但常数太大，实际不实用）。链表实现天然是O(1)空间（仅修改指针）。另外可以用自底向上迭代实现消除递归栈的O(log n)空间。

**Q8: 自顶向下和自底向上归并排序有什么区别？**
> 自顶向下用递归，从整个数组开始二分到单元素再合并；自底向上用迭代，从单元素开始逐步合并成更大的有序段。两者时间复杂度相同，但自底向上避免了递归栈开销（空间从O(n)+O(log n)降为O(n)），适合大规模数据。
