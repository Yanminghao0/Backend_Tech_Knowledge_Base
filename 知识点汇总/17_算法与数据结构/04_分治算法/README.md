# 分治算法 (Divide and Conquer)

> 分治算法是将复杂问题分解为更小子问题来解决的经典策略，包含分解、解决、合并三步骤

## 🧠 核心思想

```
分治三步骤：
┌─────────────────────────────────┐
│  1. 分解：将问题分成规模更小的子问题  │
│  2. 解决：递归求解子问题            │
│  3. 合并：将子问题的解合并为原解     │
└─────────────────────────────────┘

适用条件：
- 问题可分解为独立子问题（最优子结构）
- 子问题与原问题结构相同
- 子问题的解可合并
- 分解的收益大于分解成本
```

## 📈 学习路径

```
1. 二分查找 — 理解分治的基本思想和边界处理
      ↓
2. 归并排序 — 掌握分治的完整流程（分解→解决→合并）
      ↓
3. 快速排序 — 学习随机化算法和原地划分技术
      ↓
4. 大整数乘法 — 了解分治优化策略（Karatsuba算法）
      ↓
5. 最近点对 — 探索分治在几何问题中的应用
```

## 📊 与其他算法范式对比

| 维度 | 分治 | 动态规划 | 贪心 |
|------|------|----------|------|
| 子问题关系 | 独立 | 重叠 | 不回溯 |
| 求解方式 | 自顶向下递归 | 自底向上/记忆化 | 逐步选择 |
| 最优解保证 | 是 | 是 | 不一定 |
| 典型算法 | 归并排序、快排 | 背包问题、LCS | Dijkstra、Kruskal |
