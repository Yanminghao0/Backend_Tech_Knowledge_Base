# 快速排序

> 快速排序（Quick Sort）是一种高效的排序算法，采用分治策略实现，由Tony Hoare于1960年提出。

---

## 📋 基本信息
- **类别**：分治算法
- **难度**：中等
- **稳定性**：不稳定
- **时间复杂度**：平均O(n log n)，最坏O(n²)
- **空间复杂度**：O(log n)（递归栈）
- **关键词**：分治、基准元素、递归、原地排序

## 🎯 分治策略
快速排序的分治思想体现在三个步骤：
1. **分解**：选择一个基准元素(pivot)，将数组分为两部分，左侧元素小于等于基准，右侧元素大于基准
2. **解决**：递归地对左右两部分数组进行快速排序
3. **合并**：由于是原地排序，无需额外合并操作

### 分治过程图解

```
原始数组: [3, 6, 8, 10, 1, 2, 1]
选取基准: pivot = 1 (最后一个元素)

第1次分区:
  扫描: [3, 6, 8, 10, 1, 2, 1]
  小于区: [1, 1]  大于区: [3, 6, 8, 10, 2]
  结果: [1, 1, | 3, 6, 8, 10, 2]
                   ↑ 对右侧递归

第2次分区 (右侧 [3, 6, 8, 10, 2]):
  选取基准: pivot = 2
  小于区: [2]  大于区: [3, 6, 8, 10]
  结果: [1, 1, 2, | 3, 6, 8, 10]

第3次分区 ([3, 6, 8, 10]):
  选取基准: pivot = 10
  小于区: [3, 6, 8]  大于区: []
  结果: [1, 1, 2, 3, 6, 8, 10]
```

## 💻 代码实现
### Java实现（基本版 - Lomuto分区方案）
```java
public class QuickSort {
    public void quickSort(int[] arr, int low, int high) {
        if (low < high) {
            // 获取基准元素位置
            int pivotIndex = partition(arr, low, high);
            // 递归排序左子数组
            quickSort(arr, low, pivotIndex - 1);
            // 递归排序右子数组
            quickSort(arr, pivotIndex + 1, high);
        }
    }

    private int partition(int[] arr, int low, int high) {
        // 选择最右侧元素作为基准
        int pivot = arr[high];
        // i是小于基准区域的边界
        int i = low - 1;
        
        for (int j = low; j < high; j++) {
            // 当前元素小于等于基准，扩展小于区域
            if (arr[j] <= pivot) {
                i++;
                // 交换元素到小于区域
                int temp = arr[i];
                arr[i] = arr[j];
                arr[j] = temp;
            }
        }
        
        // 将基准元素放到最终位置
        int temp = arr[i + 1];
        arr[i + 1] = arr[high];
        arr[high] = temp;
        
        return i + 1;
    }
}
```

### Hoare分区方案（原始版本，性能更优）
```java
private int partitionHoare(int[] arr, int low, int high) {
    int pivot = arr[low + (high - low) / 2];
    int i = low - 1;
    int j = high + 1;
    
    while (true) {
        // 从左向右找第一个大于等于pivot的元素
        do { i++; } while (arr[i] < pivot);
        // 从右向左找第一个小于等于pivot的元素
        do { j--; } while (arr[j] > pivot);
        
        if (i >= j) return j;
        
        // 交换
        int temp = arr[i];
        arr[i] = arr[j];
        arr[j] = temp;
    }
}
```

### 优化策略：三数取中法（解决有序数组退化问题）
```java
private int medianOfThree(int[] arr, int low, int high) {
    int mid = low + (high - low) / 2;
    // 排序三个位置的元素
    if (arr[low] > arr[mid]) swap(arr, low, mid);
    if (arr[low] > arr[high]) swap(arr, low, high);
    if (arr[mid] > arr[high]) swap(arr, mid, high);
    // 将中间值放到high-1位置
    swap(arr, mid, high - 1);
    return arr[high - 1];
}

private void swap(int[] arr, int i, int j) {
    int temp = arr[i];
    arr[i] = arr[j];
    arr[j] = temp;
}
```

### 三向切分快速排序（处理大量重复元素）
```java
// Dutch National Flag 问题解法
public void quickSort3Way(int[] arr, int low, int high) {
    if (low < high) {
        int[] bounds = partition3Way(arr, low, high);
        quickSort3Way(arr, low, bounds[0] - 1);
        quickSort3Way(arr, bounds[1] + 1, high);
    }
}

// 返回等于pivot区域的左右边界 [lt, gt]
private int[] partition3Way(int[] arr, int low, int high) {
    int pivot = arr[low];
    int lt = low;      // arr[low..lt-1] < pivot
    int gt = high;     // arr[gt+1..high] > pivot
    int i = low + 1;   // arr[lt..i-1] == pivot
    
    while (i <= gt) {
        if (arr[i] < pivot) {
            swap(arr, lt++, i++);
        } else if (arr[i] > pivot) {
            swap(arr, i, gt--);
        } else {
            i++;
        }
    }
    return new int[]{lt, gt};
}
```

### Python实现
```python
def quick_sort(arr, low, high):
    if low < high:
        pivot_index = partition(arr, low, high)
        quick_sort(arr, low, pivot_index - 1)
        quick_sort(arr, pivot_index + 1, high)

def partition(arr, low, high):
    pivot = arr[high]
    i = low - 1
    for j in range(low, high):
        if arr[j] <= pivot:
            i += 1
            arr[i], arr[j] = arr[j], arr[i]
    arr[i + 1], arr[high] = arr[high], arr[i + 1]
    return i + 1

# Pythonic一行实现（非原地排序，仅用于理解）
def quick_sort_functional(arr):
    if len(arr) <= 1:
        return arr
    pivot = arr[len(arr) // 2]
    left = [x for x in arr if x < pivot]
    middle = [x for x in arr if x == pivot]
    right = [x for x in arr if x > pivot]
    return quick_sort_functional(left) + middle + quick_sort_functional(right)
```

### 非递归实现（使用栈模拟递归）
```java
public void quickSortIterative(int[] arr, int low, int high) {
    // 使用栈模拟递归调用
    int[] stack = new int[high - low + 1];
    int top = -1;
    
    stack[++top] = low;
    stack[++top] = high;
    
    while (top >= 0) {
        high = stack[top--];
        low = stack[top--];
        
        int pivot = partition(arr, low, high);
        
        // 先处理较短的子数组（尾递归优化）
        if (pivot - low < high - pivot) {
            if (pivot - 1 > low) {
                stack[++top] = low;
                stack[++top] = pivot - 1;
            }
            if (pivot + 1 < high) {
                stack[++top] = pivot + 1;
                stack[++top] = high;
            }
        } else {
            if (pivot + 1 < high) {
                stack[++top] = pivot + 1;
                stack[++top] = high;
            }
            if (pivot - 1 > low) {
                stack[++top] = low;
                stack[++top] = pivot - 1;
            }
        }
    }
}
```

## 📊 复杂度分析

### 时间复杂度推导

**最佳/平均情况**：每次划分将数组均匀分成两部分

```
T(n) = 2T(n/2) + O(n)
```

由主定理得 T(n) = O(n log n)

**最坏情况**：每次划分极度不平衡（如已排序数组）

```
T(n) = T(n-1) + O(n)
     = T(n-2) + O(n-1) + O(n)
     = ...
     = O(1) + O(2) + ... + O(n)
     = O(n²)
```

**随机化快速排序的期望复杂度**：

使用随机选择基准时，期望比较次数为：
```
E[C(n)] = 2n ln n ≈ 1.39 n log₂n
```

这意味着随机化快速排序的期望时间复杂度为 O(n log n)，且常数因子比归并排序更小。

### 各情况对比
| 情况 | 时间复杂度 | 空间复杂度 | 触发条件 |
|------|-----------|-----------|---------|
| 最佳 | O(n log n) | O(log n) | 每次划分均衡 |
| 平均 | O(n log n) | O(log n) | 随机输入 |
| 最坏 | O(n²) | O(n) | 已排序/逆序/所有元素相等 |

### 空间复杂度分析
- **递归栈深度**：取决于划分的平衡程度
  - 平衡划分：O(log n)
  - 极端不平衡：O(n)
- **尾递归优化**：先递归处理较短的子数组，可将最坏空间降为 O(log n)

## 🔍 应用场景
- **大规模数据排序**：平均性能优异，是实践中最快的通用排序算法之一
- **内存有限的系统**：原地排序特性，不需要额外空间
- **商业级排序库**：Java Arrays.sort()对原始类型使用双轴快速排序（Dual-Pivot Quicksort）
- **数据库索引构建**：排序环节中的核心算法
- **第K大/小元素查找**：快速选择算法（QuickSelect），平均O(n)
- **Median of Medians**：最坏O(n)选择算法的核心组件

### 快速选择算法（QuickSelect）实现
```java
// 查找第K小的元素，平均时间复杂度O(n)
public int quickSelect(int[] arr, int low, int high, int k) {
    if (low == high) return arr[low];
    
    int pivotIndex = partition(arr, low, high);
    
    if (k == pivotIndex) {
        return arr[k];
    } else if (k < pivotIndex) {
        return quickSelect(arr, low, pivotIndex - 1, k);
    } else {
        return quickSelect(arr, pivotIndex + 1, high, k);
    }
}
```

## ⚠️ 边界条件分析
1. **空数组/单元素**：`low >= high` 时直接返回
2. **两个元素**：需要一次分区和可能的交换
3. **所有元素相同**：基本版退化为O(n²)，三向切分可优化为O(n)
4. **已排序数组**：基本版退化为O(n²)，使用三数取中或随机化可避免
5. **逆序数组**：同样退化，解决方案同上
6. **大量重复元素**：使用三向切分优化

## 📝 面试常见问题

**Q1: 快速排序为什么通常比归并排序快？**
> 1. 原地排序，不需要额外数组的分配和复制开销
> 2. 缓存局部性更好，连续访问内存
> 3. 常数因子更小，实际比较和交换次数更少
> 4. 平均情况下比较次数约为 1.39n log n，归并排序为 n log n，但归并有额外复制开销

**Q2: 如何避免快速排序的最坏情况？**
> - 随机选择基准元素
> - 使用三数取中法
> - 结合插入排序处理小规模子数组
> - 使用三向切分处理重复元素
> - 使用IntroSort（内省排序）：递归深度超过阈值时切换为堆排序

**Q3: 快速排序是不稳定的，能否使其稳定？**
> 可以，但需要额外的O(n)空间来记录元素原始位置，或使用归并排序的合并方式。实际中需要稳定排序时通常直接使用归并排序。

**Q4: Lomuto分区和Hoare分区的区别？**
| 特性 | Lomuto分区 | Hoare分区 |
|------|-----------|-----------|
| 基准位置 | 最后一个元素 | 中间元素 |
| 交换次数 | 较多 | 较少（平均少3倍） |
| 实现复杂度 | 简单 | 稍复杂 |
| 返回值 | 基准的最终位置 | 分割点（不一定是基准位置） |

## 💡 最佳实践
- 结合插入排序处理小规模子数组（阈值通常为10-20）
- 使用三数取中法或随机基准选择
- 实现尾递归优化减少栈空间使用
- 对重复元素采用三向切分策略
- 对于几乎有序的数据，考虑使用归并排序或Timsort
- 大规模数据使用Dual-Pivot Quicksort（双轴快速排序）
