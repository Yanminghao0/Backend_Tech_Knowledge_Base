### 1. 分布式任务调度实战 ⭐⭐⭐ 核心
📄 [分布式任务调度实战.md](./01_分布式任务调度实战.md)

**核心内容**：

#### 调度框架对比
- ✅ **XXL-Job**：轻量级、可视化调度中心、执行器自动注册、GLUE模式
- ✅ **Elastic-Job**：分布式弹性调度、分片策略、故障转移
- ✅ **Azkaban**：工作流调度、DAG依赖管理
- ✅ **Spring Task**：轻量级单机调度、@Scheduled注解

**调度框架选型对比**：

| 框架 | 特点 | 适用场景 |
|------|------|---------|
| XXL-Job | 轻量级、可视化管理、社区活跃 | 中小规模、快速上手 |
| Elastic-Job | 弹性扩缩容、分片完善 | 大规模分布式调度 |
| Azkaban | 工作流DAG、Hadoop生态 | 大数据ETL任务 |
| Spring Task | 零依赖、简单 | 单机简单定时任务 |

#### 核心特性
- ✅ **任务分片**：大数据量任务拆分到多节点并行执行
- ✅ **故障转移**：执行器宕机后任务自动迁移
- ✅ **动态扩容**：新增执行器自动加入调度
- ✅ **执行日志**：任务执行记录、失败重试、超时告警

#### 架构设计
- ✅ **调度中心与执行器分离**：调度中心负责任务触发，执行器负责业务处理
- ✅ **任务元数据存储**：MySQL存储任务配置、执行记录
- ✅ **高可用设计**：调度中心集群部署、数据库主从、执行器多实例

#### 实践场景
- ✅ **数据同步**：定时同步异构数据源
- ✅ **报表生成**：日终统计报表、月度汇总
- ✅ **定时清理**：过期数据清理、临时文件删除
- ✅ **业务监控**：定时对账、异常数据检测
- ✅ **缓存预热**：定时刷新热点数据到缓存

**XXL-Job核心配置示例**：
```java
@XxlJob("orderSyncHandler")
public void orderSyncHandler() {
    // 获取分片参数
    int shardIndex = XxlJobHelper.getShardIndex();
    int shardTotal = XxlJobHelper.getShardTotal();
    
    // 分片查询处理
    List<Order> orders = orderMapper.selectByShard(shardIndex, shardTotal);
    for (Order order : orders) {
        syncService.sync(order);
    }
    
    XxlJobHelper.handleSuccess("同步完成，处理" + orders.size() + "条");
}
```

**适用场景**：定时数据同步、报表生成、批量处理、系统监控
