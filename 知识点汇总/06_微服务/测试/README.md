## 📊 测试金字塔

```
        /\
       /  \         端到端测试 (E2E) — 少量，慢，高成本
      /----\
     /      \       集成测试 — 中等数量，验证服务间交互
    /--------\
   /          \     契约测试 — 验证服务间接口契约
  /------------\
 /              \   单元测试 — 大量，快，低成本
/________________\
```

**比例建议**：单元测试 70% | 契约+集成测试 20% | 端到端测试 10%

## 🔧 核心工具栈

| 类别 | 工具 | 用途 |
|------|------|------|
| 单元测试 | JUnit 5 + Mockito | Service层逻辑测试 |
| 集成测试 | Spring Boot Test + Testcontainers | 仓储层、组件交互测试 |
| 契约测试 | Spring Cloud Contract / Pact | 消费者-提供者接口契约 |
| API测试 | REST Assured / MockMvc | Controller层接口测试 |
| 端到端测试 | Selenium / Cypress | 全链路用户场景测试 |
| 性能测试 | JMeter / Gatling | 接口压测、负载测试 |
| 测试数据 | Faker / DBUnit | 测试数据生成与管理 |

## 📝 关键技术点

- **测试策略**：测试金字塔在微服务中的应用、测试环境管理、测试左移
- **契约测试**：Spring Cloud Contract DSL、Pact Broker、消费者驱动契约
- **API测试**：RESTful API自动化测试、接口性能测试、Mock Server
- **端到端测试**：Selenium/Cypress在微服务场景的应用、全链路追踪辅助调试
- **测试数据**：动态测试数据生成、环境隔离方案、数据清理与回滚
- **CI集成**：测试门禁、覆盖率报告、自动化测试流水线
