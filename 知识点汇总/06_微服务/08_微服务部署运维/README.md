## 🎯 学习目标

1. **掌握部署策略**：蓝绿部署、金丝雀发布、灰度发布、影子部署的原理与适用场景
2. **熟悉容器编排**：Docker容器化、Kubernetes部署配置、Helm包管理
3. **环境管理**：多环境隔离（dev/staging/prod）、配置管理、基础设施即代码(IaC)
4. **运维自动化**：CI/CD流水线设计、自动扩缩容(HPA)、故障自愈
5. **监控告警**：部署指标监控、日志聚合分析、异常检测与告警

## 📊 部署策略对比

| 策略 | 停机时间 | 回滚速度 | 资源开销 | 适用场景 |
|------|----------|----------|----------|----------|
| 滚动更新 | 无 | 中 | 低 | 日常迭代发布 |
| 蓝绿部署 | 无 | 快(切换) | 高(2倍) | 重大版本发布 |
| 金丝雀发布 | 无 | 快(引流) | 中 | 灰度验证 |
| 影子部署 | 无 | 无需回滚 | 高 | 压测验证 |
| 重建部署 | 有 | 慢 | 低 | 非生产环境 |

## 🔧 核心工具栈

| 类别 | 工具 | 用途 |
|------|------|------|
| 容器化 | Docker / Containerd | 应用打包与隔离 |
| 编排调度 | Kubernetes / Nomad | 容器编排与调度 |
| CI/CD | GitLab CI / Jenkins / ArgoCD | 持续集成与持续部署 |
| 配置管理 | Nacos / Apollo / ConfigMap | 配置中心与环境管理 |
| 监控 | Prometheus + Grafana | 指标采集与可视化 |
| 日志 | ELK / Loki | 日志聚合与分析 |
| 服务网格 | Istio / Linkerd | 流量管理、金丝雀发布 |

---

## 📝 关键技术点

- **部署策略**：蓝绿部署、金丝雀发布、灰度发布、影子部署的原理与实施
- **容器化**：Docker镜像构建优化、多阶段构建、K8s Deployment/StatefulSet配置
- **环境管理**：多环境隔离、Namespace隔离、配置管理、基础设施即代码(Terraform)
- **运维自动化**：CI/CD流水线、GitOps(ArgoCD/Flux)、自动扩缩容(HPA/VPA)、故障自愈
- **监控告警**：部署指标监控、链路追踪、日志聚合、告警规则配置
- **安全合规**：镜像扫描(Trivy)、RBAC权限控制、NetworkPolicy网络隔离
