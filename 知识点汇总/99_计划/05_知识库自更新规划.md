# 知识库自更新规划

> 让Hermes Agent定期自主扫描、学习、补充知识库的闭环方案

---

## 一、设计思路

```
自更新闭环：

  ┌─────────────────────────────────────────┐
  │            Cron Job 定时触发              │
  │         （每周一次 / 每月一次）            │
  └──────────────────┬──────────────────────┘
                     ▼
  ┌─────────────────────────────────────────┐
  │         1. 扫描（scan_health.py）         │
  │  扫描知识库 → 发现偏薄文件/空目录/编号问题  │
  └──────────────────┬──────────────────────┘
                     ▼
  ┌─────────────────────────────────────────┐
  │         2. 诊断（Agent分析）              │
  │  分析扫描结果 → 确定补充优先级和内容方向    │
  └──────────────────┬──────────────────────┘
                     ▼
  ┌─────────────────────────────────────────┐
  │         3. 学习（Web搜索+技术追踪）       │
  │  搜索最新技术动态 → 学习新知识 → 整理成文  │
  └──────────────────┬──────────────────────┘
                     ▼
  ┌─────────────────────────────────────────┐
  │         4. 补充（写入文档）               │
  │  扩充偏薄文档 / 创建缺失文档 / 更新综述    │
  └──────────────────┬──────────────────────┘
                     ▼
  ┌─────────────────────────────────────────┐
  │         5. 提交（git commit）             │
  │  提交变更 → 更新审计报告 → 记录变更日志    │
  └─────────────────────────────────────────┘
```

---

## 二、自更新场景

### 场景1：定期健康扫描（每周）

```
触发：每周一 09:00
频率：每周一次
内容：
  1. 运行 scan_health.py 扫描知识库
  2. 发现偏薄文件/空目录/编号问题
  3. 如有问题，生成待办清单
  4. 无问题则静默（不通知用户）

输出：scan_result.json + 变更日志
```

### 场景2：技术动态追踪（每月）

```
触发：每月1日 10:00
频率：每月一次
内容：
  1. 搜索上月技术动态（Java/Spring/云原生/AI）
  2. 生成月度技术综述文档
  3. 根据新技术动态更新相关技术文档
  4. 检查是否有新的版本发布需要更新

输出：月度技术综述 + 相关文档更新
```

### 场景3：内容质量提升（每月）

```
触发：每月15日 10:00
频率：每月一次
内容：
  1. 扫描偏薄文件（<5KB）
  2. 针对每个偏薄文件，搜索最新资料扩充
  3. 检查文档中的技术版本是否过期
  4. 补充新的面试题和实战案例

输出：扩充后的文档 + 质量提升报告
```

### 场景4：知识体系扩展（每季度）

```
触发：每季度第一个月1日 10:00
频率：每季度一次
内容：
  1. 评估知识库覆盖度（是否有新技术领域缺失）
  2. 根据技术趋势新增目录或文档
  3. 更新学习路径和README导航
  4. 重新审计全局健康度

输出：新目录/新文档 + 更新后的审计报告
```

## 三、Cron Job 配置

### Job 1：周度健康扫描

```
名称：知识库健康扫描
schedule：0 9 * * 1  （每周一09:00）
脚本：python3 知识点汇总/计划/scan_health.py
模式：no_agent=True（纯脚本，静默运行）
通知：仅当发现问题（exit_code=1）时通知
```

### Job 2：月度技术综述

```
名称：月度技术综述生成
schedule：0 10 1 * *  （每月1日10:00）
模式：LLM驱动
内容：
  - 搜索上月Java/Spring/云原生/AI/数据库等领域的技术动态
  - 生成 01_月度技术综述/YYYY年M月技术综述.md
  - 更新 23_技术动态/01_月度技术综述/README.md
  - git commit
```

### Job 3：月度内容质量提升

```
名称：知识库内容质量提升
schedule：0 10 15 * *  （每月15日10:00）
模式：LLM驱动
内容：
  - 运行 scan_health.py 获取偏薄文件清单
  - 对每个偏薄文件搜索最新资料并扩充
  - 检查技术版本是否过期
  - git commit
```

## 六、质量保障

### 防止退化
- 每次扩充前必须read_file读取现有内容，保留原有结构
- 扩充而非覆盖，原有内容不丢失
- 每次变更必须git commit，保留完整历史

### 防止冗余
- scan_health.py检测偏薄文件，避免重复扩充已达标文件
- 文档风格统一检查（#标题 />引言 /---/📋目录 /##编号 /代码块 /面试题）

### 防止错误
- 技术版本信息需通过web搜索验证
- 代码示例需确保语法正确
- 面试题需确保答案准确
