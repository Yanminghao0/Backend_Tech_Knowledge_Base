# 知识库强制规范

> 所有对知识库的操作必须严格遵守以下规范，无例外

---

## 一、目录规范

### 1.1 一级目录
- 格式：`编号_中文名称`，如 `01_Java核心`、`22_源码解读`
- 编号从00开始，连续递增，不得跳号
- 00_综合 放跨领域内容，99_计划 放管理文档
- 新增目录时，取当前最大编号+1

### 1.2 子目录
- 格式：`编号_中文名称`，如 `01_核心组件`、`02_通信协议`
- 编号从01开始，连续递增
- 子目录名应与父目录有语义关联（如06_微服务下用 `01_微服务核心组件`）
- 归档目录命名为 `归档`，不加编号

### 1.3 禁止事项
- 禁止无编号目录
- 禁止编号跳号
- 禁止中英文混编命名
- 禁止用空格、特殊字符（仅允许字母、数字、下划线、中文）

---

## 二、文件命名规范

### 2.1 技术文档
- 格式：`编号_中文名称.md`，如 `01_JVM虚拟机详解.md`、`03_Spring MVC详解.md`
- 编号从01开始，连续递增
- README.md 不加编号

### 2.2 源码解读文件
- 格式：`编号.序号_中文名称.md`，如 `4.1_IoC容器源码解析.md`
- 编号与父目录编号一致，序号从1递增

### 2.3 月度综述
- 格式：`YYYY年M月技术综述.md`，如 `2026年7月技术综述.md`

### 2.4 禁止事项
- 禁止无编号文件（README.md、变更日志.md 除外）
- 禁止文件名含空格（用下划线替代）
- 禁止编号从0开始（24_Java AI框架已修正）

---

## 三、文档内容规范

### 3.1 文档结构（强制）
```markdown
# 标题

> 一句话描述

---

## 📋 目录

1. [章节1](#章节1)
2. [章节2](#章节2)
...

---

## 1. 章节标题

### 1.1 小节标题

内容...

---

## N. 面试题速查

**Q1: 问题？**

```
答案...
```

---

*最后更新：YYYY-MM-DD*
```

### 3.2 内容要求
- 每篇技术文档 ≥ 8KB（月度综述 ≥ 3KB）
- 必须包含 📋 目录
- 必须包含面试题速查（≥3道Q&A）
- 代码块必须标注语言（```java / ```bash / ```yaml / ```sql / ```mermaid）
- 全中文（技术术语可保留英文）
- 末尾标注最后更新日期

### 3.3 README要求
- 每个目录必须有README.md
- README必须包含：文档列表表格、学习路径、关联主题
- 子目录README ≥ 2KB，一级目录README ≥ 3KB

---

## 四、内容去重规范

### 4.1 定位差异化
- 专题深度文档：原理+源码+完整代码示例
- 面试速查文档：Q&A格式，概要+链接指向专题
- 全局索引文档（00_综合）：一行概述+链接，不展开

### 4.2 交叉引用
- 跨目录内容重叠时，在文档末尾加 `## 🔗 相关文档` 链接
- 面试题引用专题时写：`详见：[文档名](路径)`
- 禁止面试题和专题文档同时展开相同深度内容

---

## 五、变更规范

### 5.1 每次修改后必须
1. 运行 `python3 99_计划/scan_health.py` 检查健康度
2. `git add -A && git commit -m "描述"` 提交
3. 在 `99_计划/变更日志.md` 追加变更记录

### 5.2 commit消息格式
```
<type>: <描述>

- 具体变更1
- 具体变更2
```
type: feat(新增) / fix(修复) / docs(文档) / refactor(重构)

### 5.3 变更日志格式
```markdown
## YYYY-MM-DD 变更类型

- 操作1: 文件A (大小变化)
- 操作2: 文件B (新建/删除/修改)
- 提交: commit hash
```

---

## 六、自更新规范

### 6.1 定时任务
- 每周一09:00：健康扫描（scan_health.py，纯脚本）
- 每月1日10:00：月度技术综述生成（LLM驱动）
- 每月15日10:00：内容质量提升（扩充偏薄文档+版本检查）

### 6.2 扩充规则
- 先 read_file 读取现有内容，保留原有结构
- 在现有基础上补充深化，不覆盖原有内容
- 扩充后用 write_file 写入完整内容
- 扩充后必须 ≥ 8KB

| 禁止 | 说明 |

| 禁止无面试题速查 | 每篇≥3道Q&A |
| 禁止无代码块语言标注 | 必须```java等 |

| 禁止修改不提交 | 每次修改必须git commit |
