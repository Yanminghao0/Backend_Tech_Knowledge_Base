## 1. 处理流程概述

### 1.1 整体处理流程

```
┌─────────────────────────────────────────────────────────────┐
│                  Servlet处理流程                             │
├─────────────────────────────────────────────────────────────┤
│                                                             │
│  HTTP请求                                                   │
│     │                                                       │
│     ▼                                                       │
│  Connector (解析HTTP协议)                                   │
│     │                                                       │
│     ▼                                                       │
│  CoyoteAdapter (适配器: Coyote→Catalina)                   │
│     │                                                       │
│     ▼                                                       │
│  Engine.pipeline → Host.pipeline → Context.pipeline        │
│     │                                                       │
│     ▼                                                       │
│  Wrapper.pipeline (StandardWrapperValve)                    │
│     │                                                       │
│     ▼                                                       │
│  ApplicationFilterChain (Filter链)                          │
│     │                                                       │
│     ├── Filter1.doFilter()                                  │
│     ├── Filter2.doFilter()                                  │
│     │                                                       │
│     ▼                                                       │
│  Servlet.service()                                          │
│     │                                                       │
│     ├── doGet() / doPost() / ...                           │
│     │                                                       │
│     ▼                                                       │
│  响应处理 → HTTP响应                                         │
│                                                             │
└─────────────────────────────────────────────────────────────┘
```

### 1.2 关键组件

```java
    // 封装HTTP请求和响应（门面模式）
    Request request;
    Response response;
}
```

### 1.3 源码入口：CoyoteAdapter.service()

```java
// CoyoteAdapter是Connector和Container的桥梁
public void service(org.apache.coyote.Request req, 
                    org.apache.coyote.Response res) {
    
    // 1. 创建Catalina Request/Response对象
    Request request = new Request();
    Response response = new Response();
    request.setResponse(response);
    
    // 2. 解析请求
    request.setCoyoteRequest(req);
    response.setCoyoteResponse(res);
    
    // 3. 连接器 -> 容器
    // 从Engine开始，通过Pipeline-Valve机制逐层处理
    connector.getService().getContainer().getPipeline()
        .getFirst().invoke(request, response);
    
    // 4. 写响应
    response.finishResponse();
}
```

## 2. 请求映射机制

### 2.1 Mapper路由匹配

```java
// Mapper根据URL找到对应的Host → Context → Wrapper
// 映射规则：
// Host: www.example.com → Host对象
// Context: /myapp → Context对象（web应用）
// Wrapper: /users/* → Wrapper对象（Servlet）

// 源码核心：Mapper.internalMap()
protected void internalMap(CharChunk host, CharChunk uri,
        MappingData mappingData) {
    
    // 1. 匹配Host
    MappedHost mappedHost = exactFind(hosts, host);
    
    // 2. 匹配Context（web应用路径）
    MappedContext mappedContext = findContext(mappedHost, uri);
    
    // 3. 匹配Wrapper（Servlet路径）
    // 精确匹配 > 通配符匹配 > 默认Servlet
    MappedWrapper mappedWrapper = findWrapper(mappedContext, uri);
    
    // 4. 填充映射结果
    mappingData.host = mappedHost.object;
    mappingData.context = mappedContext.object;
    mappingData.wrapper = mappedWrapper.object;
}
```

### 2.2 匹配优先级

```
URL匹配优先级（从高到低）：

1. 精确匹配    /users/list        → 直接命中
2. 最长路径    /users/*           → 按最长路径前缀匹配
3. 扩展名匹配  *.do               → 按扩展名匹配
4. 默认Servlet  /                  → DefaultServlet处理静态资源

示例：
  web.xml配置：
  /users/list     → UserListServlet
  /users/*        → UserServlet  
  *.do            → DoServlet
  
  请求 /users/list → UserListServlet（精确匹配优先）
  请求 /users/123  → UserServlet（路径匹配）
  请求 /api.do     → DoServlet（扩展名匹配）
```

## 3. Filter链处理

### 3.1 ApplicationFilterChain核心源码

```java
// ApplicationFilterChain — 责任链模式
public final class ApplicationFilterChain implements FilterChain {

    // Filter数组
    private ApplicationFilterConfig[] filters;
    private int pos = 0;  // 当前执行位置
    private Servlet servlet;

    // 核心方法：doFilter
    @Override
    public void doFilter(ServletRequest request, ServletResponse response) {
        if (pos < filters.length) {
            // 还有Filter未执行
            ApplicationFilterConfig filterConfig = filters[pos++];
            Filter filter = filterConfig.getFilter();
            
            // 调用Filter.doFilter，Filter内部调用chain.doFilter继续
            filter.doFilter(request, response, this);
        } else {
            // 所有Filter执行完毕，调用Servlet
            servlet.service(request, response);
        }
    }
}
```

### 3.2 Filter链执行流程

```
请求 → FilterChain.doFilter()
         │
         ▼
       Filter1.doFilter(request, response, chain)
         │ 前置处理
         │ chain.doFilter() ──→ Filter2.doFilter(request, response, chain)
         │                        │ 前置处理
         │                        │ chain.doFilter() ──→ Servlet.service()
         │                        │                        │ doGet/doPost
         │                        │ 后置处理 ←─────────────┘
         │ 后置处理 ←─────────────┘
         ▼
       响应返回

特点：
- 责任链模式，递归调用
- 前置处理顺序：Filter1 → Filter2 → Servlet
- 后置处理顺序：Servlet → Filter2 → Filter1（逆序）
- Filter通过chain.doFilter()决定是否继续
```

### 3.3 Filter创建与管理

### 4.2 Servlet.service()分发

```java
// HttpServlet.service() — 根据HTTP方法分发
protected void service(HttpServletRequest req, HttpServletResponse resp) {
    String method = req.getMethod();
    
    if (method.equals(METHOD_GET)) {
        // 检查Last-Modified（缓存控制）
        long lastModified = getLastModified(req);
        if (lastModified == -1) {
            doGet(req, resp);
        } else {
            // 304缓存处理
            long ifModifiedSince = req.getDateHeader(HEADER_IFMODSINCE);
            if (ifModifiedSince < lastModified) {
                doGet(req, resp);
                resp.setDateHeader(HEADER_LASTMOD, lastModified);
            } else {
                resp.setStatus(HttpServletResponse.SC_NOT_MODIFIED);
            }
        }
    } else if (method.equals(METHOD_POST)) {
        doPost(req, resp);
    } else if (method.equals(METHOD_PUT)) {
        doPut(req, resp);
    } else if (method.equals(METHOD_DELETE)) {
        doDelete(req, resp);
    } else {
        // 不支持的方法
        resp.sendError(HttpServletResponse.SC_METHOD_NOT_ALLOWED);
    }
}
```

## 5. 异步处理机制

### 5.1 异步Servlet（Servlet 3.0+）

```java
// 异步处理：释放Tomcat线程，避免线程耗尽
@WebServlet("/async")
public class AsyncServlet extends HttpServlet {
    
    @Override
    protected void doGet(HttpServletRequest req, HttpServletResponse resp) {
        
        // 1. 开启异步
        AsyncContext asyncContext = req.startAsync();
        
        // 2. 提交到线程池处理（释放Tomcat工作线程）
        executor.submit(() -> {
            try {
                // 耗时操作
                Thread.sleep(5000);
                
                // 3. 写响应
                asyncContext.getResponse()
                    .getWriter().write("Async done");
                
                // 4. 完成异步
                asyncContext.complete();
            } catch (Exception e) {
                asyncContext.complete();
            }
        });
        
        // Tomcat线程立即返回，不等待
    }
}
```

```
同步 vs 异步：

同步处理：
  Tomcat线程 ←→ 业务执行（线程被占用5秒）
  200线程池 → 200并发请求 → 满 → 后续请求排队

异步处理：
  Tomcat线程 → startAsync → 立即释放
  业务线程池 ←→ 业务执行（5秒后回调）
  200线程池 → 瞬间处理完 → 可接受更多请求
```

### 6.1 异常处理流程

```java
// StandardWrapperValve异常处理
try {
    filterChain.doFilter(request, response);
} catch (Throwable t) {
    // 异常交给Pipeline处理
    throwable = t;
    exceptionType = t.getClass().getName();
}

// 如果有异常且响应未提交
if (throwable != null && !response.isCommitted()) {
    // 查找web.xml中配置的error-page
    // <error-page>
    //   <exception-type>java.lang.NullPointerException</exception-type>
    //   <location>/error</location>
    // </error-page>
    
    if (customErrorPage != null) {
        // 转发到错误页面
        request.getRequestDispatcher("/error")
            .forward(request, response);
    } else {
        // 默认错误页
        response.sendError(500, t.getMessage());
    }
}
```

## 7. 面试要点

### Q1: Tomcat处理HTTP请求的完整流程？

```
1. Connector接收HTTP请求，解析协议
2. CoyoteAdapter将Coyote请求转换为Catalina请求
3. Mapper根据URL匹配Host → Context → Wrapper
4. Pipeline-Valve机制：Engine → Host → Context → Wrapper
5. StandardWrapperValve创建Filter链
6. ApplicationFilterChain执行Filter.doFilter()
7. 所有Filter执行完，调用Servlet.service()
8. HttpServlet根据HTTP方法分发到doGet/doPost
9. 业务处理完成，写响应
```

### Q2: Filter链的执行顺序？

```
责任链模式，递归调用：
- 前置处理：Filter1 → Filter2 → Filter3 → Servlet
- 后置处理：Servlet → Filter3 → Filter2 → Filter1

Filter通过chain.doFilter()控制是否继续执行
如果不调用chain.doFilter()，请求被拦截
```

### Q3: Servlet是单例还是多例？

```
默认单例（多线程模型）：
- 一个Servlet类只创建一个实例
- 多个请求线程并发访问同一实例
- Servlet成员变量线程不安全（不要用实例变量存请求状态）

SingleThreadModel（已废弃）：
- 每个请求创建新实例
- 性能差，Servlet 2.4后废弃
```

### Q4: 异步Servlet解决什么问题？

```
问题：Tomcat线程池有限（默认200），耗时操作（RPC/DB）占用线程
导致线程耗尽，无法处理新请求

解决：startAsync()释放Tomcat线程
- Tomcat线程立即返回
- 业务在独立线程池执行
- 完成后asyncContext.complete()写响应

适用场景：长耗时IO操作（文件上传/大文件下载/推送）
不适用：CPU密集型操作
```
