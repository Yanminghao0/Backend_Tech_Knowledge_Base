```java
 * LoadBalance - 负载均衡接口

@SPI(RandomLoadBalance.NAME)
public interface LoadBalance {
    
    /**
     * 🔥 从多个Invoker中选择一个
     * @param invokers 服务提供者列表
     * @param url 服务URL
     * @param invocation 调用信息
     * @return 选中的Invoker
```

### 2.2 AbstractLoadBalance

```java
/**
 * AbstractLoadBalance - 负载均衡抽象基类
 */
public abstract class AbstractLoadBalance implements LoadBalance {
    
    /**
     * 计算预热权重
     */
    static int calculateWarmupWeight(int uptime, int warmup, int weight) {
        // 🔥 预热期间权重逐渐增加
        int ww = (int) (uptime / ((float) warmup / weight));
        return ww < 1 ? 1 : (Math.min(ww, weight));
    }
    
    @Override
    public <T> Invoker<T> select(List<Invoker<T>> invokers, URL url, Invocation invocation) {
        if (CollectionUtils.isEmpty(invokers)) {
            return null;
        }
        // 只有一个Invoker，直接返回
        if (invokers.size() == 1) {
            return invokers.get(0);
        }
        // 🔥 调用子类实现
        return doSelect(invokers, url, invocation);
    }
    
    /**
     * 子类实现的选择逻辑
     */
    protected abstract <T> Invoker<T> doSelect(List<Invoker<T>> invokers, 
        URL url, Invocation invocation);
    
    /**
     * 🔥 获取Invoker权重
     */
    int getWeight(Invoker<?> invoker, Invocation invocation) {
        int weight;
        URL url = invoker.getUrl();
        
        // 注册中心不参与负载均衡
        if (UrlUtils.isRegistryService(url)) {
            weight = url.getParameter(REGISTRY_KEY + "." + WEIGHT_KEY, DEFAULT_WEIGHT);
        } else {
            weight = url.getMethodParameter(invocation.getMethodName(), WEIGHT_KEY, DEFAULT_WEIGHT);
            if (weight > 0) {
                // 🔥 获取服务启动时间
                long timestamp = invoker.getUrl().getParameter(TIMESTAMP_KEY, 0L);
                if (timestamp > 0L) {
                    long uptime = System.currentTimeMillis() - timestamp;
                    if (uptime < 0) {
                        return 1;
                    }
                    // 获取预热时间
                    int warmup = invoker.getUrl().getParameter(WARMUP_KEY, DEFAULT_WARMUP);
                    // 🔥 预热期间降低权重
                    if (uptime > 0 && uptime < warmup) {
                        weight = calculateWarmupWeight((int) uptime, warmup, weight);
                    }
                }
            }
        }
        return Math.max(weight, 0);
    }
}
```

## 3. 随机负载均衡

### 3.1 RandomLoadBalance

```java
/**
 * RandomLoadBalance - 随机负载均衡（默认）
 * 按权重随机选择
 */
public class RandomLoadBalance extends AbstractLoadBalance {
    
    public static final String NAME = "random";
    
    @Override
    protected <T> Invoker<T> doSelect(List<Invoker<T>> invokers, URL url, 
            Invocation invocation) {
        
        int length = invokers.size();
        
        // 是否所有权重相同
        boolean sameWeight = true;
        
        // 每个Invoker的权重
        int[] weights = new int[length];
        
        // 第一个Invoker的权重
        int firstWeight = getWeight(invokers.get(0), invocation);
        weights[0] = firstWeight;
        
        // 权重总和
        int totalWeight = firstWeight;
        
        // 🔥 计算所有权重
        for (int i = 1; i < length; i++) {
            int weight = getWeight(invokers.get(i), invocation);
            weights[i] = weight;
            totalWeight += weight;
            
            if (sameWeight && weight != firstWeight) {
                sameWeight = false;
            }
        }
        
        // 🔥 权重不同，按权重随机
        if (totalWeight > 0 && !sameWeight) {
            // 生成随机数
            int offset = ThreadLocalRandom.current().nextInt(totalWeight);
            
            // 确定随机数落在哪个区间
            for (int i = 0; i < length; i++) {
                offset -= weights[i];
                if (offset < 0) {
                    return invokers.get(i);
                }
            }
        }
        
        // 🔥 权重相同，完全随机
        return invokers.get(ThreadLocalRandom.current().nextInt(length));
    }
}
```

## 4. 轮询负载均衡

### 4.1 RoundRobinLoadBalance

```java
/**
 * RoundRobinLoadBalance - 轮询负载均衡
 * 加权轮询算法
 */
public class RoundRobinLoadBalance extends AbstractLoadBalance {
    
    public static final String NAME = "roundrobin";
    
    // 回收周期
    private static final int RECYCLE_PERIOD = 60000;
    
    // 方法级别的权重信息
    private ConcurrentMap<String, ConcurrentMap<String, WeightedRoundRobin>> 
        methodWeightMap = new ConcurrentHashMap<>();
    
    /**
     * 加权轮询信息
     */
    protected static class WeightedRoundRobin {
        private int weight;
        private AtomicLong current = new AtomicLong(0);
        private long lastUpdate;
        
        public int getWeight() {
            return weight;
        }
        
        public void setWeight(int weight) {
            this.weight = weight;
            current.set(0);
        }
        
        public long increaseCurrent() {
            return current.addAndGet(weight);
        }
        
        public void sel(int total) {
            current.addAndGet(-1 * total);
        }
        
        public long getLastUpdate() {
            return lastUpdate;
        }
        
        public void setLastUpdate(long lastUpdate) {
            this.lastUpdate = lastUpdate;
        }
    }
    
    @Override
    protected <T> Invoker<T> doSelect(List<Invoker<T>> invokers, URL url, 
            Invocation invocation) {
        
        String key = invokers.get(0).getUrl().getServiceKey() + "." + 
            invocation.getMethodName();
        
        // 获取或创建方法级别的权重Map
        ConcurrentMap<String, WeightedRoundRobin> map = 
            methodWeightMap.computeIfAbsent(key, k -> new ConcurrentHashMap<>());
        
        int totalWeight = 0;
        long maxCurrent = Long.MIN_VALUE;
        long now = System.currentTimeMillis();
        Invoker<T> selectedInvoker = null;
        WeightedRoundRobin selectedWRR = null;
        
        // 🔥 遍历所有Invoker
        for (Invoker<T> invoker : invokers) {
            String identifyString = invoker.getUrl().toIdentityString();
            int weight = getWeight(invoker, invocation);
            
            // 获取或创建WeightedRoundRobin
            WeightedRoundRobin weightedRoundRobin = 
                map.computeIfAbsent(identifyString, k -> {
                    WeightedRoundRobin wrr = new WeightedRoundRobin();
                    wrr.setWeight(weight);
                    return wrr;
                });
            
            // 更新权重
            if (weight != weightedRoundRobin.getWeight()) {
                weightedRoundRobin.setWeight(weight);
            }
            
            // 🔥 增加当前权重
            long cur = weightedRoundRobin.increaseCurrent();
            weightedRoundRobin.setLastUpdate(now);
            
            // 🔥 选择当前权重最大的
            if (cur > maxCurrent) {
                maxCurrent = cur;
                selectedInvoker = invoker;
                selectedWRR = weightedRoundRobin;
            }
            
            totalWeight += weight;
        }
        
        // 清理过期数据
        if (invokers.size() != map.size()) {
            map.entrySet().removeIf(item -> 
                now - item.getValue().getLastUpdate() > RECYCLE_PERIOD);
        }
        
        if (selectedInvoker != null) {
            // 🔥 减去总权重
            selectedWRR.sel(totalWeight);
            return selectedInvoker;
        }
        
        return invokers.get(0);
    }
}
```

## 5. 最少活跃数

### 5.1 LeastActiveLoadBalance

```java
/**
 * LeastActiveLoadBalance - 最少活跃数负载均衡
 * 活跃数越少，说明处理能力越强
 */
public class LeastActiveLoadBalance extends AbstractLoadBalance {
    
    public static final String NAME = "leastactive";
    
    @Override
    protected <T> Invoker<T> doSelect(List<Invoker<T>> invokers, URL url, 
            Invocation invocation) {
        
        int length = invokers.size();
        
        // 最小活跃数
        int leastActive = -1;
        
        // 具有相同最小活跃数的Invoker数量
        int leastCount = 0;
        
        // 具有相同最小活跃数的Invoker索引
        int[] leastIndexes = new int[length];
        
        // 每个Invoker的权重
        int[] weights = new int[length];
        
        // 最小活跃数Invoker的权重总和
        int totalWeight = 0;
        
        // 第一个最小活跃数Invoker的权重
        int firstWeight = 0;
        
        // 是否所有最小活跃数Invoker权重相同
        boolean sameWeight = true;
        
        // 🔥 遍历所有Invoker
        for (int i = 0; i < length; i++) {
            Invoker<T> invoker = invokers.get(i);
            
            // 获取活跃数
            int active = RpcStatus.getStatus(invoker.getUrl(), 
                invocation.getMethodName()).getActive();
            
            // 获取权重
            int afterWarmup = getWeight(invoker, invocation);
            weights[i] = afterWarmup;
            
            // 🔥 发现更小的活跃数
            if (leastActive == -1 || active < leastActive) {
                leastActive = active;
                leastCount = 1;
                leastIndexes[0] = i;
                totalWeight = afterWarmup;
                firstWeight = afterWarmup;
                sameWeight = true;
                
            // 🔥 活跃数相同
            } else if (active == leastActive) {
                leastIndexes[leastCount++] = i;
                totalWeight += afterWarmup;
                
                if (sameWeight && afterWarmup != firstWeight) {
                    sameWeight = false;
                }
            }
        }
        
        // 🔥 只有一个最小活跃数Invoker
        if (leastCount == 1) {
            return invokers.get(leastIndexes[0]);
        }
        
        // 🔥 多个最小活跃数Invoker，按权重随机
        if (!sameWeight && totalWeight > 0) {
            int offsetWeight = ThreadLocalRandom.current().nextInt(totalWeight);
            for (int i = 0; i < leastCount; i++) {
                int leastIndex = leastIndexes[i];
                offsetWeight -= weights[leastIndex];
                if (offsetWeight < 0) {
                    return invokers.get(leastIndex);
                }
            }
        }
        
        // 🔥 权重相同，随机选择
        return invokers.get(leastIndexes[ThreadLocalRandom.current().nextInt(leastCount)]);
    }
}
```

### 5.2 活跃数统计

```java
/**
 * RpcStatus - RPC状态统计
 */
public class RpcStatus {
    
    // 服务级别统计
    private static final ConcurrentMap<String, RpcStatus> SERVICE_STATISTICS = 
        new ConcurrentHashMap<>();
    
    // 方法级别统计
    private static final ConcurrentMap<String, ConcurrentMap<String, RpcStatus>> 
        METHOD_STATISTICS = new ConcurrentHashMap<>();
    
    // 🔥 活跃数（正在处理的请求数）
    private final AtomicInteger active = new AtomicInteger();
    
    // 总请求数
    private final AtomicLong total = new AtomicLong();
    
    // 失败请求数
    private final AtomicInteger failed = new AtomicInteger();
    
    // 总耗时
    private final AtomicLong totalElapsed = new AtomicLong();
    
    // 失败耗时
    private final AtomicLong failedElapsed = new AtomicLong();
    
    // 最大耗时
    private final AtomicLong maxElapsed = new AtomicLong();
    
    // 失败最大耗时
    private final AtomicLong failedMaxElapsed = new AtomicLong();
    
    // 成功最大耗时
    private final AtomicLong succeededMaxElapsed = new AtomicLong();
    
    /**
     * 🔥 开始计数
     */
    public static boolean beginCount(URL url, String methodName, int max) {
        max = (max <= 0) ? Integer.MAX_VALUE : max;
        RpcStatus appStatus = getStatus(url);
        RpcStatus methodStatus = getStatus(url, methodName);
        
        if (methodStatus.active.incrementAndGet() > max) {
            methodStatus.active.decrementAndGet();
            return false;
        }
        
        appStatus.active.incrementAndGet();
        return true;
    }
    
    /**
     * 🔥 结束计数
     */
    public static void endCount(URL url, String methodName, long elapsed, boolean succeeded) {
        endCount(getStatus(url), elapsed, succeeded);
        endCount(getStatus(url, methodName), elapsed, succeeded);
    }
    
    private static void endCount(RpcStatus status, long elapsed, boolean succeeded) {
        // 减少活跃数
        status.active.decrementAndGet();
        status.total.incrementAndGet();
        status.totalElapsed.addAndGet(elapsed);
        
        // 更新最大耗时
        if (status.maxElapsed.get() < elapsed) {
            status.maxElapsed.set(elapsed);
        }
        
        if (succeeded) {
            if (status.succeededMaxElapsed.get() < elapsed) {
                status.succeededMaxElapsed.set(elapsed);
            }
        } else {
            status.failed.incrementAndGet();
            status.failedElapsed.addAndGet(elapsed);
            if (status.failedMaxElapsed.get() < elapsed) {
                status.failedMaxElapsed.set(elapsed);
            }
        }
    }
}
```

## 6. 一致性哈希

### 6.1 ConsistentHashLoadBalance

```java
/**
 * ConsistentHashLoadBalance - 一致性哈希负载均衡
 * 相同参数的请求总是发到同一提供者
 */
public class ConsistentHashLoadBalance extends AbstractLoadBalance {
    
    public static final String NAME = "consistenthash";
    
    // 选择器缓存
    private final ConcurrentMap<String, ConsistentHashSelector<?>> selectors = 
        new ConcurrentHashMap<>();
    
    @Override
    protected <T> Invoker<T> doSelect(List<Invoker<T>> invokers, URL url, 
            Invocation invocation) {
        
        String methodName = RpcUtils.getMethodName(invocation);
        String key = invokers.get(0).getUrl().getServiceKey() + "." + methodName;
        
        // 计算Invoker列表的hashCode
        int invokersHashCode = invokers.hashCode();
        
        // 获取选择器
        ConsistentHashSelector<T> selector = (ConsistentHashSelector<T>) selectors.get(key);
        
        // 🔥 Invoker列表变化，重新创建选择器
        if (selector == null || selector.identityHashCode != invokersHashCode) {
            selectors.put(key, new ConsistentHashSelector<>(invokers, methodName, 
                invokersHashCode));
            selector = (ConsistentHashSelector<T>) selectors.get(key);
        }
        
        // 🔥 选择Invoker
        return selector.select(invocation);
    }
    
    /**
     * 一致性哈希选择器
     */
    private static final class ConsistentHashSelector<T> {
        
        // 🔥 虚拟节点到Invoker的映射
        private final TreeMap<Long, Invoker<T>> virtualInvokers;
        
        // 虚拟节点数
        private final int replicaNumber;
        
        // Invoker列表的hashCode
        private final int identityHashCode;
        
        // 参与哈希计算的参数索引
        private final int[] argumentIndex;
        
        ConsistentHashSelector(List<Invoker<T>> invokers, String methodName, 
                int identityHashCode) {
            
            this.virtualInvokers = new TreeMap<>();
            this.identityHashCode = identityHashCode;
            
            URL url = invokers.get(0).getUrl();
            
            // 获取虚拟节点数，默认160
            this.replicaNumber = url.getMethodParameter(methodName, 
                HASH_NODES, 160);
            
            // 获取参与哈希的参数索引
            String[] index = COMMA_SPLIT_PATTERN.split(
                url.getMethodParameter(methodName, HASH_ARGUMENTS, "0"));
            argumentIndex = new int[index.length];
            for (int i = 0; i < index.length; i++) {
                argumentIndex[i] = Integer.parseInt(index[i]);
            }
            
            // 🔥 为每个Invoker创建虚拟节点
            for (Invoker<T> invoker : invokers) {
                String address = invoker.getUrl().getAddress();
                for (int i = 0; i < replicaNumber / 4; i++) {
                    byte[] digest = Bytes.getMD5(address + i);
                    for (int h = 0; h < 4; h++) {
                        long m = hash(digest, h);
                        virtualInvokers.put(m, invoker);
                    }
                }
            }
        }
        
        /**
         * 🔥 选择Invoker
         */
        public Invoker<T> select(Invocation invocation) {
            // 获取参与哈希的参数值
            String key = toKey(invocation.getArguments());
            
            // 计算哈希值
            byte[] digest = Bytes.getMD5(key);
            
            // 选择节点
            return selectForKey(hash(digest, 0));
        }
        
        private String toKey(Object[] args) {
            StringBuilder buf = new StringBuilder();
            for (int i : argumentIndex) {
                if (i >= 0 && i < args.length) {
                    buf.append(args[i]);
                }
            }
            return buf.toString();
        }
        
        /**
         * 🔥 根据哈希值选择节点
         */
        private Invoker<T> selectForKey(long hash) {
            // 找到第一个大于等于hash的节点
            Map.Entry<Long, Invoker<T>> entry = virtualInvokers.ceilingEntry(hash);
            
            // 如果没有，取第一个节点（环形）
            if (entry == null) {
                entry = virtualInvokers.firstEntry();
            }
            
            return entry.getValue();
        }
        
        private long hash(byte[] digest, int number) {
            return (((long) (digest[3 + number * 4] & 0xFF) << 24)
                | ((long) (digest[2 + number * 4] & 0xFF) << 16)
                | ((long) (digest[1 + number * 4] & 0xFF) << 8)
                | (digest[number * 4] & 0xFF))
                & 0xFFFFFFFFL;
        }
    }
}
```

## 7. 面试要点

### 7.1 高频面试题

#### Q1: Dubbo有哪些负载均衡策略？

```
Dubbo负载均衡策略：

1. Random（随机）- 默认
   - 按权重随机选择
   - 调用量越大分布越均匀

2. RoundRobin（轮询）
   - 按权重轮询
   - 平滑加权轮询算法

3. LeastActive（最少活跃数）
   - 活跃数越少优先级越高
   - 慢提供者收到更少请求

4. ConsistentHash（一致性哈希）
   - 相同参数请求到同一提供者
   - 适合有状态服务

5. ShortestResponse（最短响应时间）
   - 响应时间越短优先级越高
   - Dubbo 2.7+新增
```

#### Q2: 随机负载均衡如何实现权重？

```java
/**
 * 随机负载均衡权重实现：
 * 
 * 1. 计算所有Invoker的权重总和
 * 2. 生成[0, totalWeight)的随机数
 * 3. 遍历Invoker，减去各自权重
 * 4. 当随机数小于0时，选中当前Invoker
 * 
 * 示例：
 * A:5, B:3, C:2，总权重10
 * 随机数7
 * 7-5=2 (>0, 继续)
 * 2-3=-1 (<0, 选中B)
 */
```

#### Q3: 一致性哈希的优势？

```
一致性哈希优势：

1. 相同参数请求到同一提供者
   - 适合有状态服务
   - 利于缓存命中

2. 节点增减影响小
   - 只影响相邻节点
   - 不会导致全部重新分配

3. 虚拟节点解决分布不均
   - 每个物理节点对应多个虚拟节点
   - 请求分布更均匀
```

#### Q4: 最少活跃数如何统计？

```java
/**
 * 活跃数统计：
 * 
 * 1. 请求开始时：active++
 * 2. 请求结束时：active--
 * 
 * 通过ActiveLimitFilter实现：
 * - 调用前：RpcStatus.beginCount()
 * - 调用后：RpcStatus.endCount()
 * 
 * 活跃数含义：
 * - 正在处理的请求数
 * - 活跃数越少，说明处理能力越强
 */
```

## 📚 总结

### 核心要点

```
1. LoadBalance接口定义负载均衡
   - select()方法选择Invoker
   - @SPI默认random

2. AbstractLoadBalance提供基础实现
   - 权重计算
   - 预热处理

3. 四种主要策略
   - Random：按权重随机
   - RoundRobin：平滑加权轮询
   - LeastActive：最少活跃数优先
   - ConsistentHash：一致性哈希

4. 权重影响选择概率
   - 预热期间权重逐渐增加
   - 避免新启动服务压力过大

5. 可通过配置灵活选择
   - 服务级、方法级配置
   - 消费端、服务端配置
```
