## 1. String类概述与不可变性

### 1.1 String类定义

```java
public final class String
    implements java.io.Serializable, Comparable<String>, CharSequence {

    /** 字符串的值，使用byte数组存储 */
    private final byte[] value;

    /** 编码标识：0=Latin1，1=UTF16 */
    private final byte coder;

    /** 缓存的哈希码，默认0表示未计算 */
    private int hash;

    /** 序列化版本号 */
    private static final long serialVersionUID = -6849794470754667710L;
}
```

关键设计决策：
1. **`final class`**：String不能被继承，防止子类破坏不可变性
2. **`final byte[] value`**：数组是final的，引用不可变；数组内容也通过不暴露修改方法保证不变
3. **`final byte coder`**：编码标识不可变

### 1.2 不可变性的体现

```java
String s = "hello";
s = s + " world";  // 不是修改原对象，而是创建新对象
// s现在指向"hello world"这个新对象

// 所有"修改"String的方法都返回新对象
String upper = "hello".toUpperCase(); // 返回新对象"HELLO"
String sub = "hello".substring(1);    // 返回新对象"ello"
// 原始字符串"hello"始终不变
```

### 1.3 为什么String要设计成不可变？

```
1. 线程安全
   └── 不可变对象天然线程安全，无需同步

2. 安全性
   └── 防止敏感信息（如数据库URL、密码）被意外修改
   └── 作为HashMap的key不会发生变化

3. hashCode缓存
   └── 不可变所以hashCode可以缓存，首次计算后复用
   └── 作为HashMap key时性能优势明显

4. 字符串常量池
   └── 不可变才能安全地共享同一个实例

5. Class Loading安全
   └── 类加载器以String作为类名，不可变防止恶意篡改
```

### 1.4 看似可变的反射漏洞

```java
// 通过反射可以修改String的内部value（不推荐！危险！）
String s = "Hello";
System.out.println(s); // Hello

Field field = String.class.getDeclaredField("value");
field.setAccessible(true);
byte[] bytes = (byte[]) field.get(s);
// 修改内部数组（仅限Java 8的char[]，Java 9+是byte[]）
// Java 8: field.get(s) 返回 char[]
// bytes[0] = 'h';

System.out.println(s); // 可能变成"hello"
// 但这会破坏字符串常量池，导致不可预知的bug！
```

## 2. 内部存储：从char[]到byte[]

### 2.1 Java 8及之前：char[]

```java
// Java 8 的String
public final class String {
    private final char[] value;  // 每个char占2个字节（UTF-16）
}
```

在Java 8中，String内部使用`char[]`存储，每个字符占2个字节（Java的char是UTF-16编码）。但大多数应用的字符串都是Latin1（ISO-8859-1）可表示的ASCII字符，只需要1个字节。这意味着**一半的内存被浪费了**。

### 2.2 Java 9及之后：byte[] + coder

```java
// Java 9+ 的String
public final class String {
    private final byte[] value;   // 字节数组
    private final byte coder;     // 编码标识
}
```

Java 9引入了JEP 254: Compact Strings，将内部存储从`char[]`改为`byte[]`，并增加`coder`字段标识编码：

| coder值 | 编码 | 每字符占用 | 适用场景 |
|---------|------|-----------|---------|
| 0 | Latin1 (ISO-8859-1) | 1 byte | 纯ASCII/Latin1字符 |
| 1 | UTF-16 | 2 bytes | 含非Latin1字符（如中文） |

### 2.3 内存节省效果

```java
// 假设存储"hello"（5个ASCII字符）

// Java 8 (char[])
// char[] value = new char[5]; → 10 bytes + 数组头开销
// 每个char 2字节，5个字符 = 10字节

// Java 9+ (byte[], Latin1)
// byte[] value = new byte[5]; → 5 bytes + 数组头开销
// 每个byte 1字节，5个字符 = 5字节
// 节省50%内存！

// 含中文的情况 "你好"（2个中文字符）
// coder = 1 (UTF-16)
// byte[] value = new byte[4]; → 4 bytes
// 与Java 8相同
```

### 2.4 编码判断

```java
// String内部判断编码的方式
private static boolean isLatin1(byte[] bytes) {
    for (byte b : bytes) {
        if (b < 0) return false; // 非Latin1字符
    }
    return true;
}

// 创建String时自动选择编码
String s1 = "hello";      // coder = 0 (Latin1)
String s2 = "你好";        // coder = 1 (UTF-16)
String s3 = "hello世界";   // coder = 1 (UTF-16)，因为含非Latin1字符
```

### 3.3 字符操作中的编码处理

```java
// length()方法——根据编码返回不同的长度
public int length() {
    return value.length >> coder;  // Latin1: >>0, UTF16: >>1
}

// charAt()方法
public char charAt(int index) {
    if (isLatin1()) {
        return StringLatin1.charAt(value, index);
    } else {
        return StringUTF16.charAt(value, index);
    }
}

// Latin1的charAt
public static char charAt(byte[] value, int index) {
    if (index < 0 || index >= value.length) {
        throw new StringIndexOutOfBoundsException(index);
    }
    return (char)(value[index] & 0xff);
}

// UTF16的charAt
public static char charAt(byte[] value, int index) {
    checkIndex(index, value);
    return getChar(value, index);
}

// getChar——从byte数组中读取UTF-16字符
@IntrinsicCandidate
public static char getChar(byte[] val, int index) {
    index <<= 1; // 每个字符2字节
    return (char)(((val[index++] & 0xFF) << 8) | (val[index] & 0xFF));
}
```

### 3.4 编码转换

```java
// 当Latin1字符串与UTF16字符串拼接时，需要编码升级
private void inflate(byte[] src, int srcPos, byte[] dst, int dstPos, int len) {
    // 将Latin1的byte[]转换为UTF16的byte[]
    for (int i = 0; i < len; i++) {
        StringUTF16.putChar(dst, dstPos++, src[srcPos++] & 0xff);
    }
}

// 拼接时自动处理编码
public String concat(String str) {
    if (str.isEmpty()) return this;

    if (coder() == str.coder()) {
        // 同编码——直接数组复制
        byte[] buf = Arrays.copyOf(value, length() + str.length());
        System.arraycopy(str.value, 0, buf, value.length, str.value.length);
        return new String(buf, coder);
    } else {
        // 不同编码——升级为UTF16
        byte[] buf = StringUTF16.newBytesFor(length() + str.length());
        // 先把自己写入（可能需要inflate）
        // 再把str写入
        // ...
        return new String(buf, UTF16);
    }
}
```

## 4. 字符串常量池

### 4.1 常量池的位置

```
JDK 6 及之前：永久代（PermGen）
JDK 7+：堆（Heap）
```

JDK 6将字符串常量池放在永久代，永久代空间有限且GC频率低，容易导致OOM。JDK 7移到了堆中，可以利用常规GC管理。

### 4.2 常量池的工作原理

```java
// 字面量赋值——从常量池获取
String s1 = "hello";       // 常量池中没有"hello"，创建并放入常量池
String s2 = "hello";       // 常量池中已有"hello"，直接引用
System.out.println(s1 == s2); // true —— 同一个常量池实例

// new创建——堆中创建新对象
String s3 = new String("hello"); // 堆中创建新对象，但字符值仍来自常量池
System.out.println(s1 == s3); // false —— s3是堆中新对象

// intern()——从常量池获取或放入
String s4 = s3.intern();    // s3.intern()返回常量池中的"hello"
System.out.println(s1 == s4); // true —— s4是常量池中的实例
```

### 4.3 常量池内存模型

```
                    堆内存
    ┌──────────────────────────────────────┐
    │                                      │
    │  字符串常量池 (String Pool)            │
    │  ┌─────────┐ ┌─────────┐ ┌─────────┐│
    │  │ "hello" │ │ "world" │ │ "java"  ││
    │  └─────────┘ └─────────┘ └─────────┘│
    │       ↑                              │
    │       │                              │
    │  s1 ──┘ s2 ──┘ (引用同一个对象)       │
    │                                      │
    │  s3 ──→ [String对象] (堆中独立对象)   │
    │         value ──→ "hello"(常量池引用) │
    │                                      │
    └──────────────────────────────────────┘
```

### 4.4 编译期常量与运行期常量

```java
// 编译期常量——编译器优化
String s1 = "hello" + "world";  // 编译器直接优化为 "helloworld"
String s2 = "helloworld";
System.out.println(s1 == s2); // true —— 编译期已确定

// 运行期拼接——创建新对象
String a = "hello";
String b = "world";
String s3 = a + b;  // 运行期拼接，创建新对象
System.out.println(s3 == s2); // false —— s3是运行期创建的新对象

// final修饰的变量参与拼接——编译期确定
final String fa = "hello";
final String fb = "world";
String s4 = fa + fb;  // final变量在编译期被替换
System.out.println(s4 == s2); // true —— 等同于"helloworld"
```

### 4.5 常量池的GC

```java
// JDK 7+ 字符串常量池在堆中，可以被Full GC回收
// 当常量池中的字符串没有被任何引用指向时，可以被GC回收

// 验证常量池GC
List<String> list = new ArrayList<>();
int i = 0;
while (true) {
    // intern()会不断向常量池添加字符串
    // 如果常量池不能GC，最终会OOM
    list.add(("string" + i++).intern());
}
// 在JDK 8+中，Full GC会清理无引用的常量池字符串
// 不会OOM（但会频繁GC）
```

## 5. intern()方法深度解析

### 5.1 源码

```java
/**
 * 返回字符串规范表示。
 * 当调用intern方法时，如果常量池中已经包含一个等于此String对象的字符串
 * （由equals方法确定），则返回常量池中的字符串。
 * 否则，将此String对象添加到常量池中，并返回对此String对象的引用。
 *
 * 对于任意两个字符串s和t，当且仅当s.equals(t)为true时，
 * s.intern() == t.intern()为true。
 */
public native String intern();
```

intern()是native方法，由JVM实现。

### 5.2 intern()的工作流程

```
String s = new String("hello");
String interned = s.intern();

执行流程：
1. 查找字符串常量池中是否有equals("hello")的字符串
2. 如果有 → 返回常量池中的引用
3. 如果没有：
   a. JDK 6: 将s的副本复制到永久代，返回永久代中的引用
   b. JDK 7+: 将s的引用存入常量池（StringTable），返回s本身的引用
```

### 5.3 经典面试题

```java
// 面试题1
String s1 = new String("1");
// 创建了2个对象：常量池中的"1" + 堆中的new String("1")
// s1指向堆中的对象

s1.intern();
String s2 = "1";
System.out.println(s1 == s2); // false
// s1是堆中的对象，s2是常量池中的"1"
// s1.intern()返回常量池中的"1"，但s1本身没变

// 面试题2
String s3 = new String("1") + new String("1");
// s3是动态拼接的结果，值为"11"
// 注意："11"此时不在常量池中！只有"1"在常量池

s3.intern();
String s4 = "11";
System.out.println(s3 == s4); // JDK7+ true, JDK6 false
// JDK 7+: s3.intern()将s3的引用放入常量池，s4直接取常量池引用=s3
// JDK 6:  s3.intern()将s3的副本放入永久代，s4取永久代引用≠s3
```

### 5.4 intern()的性能考虑

```java
// intern()可以减少内存——但不要滥用！

// 坏例子：对大量不同的字符串调用intern()
// StringTable本质是HashTable，条目过多会导致哈希冲突增加
// 查找性能从O(1)退化为O(n)

// 可以通过JVM参数调整StringTable大小
// -XX:StringTableSize=1000003（默认大小因JDK版本而异）

// 合理使用场景
public class User {
    private String status; // "ACTIVE", "INACTIVE", "BANNED" 等有限值

    public void setStatus(String status) {
        this.status = status.intern(); // 有限值集合适合intern
    }
}

// 不合理使用
String randomId = UUID.randomUUID().toString().intern(); // 每个都不同，浪费StringTable
```

### 5.5 StringTable实现

```java
// JVM中StringTable的本质是一个HashTable
// key: 字符串的hash值
// value: 字符串对象的引用

// OpenJDK中的C++实现（简化）
class StringTable : public Hashtable<oop, mtSymbol> {
    // 桶数组
    // 每个桶是一个链表
    // 存储字符串引用

    oop intern(oop string, TRAPS) {
        // 1. 计算hash
        unsigned int hash = java_lang_String::hash_code(string);

        // 2. 查找
        oop found = lookup(string, hash);
        if (found != NULL) return found;

        // 3. 未找到——添加
        return basic_add(string, hash);
    }
};

// JDK 9+ 使用链表转红黑树优化（类似HashMap）
// 当桶中元素超过阈值时，链表转红黑树
```

## 7. 核心方法源码解析

### 7.1 equals方法

```java
public boolean equals(Object anObject) {
    // 1. 先比较引用（快速路径）
    if (this == anObject) {
        return true;
    }
    // 2. 类型检查
    if (anObject instanceof String) {
        String aString = (String)anObject;
        // 3. 编码相同才比较（不同编码需要特殊处理）
        if (coder() == aString.coder()) {
            return StringLatin1.equals(value, aString.value);
        }
    }
    return false;
}

// Latin1的equals
public static boolean equals(byte[] value, byte[] other) {
    if (value.length == other.length) {
        for (int i = 0; i < value.length; i++) {
            if (value[i] != other[i]) {
                return false;
            }
        }
        return true;
    }
    return false;
}
```

### 7.2 hashCode方法

```java
// String的hashCode算法
public int hashCode() {
    int h = hash;
    if (h == 0 && value.length > 0) {
        // 只在第一次调用时计算
        hash = h = isLatin1() ?
            StringLatin1.hashCode(value) :
            StringUTF16.hashCode(value);
    }
    return h;
}

// Latin1的hashCode
public static int hashCode(byte[] value) {
    int h = 0;
    for (byte v : value) {
        h = 31 * h + (v & 0xff);
    }
    return h;
}

// 算法：s[0]*31^(n-1) + s[1]*31^(n-2) + ... + s[n-1]
// 与Object.hashCode不同，String的hashCode是确定性的——
// 相同内容的String始终返回相同的hashCode
```

### 7.3 substring方法

```java
// Java 7+ 的substring实现
public String substring(int beginIndex, int endIndex) {
    // 边界检查
    int length = length();
    checkBoundsBeginEnd(beginIndex, endIndex, length);

    // 范围检查
    int subLen = endIndex - beginIndex;
    if (subLen == 0) {
        return "";
    }

    // 创建新对象，复制子数组
    if (isLatin1()) {
        return new String(value, beginIndex, subLen, Latin1);
    } else {
        // UTF16每个字符2字节，beginIndex需要乘2
        return new String(value, beginIndex << 1, subLen << 1, UTF16);
    }
}

// 内部构造方法——直接从byte数组截取
private String(byte[] value, byte coder) {
    this.value = value;
    this.coder = coder;
}
```

### 7.4 indexOf方法

```java
public int indexOf(String str) {
    return indexOf(str, 0);
}

public int indexOf(String str, int fromIndex) {
    return indexOf(value, coder, length(), str, fromIndex);
}

// Latin1的indexOf
static int indexOf(byte[] value, int valueCount, byte[] str, int strCount, int fromIndex) {
    byte first = str[0];
    int max = valueCount - strCount;  // 搜索范围上限

    for (int i = fromIndex; i <= max; i++) {
        // 找到第一个字符匹配的位置
        if (value[i] != first) {
            while (++i <= max && value[i] != first);
        }

        // 找到首字符后，比较剩余部分
        if (i <= max) {
            int j = i + 1;
            int k = 1;
            while (k < strCount && value[j] == str[k]) {
                j++;
                k++;
            }
            if (k == strCount) {
                return i; // 找到
            }
        }
    }
    return -1; // 未找到
}
```

```java
    if (((regex.value.length == 1 &&
         ".$|()[{^?*+\\".indexOf(ch = regex.charAt(0)) == -1)) ||
         (regex.length() == 2 &&
          regex.charAt(0) == '\\' &&
          (((ch = regex.charAt(1))-'0')|('9'-ch)) < 0 &&
          ((ch-'a')|('z'-ch)) < 0 &&
          ((ch-'A')|('Z'-ch)) < 0)) &&
         (ch < Character.MIN_HIGH_SURROGATE ||
          ch > Character.MAX_LOW_SURROGATE)) {
        // 快速路径——直接字符匹配，不走正则引擎
        return splitFastPath(ch, limit);
    }

    // 慢路径——使用正则表达式
    return Pattern.compile(regex).split(this, limit);
}
```

### 8.3 StringBuilder源码要点

```java
public final class StringBuilder
    extends AbstractStringBuilder implements Appendable, Serializable {

    // 继承自AbstractStringBuilder
    // byte[] value;  // 可变数组
    // int count;     // 实际使用长度

    public StringBuilder() {
        super(16); // 默认容量16
    }

    public StringBuilder(int capacity) {
        super(capacity);
    }

    public StringBuilder append(String str) {
        super.append(str);
        return this;
    }

    @Override
    public String toString() {
        // 创建新String，复制有效部分
        byte[] val = new byte[count << coder];
        System.arraycopy(value, 0, val, 0, count << coder);
        return new String(val, coder);
    }
}

// AbstractStringBuilder.append
public AbstractStringBuilder append(String str) {
    if (str == null) {
        return appendNull();
    }
    int len = str.length();
    ensureCapacityInternal(count + len); // 扩容
    putStringAt(count, str);
    count += len;
    return this;
}

// 扩容策略
private void ensureCapacityInternal(int minimumCapacity) {
    int oldCapacity = value.length >> coder;
    if (minimumCapacity - oldCapacity > 0) {
        // 新容量 = 旧容量 * 2 + 2
        int newCapacity = (oldCapacity << 1) + 2;
        if (newCapacity - minimumCapacity < 0) {
            newCapacity = minimumCapacity;
        }
        // 复制到新数组
        value = Arrays.copyOf(value, newCapacity << coder);
    }
}
```

## 10. substring与内存泄漏

### 10.1 JDK 6的内存泄漏

```java
// JDK 6 的substring实现
public String substring(int beginIndex, int endIndex) {
    return new String(offset + beginIndex, endIndex - beginIndex, value);
    // 内部共享同一个char[] value，只是改变offset和count
}

// JDK 6 的String内部
class String {
    char[] value;  // 共享数组
    int offset;    // 起始偏移
    int count;     // 字符数
}

// 问题场景
String huge = new String(new char[1000000]); // 1MB的大字符串
String tiny = huge.substring(0, 10);          // 只取10个字符

// tiny的value仍然指向huge的char[1000000]！
// huge对象可以被GC，但char[1000000]不能被回收（tiny还在引用）
// 这就是"内存泄漏"——10个字符占用了1MB内存
```

### 10.2 JDK 7+的修复

```java
// JDK 7+ 的substring实现
public String substring(int beginIndex, int endIndex) {
    int subLen = endIndex - beginIndex;
    // 创建新数组，只复制需要的部分
    return new String(value, beginIndex, subLen);
    // 新String有自己的char[]/byte[]，不共享
}

// JDK 7+不再有offset和count字段
// substring会复制子数组，彻底解决了内存泄漏问题
```

### 10.3 性能权衡

```
JDK 6（共享数组）:
  优点: substring O(1)时间，不复制数组
  缺点: 可能导致内存泄漏

JDK 7+（复制数组）:
  优点: 不会内存泄漏
  缺点: substring O(n)时间，需要复制数组

权衡: 内存正确性 > 性能，JDK 7+的选择是正确的
```

## 11. 面试题速查

### Q1: String为什么是不可变的？

1. final class防止子类破坏不可变性。2. private final byte[] value，引用不可变且不暴露修改方法。3. 所有"修改"方法都返回新对象。4. 不可变带来线程安全、hashCode缓存、安全性、常量池共享等好处。

### Q2: String、StringBuilder、StringBuffer的区别？

String不可变；StringBuilder可变，非线程安全，性能好；StringBuffer可变，线程安全（方法用synchronized修饰），性能略差。单线程拼接用StringBuilder，多线程用StringBuffer或使用StringBuilder + 外部同步。

### Q3: new String("abc")创建了几个对象？

如果常量池中没有"abc"，创建2个对象：1个在常量池，1个在堆。如果常量池中已有"abc"，创建1个对象：只在堆中创建。另外，new String(char[])等构造方法不会在常量池中创建对象。

### Q4: intern()方法的作用和原理？

intern()查找字符串常量池：如果常量池中已有equals相等的字符串，返回常量池中的引用；如果没有，将当前字符串的引用（JDK 7+）或副本（JDK 6）放入常量池并返回。intern()是native方法，由JVM实现。JDK 7+常量池在堆中，可以存引用而非副本。

### Q5: Java 9的Compact Strings是什么？

将String内部存储从char[]改为byte[]，增加coder字段标识编码。Latin1字符用1字节存储（之前用2字节），节省约50%内存。含非Latin1字符（如中文）时使用UTF-16编码。通过-XX:-CompactStrings可关闭。

### Q6: String的hashCode算法是什么？

`s[0]*31^(n-1) + s[1]*31^(n-2) + ... + s[n-1]`。使用31作为乘数因为31是奇素数且可优化为位运算（31*i = (i<<5)-i）。hashCode只在首次调用时计算，之后缓存在hash字段中。相同内容的String始终返回相同的hashCode。

### Q7: JDK 6的substring有什么问题？

JDK 6的substring共享原字符串的char[]，只改变offset和count。如果用substring截取大字符串的一小部分，大字符串的char[]不会被GC回收，导致内存泄漏。JDK 7+改为复制子数组，解决了这个问题。

### Q8: 字符串常量池在哪里？

JDK 6及之前在永久代（PermGen），JDK 7移到堆中。JDK 8移除了永久代改为元空间（Metaspace），但字符串常量池仍在堆中。常量池中的字符串如果没有被引用，可以被Full GC回收。

### Q9: String s = "a"+"b"; 编译器做了什么？

如果a和b都是字面量，编译器在编译期直接合并为"ab"（常量折叠）。如果a和b是变量，编译器生成StringBuilder.append()代码（JDK 8）或invokedynamic（JDK 9+）。final修饰的变量在编译期会被替换为字面量，等同于字面量拼接。

### Q10: String的equals方法流程是什么？

先比较引用（this==obj），再检查类型（instanceof String），再检查编码是否相同，最后逐字节比较。Latin1编码的比较调用StringLatin1.equals，UTF-16的调用StringUTF16.equals。不同编码的字符串直接返回false（内容不可能相同）。
