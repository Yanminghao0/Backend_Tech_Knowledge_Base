## 1. Thread 类概述

```java
public class Thread implements Runnable {
    // 线程名
    private volatile String name;

    // 线程优先级 (1~10)
    private int priority;

    // 是否为守护线程
    private boolean daemon = false;

    // 线程要执行的任务
    private Runnable target;

    // 线程组
    private ThreadGroup group;

    // 线程 ID
    private long tid;

    // 线程状态
    private volatile int threadStatus;

    // 未捕获异常处理器
    private volatile UncaughtExceptionHandler uncaughtExceptionHandler;

    // ...
}
```

### 1.1 创建线程的方式

```java
// 方式1: 继承 Thread
class MyThread extends Thread {
    @Override
    public void run() {
        System.out.println("running");
    }
}
new MyThread().start();

// 方式2: 实现 Runnable
class MyTask implements Runnable {
    @Override
    public void run() {
        System.out.println("running");
    }
}
new Thread(new MyTask()).start();

// 方式3: Lambda（本质还是 Runnable）
new Thread(() -> System.out.println("running")).start();

// 方式4: 实现 Callable + FutureTask（有返回值）
FutureTask<String> future = new FutureTask<>(() -> "result");
new Thread(future).start();
System.out.println(future.get()); // "result"
```

> 本质上只有一种方式：**创建 Thread 对象并调用 start()**。`Runnable`/`Callable` 只是任务的定义方式，最终还是通过 Thread 来执行。

## 2. 线程状态（六种状态）

Java 线程有 6 种状态，定义在 `Thread.State` 枚举中：

```java
public enum State {
    NEW,           // 新建
    RUNNABLE,      // 可运行（就绪 + 运行中）
    BLOCKED,       // 阻塞（等待 monitor 锁）
    WAITING,       // 等待（无限期等待）
    TIMED_WAITING, // 限时等待
    TERMINATED;    // 终止
}
```

### 2.1 状态转换图

```
                        start()
     ┌──────────┐ ──────────────→ ┌──────────────┐
     │   NEW    │                 │   RUNNABLE   │ ←──────────┐
     └──────────┘                 └──────┬───────┘            │
                                         │                    │
                    ┌────────────────────┼────────────────────┤
                    │                    │                    │
                    ▼                    ▼                    ▼
             ┌──────────┐        ┌──────────────┐    ┌───────────────┐
             │ BLOCKED  │        │   WAITING    │    │ TIMED_WAITING │
             │          │        │              │    │               │
             │ 等待锁    │        │ wait()       │    │ sleep(ms)     │
             │          │        │ join()       │    │ wait(ms)      │
             │          │        │ LockSupport  │    │ join(ms)      │
             └────┬─────┘        └──────┬───────┘    └───────┬───────┘
                  │                     │                    │
                  │  获取锁             │ notify/notifyAll   │ 超时/notify
                  └─────────────────────┴────────────────────┘
                                         │
                                         │ run() 结束
                                         ▼
                                  ┌──────────────┐
                                  │  TERMINATED  │
                                  └──────────────┘
```

### 2.2 各状态详解

```java
// NEW: 创建了 Thread 对象但未调用 start()
Thread t1 = new Thread(() -> {});
System.out.println(t1.getState()); // NEW

// RUNNABLE: 调用了 start()，包括就绪和运行中
// Java 将操作系统层面的"就绪"和"运行中"合并为 RUNNABLE
// 还包括 IO 阻塞（如 socket.read()）也算 RUNNABLE
Thread t2 = new Thread(() -> {
    while (true) {}
});
t2.start();
System.out.println(t2.getState()); // RUNNABLE

// BLOCKED: 等待 synchronized 锁
Object lock = new Object();
Thread t3a = new Thread(() -> {
    synchronized (lock) {
        try { Thread.sleep(10000); } catch (Exception e) {}
    }
});
Thread t3b = new Thread(() -> {
    synchronized (lock) {} // 阻塞等锁
});
t3a.start();
Thread.sleep(100);
t3b.start();
Thread.sleep(100);
System.out.println(t3b.getState()); // BLOCKED

// WAITING: 无限等待
Thread t4 = new Thread(() -> {
    synchronized (lock) {
        try { lock.wait(); } catch (Exception e) {}
    }
});
t4.start();
Thread.sleep(100);
System.out.println(t4.getState()); // WAITING

// TIMED_WAITING: 限时等待
Thread t5 = new Thread(() -> {
    try { Thread.sleep(10000); } catch (Exception e) {}
});
t5.start();
Thread.sleep(100);
System.out.println(t5.getState()); // TIMED_WAITING

// TERMINATED: 执行完毕
Thread t6 = new Thread(() -> {});
t6.start();
t6.join();
System.out.println(t6.getState()); // TERMINATED
```

### 2.3 注意事项

- **RUNNABLE 包含 IO 阻塞**：线程在 `socket.read()` 阻塞时，Java 层面状态仍是 `RUNNABLE`，因为 JVM 无法感知操作系统层面的阻塞。
- **BLOCKED 仅限 synchronized**：等待 `ReentrantLock` 时状态是 `WAITING`（因为 `LockSupport.park`），不是 `BLOCKED`。

## 3. start() vs run()

### 3.1 start() 源码

```java
public synchronized void start() {
    // 检查线程状态，必须为 NEW（0）
    if (threadStatus != 0)
        throw new IllegalThreadStateException();

    // 加入线程组
    group.add(this);

    boolean started = false;
    try {
        // 调用 native 方法创建并启动操作系统线程
        start0();
        started = true;
    } finally {
        try {
            if (!started) {
                group.threadStartFailed(this);
            }
        } catch (Throwable ignore) {}
    }
}

private native void start0();
```

### 3.2 run() 源码

```java
@Override
public void run() {
    if (target != null) {
        target.run(); // 执行传入的 Runnable
    }
}
```

### 3.3 核心区别

| 维度 | start() | run() |
|---|---|---|
| 作用 | 创建新线程并调用 run() | 普通方法调用 |
| 线程 | 在新线程中执行 | 在当前调用者线程中执行 |
| 调用次数 | 只能调用一次 | 可以多次调用 |
| JVM 层面 | 调用 native start0() 创建 OS 线程 | 普通 Java 方法调用 |

```java
Thread t = new Thread(() -> {
    System.out.println("Thread: " + Thread.currentThread().getName());
});

t.start(); // 输出: Thread: Thread-0（新线程）
t.run();   // 输出: Thread: main（当前线程，重复执行）
```

### 3.4 start0() 的底层流程

```
Java Thread.start()
    → JVM Thread::start()
        → 创建操作系统线程 (pthread_create on Linux, CreateThread on Windows)
            → 新线程入口: JavaThread::run()
                → callback: thread_entry()
                    → 调用 Java 层的 Thread.run()
```

> `start()` 只能调用一次。第二次调用会抛出 `IllegalThreadStateException`，因为线程状态已不是 `NEW`。

## 4. join() 方法原理

### 4.1 源码解析

```java
public final void join() throws InterruptedException {
    join(0); // 委托给 join(long millis)
}

public final synchronized void join(long millis) throws InterruptedException {
    long base = System.currentTimeMillis();
    long now = 0;

    if (millis < 0) {
        throw new IllegalArgumentException("timeout value is negative");
    }

    if (millis == 0) {
        // 无限等待
        while (isAlive()) {
            wait(0); // 调用 Object.wait()，释放 this 锁
        }
    } else {
        // 限时等待
        while (isAlive()) {
            long delay = millis - now;
            if (delay <= 0) {
                break; // 超时
            }
            wait(delay); // 限时等待
            now = System.currentTimeMillis() - base;
        }
    }
}

public final synchronized void join(long millis, int nanos) throws InterruptedException {
    // 参数校验后委托给 join(long)
    if (millis < 0) throw new IllegalArgumentException("timeout value is negative");
    if (nanos < 0 || nanos > 999999) throw new IllegalArgumentException("nanosecond timeout value out of range");
    if (nanos > 0 && millis < Long.MAX_VALUE) millis++;
    join(millis);
}
```

### 4.2 join 的本质

**`join()` 的本质就是 `wait()`** —— 当前线程在 Thread 对象上等待，直到目标线程结束。

```java
// join 的等价写法
Thread t = new Thread(() -> { /* ... */ });
t.start();

synchronized (t) {
    while (t.isAlive()) {
        t.wait(); // 在 Thread 对象上等待
    }
}
```

### 4.3 谁调用了 notify?

```java
// JVM 在线程结束时自动调用 exit() 方法
private void exit() {
    // ...
    // 线程结束后，JVM 会调用 this.notifyAll()
    // 唤醒所有在该 Thread 对象上 wait 的线程
    group.threadTerminated(this);
    // ...
}
```

> 线程结束时，JVM 会自动调用 `notifyAll()` 唤醒所有通过 `join()` 等待的线程。

### 4.4 使用示例

```java
Thread t1 = new Thread(() -> {
    try { Thread.sleep(2000); } catch (Exception e) {}
    System.out.println("t1 done");
});

Thread t2 = new Thread(() -> {
    try { t1.join(); } catch (InterruptedException e) {}
    System.out.println("t2 done");
});

t1.start();
t2.start();
// 输出顺序：t1 done → t2 done
```

## 5. interrupt() 协作式中断

### 5.1 核心方法

```java
// 设置中断标志
public void interrupt() {
    if (this != Thread.currentThread())
        checkAccess(); // 安全检查

    synchronized (blockerLock) {
        Interruptible b = blocker;
        if (b != null) {
            interrupt0();  // 设置中断标志
            b.interrupt(this);
            return;
        }
    }
    interrupt0(); // native 方法，设置中断标志
}

// 检查并清除中断标志
public static boolean interrupted() {
    return currentThread().isInterrupted(true); // 清除标志
}

// 检查中断标志（不清除）
public boolean isInterrupted() {
    return isInterrupted(false); // 不清除标志
}

// native 方法
private native boolean isInterrupted(boolean ClearInterrupted);
```

### 5.2 协作式中断的含义

Java 的中断是**协作式**的——`interrupt()` 只是设置一个标志位，不会强制停止线程。线程需要自己检查标志并决定如何响应。

```java
// 正确的中断响应模式
Thread t = new Thread(() -> {
    while (!Thread.currentThread().isInterrupted()) {
        // 执行任务
        doWork();
    }
    // 清理资源
    cleanup();
});
t.start();
// 需要停止时
t.interrupt(); // 设置中断标志，线程下次循环检查到标志后退出
```

### 5.3 阻塞方法的中断响应

当线程在 `sleep()`、`wait()`、`join()` 等可中断的阻塞方法上等待时，如果被 `interrupt()`，会：

1. **清除中断标志**
2. **抛出 `InterruptedException`**

```java
Thread t = new Thread(() -> {
    try {
        Thread.sleep(10000); // 被中断时抛出 InterruptedException
    } catch (InterruptedException e) {
        // 此时中断标志已被清除！
        System.out.println(Thread.currentThread().isInterrupted()); // false

        // 正确做法：恢复中断标志
        Thread.currentThread().interrupt();
        // 或: 处理中断后退出
    }
});
t.start();
Thread.sleep(100);
t.interrupt();
```

### 5.4 为什么不能强制停止线程？

强制停止线程（如 `Thread.stop()`）会导致：

1. **数据不一致**：线程可能在写操作进行到一半时被停止。
2. **锁不释放**：持有的锁不会正常释放，导致死锁。
3. **资源泄漏**：打开的文件、连接等不会被正确关闭。

> `Thread.stop()`、`Thread.suspend()`、`Thread.resume()` 已被废弃，永远不要使用。

### 5.5 正确处理 InterruptedException 的原则

```java
// ❌ 错误1：吞掉异常
try {
    Thread.sleep(1000);
} catch (InterruptedException e) {
    // 什么都不做 → 中断信号丢失，上层调用者无法感知
}

// ❌ 错误2：只打印日志
try {
    Thread.sleep(1000);
} catch (InterruptedException e) {
    e.printStackTrace(); // 中断信号仍丢失
}

// ✅ 正确1：恢复中断标志
try {
    Thread.sleep(1000);
} catch (InterruptedException e) {
    Thread.currentThread().interrupt(); // 恢复中断标志
}

// ✅ 正确2：向上抛出
public void doWork() throws InterruptedException {
    Thread.sleep(1000); // 让调用者决定如何处理中断
}

// ✅ 正确3：退出循环
while (!Thread.currentThread().isInterrupted()) {
    try {
        Thread.sleep(1000);
        doWork();
    } catch (InterruptedException e) {
        break; // 退出循环
    }
}
```

## 6. sleep() vs wait()

### 6.1 对比表

| 维度 | sleep() | wait() |
|---|---|---|
| 所属类 | Thread | Object |
| 是否释放锁 | **否** | **是** |
| 使用位置 | 任意 | synchronized 块内 |
| 唤醒方式 | 超时 / interrupt | notify / notifyAll / 超时 / interrupt |
| 用途 | 暂停当前线程 | 线程间通信/协作 |
| 状态变化 | TIMED_WAITING | WAITING / TIMED_WAITING |

### 6.2 sleep() 源码

```java
public static void sleep(long millis) throws InterruptedException {
    if (millis < 0) throw new IllegalArgumentException("timeout value is negative");
    sleep(millis, 0);
}

public static void sleep(long millis, int nanos) throws InterruptedException {
    if (millis < 0) throw new IllegalArgumentException("timeout value is negative");
    if (nanos < 0 || nanos > 999999) throw new IllegalArgumentException("nanosecond timeout value out of range");
    if (nanos > 0 && millis < Long.MAX_VALUE) millis++;

    // 调用 native 方法
    sleep0(millis);
}

private static native void sleep0(long millis) throws InterruptedException;
```

### 6.3 sleep 不释放锁的证明

```java
Object lock = new Object();

Thread t1 = new Thread(() -> {
    synchronized (lock) {
        System.out.println("t1 获取锁");
        try {
            Thread.sleep(5000); // sleep 不释放锁
        } catch (InterruptedException e) {}
        System.out.println("t1 释放锁");
    }
});

Thread t2 = new Thread(() -> {
    try { Thread.sleep(1000); } catch (Exception e) {}
    synchronized (lock) {
        System.out.println("t2 获取锁"); // 必须等 t1 执行完才能获取
    }
});

t1.start();
t2.start();
// t1 获取锁 → (5秒后) t1 释放锁 → t2 获取锁
```

### 6.4 wait 释放锁的证明

```java
Object lock = new Object();

Thread t1 = new Thread(() -> {
    synchronized (lock) {
        System.out.println("t1 获取锁");
        try {
            lock.wait(5000); // wait 释放锁
        } catch (InterruptedException e) {}
        System.out.println("t1 重新获取锁并执行完");
    }
});

Thread t2 = new Thread(() -> {
    try { Thread.sleep(1000); } catch (Exception e) {}
    synchronized (lock) {
        System.out.println("t2 获取锁"); // t1 wait 后 t2 立即获取锁
    }
});

t1.start();
t2.start();
// t1 获取锁 → (1秒后) t2 获取锁 → (5秒后) t1 重新获取锁并执行完
```

### 6.5 TimeUnit 的便捷方法

```java
// 推荐：使用 TimeUnit，可读性更好
TimeUnit.SECONDS.sleep(1);      // 等价于 Thread.sleep(1000)
TimeUnit.MINUTES.sleep(1);      // 等价于 Thread.sleep(60000)
TimeUnit.MILLISECONDS.sleep(500); // 等价于 Thread.sleep(500)
```

## 7. 线程优先级

### 7.1 源码

```java
// 优先级范围
public static final int MIN_PRIORITY = 1;
public static final int NORM_PRIORITY = 5;  // 默认优先级
public static final int MAX_PRIORITY = 10;

public final void setPriority(int newPriority) {
    ThreadGroup g;
    checkAccess();
    if (newPriority > MAX_PRIORITY || newPriority < MIN_PRIORITY) {
        throw new IllegalArgumentException();
    }
    if ((g = getThreadGroup()) != null) {
        // 优先级不能超过线程组的最大优先级
        if (newPriority > g.getMaxPriority()) {
            newPriority = g.getMaxPriority();
        }
        setPriority0(priority = newPriority); // native 方法
    }
}

public final int getPriority() {
    return priority;
}
```

### 7.2 优先级的实际效果

```java
Thread highPriority = new Thread(() -> { /* ... */ });
highPriority.setPriority(Thread.MAX_PRIORITY); // 10

Thread lowPriority = new Thread(() -> { /* ... */ });
lowPriority.setPriority(Thread.MIN_PRIORITY); // 1
```

> **注意**：线程优先级映射到操作系统优先级的方式因平台而异。Java 的 1~10 会被映射到操作系统的优先级范围。**优先级不保证执行顺序**，只是给操作系统的建议。在某些操作系统上（如 Linux），优先级可能被忽略。

### 7.3 优先级继承

```java
// 新创建的线程默认继承创建它的线程的优先级
Thread.currentThread().setPriority(8);
Thread t = new Thread(() -> {});
System.out.println(t.getPriority()); // 8（继承）

// 但如果父线程优先级超过线程组最大优先级，会被截断
```

## 8. 守护线程

### 8.1 源码

```java
public final void setDaemon(boolean on) {
    checkAccess();
    // 只能在线程启动前设置
    if (isAlive()) {
        throw new IllegalThreadStateException();
    }
    daemon = on;
}

public final boolean isDaemon() {
    return daemon;
}
```

### 8.2 守护线程的特点

```java
// 用户线程 vs 守护线程
Thread userThread = new Thread(() -> {
    while (true) {
        try { Thread.sleep(1000); } catch (Exception e) {}
        System.out.println("用户线程运行中...");
    }
});

Thread daemonThread = new Thread(() -> {
    while (true) {
        try { Thread.sleep(1000); } catch (Exception e) {}
        System.out.println("守护线程运行中...");
    }
});
daemonThread.setDaemon(true); // 必须在 start() 前设置

userThread.start();
daemonThread.start();

// 当 userThread 被中断/结束时，JVM 发现没有用户线程了，
// 会直接退出，守护线程会被强制终止（finally 块不保证执行）
```

### 8.3 守护线程 vs 用户线程

| 维度 | 用户线程 | 守护线程 |
|---|---|---|
| JVM 退出 | 所有用户线程结束后才退出 | 不影响 JVM 退出 |
| 设置时机 | 任意（但实际建议 start 前） | 必须在 start() 前 |
| 典型用途 | 业务线程 | GC 线程、Finalizer 线程、心跳检测 |
| finally 执行 | 保证执行 | 不保证执行 |
| 子线程继承 | 不继承 | 守护线程创建的子线程默认也是守护线程 |

### 8.4 守护线程的陷阱

```java
Thread daemon = new Thread(() -> {
    try {
        // 执行一些重要的清理工作
        doImportantWork();
    } finally {
        // ❌ 守护线程的 finally 不保证执行！
        cleanupResources();
    }
});
daemon.setDaemon(true);
daemon.start();

// 当主线程结束时，JVM 退出，daemon 被强制终止
// finally 块可能不会执行！
```

> **最佳实践**：不要在守护线程中执行需要可靠完成的操作（如文件写入、数据库提交）。守护线程适合执行"可中断"的后台任务（如日志缓冲刷新、心跳检测）。

### 8.5 GC 线程就是守护线程

```java
// 查看所有线程
ThreadMXBean bean = ManagementFactory.getThreadMXBean();
for (ThreadInfo info : bean.dumpAllThreads(false, false)) {
    System.out.println(info.getThreadName() + " daemon=" + info.isDaemon());
}
// 输出示例:
// main daemon=false
// Reference Handler daemon=true
// Finalizer daemon=true
// Signal Dispatcher daemon=true
// Common-Cleaner daemon=true
// G1 GC 线程 daemon=true (JDK 9+)
```

## 9. 线程未捕获异常处理

### 9.1 默认行为

当线程抛出未捕获的异常时，默认行为是打印堆栈跟踪并终止线程。

### 9.2 UncaughtExceptionHandler

```java
@FunctionalInterface
public interface UncaughtExceptionHandler {
    void uncaughtException(Thread t, Throwable e);
}

// Thread 类中的相关字段和方法
private volatile UncaughtExceptionHandler uncaughtExceptionHandler;
private static volatile UncaughtExceptionHandler defaultUncaughtExceptionHandler;

// 设置某个线程的异常处理器
public void setUncaughtExceptionHandler(UncaughtExceptionHandler eh) {
    checkAccess();
    uncaughtExceptionHandler = eh;
}

// 设置全局默认异常处理器
public static void setDefaultUncaughtExceptionHandler(UncaughtExceptionHandler eh) {
    checkAccess();
    defaultUncaughtExceptionHandler = eh;
}

// 分发异常处理的逻辑
private void dispatchUncaughtException(Throwable e) {
    getUncaughtExceptionHandler().uncaughtException(this, e);
}

public UncaughtExceptionHandler getUncaughtExceptionHandler() {
    return uncaughtExceptionHandler != null ?
        uncaughtExceptionHandler : group; // 默认委托给 ThreadGroup
}
```

### 9.3 使用示例

```java
// 设置全局默认处理器
Thread.setDefaultUncaughtExceptionHandler((t, e) -> {
    System.err.println("线程 " + t.getName() + " 发生异常: " + e.getMessage());
    // 记录日志、发送告警等
});

// 设置单个线程的处理器
Thread t = new Thread(() -> {
    throw new RuntimeException("出错了！");
});
t.setUncaughtExceptionHandler((thread, ex) -> {
    System.err.println(thread.getName() + ": " + ex.getMessage());
});
t.start();
// 输出: Thread-0: 出错了！

// ThreadGroup 作为默认处理器
// ThreadGroup 实现了 UncaughtExceptionHandler 接口
// 默认行为: 打印异常堆栈
```

### 9.4 线程池中的异常处理

```java
// 线程池中 submit() 会吞掉异常（封装在 Future 中）
ExecutorService pool = Executors.newFixedThreadPool(1);
Future<?> future = pool.submit(() -> {
    throw new RuntimeException("submit 异常");
});
try {
    future.get(); // 异常在这里抛出 ExecutionException
} catch (ExecutionException e) {
    System.out.println(e.getCause().getMessage()); // submit 异常
}

// 线程池中 execute() 不会吞掉异常
pool.execute(() -> {
    throw new RuntimeException("execute 异常"); // 触发 UncaughtExceptionHandler
});
```

## 10. 面试题速查

### Q1: Java 线程有几种状态？

6 种：NEW（新建）、RUNNABLE（可运行）、BLOCKED（阻塞等待锁）、WAITING（无限等待）、TIMED_WAITING（限时等待）、TERMINATED（终止）。

### Q2: start() 和 run() 的区别？

`start()` 创建新的操作系统线程并执行 `run()`，`run()` 只是普通方法调用，在调用者线程中执行。`start()` 只能调用一次，`run()` 可以多次调用。

### Q3: 如何正确停止一个线程？

使用 `interrupt()` 设置中断标志，线程在循环中检查 `isInterrupted()` 并自行退出。对于阻塞方法（sleep/wait/join），捕获 `InterruptedException` 后恢复中断标志或退出。

### Q4: sleep() 和 wait() 的区别？

`sleep()` 是 Thread 的静态方法，不释放锁，用于暂停当前线程；`wait()` 是 Object 的方法，释放锁，用于线程间通信，必须在 synchronized 块中调用。

### Q5: 为什么 wait/notify 定义在 Object 而不是 Thread？

Java 的锁是基于对象的（每个对象都有 Monitor），任意对象都可以作为锁。wait/notify 操作的是对象的 Monitor 等待队列，所以定义在 Object 上。

### Q6: 守护线程和用户线程的区别？

JVM 等待所有用户线程结束后才退出，守护线程不影响 JVM 退出。守护线程的 finally 块不保证执行。GC、Finalizer 等都是守护线程。

### Q7: interrupt() 会立即停止线程吗？

不会。`interrupt()` 只是设置中断标志位，是协作式中断。线程需要自行检查标志并决定如何响应。阻塞方法（sleep/wait/join）被中断时会抛出 InterruptedException。

### Q8: 线程优先级有保证吗？

没有。线程优先级只是给操作系统的建议，实际调度由操作系统决定。不同平台映射不同，有些平台可能忽略优先级。不应依赖优先级来保证执行顺序。

### Q9: join() 的原理是什么？

`join()` 的本质是 `wait()`。当前线程在目标 Thread 对象上 wait，目标线程结束时 JVM 自动调用 `notifyAll()` 唤醒等待的线程。

### Q10: 线程池中 submit() 和 execute() 的异常处理有什么不同？

`execute()` 抛出未捕获异常会触发 `UncaughtExceptionHandler`；`submit()` 将异常封装在 `Future` 中，调用 `future.get()` 时抛出 `ExecutionException`。

### Q11: 一个线程两次调用 start() 会怎样？

抛出 `IllegalThreadStateException`。`start()` 会检查线程状态，只有 `NEW` 状态才能调用。

### Q12: BLOCKED 和 WAITING 有什么区别？

`BLOCKED` 是等待 synchronized 锁（被动阻塞），`WAITING` 是主动调用 wait/park 进入等待。等待 `ReentrantLock` 时是 `WAITING`（基于 LockSupport.park），不是 `BLOCKED`。
