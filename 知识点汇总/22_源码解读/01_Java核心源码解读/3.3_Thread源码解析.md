## 1. Thread类概述

### 1.1 Thread类定义

```java
public class Thread implements Runnable {

    // 线程名称
    private volatile String name;

    // 线程优先级（1-10）
    private int priority;

    // 是否为守护线程
    private boolean daemon = false;

    // Runnable目标
    private Runnable target;

    // 线程组
    private ThreadGroup group;

    // 线程ID
    private final long tid;

    // 线程状态
    private volatile int threadStatus;

    // 未捕获异常处理器
    private volatile UncaughtExceptionHandler uncaughtExceptionHandler;

    // 线程局部变量
    ThreadLocal.ThreadLocalMap threadLocals;
    ThreadLocal.ThreadLocalMap inheritableThreadLocals;
}
```

### 1.2 创建线程的方式

```java
// 方式1：继承Thread
class MyThread extends Thread {
    @Override
    public void run() {
        System.out.println("Thread: " + getName());
    }
}
new MyThread().start();

// 方式2：实现Runnable
class MyTask implements Runnable {
    @Override
    public void run() {
        System.out.println("Runnable: " + Thread.currentThread().getName());
    }
}
new Thread(new MyTask()).start();

// 方式3：Lambda表达式
new Thread(() -> System.out.println("Lambda")).start();

// 方式4：Callable + FutureTask
FutureTask<String> future = new FutureTask<>(() -> {
    return "result";
});
new Thread(future).start();
String result = future.get();

// 方式5：线程池（推荐的生产环境方式）
ExecutorService pool = Executors.newFixedThreadPool(4);
pool.submit(() -> System.out.println("Pool thread"));
pool.shutdown();
```

### 1.3 Thread与Runnable的关系

```java
// Thread类本身就实现了Runnable
public class Thread implements Runnable {
    private Runnable target;  // 持有另一个Runnable

    @Override
    public void run() {
        if (target != null) {
            target.run();  // 委托给target执行
        }
    }
}

// 继承Thread重写run() vs 传入Runnable
// 本质都是覆盖run()方法的行为
// 传入Runnable更灵活——不占用继承位
```

## 2. 线程状态转换

### 2.1 六种线程状态

```java
public enum State {
    NEW,           // 新建：已创建但未start
    RUNNABLE,      // 可运行：正在JVM中执行（可能等待OS调度）
    BLOCKED,       // 阻塞：等待监视器锁（synchronized）
    WAITING,       // 等待：无限期等待（wait/join/LockSupport.park）
    TIMED_WAITING, // 限时等待：sleep/wait(timeout)/join(timeout)
    TERMINATED     // 终止：run方法执行完毕
}
```

### 2.2 状态转换图

```
                    start()
    NEW ───────────────────────→ RUNNABLE
                                    │
                          ┌─────────┼─────────┐
                          ↓         │         ↓
                     BLOCKED        │   TERMINATED
                  (等synchronized)  │    (run结束)
                          │         │         ↑
                          ↓         │         │
                      RUNNABLE      │    RUNNABLE
                                    │         ↑
                          ┌─────────┼─────────┤
                          ↓         │         │
                      WAITING       │   TIMED_WAITING
                   (wait/join/park)  │  (sleep/wait(t)/join(t))
                          │         │         │
                          ↓         │         ↓
                      RUNNABLE ←────┴─────────┘
```

### 2.3 各状态的触发条件

```java
// NEW → RUNNABLE
Thread t = new Thread(); // NEW
t.start();               // RUNNABLE

// RUNNABLE → BLOCKED
synchronized (lock) {
    // 如果锁被其他线程持有，当前线程进入BLOCKED
}

// RUNNABLE → WAITING
Object.wait();      // 无限等待
Thread.join();      // 等待线程结束
LockSupport.park(); // 无限等待

// RUNNABLE → TIMED_WAITING
Thread.sleep(1000);     // 限时睡眠
Object.wait(1000);      // 限时等待
Thread.join(1000);      // 限时等待线程结束

// 任何状态 → TERMINATED
// run()方法执行完毕或抛出未捕获异常
```

### 2.4 BLOCKED vs WAITING的区别

```java
// BLOCKED：竞争synchronized锁失败
// 线程在等EntryList排队，不响应interrupt
synchronized (lock) {
    // 如果lock被占用，当前线程BLOCKED
}

// WAITING：主动调用wait/join/park
// 线程在WaitSet中等待，可以被interrupt唤醒
synchronized (lock) {
    lock.wait(); // 释放锁，进入WAITING
}

// 关键区别：
// BLOCKED是被动等待锁，WAITING是主动等待通知
// BLOCKED不释放锁，WAITING（wait()）释放锁
// BLOCKED只发生在synchronized，WAITING发生在wait/join/park
```

### 2.5 获取和监控线程状态

```java
Thread t = new Thread(() -> {
    try {
        Thread.sleep(1000);
    } catch (InterruptedException e) {}
});

System.out.println(t.getState()); // NEW
t.start();
System.out.println(t.getState()); // RUNNABLE

// 等待线程进入TIMED_WAITING
Thread.sleep(100);
System.out.println(t.getState()); // TIMED_WAITING

t.join();
System.out.println(t.getState()); // TERMINATED

// 线程dump（获取所有线程状态）
Map<Thread, StackTraceElement[]> allStackTraces = Thread.getAllStackTraces();
for (Map.Entry<Thread, StackTraceElement[]> entry : allStackTraces.entrySet()) {
    System.out.println(entry.getKey().getName() + ": " + entry.getKey().getState());
}
```

## 3. start() vs run()

### 3.1 start()源码

```java
/**
 * 使线程开始执行。JVM将调用此线程的run方法。
 * 结果是两个线程并发执行：当前线程（start()返回后继续）和另一个线程（执行run()）。
 *
 * 一个线程不能被启动两次。线程启动后不能重新启动。
 */
public synchronized void start() {
    // 1. 检查线程状态（0=NEW）
    if (threadStatus != 0)
        throw new IllegalThreadStateException();

    // 2. 通知线程组
    group.add(this);

    boolean started = false;
    try {
        // 3. 调用native方法创建并启动OS线程
        start0();
        started = true;
    } finally {
        try {
            if (!started) {
                group.threadStartFailed(this);
            }
        } catch (Throwable ignore) {}
    }
}

// native方法——由JVM实现，创建OS线程
private native void start0();
```

### 3.2 run()源码

```java
/**
 * 如果线程使用Runnable构造，则调用Runnable的run方法。
 * 否则此方法什么都不做（子类应重写此方法）。
 */
@Override
public void run() {
    if (target != null) {
        target.run();
    }
}
```

### 3.3 核心区别

```java
// start()：创建新的OS线程，在新线程中执行run()
Thread t = new Thread(() -> {
    System.out.println("Thread: " + Thread.currentThread().getName());
});

t.start(); // 创建新线程"Thread-0"，在新线程中执行
// 输出: Thread: Thread-0

// run()：在当前线程中执行，不创建新线程
t.run(); // 在main线程中执行
// 输出: Thread: main
```

### 3.4 start()只能调用一次

```java
Thread t = new Thread(() -> {});
t.start(); // OK
t.start(); // 抛出 IllegalThreadStateException
// threadStatus已经不是0（NEW）了

// run()可以多次调用（但通常不需要）
Thread t2 = new Thread(() -> System.out.println("run"));
t2.run(); // OK
t2.run(); // OK——只是普通方法调用
```

### 3.5 start0()的JVM实现

```c
// OpenJDK中 start0 → JVM_StartThread 的简化逻辑

JVM_ENTRY(void, JVM_StartThread(JNIEnv* env, jobject jthread))
    // 1. 创建JavaThread对象
    JavaThread* native_thread = new JavaThread(&thread_entry, sz);

    // 2. 创建OS线程（pthread_create）
    // 这一步真正创建了操作系统级别的线程

    // 3. 设置线程状态
    java_lang_Thread::set_thread_status(jthread, RUNNABLE);

    // 4. 调用Thread.run()
    // OS线程的入口函数最终会调用Java层面的run()
JVM_END

// thread_entry —— OS线程的入口函数
static void thread_entry(JavaThread* thread, TRAPS) {
    // 获取Java Thread对象
    Handle obj(THREAD, thread->threadObj());
    // 调用 run() 方法
    JavaValue result(T_VOID);
    JavaCalls::call_virtual(&result, obj,
        SystemDictionary::Thread_klass(),
        vmSymbols::run_method_name(),
        vmSymbols::void_method_signature(),
        THREAD);
}
```

## 4. interrupt机制

### 4.1 interrupt相关方法

```java
/**
 * 中断此线程。
 * 如果线程在wait/join/sleep中，会抛出InterruptedException。
 * 否则只是设置中断标志位。
 */
public void interrupt() {
    if (this != Thread.currentThread())
        checkAccess();  // 安全检查

    synchronized (blockerLock) {
        Interruptible b = blocker;
        if (b != null) {
            interrupt0();  // 设置中断标志
            b.interrupt(this);
            return;
        }
    }
    interrupt0();  // native方法——设置中断标志
}

/**
 * 检查此线程是否被中断。会清除中断标志。
 */
public static boolean interrupted() {
    return currentThread().isInterrupted(true); // true=清除标志
}

/**
 * 检查此线程是否被中断。不会清除中断标志。
 */
public boolean isInterrupted() {
    return isInterrupted(false); // false=不清除标志
}

// native方法
private native boolean isInterrupted(boolean ClearInterrupted);
```

### 4.2 interrupt的工作机制

```java
// interrupt只是设置标志位，不是强制停止线程！
Thread t = new Thread(() -> {
    // 如果不检查中断标志，线程不会停下来
    while (true) {
        // 一直运行——interrupt只是设置了标志位
    }
});
t.start();
t.interrupt(); // 线程不会停下来！
```

### 4.3 正确处理中断

```java
// 方式1：主动检查中断标志
Thread t = new Thread(() -> {
    while (!Thread.currentThread().isInterrupted()) {
        // 执行任务
        doWork();
    }
    // 检测到中断，优雅退出
    cleanup();
});

// 方式2：响应InterruptedException
Thread t2 = new Thread(() -> {
    try {
        while (true) {
            // sleep/wait/join会检查中断标志
            Thread.sleep(1000); // 如果被interrupt，抛出InterruptedException
            doWork();
        }
    } catch (InterruptedException e) {
        // InterruptedException会清除中断标志！
        // 线程需要决定如何响应中断
        System.out.println("线程被中断，退出");
        cleanup();
    }
});

// 方式3：重新设置中断标志（当不能抛出InterruptedException时）
public void run() {
    while (!Thread.currentThread().isInterrupted()) {
        try {
            Thread.sleep(1000);
            doWork();
        } catch (InterruptedException e) {
            // sleep抛出InterruptedException时，中断标志已被清除
            // 需要重新设置，否则循环不会退出
            Thread.currentThread().interrupt();
        }
    }
}
```

### 4.4 interrupted() vs isInterrupted()

```java
// static interrupted()：检查当前线程并清除标志
Thread.currentThread().interrupt();
System.out.println(Thread.interrupted()); // true（并清除标志）
System.out.println(Thread.interrupted()); // false（标志已被清除）

// isInterrupted()：检查指定线程但不清除标志
Thread t = new Thread(() -> {
    while (!Thread.currentThread().isInterrupted()) {
        // 循环
    }
});
t.start();
t.interrupt();
System.out.println(t.isInterrupted()); // true
System.out.println(t.isInterrupted()); // true（没有被清除）
```

### 4.5 interrupt与synchronized

```java
// interrupt不能中断等待synchronized锁的线程（BLOCKED状态）
Object lock = new Object();

Thread t1 = new Thread(() -> {
    synchronized (lock) {
        try { Thread.sleep(10000); } catch (Exception e) {}
    }
});

Thread t2 = new Thread(() -> {
    synchronized (lock) {
        // t2在等t1释放lock，处于BLOCKED状态
    }
});

t1.start();
Thread.sleep(100);
t2.start();
t2.interrupt(); // t2不会响应！仍处于BLOCKED等待锁

// ReentrantLock.lock()也不响应interrupt
// 但 ReentrantLock.lockInterruptibly() 可以响应！
ReentrantLock reentrantLock = new ReentrantLock();

Thread t3 = new Thread(() -> {
    try {
        reentrantLock.lockInterruptibly(); // 可被interrupt中断
        // ...
    } catch (InterruptedException e) {
        System.out.println("被中断");
    }
});
```

## 5. join()原理

### 5.1 join()源码

```java
/**
 * 等待此线程终止，最多等待millis毫秒。
 */
public final synchronized void join(long millis)
        throws InterruptedException {

    long base = System.currentTimeMillis();
    long now = 0;

    if (millis < 0) {
        throw new IllegalArgumentException("timeout value is negative");
    }

    if (millis == 0) {
        // 无限等待，直到线程终止
        while (isAlive()) {
            wait(0);  // 调用Object.wait()！
        }
    } else {
        // 限时等待
        while (isAlive()) {
            long delay = millis - now;
            if (delay <= 0) {
                break;  // 超时
            }
            wait(delay);
            now = System.currentTimeMillis() - base;
        }
    }
}

public final void join() throws InterruptedException {
    join(0); // 无限等待
}
```

### 5.2 join()的本质

```java
// join()底层就是wait()！
// 当线程终止时，JVM会自动调用notifyAll()

// 等价代码：
public void join() throws InterruptedException {
    synchronized (this) {  // 以Thread对象本身作为锁
        while (this.isAlive()) {
            this.wait();   // 等待线程终止
        }
    }
}

// JVM在线程终止时的回调（C++层面）
// 当线程的run()方法结束（正常或异常），JVM会调用：
void JavaThread::exit(bool destroy_vm, ExitType exit_type) {
    // ... 清理资源 ...
    // 通知所有等待此线程的线程
    ensure_join(this);
}

static void ensure_join(JavaThread* thread) {
    Handle threadObj(thread, thread->threadObj());
    ObjectLocker lock(threadObj, thread);
    // 设置线程状态为TERMINATED
    java_lang_Thread::set_thread_status(threadObj(), TERMINATED);
    // 清除thread字段
    java_lang_Thread::set_thread(threadObj(), NULL);
    // 调用notifyAll——唤醒所有join等待的线程
    lock.notify_all(thread);
}
```

### 5.3 join()使用示例

```java
Thread t1 = new Thread(() -> {
    try { Thread.sleep(2000); } catch (Exception e) {}
    System.out.println("t1完成");
});

Thread t2 = new Thread(() -> {
    try { Thread.sleep(1000); } catch (Exception e) {}
    System.out.println("t2完成");
});

t1.start();
t2.start();

// main线程等待t1和t2都完成
t1.join(); // main线程阻塞，直到t1终止
t2.join(); // main线程阻塞，直到t2终止

System.out.println("所有线程完成");

// 输出顺序：
// t2完成（1秒后）
// t1完成（2秒后）
// 所有线程完成
```

### 5.4 join(timeout)

```java
Thread t = new Thread(() -> {
    try { Thread.sleep(10000); } catch (Exception e) {}
});

t.start();
t.join(3000); // 最多等3秒

if (t.isAlive()) {
    System.out.println("超时了，线程还在运行");
    t.interrupt(); // 可选：中断线程
} else {
    System.out.println("线程已完成");
}
```

## 6. sleep()与yield()

### 6.1 sleep()源码

```java
/**
 * 使当前线程暂停执行指定的毫秒数。
 * 线程不会丢失任何监视器的所有权（不释放锁）。
 */
public static void sleep(long millis) throws InterruptedException {
    if (millis < 0) {
        throw new IllegalArgumentException("timeout value is negative");
    }
    // 调用native方法
    sleep(millis, 0);
}

public static void sleep(long millis, int nanos) throws InterruptedException {
    if (millis < 0) {
        throw new IllegalArgumentException("timeout value is negative");
    }
    if (nanos < 0 || nanos > 999999) {
        throw new IllegalArgumentException("nanosecond timeout value out of range");
    }
    if (nanos > 0 && millis < Long.MAX_VALUE) {
        millis++;
    }
    sleep0(millis); // native方法
}

private static native void sleep0(long millis);
```

### 6.2 sleep vs wait

```java
// sleep：Thread的静态方法，不释放锁
synchronized (lock) {
    Thread.sleep(1000); // 持有lock，其他线程拿不到锁
}

// wait：Object的方法，释放锁
synchronized (lock) {
    lock.wait(1000); // 释放lock，其他线程可以获取锁
}
```

| 特性 | sleep() | wait() |
|------|---------|--------|
| 所属类 | Thread | Object |
| 释放锁 | 否 | 是 |
| 使用位置 | 任意 | synchronized块内 |
| 唤醒方式 | 超时 | notify/notifyAll/超时 |
| 抛出异常 | InterruptedException | InterruptedException |

### 6.3 yield()源码

```java
/**
 * 提示调度器当前线程愿意让出CPU。
 * 调度器可以忽略此提示。
 */
public static native void yield();
```

yield()只是一个**提示**，JVM和OS可以忽略它。它将线程从RUNNING状态转为RUNNABLE状态，让出CPU给同优先级或更高优先级的线程。

### 6.4 TimeUnit——更优雅的sleep

```java
// 不优雅
Thread.sleep(5000); // 5000毫秒是多少秒？不直观

// 优雅
TimeUnit.SECONDS.sleep(5);
TimeUnit.MINUTES.sleep(2);
TimeUnit.MILLISECONDS.sleep(500);
```

## 7. 线程优先级与守护线程

### 7.1 线程优先级

```java
// 线程优先级1-10，默认5
public static final int MIN_PRIORITY = 1;
public static final int NORM_PRIORITY = 5;
public static final int MAX_PRIORITY = 10;

// 设置优先级
Thread t = new Thread(() -> {});
t.setPriority(Thread.MAX_PRIORITY); // 10
t.setPriority(Thread.MIN_PRIORITY); // 1

// 优先级继承
// 子线程默认继承父线程的优先级
Thread.currentThread().setPriority(8);
Thread child = new Thread(() -> {
    System.out.println(Thread.currentThread().getPriority()); // 8
});
child.start();
```

> ⚠️ **注意**：线程优先级依赖于OS调度器，不同平台行为不同。不能依赖优先级保证执行顺序。

### 7.2 守护线程

```java
/**
 * 将此线程标记为守护线程或用户线程。
 * 当JVM中只剩守护线程时，JVM退出。
 * 必须在线程启动前调用。
 */
public final void setDaemon(boolean on) {
    if (isAlive()) {
        throw new IllegalThreadStateException();
    }
    daemon = on;
}

public final boolean isDaemon() {
    return daemon;
}
```

```java
// 守护线程示例
Thread daemon = new Thread(() -> {
    while (true) {
        try {
            Thread.sleep(1000);
            System.out.println("守护线程运行中...");
        } catch (InterruptedException e) {
            break;
        }
    }
});
daemon.setDaemon(true); // 必须在start()前设置
daemon.start();

// main线程结束后，JVM检查只剩守护线程，直接退出
// 守护线程的finally块不一定会执行！
```

### 7.3 守护线程的注意事项

```java
// 1. 守护线程的finally不保证执行
Thread daemon = new Thread(() -> {
    try {
        Thread.sleep(10000);
    } catch (InterruptedException e) {
    } finally {
        // JVM退出时不保证执行！
        System.out.println("cleanup");
    }
});
daemon.setDaemon(true);
daemon.start();
// main线程结束后JVM直接退出，finally可能不执行

// 2. 守护线程创建的子线程默认也是守护线程
Thread t = new Thread(() -> {
    Thread child = Thread.currentThread();
    System.out.println("child isDaemon: " + child.isDaemon()); // true
});
t.setDaemon(true);
t.start();

// 3. 守护线程不适合做IO/资源清理
// 因为JVM可能在任何时候退出
// 适合的场景：GC线程、心跳检测、定时清理
```

### 8.2 为什么需要上下文类加载器

```
问题场景：
    BootstrapClassLoader（JVM内置）
        ↓ 委托
    AppClassLoader（应用类加载器）
        ↓ 加载
    用户代码
        ↓ 调用
    JDBC API（由Bootstrap加载）
        ↓ 需要
    JDBC Driver实现（由App加载）

    JDBC API由BootstrapClassLoader加载，看不到用户classpath上的Driver实现
    解决方案：JDBC API通过Thread.currentThread().getContextClassLoader()
    获取AppClassLoader，用它来加载Driver实现
```

## 9. UncaughtExceptionHandler

### 9.1 线程未捕获异常处理

```java
// 当线程的run()方法抛出未捕获异常时，JVM会调用UncaughtExceptionHandler

@FunctionalInterface
public interface UncaughtExceptionHandler {
    void uncaughtException(Thread t, Throwable e);
}

// 设置方式
Thread t = new Thread(() -> {
    throw new RuntimeException("线程异常");
});

t.setUncaughtExceptionHandler((thread, ex) -> {
    System.out.println("线程 " + thread.getName() + " 异常: " + ex.getMessage());
});

t.start();
// 输出: 线程 Thread-0 异常: 线程异常
```

### 9.2 全局默认处理器

```java
// 设置所有线程的默认异常处理器
Thread.setDefaultUncaughtExceptionHandler((thread, ex) -> {
    System.err.println("全局异常处理: " + thread.getName());
    ex.printStackTrace();
    // 发送告警、记录日志等
});

// 之后创建的线程如果没有单独设置handler，就使用这个默认的
new Thread(() -> {
    throw new RuntimeException("oops");
}).start();
// 输出: 全局异常处理: Thread-0
```

### 9.3 线程池中的异常处理

```java
// 线程池中submit()的异常被Future.get()封装
ExecutorService pool = Executors.newFixedThreadPool(1);
Future<?> future = pool.submit(() -> {
    throw new RuntimeException("task error");
});

try {
    future.get(); // 抛出 ExecutionException 包装的 RuntimeException
} catch (ExecutionException e) {
    System.out.println(e.getCause().getMessage()); // task error
}

// execute()的异常会触发UncaughtExceptionHandler
pool.execute(() -> {
    throw new RuntimeException("execute error");
});
// 如果没有设置handler，异常直接打到stderr

// 自定义ThreadPoolExecutor重写afterExecute
class MyThreadPool extends ThreadPoolExecutor {
    @Override
    protected void afterExecute(Runnable r, Throwable t) {
        super.afterExecute(r, t);
        if (t != null) {
            System.err.println("任务异常: " + t.getMessage());
        }
        // submit的情况需要额外处理
        if (t == null && r instanceof Future<?>) {
            try {
                ((Future<?>) r).get();
            } catch (Exception e) {
                System.err.println("Future任务异常: " + e.getCause());
            }
        }
    }
}
```

## 10. 虚拟线程（Java 21）

### 10.1 虚拟线程概述

```java
// Java 21引入虚拟线程（Project Loom）
// 虚拟线程是用户态线程，由JVM调度，不是OS线程

// 创建虚拟线程
Thread vThread = Thread.ofVirtual().start(() -> {
    System.out.println("Virtual thread: " + Thread.currentThread());
});

// 虚拟线程工厂
ThreadFactory factory = Thread.ofVirtual().factory();

// 虚拟线程执行器
try (var executor = Executors.newVirtualThreadPerTaskExecutor()) {
    for (int i = 0; i < 10000; i++) {
        executor.submit(() -> {
            // 可以创建成千上万个虚拟线程
            Thread.sleep(Duration.ofSeconds(1));
            return "done";
        });
    }
}
```

### 10.2 虚拟线程 vs 平台线程

```java
// 平台线程（传统Thread）
Thread platformThread = new Thread(() -> {});
platformThread.start();
// 1:1映射到OS线程，创建成本高，数量有限（通常几千）

// 虚拟线程
Thread virtualThread = Thread.ofVirtual().unstarted(() -> {});
virtualThread.start();
// N:M映射，多个虚拟线程调度到少量载体线程（平台线程）
// 创建成本极低，可以创建数百万个

// 判断是否为虚拟线程
System.out.println(Thread.currentThread().isVirtual()); // false（main线程）
```

### 10.3 虚拟线程的适用场景

```java
// 虚拟线程适合IO密集型任务
// 不适合CPU密集型任务

// 场景1：高并发HTTP请求
try (var executor = Executors.newVirtualThreadPerTaskExecutor()) {
    List<Future<String>> futures = urls.stream()
        .map(url -> executor.submit(() -> fetchUrl(url)))
        .collect(Collectors.toList());

    for (Future<String> f : futures) {
        System.out.println(f.get());
    }
}
// 可以同时发起数万个HTTP请求

// 场景2：数据库查询
try (var executor = Executors.newVirtualThreadPerTaskExecutor()) {
    List<Future<List<User>>> futures = userIds.stream()
        .map(id -> executor.submit(() -> queryUser(id)))
        .collect(Collectors.toList());
    // ...
}

// 虚拟线程的注意事项：
// 1. 不要池化虚拟线程——用完即弃
// 2. 避免在synchronized块中执行IO——会钉住载体线程
// 3. 使用ReentrantLock替代synchronized
```

## 11. 面试题速查

### Q1: start()和run()的区别？

start()创建新的OS线程并在新线程中执行run()；run()只是普通方法调用，在当前线程执行。start()只能调用一次（再次调用抛IllegalThreadStateException），run()可以多次调用。应该调用start()来启动线程，直接调run()不会创建新线程。

### Q2: Java线程有几种状态？

6种：NEW（新建未启动）、RUNNABLE（可运行，包括就绪和运行中）、BLOCKED（等待synchronized锁）、WAITING（无限期等待）、TIMED_WAITING（限时等待）、TERMINATED（终止）。注意Java的RUNNABLE对应OS的就绪+运行两个状态。

### Q3: BLOCKED和WAITING的区别？

BLOCKED是线程竞争synchronized锁失败后的被动等待，不响应interrupt。WAITING是线程主动调用wait()/join()/park()后的等待，可以响应interrupt。wait()会释放锁，BLOCKED没有持有锁。BLOCKED只发生在synchronized，WAITING发生在wait/join/park。

### Q4: interrupt()会停止线程吗？

不会。interrupt()只是设置中断标志位。线程需要自己检查标志位（isInterrupted()）或响应InterruptedException来决定是否退出。如果线程在sleep/wait/join中被interrupt，会抛出InterruptedException（同时清除中断标志）。

### Q5: interrupted()和isInterrupted()的区别？

interrupted()是静态方法，检查当前线程的中断标志并清除它。isInterrupted()是实例方法，检查指定线程的中断标志但不清除。InterruptedException被捕获后中断标志已被清除，如需保持中断状态应重新调用interrupt()。

### Q6: join()的实现原理是什么？

join()底层调用wait()。在Thread对象上同步，然后while(isAlive()) wait()。当线程终止时，JVM自动调用该Thread对象的notifyAll()，唤醒所有join()等待的线程。所以join()的本质是wait/notify机制。

### Q7: sleep()和wait()的区别？

sleep()是Thread的静态方法，不释放锁，可以在任意位置调用，超时后自动唤醒。wait()是Object的方法，释放锁，必须在synchronized块内调用，需要notify/notifyAll或超时唤醒。sleep用于暂停执行，wait用于线程间通信。

### Q8: 守护线程有什么特点？

守护线程的退出不影响JVM——当所有非守护线程结束时JVM退出，守护线程被直接终止。守护线程的finally块不保证执行。守护线程创建的子线程默认也是守护线程。适合GC、心跳检测等辅助任务，不适合IO和资源清理。

### Q9: 虚拟线程和平台线程的区别？

虚拟线程是JVM管理的用户态线程，N:M映射到OS线程，创建成本极低可以创建数百万个。平台线程1:1映射OS线程，创建成本高数量有限。虚拟线程适合IO密集型高并发场景，不适合CPU密集型。虚拟线程不要池化，用完即弃。

### Q10: 线程抛出未捕获异常会怎样？

线程终止。如果有UncaughtExceptionHandler，调用它处理异常。否则异常栈打印到stderr。线程池中submit()的异常被Future封装，调用get()时抛出ExecutionException。execute()的异常直接触发handler。线程异常不会影响其他线程。
