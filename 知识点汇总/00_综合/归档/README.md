## 📋 归档原则

### 什么文档应该归档？

1. **历史状态文档**
   - 版本升级说明
   - 单个文档完成状态
   - 已过期的待办清单

2. **历史性说明文档**
   - 目录调整说明
   - 结构变更说明
   - 临时性指南

3. **已被替代的文档**
   - 被更新版本替代
   - 被更详细文档替代
   - 内容已合并到其他文档

### 什么文档不应该归档？

1. **技术性文档** - 永远保留
   - Java核心原理
   - Spring详解
   - MySQL详解
   - ...所有技术知识文档

2. **活跃的工具文档** - 持续更新
   - README.md
   - 学习路线
   - 快速索引

3. **有参考价值的规划** - 需要更新而非归档
   - 待补充清单（应更新到最新）
   - 学习计划
   - 技术路线图
