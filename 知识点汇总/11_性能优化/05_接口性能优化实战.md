## 接口性能指标

### 1.1 核心性能指标

```
┌─────────────────────────────────────────────────────────────────┐
│                    接口性能指标体系                                │
├──────────────┬──────────────────────────────────────────────────┤
│ 指标          │ 说明                                              │
├──────────────┼──────────────────────────────────────────────────┤
│ RT (响应时间) │ 从请求发出到收到响应的总时间                        │
│              │ P50: 50%的请求在此时间内完成                       │
│              │ P90: 90%的请求在此时间内完成                       │
│              │ P99: 99%的请求在此时间内完成                       │
│              │ P999: 99.9%的请求在此时间内完成                    │
├──────────────┼──────────────────────────────────────────────────┤
│ QPS          │ Queries Per Second，每秒请求数                    │
│ (吞吐量)     │ = 并发数 / 平均响应时间                            │
├──────────────┼──────────────────────────────────────────────────┤
│ TPS          │ Transactions Per Second，每秒事务数               │
├──────────────┼──────────────────────────────────────────────────┤
│ 并发数        │ 同时处理的请求数量                                │
├──────────────┼──────────────────────────────────────────────────┤
│ 错误率        │ 失败请求数 / 总请求数 × 100%                      │
│              │ 生产环境应 < 0.1%                                 │
├──────────────┼──────────────────────────────────────────────────┤
│ 可用性        │ 系统可用时间 / 总时间 × 100%                      │
│              │ 目标：99.99%（年宕机 < 52.6分钟）                 │
└──────────────┴──────────────────────────────────────────────────┘
```

### 1.2 性能标准参考

```
接口RT分级标准：
✅ 优秀：  P99 < 100ms
✅ 良好：  P99 < 200ms
⚠️ 一般：  P99 < 500ms
❌ 较差：  P99 < 1000ms
❌ 不可接受：P99 > 1000ms

不同场景的RT要求：
- 首页/搜索：     P99 < 200ms（用户体验敏感）
- 核心交易：      P99 < 500ms（一致性要求高）
- 报表/导出：     P99 < 5000ms（可异步化）
- 后台任务：      无严格要求（异步处理）

QPS参考：
- 单机QPS：       500~5000（取决于业务复杂度）
- 集群QPS：       单机QPS × 机器数 × 0.8（预留20%余量）
```

## 串行转并行

### 3.1 CompletableFuture并行编排

```java
@Service
public class OrderService {

    @Autowired
    private OrderMapper orderMapper;
    @Autowired
    private UserService userService;
    @Autowired
    private OrderItemService orderItemService;
    @Autowired
    private LogisticsService logisticsService;

    // ❌ 串行调用：总耗时 = 800 + 500 + 600 + 100 = 2000ms
    public OrderDetailVO getOrderDetailSerial(Long orderId) {
        Order order = orderMapper.selectById(orderId);        // 800ms
        User user = userService.getUserById(order.getUserId()); // 500ms
        List<OrderItem> items = orderItemService.getItems(orderId); // 600ms
        Logistics logistics = logisticsService.getInfo(orderId);    // 100ms
        
        return buildOrderDetailVO(order, user, items, logistics);
    }

    // ✅ 并行调用：总耗时 = max(800, 500, 600, 100) = 800ms
    @Async("orderDetailExecutor")
    public CompletableFuture<OrderDetailVO> getOrderDetailParallel(Long orderId) {
        // 并行获取各项数据
        CompletableFuture<Order> orderFuture = CompletableFuture.supplyAsync(
            () -> orderMapper.selectById(orderId), orderDetailExecutor);
        
        CompletableFuture<List<OrderItem>> itemsFuture = CompletableFuture.supplyAsync(
            () -> orderItemService.getItems(orderId), orderDetailExecutor);
        
        CompletableFuture<Logistics> logisticsFuture = CompletableFuture.supplyAsync(
            () -> logisticsService.getInfo(orderId), orderDetailExecutor);
        
        // order获取后再并行获取user
        CompletableFuture<User> userFuture = orderFuture.thenApplyAsync(
            order -> userService.getUserById(order.getUserId()), orderDetailExecutor);
        
        // 等待所有任务完成
        CompletableFuture.allOf(orderFuture, userFuture, itemsFuture, logisticsFuture).join();
        
        return CompletableFuture.completedFuture(
            buildOrderDetailVO(
                orderFuture.join(),
                userFuture.join(),
                itemsFuture.join(),
                logisticsFuture.join()
            )
        );
    }

    // 更复杂的编排场景
    public OrderDetailVO getOrderDetailComplex(Long orderId) {
        // 第一阶段：并行获取基础数据
        CompletableFuture<Order> orderFuture = CompletableFuture.supplyAsync(
            () -> orderMapper.selectById(orderId));
        
        CompletableFuture<List<OrderItem>> itemsFuture = CompletableFuture.supplyAsync(
            () -> orderItemService.getItems(orderId));
        
        // 第二阶段：order获取后，并行获取user和coupon信息
        CompletableFuture<User> userFuture = orderFuture.thenComposeAsync(
            order -> CompletableFuture.supplyAsync(
                () -> userService.getUserById(order.getUserId())));
        
        CompletableFuture<Coupon> couponFuture = orderFuture.thenComposeAsync(
            order -> CompletableFuture.supplyAsync(
                () -> couponService.getByOrderId(order.getCouponId())));
        
        // 第三阶段：所有数据到齐后组装
        return CompletableFuture.allOf(
            orderFuture, itemsFuture, userFuture, couponFuture
        ).thenApplyAsync(v -> {
            OrderDetailVO vo = new OrderDetailVO();
            vo.setOrder(orderFuture.join());
            vo.setItems(itemsFuture.join());
            vo.setUser(userFuture.join());
            vo.setCoupon(couponFuture.join());
            return vo;
        }).join();  // 总耗时 ≈ max(800, 600) + max(500, 200) = 1300ms
    }
}
```

### 3.2 并行注意事项

```java
// ⚠️ 并行化需要注意的问题：

// 1. 线程安全问题
// ❌ 非线程安全的集合
List<Order> results = new ArrayList<>();
futures.forEach(f -> {
    f.thenAccept(order -> results.add(order)); // ArrayList不是线程安全的
});

// ✅ 使用线程安全集合
List<Order> results = Collections.synchronizedList(new ArrayList<>());
// 或使用CopyOnWriteArrayList
List<Order> results = new CopyOnWriteArrayList<>();

// 2. 合理设置超时时间
OrderDetailVO result = future.get(2, TimeUnit.SECONDS); // 设置超时
// 或使用CompletableFuture.orTimeout (Java 9+)
CompletableFuture<OrderDetailVO> futureWithTimeout = future
    .orTimeout(2, TimeUnit.SECONDS)
    .exceptionally(ex -> {
        log.warn("获取订单详情超时，返回降级数据", ex);
        return getDefaultOrderDetail();
    });

// 3. 并行度不是越高越好
// 核心公式：并行度 = min(任务数, CPU核心数 × 线程池大小)
// 过多并行任务会导致线程切换开销增大
```

## 缓存策略

### 5.1 多级缓存架构

```
┌─────────────────────────────────────────────────────────────┐
│                    多级缓存架构                                │
├─────────────────────────────────────────────────────────────┤
│                                                              │
│   请求 → Nginx缓存 → 本地缓存(Caffeine) → Redis → DB       │
│           (L1)        (L2)              (L3)    (L4)       │
│                                                              │
│   L1 Nginx缓存：    静态资源、不常变化的页面                   │
│   L2 本地缓存：      进程内缓存，访问延迟 < 1ms               │
│   L3 Redis缓存：     分布式缓存，访问延迟 < 1ms~3ms          │
│   L4 数据库：         访问延迟 > 10ms                        │
│                                                              │
│   缓存命中率目标：L2 > 80%，L3 > 95%                         │
└─────────────────────────────────────────────────────────────┘
```

```java
// Caffeine本地缓存 + Redis分布式缓存（多级缓存）
@Configuration
public class CacheConfig {

    @Bean
    public Cache<String, Object> localCache() {
        return Caffeine.newBuilder()
            .maximumSize(10000)                    // 最大缓存条目数
            .expireAfterWrite(5, TimeUnit.MINUTES)  // 写入后5分钟过期
            .expireAfterAccess(3, TimeUnit.MINUTES) // 访问后3分钟过期
            .recordStats()                          // 开启统计
            .build();
    }
}

@Service
public class MultiLevelCacheService {

    @Autowired
    private Cache<String, Object> localCache;
    
    @Autowired
    private RedisTemplate<String, Object> redisTemplate;
    
    @Autowired
    private OrderMapper orderMapper;

    /**
     * 多级缓存查询：
     * L1 本地缓存 → L2 Redis → L3 数据库
     */
    public Order getOrderById(Long orderId) {
        String key = "order:detail:" + orderId;
        
        // L1: 查询本地缓存
        Order order = (Order) localCache.getIfPresent(key);
        if (order != null) {
            return order;
        }
        
        // L2: 查询Redis缓存
        order = (Order) redisTemplate.opsForValue().get(key);
        if (order != null) {
            // 回填本地缓存
            localCache.put(key, order);
            return order;
        }
        
        // L3: 查询数据库
        order = orderMapper.selectById(orderId);
        if (order != null) {
            // 回填两级缓存
            redisTemplate.opsForValue().set(key, order, 30, TimeUnit.MINUTES);
            localCache.put(key, order);
        }
        return order;
    }
    
    /**
     * 更新缓存（旁路缓存模式）
     */
    @Transactional
    public void updateOrder(Order order) {
        // 1. 先更新数据库
        orderMapper.updateById(order);
        
        // 2. 删除缓存（而非更新，避免并发一致性问题）
        String key = "order:detail:" + order.getOrderId();
        redisTemplate.delete(key);
        localCache.invalidate(key);
    }
}
```

### 5.3 缓存穿透/击穿/雪崩防护

```java
@Service
public class CacheProtectionService {

    @Autowired
    private RedisTemplate<String, String> redisTemplate;
    
    @Autowired
    private BloomFilter<Long> orderBloomFilter;  // 布隆过滤器

    // ========== 缓存穿透防护 ==========
    // 查询不存在的数据，每次都打到数据库
    
    public Order getOrderWithProtection(Long orderId) {
        String key = "order:detail:" + orderId;
        String nullKey = "order:null:" + orderId;
        
        // 方案1：布隆过滤器拦截不存在的key
        if (!orderBloomFilter.mightContain(orderId)) {
            log.debug("布隆过滤器拦截，orderId={}不存在", orderId);
            return null;
        }
        
        // 方案2：缓存空值（防止反复查询同一不存在的key）
        Order order = (Order) redisTemplate.opsForValue().get(key);
        if (order != null) {
            return order;
        }
        
        // 检查空值缓存
        if (Boolean.TRUE.equals(redisTemplate.hasKey(nullKey))) {
            return null;  // 之前查过，确实不存在
        }
        
        // 查询数据库
        order = orderMapper.selectById(orderId);
        if (order != null) {
            redisTemplate.opsForValue().set(key, order, 30, TimeUnit.MINUTES);
        } else {
            // 缓存空值，短过期时间
            redisTemplate.opsForValue().set(nullKey, "", 2, TimeUnit.MINUTES);
        }
        return order;
    }

    // ========== 缓存击穿防护 ==========
    // 热点key过期瞬间，大量请求同时打到数据库
    
    public Order getOrderWithLock(Long orderId) {
        String key = "order:detail:" + orderId;
        
        // 1. 先查缓存
        Order order = (Order) redisTemplate.opsForValue().get(key);
        if (order != null) {
            return order;
        }
        
        // 2. 获取分布式锁（防止缓存击穿）
        String lockKey = "lock:order:" + orderId;
        String lockValue = UUID.randomUUID().toString();
        
        try {
            // 尝试获取锁，等待3秒
            Boolean locked = redisTemplate.opsForValue()
                .setIfAbsent(lockKey, lockValue, 10, TimeUnit.SECONDS);
            
            if (Boolean.TRUE.equals(locked)) {
                // 获取锁成功，再次检查缓存（双重检查）
                order = (Order) redisTemplate.opsForValue().get(key);
                if (order != null) {
                    return order;
                }
                
                // 查询数据库并回填缓存
                order = orderMapper.selectById(orderId);
                if (order != null) {
                    // 随机过期时间，防止缓存雪崩
                    int expire = 30 * 60 + new Random().nextInt(300);
                    redisTemplate.opsForValue().set(key, order, expire, TimeUnit.SECONDS);
                }
                return order;
            } else {
                // 获取锁失败，短暂等待后重试读缓存
                Thread.sleep(50);
                return (Order) redisTemplate.opsForValue().get(key);
            }
        } finally {
            // 释放锁（Lua脚本保证原子性）
            String script = 
                "if redis.call('get', KEYS[1]) == ARGV[1] then " +
                "  return redis.call('del', KEYS[1]) " +
                "else return 0 end";
            redisTemplate.execute(
                new DefaultRedisScript<>(script, Long.class),
                Collections.singletonList(lockKey),
                lockValue
            );
        }
    }

    // ========== 缓存雪崩防护 ==========
    // 大量key同时过期，或Redis宕机
    
    public void cacheOrderWithRandomExpiry(Long orderId, Order order) {
        String key = "order:detail:" + orderId;
        // 基础过期时间30分钟 + 随机0~5分钟
        int baseExpire = 1800;
        int randomExpire = new Random().nextInt(300);
        redisTemplate.opsForValue().set(key, order, 
            baseExpire + randomExpire, TimeUnit.SECONDS);
    }
}
```

```java
@Service
public class OrderBatchService {

    // ❌ 逐条插入：1000条数据需要1000次DB往返
    public void insertOneByOne(List<Order> orders) {
        for (Order order : orders) {
            orderMapper.insert(order);  // 每次一条
        }
    }
    // 耗时：约10~15秒

    // ✅ 批量插入：一次SQL插入所有数据
    public void batchInsert(List<Order> orders) {
        int batchSize = 500;
        for (int i = 0; i < orders.size(); i += batchSize) {
            int end = Math.min(i + batchSize, orders.size());
            List<Order> batch = orders.subList(i, end);
            orderMapper.batchInsert(batch);
        }
    }
    // 耗时：约200ms（提升50~75倍）

    // ✅ 使用MyBatis的Batch执行器
    @Transactional
    public void batchInsertWithExecutor(List<Order> orders) {
        SqlSessionFactory sqlSessionFactory = ...;
        try (SqlSession session = sqlSessionFactory.openSession(ExecutorType.BATCH, false)) {
            OrderMapper mapper = session.getMapper(OrderMapper.class);
            for (int i = 0; i < orders.size(); i++) {
                mapper.insert(orders.get(i));
                if (i % 500 == 0 || i == orders.size() - 1) {
                    session.flushStatements();  // 刷新批处理
                }
            }
            session.commit();
        }
    }
    // 耗时：约150ms

    // ✅ 批量查询避免N+1问题
    // ❌ N+1查询：先查列表（1次），再逐条查关联数据（N次）
    public List<OrderVO> getOrdersWithNPlus1(List<Long> orderIds) {
        List<Order> orders = orderMapper.selectByIds(orderIds);  // 1次
        List<OrderVO> result = new ArrayList<>();
        for (Order order : orders) {
            User user = userMapper.selectById(order.getUserId());  // N次
            result.add(new OrderVO(order, user));
        }
        return result;
    }
    
    // ✅ 批量查询：1次查询获取所有关联数据
    public List<OrderVO> getOrdersBatch(List<Long> orderIds) {
        List<Order> orders = orderMapper.selectByIds(orderIds);
        // 提取所有userId，一次性查询
        Set<Long> userIds = orders.stream()
            .map(Order::getUserId)
            .collect(Collectors.toSet());
        Map<Long, User> userMap = userMapper.selectByIds(userIds).stream()
            .collect(Collectors.toMap(User::getUserId, u -> u));
        
        return orders.stream()
            .map(order -> new OrderVO(order, userMap.get(order.getUserId())))
            .collect(Collectors.toList());
    }
    // 2次查询代替N+1次
}
```

```java
// 优化后：并行获取用户信息
public OrderDetailVO getOrderDetailOptimized(Long orderId) {
    // 1. 查订单（必须先查，后续依赖userId）
    Order order = orderMapper.selectById(orderId);  // 5ms
    
    // 2. 并行获取其他数据
    CompletableFuture<User> userFuture = CompletableFuture.supplyAsync(
        () -> userService.getUserById(order.getUserId()), executor);
    
    CompletableFuture<List<OrderItem>> itemsFuture = CompletableFuture.supplyAsync(
        () -> orderItemService.getItems(orderId), executor);

        () -> logisticsService.getInfo(orderId), executor);

    CompletableFuture.allOf(userFuture, itemsFuture, logisticsFuture).join();
    
    return buildOrderDetailVO(order, userFuture.join(), 
        itemsFuture.join(), logisticsFuture.join());
}
```

```java
// 优化后：添加多级缓存
public OrderDetailVO getOrderDetailCached(Long orderId) {
    String key = "order:detail:" + orderId;
    
    // L1: 本地缓存
    OrderDetailVO cached = (OrderDetailVO) localCache.getIfPresent(key);
    if (cached != null) {
        return cached;  // < 1ms
    }
    
    // L2: Redis缓存
    cached = (OrderDetailVO) redisTemplate.opsForValue().get(key);
    if (cached != null) {
        localCache.put(key, cached);
        return cached;  // < 3ms
    }
    
    // L3: 数据库查询 + 并行获取
    Order order = orderMapper.selectById(orderId);
    if (order == null) return null;
    
    CompletableFuture<User> userFuture = CompletableFuture.supplyAsync(
        () -> userService.getUserById(order.getUserId()), executor);
    CompletableFuture<List<OrderItem>> itemsFuture = CompletableFuture.supplyAsync(
        () -> orderItemService.getItems(orderId), executor);
    CompletableFuture<Logistics> logisticsFuture = CompletableFuture.supplyAsync(
        () -> logisticsService.getInfo(orderId), executor);
    
    CompletableFuture.allOf(userFuture, itemsFuture, logisticsFuture).join();
    
    OrderDetailVO result = buildOrderDetailVO(order, 
        userFuture.join(), itemsFuture.join(), logisticsFuture.join());
    
    // 回填缓存
    redisTemplate.opsForValue().set(key, result, 30, TimeUnit.MINUTES);
    localCache.put(key, result);
    
    return result;
}
```

**缓存优化后：**
- 缓存命中：505ms → 1ms
- 缓存未命中：505ms → 50ms（数据库查询8ms + 并行获取50ms）

```java
    public void sendNotification(Order order) { ... }

    @Lazy

    public void process(Order order) {
        self.sendNotification(order);  // 通过代理调用，@Async生效
```
