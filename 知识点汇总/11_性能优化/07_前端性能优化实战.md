## 三、加载性能优化

### 3.1 关键渲染路径优化

```
关键渲染路径（Critical Rendering Path）：

HTML下载 → HTML解析 → DOM构建
                        ↓
CSS下载 → CSS解析 → CSSOM构建 → 布局(Layout) → 绘制(Paint) → 合成(Composite)
                        ↑
JS下载 → JS解析 → JS执行 → 可能修改DOM/CSSOM

优化目标：尽快完成首屏渲染
核心思路：减少阻塞资源、优先加载关键资源、延迟非关键资源
```

```html
<!-- 1. CSS：放在<head>中，优先加载 -->
<head>
  <!-- 关键CSS内联 -->
  <style>
    /* 首屏关键样式直接内联，避免额外请求 */
    body { margin: 0; font-family: sans-serif; }
    .header { height: 60px; background: #fff; }
    .hero { min-height: 400px; }
  </style>

  <!-- 非关键CSS异步加载 -->
  <link rel="preload" href="/css/non-critical.css" as="style" onload="this.rel='stylesheet'">
  <noscript><link rel="stylesheet" href="/css/non-critical.css"></noscript>
</head>

<!-- 2. JS：放在<body>底部，或使用defer/async -->
<body>
  <div id="app"><!-- 首屏内容 --></div>

  <!-- defer：DOM解析完成后、DOMContentLoaded前执行 -->
  <script src="/js/vendor.js" defer></script>
  <script src="/js/app.js" defer></script>

  <!-- async：下载不阻塞解析，下载完立即执行（适合独立脚本） -->
  <script async src="https://www.googletagmanager.com/gtag/js?id=GA_ID"></script>
</body>
```

### 3.2 defer vs async

```
脚本加载执行时序对比：

普通 <script>:
HTML解析 ──→ [暂停] ──下载JS──执行JS── [恢复] ──→ 完成
                  ↓（阻塞解析）

async <script>:
HTML解析 ──→ ──→ ──→ ──→ [暂停]──执行JS── [恢复] ──→ 完成
               └──下载JS──┘（不阻塞，但执行时阻塞）

defer <script>:
HTML解析 ──→ ──→ ──→ ──→ ──→ 完成 ──→ 执行JS ──→ DOMContentLoaded
               └──下载JS──┘（不阻塞解析，延迟到解析完成后执行）

总结：
- 关键脚本用 defer（按顺序执行，不阻塞解析）
- 独立第三方脚本用 async（如统计、广告）
- 首屏不需要的脚本动态 import（按需加载）
```

## 四、渲染性能优化

### 4.1 浏览器渲染流水线

```
浏览器渲染完整流水线：

┌─────────┐    ┌─────────┐    ┌──────────┐    ┌─────────┐    ┌──────────┐
│  Style  │ →  │ Layout  │ →  │  Paint   │ →  │ Composite│ →  │  显示    │
│ 样式计算 │    │  布局   │    │   绘制   │    │   合成   │    │          │
└─────────┘    └─────────┘    └──────────┘    └─────────┘    └──────────┘

触发不同阶段的属性：

只触发 Composite（最优）：
  transform, opacity, filter

触发 Layout + Paint + Composite（最差）：
  width, height, margin, padding, top, left, font-size, ...

触发 Paint + Composite（中等）：
  color, background, visibility, border-color, ...

优化原则：尽量只触发 Composite，使用 transform 代替 position
```

### 4.2 使用 requestAnimationFrame

```javascript
// ❌ 错误：直接在事件中修改样式，可能导致丢帧
element.addEventListener('scroll', () => {
  element.style.transform = `translateY(${window.scrollY}px)`;
});

// ✅ 正确：使用 rAF 将视觉更新同步到浏览器刷新
let ticking = false;
element.addEventListener('scroll', () => {
  if (!ticking) {
    requestAnimationFrame(() => {
      element.style.transform = `translateY(${window.scrollY}px)`;
      ticking = false;
    });
    ticking = true;
  }
});

// rAF 动画示例
function animate(element, from, to, duration) {
  const startTime = performance.now();

  function frame(currentTime) {
    const elapsed = currentTime - startTime;
    const progress = Math.min(elapsed / duration, 1);
    // easeOutCubic 缓动
    const eased = 1 - Math.pow(1 - progress, 3);
    const current = from + (to - from) * eased;

    element.style.transform = `translateX(${current}px)`;

    if (progress < 1) {
      requestAnimationFrame(frame);
    }
  }

  requestAnimationFrame(frame);
}
```

### 4.3 虚拟列表（大数据量渲染）

```javascript
import { useRef, useState, useMemo, useCallback } from 'react';

// 虚拟列表组件：只渲染可视区域内的项
function VirtualList({ items, itemHeight, containerHeight, renderItem }) {
  const [scrollTop, setScrollTop] = useState(0);
  const containerRef = useRef(null);

  // 可视区域内可显示的项数
  const visibleCount = Math.ceil(containerHeight / itemHeight) + 2; // 多渲染2项作为缓冲

  // 可视区域起始索引
  const startIndex = Math.max(0, Math.floor(scrollTop / itemHeight) - 1);

  // 可视区域结束索引
  const endIndex = Math.min(items.length, startIndex + visibleCount);

  // 可视区域数据
  const visibleItems = useMemo(() => {
    return items.slice(startIndex, endIndex).map((item, i) => ({
      ...item,
      index: startIndex + i,
    }));
  }, [items, startIndex, endIndex]);

  // 偏移量（让渲染项出现在正确位置）
  const offsetY = startIndex * itemHeight;

  // 总高度（撑开滚动条）
  const totalHeight = items.length * itemHeight;

  const handleScroll = useCallback((e) => {
    setScrollTop(e.target.scrollTop);
  }, []);

  return (
    <div
      ref={containerRef}
      onScroll={handleScroll}
      style={{ height: containerHeight, overflow: 'auto', position: 'relative' }}
    >
      <div style={{ height: totalHeight, position: 'relative' }}>
        <div style={{
          position: 'absolute',
          top: offsetY,
          left: 0,
          right: 0,
        }}>
          {visibleItems.map((item, i) => (
            <div key={item.index} style={{ height: itemHeight }}>
              {renderItem(item, i)}
            </div>
          ))}
        </div>
      </div>
    </div>
  );
}

// 使用示例：渲染10万条数据
function App() {
  const items = useMemo(() => {
    return Array.from({ length: 100000 }, (_, i) => ({
      id: i,
      name: `Item ${i}`,
    }));
  }, []);

  return (
    <VirtualList
      items={items}
      itemHeight={50}
      containerHeight={600}
      renderItem={(item) => (
        <div style={{ padding: '12px', borderBottom: '1px solid #eee' }}>
          {item.name}
        </div>
      )}
    />
  );
}
```

### 4.4 防抖与节流

```javascript
// 防抖（Debounce）：停止触发后才执行，适合搜索框、resize
function debounce(fn, delay) {
  let timer = null;
  return function (...args) {
    if (timer) clearTimeout(timer);
    timer = setTimeout(() => {
      fn.apply(this, args);
      timer = null;
    }, delay);
  };
}

// 节流（Throttle）：固定频率执行，适合滚动、拖拽
function throttle(fn, interval) {
  let lastTime = 0;
  return function (...args) {
    const now = Date.now();
    if (now - lastTime >= interval) {
      fn.apply(this, args);
      lastTime = now;
    }
  };
}

// requestAnimationFrame 节流（更精准，与浏览器刷新同步）
function rafThrottle(fn) {
  let ticking = false;
  return function (...args) {
    if (!ticking) {
      requestAnimationFrame(() => {
        fn.apply(this, args);
        ticking = false;
      });
      ticking = true;
    }
  };
}

// 使用示例
const handleSearch = debounce((query) => {
  fetchResults(query);
}, 300);

const handleScroll = rafThrottle(() => {
  updateScrollPosition(window.scrollY);
});

// React Hook 版
function useDebounce(value, delay) {
  const [debouncedValue, setDebouncedValue] = useState(value);
  useEffect(() => {
    const timer = setTimeout(() => setDebouncedValue(value), delay);
    return () => clearTimeout(timer);
  }, [value, delay]);
  return debouncedValue;
}
```

## 六、缓存策略

### 6.1 HTTP 缓存策略

```
HTTP 缓存决策树：

                        请求资源
                           │
                    ┌──────▼──────┐
                    │ 有缓存？     │
                    └──┬──────┬───┘
                   No │      │ Yes
                      │      │
                ┌─────▼─┐  ┌─▼──────────────┐
                │ 请求   │  │ 强缓存有效？     │
                │ 服务器 │  │ (Cache-Control)  │
                └────────┘  └──┬──────────┬───┘
                           Yes │          │ No
                              │          │
                        ┌─────▼─┐  ┌─────▼─────────┐
                        │ 用缓存 │  │ 协商缓存       │
                        │ 200   │  │ (ETag/Last-    │
                        │ (disk)│  │  Modified)     │
                        └───────┘  └──┬────────┬───┘
                                  304 │        │ 200
                                     │        │
                               ┌─────▼─┐  ┌──▼──────┐
                               │ 用缓存 │  │ 请求服务器│
                               │ 304   │  │ 200     │
```

### 6.2 Service Worker 缓存

```javascript
// service-worker.js
const CACHE_VERSION = 'v3';
const STATIC_CACHE = `static-${CACHE_VERSION}`;
const RUNTIME_CACHE = `runtime-${CACHE_VERSION}`;

// 预缓存资源列表
const PRECACHE_URLS = [
  '/',
  '/index.html',
  '/css/critical.css',
  '/js/vendor.js',
  '/js/app.js',
  '/offline.html',
];

// 安装：预缓存核心资源
self.addEventListener('install', (event) => {
  event.waitUntil(
    caches.open(STATIC_CACHE)
      .then(cache => cache.addAll(PRECACHE_URLS))
      .then(() => self.skipWaiting())
  );
});

// 激活：清理旧缓存
self.addEventListener('activate', (event) => {
  event.waitUntil(
    caches.keys()
      .then(keys => Promise.all(
        keys
          .filter(key => key !== STATIC_CACHE && key !== RUNTIME_CACHE)
          .map(key => caches.delete(key))
      ))
      .then(() => self.clients.claim())
  );
});

// 请求拦截：缓存优先 + 网络更新策略
self.addEventListener('fetch', (event) => {
  const { request } = event;

  // 只缓存GET请求
  if (request.method !== 'GET') return;

  // HTML请求：网络优先，失败时用缓存
  if (request.headers.get('accept')?.includes('text/html')) {
    event.respondWith(
      fetch(request)
        .then(response => {
          const clone = response.clone();
          caches.open(RUNTIME_CACHE).then(cache => cache.put(request, clone));
          return response;
        })
        .catch(() => caches.match(request).then(cached => cached || caches.match('/offline.html')))
    );
    return;
  }

  // 静态资源：缓存优先
  event.respondWith(
    caches.match(request).then(cached => {
      return cached || fetch(request).then(response => {
        const clone = response.clone();
        caches.open(RUNTIME_CACHE).then(cache => cache.put(request, clone));
        return response;
      });
    })
  );
});
```

```javascript
// 1. IntersectionObserver 实现
class ImageLazyLoader {
  constructor() {
    this.observer = new IntersectionObserver(
      this.handleIntersection.bind(this),
      {
        rootMargin: '50px 0px',  // 提前50px开始加载
        threshold: 0.01,
      }
    );
  }

  observe(img) {
    this.observer.observe(img);
  }

  handleIntersection(entries) {
    entries.forEach(entry => {
      if (entry.isIntersecting) {
        const img = entry.target;
        const src = img.dataset.src;
        if (src) {
          img.src = src;
          img.removeAttribute('data-src');
        }
        this.observer.unobserve(img);
      }
    });
  }
}

// 使用
const loader = new ImageLazyLoader();
document.querySelectorAll('img[data-src]').forEach(img => loader.observe(img));

// 2. React 组件版
function LazyImage({ src, alt, width, height }) {
  const imgRef = useRef(null);
  const [isLoaded, setIsLoaded] = useState(false);
  const [isInView, setIsInView] = useState(false);

  useEffect(() => {
    const observer = new IntersectionObserver(([entry]) => {
      if (entry.isIntersecting) {
        setIsInView(true);
        observer.disconnect();
      }
    }, { rootMargin: '50px' });

    if (imgRef.current) observer.observe(imgRef.current);
    return () => observer.disconnect();
  }, []);

  return (
    <div style={{ width, height, position: 'relative', backgroundColor: '#f0f0f0' }}>
      {!isLoaded && <div className="skeleton" />}
      <img
        ref={imgRef}
        src={isInView ? src : undefined}
        alt={alt}
        width={width}
        height={height}
        onLoad={() => setIsLoaded(true)}
        style={{
          opacity: isLoaded ? 1 : 0,
          transition: 'opacity 0.3s',
          position: 'absolute',
          top: 0,
          left: 0,
        }}
      />
    </div>
  );
}
```

## 十一、面试题速查

### Q1：前端性能优化的核心指标有哪些？

> Web Vitals 三大核心指标：**LCP**（最大内容绘制，< 2.5s，衡量加载性能）、**FID/INP**（首次输入延迟/交互到绘制，< 100ms/200ms，衡量交互性）、**CLS**（累积布局偏移，< 0.1，衡量视觉稳定性）。此外还有 TTFB（首字节时间）、FCP（首次内容绘制）、TTI（可交互时间）等辅助指标。

### Q2：如何减少首屏加载时间？

> 从关键渲染路径入手：1）关键CSS内联到 `<head>`；2）JS 使用 `defer` 避免阻塞解析；3）预加载关键资源（`preload` font/critical CSS）；4）代码分割，首屏只加载必要代码；5）接口预请求（在JS加载前发API请求）；6）骨架屏降低用户感知等待；7）图片懒加载 + WebP格式；8）开启Gzip/Brotli压缩；9）HTTP强缓存静态资源。

### Q3：defer 和 async 的区别？

> 两者都不阻塞HTML解析。**async**：下载完成后立即执行，多个async脚本执行顺序不确定，适合独立第三方脚本（统计、广告）。**defer**：下载完成后等到HTML解析完毕、DOMContentLoaded之前按顺序执行，适合有关联依赖的应用脚本。首屏关键脚本优先用 defer。

### Q4：如何优化 CLS（累积布局偏移）？

> 1）所有图片/视频/广告位设置明确的 `width` 和 `height` 属性，或用 CSS `aspect-ratio`；2）动态内容插入到文档流末尾而非顶部；3）字体加载使用 `font-display: swap` 并预加载，配合字体度量匹配避免文字跳动；4）避免在已渲染内容上方插入弹窗/横幅。

### Q5：虚拟列表的原理是什么？

> 虚拟列表只渲染可视区域内的元素（+ 少量缓冲项），而非全部数据。通过监听滚动事件计算当前 `scrollTop`，推算出可视区域的起始索引和结束索引，只截取这段数据渲染。同时用绝对定位将渲染项偏移到正确位置，并用一个等于总高度的占位元素撑开滚动条。这样无论数据量多大，DOM节点数始终恒定。

### Q6：Service Worker 的缓存策略有哪些？

> 常用策略：1）**Cache First**（缓存优先）——先查缓存，没有再请求网络，适合静态资源；2）**Network First**（网络优先）——先请求网络，失败再用缓存，适合API数据；3）**Stale While Revalidate**——先返回缓存，同时后台更新缓存，适合非关键内容；4）**Cache Only**——只用缓存，适合离线页面。实际应用中按资源类型组合使用，如HTML用网络优先，JS/CSS/图片用缓存优先。

### Q7：React 中如何避免不必要的重渲染？

> 1）`React.memo` 对函数组件做浅比较，props不变则跳过渲染；2）`useMemo` 缓存计算结果，依赖不变则返回缓存值；3）`useCallback` 缓存函数引用，避免每次render创建新函数传递给子组件；4）状态拆分，不相关的状态分开管理，避免一个状态变更触发全组件重渲染；5）列表项使用稳定的 `key`；6）大数据量列表用虚拟列表（react-window）；7）将变化频繁的部分下沉到独立子组件，缩小重渲染范围。

### Q8：Tree Shaking 的原理和条件是什么？

> Tree Shaking 基于 ES6 模块的静态分析，在编译时确定哪些导出未被使用，然后在压缩阶段移除。条件：1）必须使用 ES6 `import/export` 语法（不能是 CommonJS `require`）；2）在 `package.json` 中设置 `"sideEffects": false` 或指定有副作用的文件；3）生产模式下开启 `usedExports` 和 `minimize`。注意 CommonJS 和动态 import 无法 Tree Shaking。
