## 测试体系概述

### 测试金字塔

```
       /\
      /E2E\        10%  - 端到端测试（慢、脆弱、昂贵）
     /------\
    /集成测试\     20%  - 集成测试（适中）
   /----------\
  /  单元测试  \   70%  - 单元测试（快、稳定、便宜）
 /--------------\
```

### 测试类型

| 测试类型 | 测试对象 | 工具 | 执行频率 |
|---------|---------|------|---------|
| **单元测试** | 单个方法/类 | JUnit 5, Mockito | 每次提交 |
| **集成测试** | 多个组件交互 | Spring Test, TestContainers | 每日构建 |
| **E2E测试** | 完整业务流程 | Selenium, REST Assured | 发版前 |

## 单元测试

### 1. JUnit 5基础

JUnit 5 = JUnit Platform + JUnit Jupiter + JUnit Vintage

**Maven依赖**：
```xml
<dependency>
    <groupId>org.junit.jupiter</groupId>
    <artifactId>junit-jupiter</artifactId>
    <version>5.10.2</version>
    <scope>test</scope>
</dependency>
```

**核心注解**：
```java
import org.junit.jupiter.api.*;

@Test           // 标记测试方法
@DisplayName    // 测试显示名称
@BeforeEach     // 每个测试前执行
@AfterEach      // 每个测试后执行
@BeforeAll      // 所有测试前执行（static方法）
@AfterAll       // 所有测试后执行（static方法）
@Nested         // 嵌套测试类
@ParameterizedTest  // 参数化测试
@RepeatedTest       // 重复测试
@Tag                 // 标签过滤
@Disabled            // 禁用测试
```

**参数化测试**：
```java
@ParameterizedTest
@ValueSource(strings = {"hello", "world", "junit"})
void testWithStrings(String word) {
    assertNotNull(word);
}

@ParameterizedTest
@CsvSource({
    "1, 1, 2",
    "2, 3, 5",
    "10, 20, 30"
})
void testAddition(int a, int b, int expected) {
    assertEquals(expected, a + b);
}
```

**断言增强**：
```java
// 分组断言，全部执行后统一报告
assertAll("person",
    () -> assertEquals("John", person.getFirstName()),
    () -> assertEquals("Doe", person.getLastName()),
    () -> assertEquals(30, person.getAge())
);

// 超时断言
assertTimeout(Duration.ofMillis(500), () -> {
    service.processData();
});

// 异常断言
Exception ex = assertThrows(IllegalArgumentException.class, () -> {
    service.validate(null);
});
assertEquals("参数不能为空", ex.getMessage());
```

### 2. Mockito使用

**Maven依赖**：
```xml
<dependency>
    <groupId>org.mockito</groupId>
    <artifactId>mockito-core</artifactId>
    <version>5.11.0</version>
    <scope>test</scope>
</dependency>
<dependency>
    <groupId>org.mockito</groupId>
    <artifactId>mockito-junit-jupiter</artifactId>
    <version>5.11.0</version>
    <scope>test</scope>
</dependency>
```

**Mock与Stub**：
```java
@ExtendWith(MockitoExtension.class)
class UserServiceTest {

    @Mock
    private UserRepository userRepository;

    @InjectMocks
    private UserServiceImpl userService;

    @Test
    @DisplayName("根据ID查询用户 - 正常情况")
    void testFindById() {
        // Given
        User mockUser = new User(1L, "张三", "zhangsan@email.com");
        when(userRepository.findById(1L)).thenReturn(Optional.of(mockUser));

        // When
        User result = userService.findById(1L);

        // Then
        assertThat(result).isNotNull();
        assertThat(result.getName()).isEqualTo("张三");
        verify(userRepository).findById(1L);
    }

    @Test
    @DisplayName("根据ID查询用户 - 用户不存在")
    void testFindById_NotFound() {
        when(userRepository.findById(999L)).thenReturn(Optional.empty());

        assertThrows(UserNotFoundException.class, () -> {
            userService.findById(999L);
        });
    }
}
```

**验证行为**：
```java
// 验证调用次数
verify(userRepository, times(1)).findById(1L);
verify(userRepository, never()).deleteById(anyLong());
verify(userRepository, atMost(3)).save(any(User.class));

// 验证调用顺序
InOrder inOrder = inOrder(userRepository);
inOrder.verify(userRepository).findById(1L);
inOrder.verify(userRepository).save(any(User.class));

// 参数捕获
ArgumentCaptor<User> captor = ArgumentCaptor.forClass(User.class);
verify(userRepository).save(captor.capture());
assertEquals("张三", captor.getValue().getName());
```

### 3. 单元测试实战

**Service层测试模板**：
```java
@ExtendWith(MockitoExtension.class)
class OrderServiceTest {

    @Mock private OrderRepository orderRepository;
    @Mock private PaymentService paymentService;
    @Mock private InventoryService inventoryService;
    @InjectMocks private OrderServiceImpl orderService;

    private OrderRequest validRequest;

    @BeforeEach
    void setUp() {
        validRequest = OrderRequest.builder()
            .userId(1L)
            .productId(100L)
            .quantity(2)
            .build();
    }

    @Nested
    @DisplayName("创建订单")
    class CreateOrder {

        @Test
        @DisplayName("成功创建订单")
        void createOrder_Success() {
            // Given
            when(inventoryService.checkStock(100L, 2)).thenReturn(true);
            when(paymentService.charge(any(PaymentRequest.class))).thenReturn(true);
            when(orderRepository.save(any(Order.class))).thenAnswer(inv -> inv.getArgument(0));

            // When
            Order order = orderService.createOrder(validRequest);

            // Then
            assertThat(order.getStatus()).isEqualTo(OrderStatus.CREATED);
            verify(inventoryService).deductStock(100L, 2);
        }

        @Test
        @DisplayName("库存不足时抛出异常")
        void createOrder_OutOfStock() {
            when(inventoryService.checkStock(100L, 2)).thenReturn(false);

            assertThrows(OutOfStockException.class, () -> {
                orderService.createOrder(validRequest);
            });

            verify(paymentService, never()).charge(any());
        }
    }
}
```

```java
@SpringBootTest
@ActiveProfiles("test")
@Transactional  // 测试后自动回滚
class UserControllerIntegrationTest {

    @Autowired
    private TestRestTemplate restTemplate;

    @Autowired
    private UserRepository userRepository;

    @Test
    void testCreateUser() {
        // Given
        UserRequest request = new UserRequest("张三", "zhangsan@test.com");

        // When
        ResponseEntity<User> response = restTemplate.postForEntity(
            "/api/users", request, User.class);

        // Then
        assertThat(response.getStatusCode()).isEqualTo(HttpStatus.CREATED);
        assertThat(response.getBody().getName()).isEqualTo("张三");
    }
}
```

**分层测试**：

```java
@SpringBootTest
@Testcontainers
class OrderRepositoryIT {

    @Container
    static MySQLContainer<?> mysql = new MySQLContainer<>("mysql:8.0")
        .withDatabaseName("testdb")
        .withUsername("test")
        .withPassword("test");

    @DynamicPropertySource
    static void configureProperties(DynamicPropertyRegistry registry) {
        registry.add("spring.datasource.url", mysql::getJdbcUrl);
        registry.add("spring.datasource.username", mysql::getUsername);
        registry.add("spring.datasource.password", mysql::getPassword);
    }

    @Autowired
    private OrderRepository orderRepository;

    @Test
    void testSaveAndFindOrder() {
        Order order = new Order(1L, BigDecimal.valueOf(99.99));
        Order saved = orderRepository.save(order);

        assertThat(saved.getId()).isNotNull();
        assertThat(orderRepository.findById(saved.getId()))
            .isPresent()
            .get()
            .extracting(Order::getTotalAmount)
            .isEqualByComparingTo(BigDecimal.valueOf(99.99));
    }
}
```

### 3. API测试（REST Assured）

**Maven依赖**：
```xml
<dependency>
    <groupId>io.rest-assured</groupId>
    <artifactId>rest-assured</artifactId>
    <version>5.4.0</version>
    <scope>test</scope>
</dependency>
```

```java
@SpringBootTest(webEnvironment = SpringBootTest.WebEnvironment.RANDOM_PORT)
class UserApiTest {

    @LocalServerPort
    private int port;

    @BeforeEach
    void setUp() {
        RestAssured.port = port;
        RestAssured.baseURI = "http://localhost";
    }

    @Test
    void testCreateAndRetrieveUser() {
        // 创建用户
        User created = given()
            .contentType(ContentType.JSON)
            .body(new UserRequest("张三", "zhangsan@test.com"))
        .when()
            .post("/api/users")
        .then()
            .statusCode(201)
            .extract().as(User.class);

        // 查询用户
        given()
            .pathParam("id", created.getId())
        .when()
            .get("/api/users/{id}")
        .then()
            .statusCode(200)
            .body("name", equalTo("张三"))
            .body("email", equalTo("zhangsan@test.com"));
    }
}
```

## 端到端测试

### 1. Selenium自动化

**Maven依赖**：
```xml
<dependency>
    <groupId>org.seleniumhq.selenium</groupId>
    <artifactId>selenium-java</artifactId>
    <version>4.18.1</version>
    <scope>test</scope>
</dependency>
```

```java
class LoginE2ETest {

    private WebDriver driver;

    @BeforeEach
    void setUp() {
        ChromeOptions options = new ChromeOptions();
        options.addArguments("--headless=new");
        options.addArguments("--no-sandbox");
        driver = new ChromeDriver(options);
    }

    @Test
    void testLoginFlow() {
        driver.get("http://localhost:8080/login");

        driver.findElement(By.id("username")).sendKeys("admin");
        driver.findElement(By.id("password")).sendKeys("password123");
        driver.findElement(By.id("login-btn")).click();

        WebDriverWait wait = new WebDriverWait(driver, Duration.ofSeconds(10));
        wait.until(ExpectedConditions.urlContains("/dashboard"));

        assertThat(driver.getCurrentUrl()).contains("/dashboard");
        assertThat(driver.findElement(By.id("welcome")).getText())
            .contains("欢迎, admin");
    }

    @AfterEach
    void tearDown() {
        if (driver != null) {
            driver.quit();
        }
    }
}
```

```java
class PlaywrightE2ETest {

        playwright.close();
    }
}
```

## 测试覆盖率

### 1. JaCoCo配置

**Maven插件**：
```xml
<plugin>
    <groupId>org.jacoco</groupId>
    <artifactId>jacoco-maven-plugin</artifactId>
    <version>0.8.11</version>
    <executions>
        <execution>
            <goals>
                <goal>prepare-agent</goal>
            </goals>
        </execution>
        <execution>
            <id>report</id>
            <phase>test</phase>
            <goals>
                <goal>report</goal>
            </goals>
        </execution>
        <execution>
            <id>check</id>
            <goals>
                <goal>check</goal>
            </goals>
            <configuration>
                <rules>
                    <rule>
                        <element>BUNDLE</element>
                        <limits>
                            <limit>
                                <counter>LINE</counter>
                                <value>COVEREDRATIO</value>
                                <minimum>0.80</minimum>
                            </limit>
                            <limit>
                                <counter>BRANCH</counter>
                                <value>COVEREDRATIO</value>
                                <minimum>0.70</minimum>
                            </limit>
                        </limits>
                    </rule>
                </rules>
            </configuration>
        </execution>
    </executions>
</plugin>
```

### 2. 覆盖率报告

执行 `mvn test` 后在 `target/site/jacoco/` 生成HTML报告。

**报告解读**：
- **行覆盖率(LINE)**：被执行的代码行比例
- **分支覆盖率(BRANCH)**：if/switch分支覆盖比例
- **方法覆盖率(METHOD)**：被调用的方法比例
- **类覆盖率(CLASS)**：被测试的类比例

### 3. 覆盖率要求

| 模块 | 行覆盖率 | 分支覆盖率 | 说明 |
|------|---------|-----------|------|
| **核心业务** | ≥80% | ≥70% | 订单、支付、用户 |
| **通用工具** | ≥90% | ≥80% | 工具类、帮助类 |
| **Controller** | ≥60% | ≥50% | 主要测API契约 |
| **配置类** | 可豁免 | 可豁免 | 自动配置类 |

## 最佳实践

### 1. 测试命名规范

```java
// 方法命名: test_{方法名}_{场景}_{预期结果}
@Test
void test_createOrder_whenStockSufficient_thenOrderCreated() {}

// 或使用DisplayName
@Test
@DisplayName("创建订单 - 库存充足时 - 订单创建成功")
void createOrder_success() {}
```

### 2. 测试数据管理

```java
// 使用TestFixture模式
@TestInstance(TestInstance.Lifecycle.PER_CLASS)
class OrderServiceTest {

    private OrderFixture fixture;

    @BeforeAll
    void initFixture() {
        fixture = new OrderFixture();
    }

    @Test
    void testWithDefaultOrder() {
        Order order = fixture.defaultOrder();
        // ...
    }

    @Test
    void testWithHighValueOrder() {
        Order order = fixture.defaultOrder()
            .withTotalAmount(BigDecimal.valueOf(99999));
        // ...
    }
}

// Fixture类
class OrderFixture {
    public Order defaultOrder() {
        return Order.builder()
            .userId(1L)
            .productId(100L)
            .quantity(1)
            .totalAmount(BigDecimal.valueOf(99.99))
            .build();
    }
}
```

### 3. 测试维护策略

- **消除重复**：使用`@BeforeEach`初始化公共数据，提取工具方法
- **避免测试间依赖**：每个测试独立运行，不依赖执行顺序
- **及时清理**：`@Transactional`自动回滚，或手动清理测试数据
- **CI集成**：在CI流水线中强制执行测试，覆盖率不达标阻止合并
- **定期审查**：移除过时测试，更新与业务变更不符的测试用例
