## 1. 手写快速排序与归并排序

### 1.1 面试题

```
Q1: 手写快速排序，说明时间/空间复杂度
Q2: 手写归并排序，说明稳定性和适用场景
```

### 1.2 解答思路

```
快速排序:
  分治思想: 选基准pivot → 小的放左大的放右 → 递归左右子数组
  时间: 平均O(NlogN), 最坏O(N²)(已排序数组+首元素作pivot)
  空间: O(logN)递归栈
  不稳定排序

归并排序:
  分治思想: 对半拆分 → 分别排序 → 合并两个有序数组
  时间: 始终O(NlogN)
  空间: O(N)辅助数组
  稳定排序，适合链表排序和外部排序
```

### 1.3 详细实现

**快速排序：**

```java
public class QuickSort {

    public static void sort(int[] arr) {
        if (arr == null || arr.length <= 1) return;
        quickSort(arr, 0, arr.length - 1);
    }

    private static void quickSort(int[] arr, int left, int right) {
        if (left >= right) return;

        // 随机选pivot避免最坏情况
        int pivotIdx = left + new Random().nextInt(right - left + 1);
        swap(arr, pivotIdx, right); // pivot放到最右

        int pivot = partition(arr, left, right);
        quickSort(arr, left, pivot - 1);
        quickSort(arr, pivot + 1, right);
    }

    // Lomuto分区: 小于pivot的放左边
    private static int partition(int[] arr, int left, int right) {
        int pivot = arr[right];
        int i = left; // i指向"小于pivot区域"的下一个位置
        for (int j = left; j < right; j++) {
            if (arr[j] < pivot) {
                swap(arr, i, j);
                i++;
            }
        }
        swap(arr, i, right); // pivot归位
        return i;
    }

    private static void swap(int[] arr, int i, int j) {
        int tmp = arr[i];
        arr[i] = arr[j];
        arr[j] = tmp;
    }
}
```

**归并排序：**

```java
public class MergeSort {

    public static void sort(int[] arr) {
        if (arr == null || arr.length <= 1) return;
        int[] temp = new int[arr.length]; // 辅助数组
        mergeSort(arr, 0, arr.length - 1, temp);
    }

    private static void mergeSort(int[] arr, int left, int right, int[] temp) {
        if (left >= right) return;
        int mid = left + (right - left) / 2;
        mergeSort(arr, left, mid, temp);      // 排左半
        mergeSort(arr, mid + 1, right, temp);  // 排右半
        merge(arr, left, mid, right, temp);     // 合并
    }

    // 合并两个有序子数组
    private static void merge(int[] arr, int left, int mid, int right, int[] temp) {
        int i = left, j = mid + 1, k = 0;

        while (i <= mid && j <= right) {
            if (arr[i] <= arr[j]) { // <=保证稳定性
                temp[k++] = arr[i++];
            } else {
                temp[k++] = arr[j++];
            }
        }
        while (i <= mid)  temp[k++] = arr[i++];
        while (j <= right) temp[k++] = arr[j++];

        // 复制回原数组
        System.arraycopy(temp, 0, arr, left, k);
    }
}
```

## 2. 手写反转链表/链表相交/环形链表

### 2.1 面试题

```
Q1: 手写反转链表(迭代+递归)
Q2: 判断链表是否有环，有环则找入环节点
Q3: 找两个链表的相交节点
```

### 2.2 解答思路

```
反转链表:
  迭代: pre/cur/next三指针，逐个反转指向
  递归: 反转后面的链表，让next.next指向自己

环形链表:
  快慢指针: slow走1步，fast走2步，相遇则有环
  找入环: 相遇后，一个从head出发，一个从相遇点出发，再相遇即为入环节点
  证明: 设head到入环距离a，入环到相遇点距离b，环长c
        slow走a+b, fast走a+b+kc → 2(a+b)=a+b+kc → a+b=kc → a=kc-b
        所以从head走a步 = 从相遇点走kc-b步 = 入环点

链表相交:
  双指针法: pA走A链表+B链表，pB走B链表+A链表
  两个指针走的总长度相同，必在相交点相遇(或同时到null)
```

### 2.3 详细实现

```java
public class LinkedListAlgorithms {

    static class ListNode {
        int val;
        ListNode next;
        ListNode(int val) { this.val = val; }
    }

    // 反转链表(迭代)
    public static ListNode reverseList(ListNode head) {
        ListNode prev = null;
        ListNode curr = head;
        while (curr != null) {
            ListNode next = curr.next;
            curr.next = prev; // 反转指向
            prev = curr;
            curr = next;
        }
        return prev; // 新头
    }

    // 反转链表(递归)
    public static ListNode reverseListRecursive(ListNode head) {
        if (head == null || head.next == null) return head;
        ListNode newHead = reverseListRecursive(head.next);
        head.next.next = head; // next指向自己
        head.next = null;      // 断开原指向
        return newHead;
    }

    // 环形链表检测
    public static boolean hasCycle(ListNode head) {
        ListNode slow = head, fast = head;
        while (fast != null && fast.next != null) {
            slow = slow.next;
            fast = fast.next.next;
            if (slow == fast) return true;
        }
        return false;
    }

    // 找入环节点
    public static ListNode detectCycle(ListNode head) {
        ListNode slow = head, fast = head;
        while (fast != null && fast.next != null) {
            slow = slow.next;
            fast = fast.next.next;
            if (slow == fast) {
                ListNode p = head;
                while (p != slow) {
                    p = p.next;
                    slow = slow.next;
                }
                return p; // 入环节点
            }
        }
        return null;
    }

    // 链表相交
    public static ListNode getIntersectionNode(ListNode headA, ListNode headB) {
        ListNode pA = headA, pB = headB;
        while (pA != pB) {
            pA = (pA == null) ? headB : pA.next;
            pB = (pB == null) ? headA : pB.next;
        }
        return pA; // 相交点或null
    }
}
```

## 3. 手写二叉树遍历

### 3.1 面试题

```
Q1: 手写二叉树的前中后序遍历(递归+迭代)
Q2: 手写层序遍历
Q3: 不用递归，用Morris遍历实现O(1)空间中序遍历
```

### 3.2 解答思路

```
递归遍历: 简单直观，但面试官常追问迭代版
迭代遍历: 用栈模拟递归
  前序: 入栈顺序 右→左(保证左先出)
  中序: 一路向左入栈，弹出后转向右子树
  后序: 前序的"根右左"反转 → "左右根"
层序: 队列BFS
Morris: 利用空闲right指针建立线索，O(1)空间
```

### 3.3 详细实现

```java
public class TreeTraversal {

    static class TreeNode {
        int val;
        TreeNode left, right;
        TreeNode(int val) { this.val = val; }
    }

    // === 递归版 ===
    public static void preOrder(TreeNode root) {
        if (root == null) return;
        System.out.print(root.val + " ");
        preOrder(root.left);
        preOrder(root.right);
    }

    public static void inOrder(TreeNode root) {
        if (root == null) return;
        inOrder(root.left);
        System.out.print(root.val + " ");
        inOrder(root.right);
    }

    public static void postOrder(TreeNode root) {
        if (root == null) return;
        postOrder(root.left);
        postOrder(root.right);
        System.out.print(root.val + " ");
    }

    // === 迭代版 ===

    // 前序迭代
    public static List<Integer> preOrderIter(TreeNode root) {
        List<Integer> result = new ArrayList<>();
        if (root == null) return result;
        Deque<TreeNode> stack = new ArrayDeque<>();
        stack.push(root);
        while (!stack.isEmpty()) {
            TreeNode node = stack.pop();
            result.add(node.val);
            if (node.right != null) stack.push(node.right);
            if (node.left != null) stack.push(node.left);
        }
        return result;
    }

    // 中序迭代
    public static List<Integer> inOrderIter(TreeNode root) {
        List<Integer> result = new ArrayList<>();
        Deque<TreeNode> stack = new ArrayDeque<>();
        TreeNode curr = root;
        while (curr != null || !stack.isEmpty()) {
            while (curr != null) {
                stack.push(curr);
                curr = curr.left;
            }
            curr = stack.pop();
            result.add(curr.val);
            curr = curr.right;
        }
        return result;
    }

    // 后序迭代: 前序(根右左)反转 → 后序(左右根)
    public static List<Integer> postOrderIter(TreeNode root) {
        List<Integer> result = new ArrayList<>();
        if (root == null) return result;
        Deque<TreeNode> stack = new ArrayDeque<>();
        stack.push(root);
        while (!stack.isEmpty()) {
            TreeNode node = stack.pop();
            result.add(node.val);
            if (node.left != null) stack.push(node.left);
            if (node.right != null) stack.push(node.right);
        }
        Collections.reverse(result);
        return result;
    }

    // 层序遍历(BFS)
    public static List<List<Integer>> levelOrder(TreeNode root) {
        List<List<Integer>> result = new ArrayList<>();
        if (root == null) return result;
        Queue<TreeNode> queue = new LinkedList<>();
        queue.offer(root);
        while (!queue.isEmpty()) {
            int levelSize = queue.size();
            List<Integer> level = new ArrayList<>();
            for (int i = 0; i < levelSize; i++) {
                TreeNode node = queue.poll();
                level.add(node.val);
                if (node.left != null) queue.offer(node.left);
                if (node.right != null) queue.offer(node.right);
            }
            result.add(level);
        }
        return result;
    }

    // Morris中序遍历: O(1)空间
    public static List<Integer> morrisInOrder(TreeNode root) {
        List<Integer> result = new ArrayList<>();
        TreeNode curr = root;
        while (curr != null) {
            if (curr.left == null) {
                result.add(curr.val);
                curr = curr.right;
            } else {
                TreeNode prev = curr.left;
                while (prev.right != null && prev.right != curr) {
                    prev = prev.right;
                }
                if (prev.right == null) {
                    prev.right = curr; // 建立线索
                    curr = curr.left;
                } else {
                    prev.right = null;  // 断开线索
                    result.add(curr.val);
                    curr = curr.right;
                }
            }
        }
        return result;
    }
}
```

## 4. 手写单例模式

### 4.1 面试题

```
Q: 手写5种单例模式实现，说明各自的优缺点和线程安全性
  1. 饿汉式  2. 懒汉式(synchronized)  3. DCL双重检查锁
  4. 静态内部类  5. 枚举单例
```

### 4.2 解答思路

```
核心问题: 线程安全 + 延迟加载 + 防止反射/序列化破坏
  饿汉式: 类加载时创建，线程安全但不能延迟加载
  懒汉式: synchronized保证安全但性能差
  DCL: 双重检查+volatile，安全且高性能
  静态内部类: 利用类加载机制，安全+延迟(推荐)
  枚举: 天然防反射和序列化(最佳)
```

### 4.3 详细实现

```java
// 1. 饿汉式(线程安全，非延迟加载)
public class EagerSingleton {
    private static final EagerSingleton INSTANCE = new EagerSingleton();
    private EagerSingleton() {}
    public static EagerSingleton getInstance() { return INSTANCE; }
}

// 2. 懒汉式(线程安全，性能差)
public class LazySingleton {
    private static LazySingleton instance;
    private LazySingleton() {}
    public static synchronized LazySingleton getInstance() {
        if (instance == null) instance = new LazySingleton();
        return instance;
    }
}

// 3. DCL双重检查锁(线程安全，高性能) ★常用
public class DCLSingleton {
    private static volatile DCLSingleton instance; // volatile禁止指令重排序
    private DCLSingleton() {}
    public static DCLSingleton getInstance() {
        if (instance == null) {              // 第一次检查(无锁)
            synchronized (DCLSingleton.class) {
                if (instance == null) {      // 第二次检查(有锁)
                    instance = new DCLSingleton();
                }
            }
        }
        return instance;
    }
}

// 4. 静态内部类(线程安全，延迟加载) ★推荐
public class InnerClassSingleton {
    private InnerClassSingleton() {}
    private static class Holder {
        static final InnerClassSingleton INSTANCE = new InnerClassSingleton();
    }
    public static InnerClassSingleton getInstance() { return Holder.INSTANCE; }
}

// 5. 枚举单例(最佳) ★★★
public enum EnumSingleton {
    INSTANCE;
    public void doSomething() { /* ... */ }
}
```

### 4.4 防止破坏单例

```java
// 防反射: 构造器检查
private DCLSingleton() {
    if (instance != null) throw new RuntimeException("不允许反射创建");
}

// 防序列化: readResolve
private Object readResolve() { return INSTANCE; }
```

## 5. 手写观察者模式

### 5.1 面试题

```
Q: 手写观察者模式，实现一个事件发布订阅系统
   要求: 注册观察者、移除观察者、通知所有观察者
```

### 5.2 解答思路

```
核心角色:
  Subject(被观察者): 维护观察者列表，状态变化时通知
  Observer(观察者): 定义更新接口
  ConcreteSubject/ConcreteObserver: 具体实现

  JDK中的应用:
    java.util.Observable(已废弃) / java.util.Observer(已废弃)
    替代方案: java.beans.PropertyChangeListener / EventBus / RxJava

  Spring中的应用:
    ApplicationEvent + ApplicationListener (事件驱动)
```

### 5.3 详细实现

```java
// 观察者接口
public interface Observer {
    void update(String event, Object data);
}

// 被观察者(事件总线)
public class EventBus {
    private final Map<String, List<Observer>> listeners = new ConcurrentHashMap<>();

    // 注册
    public void subscribe(String event, Observer observer) {
        listeners.computeIfAbsent(event, k -> new CopyOnWriteArrayList<>()).add(observer);
    }

    // 移除
    public void unsubscribe(String event, Observer observer) {
        List<Observer> list = listeners.get(event);
        if (list != null) list.remove(observer);
    }

    // 通知
    public void publish(String event, Object data) {
        List<Observer> list = listeners.get(event);
        if (list != null) {
            for (Observer observer : list) {
                observer.update(event, data);
            }
        }
    }
}

// 使用示例
EventBus bus = new EventBus();
bus.subscribe("order.created", (event, data) ->
    System.out.println("发送邮件: " + data));
bus.subscribe("order.created", (event, data) ->
    System.out.println("扣减库存: " + data));
bus.publish("order.created", "订单#12345");
```

## 6. 手写策略模式+工厂模式

### 6.1 面试题

```
Q: 用策略模式+工厂模式消除if-else，实现支付方式选择
   原始代码:
     if (type.equals("alipay")) { ... }
     else if (type.equals("wechat")) { ... }
     else if (type.equals("bank")) { ... }
```

### 6.2 解答思路

```
策略模式: 定义算法族，分别封装，让它们可以互相替换
工厂模式: 根据类型创建对应的策略对象
组合: 工厂负责创建，策略负责执行，客户端只依赖接口

  PayStrategy(策略接口)
    ├── AlipayStrategy
    ├── WechatPayStrategy
    └── BankPayStrategy
  PayStrategyFactory(工厂)
    └── getStrategy(type) → 对应Strategy实例
```

### 6.3 详细实现

```java
// 策略接口
public interface PayStrategy {
    PayResult pay(PayOrder order);
    String getType(); // Spring注入时自动注册
}

// 支付宝策略
@Component
public class AlipayStrategy implements PayStrategy {
    @Override
    public PayResult pay(PayOrder order) {
        // 调用支付宝SDK...
        return new PayResult(true, "支付宝支付成功");
    }
    @Override
    public String getType() { return "alipay"; }
}

// 微信策略
@Component
public class WechatPayStrategy implements PayStrategy {
    @Override
    public PayResult pay(PayOrder order) {
        return new PayResult(true, "微信支付成功");
    }
    @Override
    public String getType() { return "wechat"; }
}

// 策略工厂(Spring自动注册)
@Component
public class PayStrategyFactory implements InitializingBean {
    @Autowired
    private List<PayStrategy> strategies;
    private final Map<String, PayStrategy> strategyMap = new HashMap<>();

    @Override
    public void afterPropertiesSet() {
        for (PayStrategy s : strategies) {
            strategyMap.put(s.getType(), s);
        }
    }

    public PayStrategy getStrategy(String type) {
        PayStrategy strategy = strategyMap.get(type);
        if (strategy == null) throw new IllegalArgumentException("不支持的支付方式: " + type);
        return strategy;
    }
}

// 使用: 一行代码替代所有if-else
@Service
public class PayService {
    @Autowired private PayStrategyFactory factory;
    public PayResult pay(String type, PayOrder order) {
        return factory.getStrategy(type).pay(order); // 无if-else
    }
}
```

## 7. 手写简易版Spring Boot自动配置

### 7.1 面试题

```
Q: 手写简易版Spring Boot自动配置，说明@EnableAutoConfiguration原理
   要求: 自定义@EnableXxx + SPI自动加载 + 条件装配
```

### 7.2 解答思路

```
Spring Boot自动配置原理:
  1. @SpringBootApplication → @EnableAutoConfiguration
  2. @Import(AutoConfigurationImportSelector)
  3. 读取META-INF/spring.factories中的配置类
  4. @ConditionalOnXxx按条件决定是否装配
  5. 符合条件的@Bean注册到IoC容器
```

### 7.3 详细实现

```java
// 1. 自动配置类(条件装配)
@AutoConfiguration
@ConditionalOnClass(RedisTemplate.class)
@ConditionalOnProperty(name = "my.redis.enabled", havingValue = "true", matchIfMissing = true)
@EnableConfigurationProperties(MyRedisProperties.class)
public class MyRedisAutoConfiguration {

    @Bean
    @ConditionalOnMissingBean
    public RedisTemplate<String, Object> redisTemplate(
            RedisConnectionFactory factory) {
        RedisTemplate<String, Object> template = new RedisTemplate<>();
        template.setConnectionFactory(factory);
        template.setKeySerializer(new StringRedisSerializer());
        template.setValueSerializer(new GenericJackson2JsonRedisSerializer());
        return template;
    }
}

// 2. 配置属性
@ConfigurationProperties(prefix = "my.redis")
public class MyRedisProperties {
    private boolean enabled = true;
    private String prefix;
}

// 3. spring.factories注册
// META-INF/spring.factories:
// org.springframework.boot.autoconfigure.EnableAutoConfiguration=\
//   com.example.starter.MyRedisAutoConfiguration

// 4. 自定义@EnableXxx
@Target(ElementType.TYPE)
@Retention(RetentionPolicy.RUNTIME)
@Import(MyRedisAutoConfiguration.class)
public @interface EnableMyRedis {}

// 5. ImportSelector方式(更灵活，批量导入)
public class MyImportSelector implements ImportSelector {
    @Override
    public String[] selectImports(AnnotationMetadata metadata) {
        return new String[]{
            "com.example.starter.MyRedisAutoConfiguration",
            "com.example.starter.MyCacheAutoConfiguration"
        };
    }
}
```

## 8. 手写简易版MyBatis

### 8.1 面试题

```
Q: 手写简易版MyBatis，实现Mapper接口代理和SQL执行
   要求: @Select/@Insert注解 + MapperProxy动态代理 + SqlSession执行
```

### 8.2 解答思路

```
MyBatis核心流程:
  SqlSessionFactory → SqlSession → getMapper() → JDK动态代理
  → InvocationHandler拦截 → 解析注解获取SQL → JDBC执行 → 结果映射
```

### 8.3 详细实现

```java
// 1. SQL注解
@Target(ElementType.METHOD) @Retention(RetentionPolicy.RUNTIME)
public @interface Select { String value(); }

@Target(ElementType.METHOD) @Retention(RetentionPolicy.RUNTIME)
public @interface Insert { String value(); }

// 2. Mapper接口
public interface UserMapper {
    @Select("SELECT id, name, email FROM user WHERE id = #{id}")
    User findById(int id);
    @Insert("INSERT INTO user(name, email) VALUES(#{name}, #{email})")
    int insert(User user);
}

// 3. MapperProxy(JDK动态代理)
public class MapperProxy<T> implements InvocationHandler {
    private final Class<T> mapperInterface;
    private final SqlSession sqlSession;

    public MapperProxy(Class<T> mapperInterface, SqlSession sqlSession) {
        this.mapperInterface = mapperInterface;
        this.sqlSession = sqlSession;
    }

    @Override
    public Object invoke(Object proxy, Method method, Object[] args) throws Throwable {
        if (method.getDeclaringClass() == Object.class) return method.invoke(this, args);
        if (method.isAnnotationPresent(Select.class)) {
            String sql = method.getAnnotation(Select.class).value();
            sql = sql.replaceAll("#\\{\\w+}", "?"); // #{id} → ?
            return sqlSession.selectList(sql, args, method.getReturnType());
        }
        if (method.isAnnotationPresent(Insert.class)) {
            String sql = method.getAnnotation(Insert.class).value();
            return sqlSession.update(sql, args);
        }
        throw new UnsupportedOperationException("不支持: " + method.getName());
    }
}

// 4. SqlSession
public class SqlSession {
    private final Connection connection;
    public SqlSession(Connection connection) { this.connection = connection; }

    public <T> List<T> selectList(String sql, Object[] args, Class<?> returnType) {
        try (PreparedStatement ps = connection.prepareStatement(sql)) {
            if (args != null) for (int i = 0; i < args.length; i++) ps.setObject(i+1, args[i]);
            ResultSet rs = ps.executeQuery();
            List<T> result = new ArrayList<>();
            while (rs.next()) {
                T obj = (T) returnType.getDeclaredConstructor().newInstance();
                for (Field f : returnType.getDeclaredFields()) {
                    f.setAccessible(true);
                    try { f.set(obj, rs.getObject(f.getName())); } catch (SQLException ignored) {}
                }
                result.add(obj);
            }
            return result;
        } catch (Exception e) { throw new RuntimeException("SQL执行失败", e); }
    }

    public int update(String sql, Object[] args) {
        try (PreparedStatement ps = connection.prepareStatement(sql)) {
            if (args != null) for (int i = 0; i < args.length; i++) ps.setObject(i+1, args[i]);
            return ps.executeUpdate();
        } catch (SQLException e) { throw new RuntimeException("SQL执行失败", e); }
    }

    @SuppressWarnings("unchecked")
    public <T> T getMapper(Class<T> mapperInterface) {
        return (T) Proxy.newProxyInstance(
            mapperInterface.getClassLoader(),
            new Class[]{mapperInterface},
            new MapperProxy<>(mapperInterface, this));
    }
}

// 5. 使用
SqlSession session = new SqlSession(conn);
UserMapper mapper = session.getMapper(UserMapper.class);
User user = mapper.findById(1);
```
