## 1. 手写HashMap

### 1.1 面试题

```
Q: 请手写一个简易版HashMap，要求：
  1. 实现put(key, value)和get(key)方法
  2. 使用数组+链表解决哈希冲突
  3. 支持扩容
  4. 说明与JDK HashMap的区别
```

### 1.2 解答思路

```
核心设计:
  1. 数据结构: Node数组(table) + 链表(拉链法)
  2. 哈希函数: hash(key) % table.length
  3. 扩容: 当size > capacity * loadFactor时，数组扩容2倍
  4. 查找: hash定位桶 → 遍历链表比较key

  与JDK HashMap的区别:
    - JDK 8+ 链表长度>8转红黑树(这里不实现)
    - JDK用扰动函数(hash ^ hash>>>16)减少碰撞
    - JDK扩容时优化了rehash(高位bit判断)
    - JDK有并发安全版本ConcurrentHashMap
```

### 1.3 详细实现

```java
public class MyHashMap<K, V> {

    // Node节点
    private static class Node<K, V> {
        final K key;
        V value;
        Node<K, V> next;

        Node(K key, V value) {
            this.key = key;
            this.value = value;
        }
    }

    private Node<K, V>[] table;
    private int size;
    private final float loadFactor;
    private static final int DEFAULT_CAPACITY = 16;
    private static final float DEFAULT_LOAD_FACTOR = 0.75f;

    @SuppressWarnings("unchecked")
    public MyHashMap() {
        this.table = new Node[DEFAULT_CAPACITY];
        this.loadFactor = DEFAULT_LOAD_FACTOR;
        this.size = 0;
    }

    public MyHashMap(int capacity, float loadFactor) {
        this.table = new Node[capacity];
        this.loadFactor = loadFactor;
        this.size = 0;
    }

    // 哈希函数(简化版，JDK用扰动函数)
    private int hash(K key) {
        if (key == null) return 0;
        int h = key.hashCode();
        // 扰动函数: 高16位异或低16位，减少碰撞
        return (h ^ (h >>> 16)) & (table.length - 1);
    }

    // put: 插入或更新
    public void put(K key, V value) {
        // 检查是否需要扩容
        if (size >= table.length * loadFactor) {
            resize();
        }

        int index = hash(key);
        Node<K, V> node = table[index];

        if (node == null) {
            // 桶为空，直接放入
            table[index] = new Node<>(key, value);
            size++;
            return;
        }

        // 遍历链表
        Node<K, V> prev = null;
        while (node != null) {
            // key相同则更新
            if (node.key.equals(key)) {
                node.value = value;
                return;
            }
            prev = node;
            node = node.next;
        }

        // 链表尾部追加(尾插法，JDK8用尾插，JDK7用头插)
        prev.next = new Node<>(key, value);
        size++;
    }

    // get: 查找
    public V get(K key) {
        int index = hash(key);
        Node<K, V> node = table[index];

        while (node != null) {
            if (node.key.equals(key)) {
                return node.value;
            }
            node = node.next;
        }
        return null;
    }

    // remove: 删除
    public V remove(K key) {
        int index = hash(key);
        Node<K, V> node = table[index];
        Node<K, V> prev = null;

        while (node != null) {
            if (node.key.equals(key)) {
                if (prev == null) {
                    table[index] = node.next; // 删除头节点
                } else {
                    prev.next = node.next;    // 删除中间/尾节点
                }
                size--;
                return node.value;
            }
            prev = node;
            node = node.next;
        }
        return null;
    }

    // 扩容: 数组容量翻倍，所有元素rehash
    @SuppressWarnings("unchecked")
    private void resize() {
        Node<K, V>[] oldTable = table;
        table = new Node[oldTable.length * 2];
        size = 0; // 重新计数

        for (Node<K, V> head : oldTable) {
            Node<K, V> node = head;
            while (node != null) {
                put(node.key, node.value); // rehash
                node = node.next;
            }
        }
    }

    public int size() {
        return size;
    }
}
```

### 1.4 扩展追问

```
Q: JDK 8的HashMap为什么链表长度>8要转红黑树？
A: 红黑树查找O(logN) vs 链表O(N)。哈希分布均匀时链表极短(泊松分布概率0.00000006)，
   转5红黑树是极端情况兜底。选8是权衡: 太小浪费树化开销, 太大失去意义。

Q: 为什么容量必须是2的幂？
A: hash & (n-1) 等价于 hash % n，但位运算更快。扩容2倍时只需判断高位bit即可rehash。

Q: JDK 7头插法为什么会导致死循环？
A: 多线程扩容时，头插法会反转链表顺序，两个线程同时扩容可能形成环形链表，
   get时死循环。JDK 8改用尾插法解决了这个问题(但HashMap仍然不是线程安全的)。
```

## 2. 手写ArrayList

### 2.1 面试题

```
Q: 请手写一个简易版ArrayList，要求：
  1. 实现add(E)、get(int)、remove(int)、size()方法
  2. 支持动态扩容
  3. 说明扩容策略
```

### 2.2 解答思路

```
核心设计:
  1. 底层: Object数组
  2. 扩容: 当size == capacity时，新容量 = 旧容量 * 1.5(JDK策略)
  3. 删除: System.arraycopy移动元素
  4. 范围检查: index >= 0 && index < size

  JDK ArrayList扩容策略:
    - 初始容量10(首次add时创建)
    - 新容量 = 旧容量 + 旧容量 >> 1 (即1.5倍)
    - Arrays.copyOf复制数组
```

### 2.3 详细实现

```java
public class MyArrayList<E> {

    private Object[] elementData;
    private int size;
    private static final int DEFAULT_CAPACITY = 10;

    public MyArrayList() {
        this.elementData = new Object[DEFAULT_CAPACITY];
    }

    public MyArrayList(int initialCapacity) {
        if (initialCapacity < 0) {
            throw new IllegalArgumentException("Illegal capacity: " + initialCapacity);
        }
        this.elementData = new Object[initialCapacity];
    }

    // add: 添加元素
    public boolean add(E e) {
        ensureCapacity(size + 1);
        elementData[size++] = e;
        return true;
    }

    // add(index): 指定位置插入
    public void add(int index, E e) {
        rangeCheckForAdd(index);
        ensureCapacity(size + 1);
        // 将index及之后的元素后移一位
        System.arraycopy(elementData, index, elementData, index + 1, size - index);
        elementData[index] = e;
        size++;
    }

    // get: 获取元素
    @SuppressWarnings("unchecked")
    public E get(int index) {
        rangeCheck(index);
        return (E) elementData[index];
    }

    // remove: 删除元素
    @SuppressWarnings("unchecked")
    public E remove(int index) {
        rangeCheck(index);
        E oldValue = (E) elementData[index];
        // 将index之后的元素前移一位
        int numMoved = size - index - 1;
        if (numMoved > 0) {
            System.arraycopy(elementData, index + 1, elementData, index, numMoved);
        }
        elementData[--size] = null; // 帮助GC
        return oldValue;
    }

    // set: 替换元素
    @SuppressWarnings("unchecked")
    public E set(int index, E e) {
        rangeCheck(index);
        E oldValue = (E) elementData[index];
        elementData[index] = e;
        return oldValue;
    }

    // 确保容量
    private void ensureCapacity(int minCapacity) {
        if (minCapacity > elementData.length) {
            grow();
        }
    }

    // 扩容: 1.5倍
    private void grow() {
        int oldCapacity = elementData.length;
        int newCapacity = oldCapacity + (oldCapacity >> 1); // 1.5倍
        elementData = Arrays.copyOf(elementData, newCapacity);
    }

    private void rangeCheck(int index) {
        if (index < 0 || index >= size) {
            throw new IndexOutOfBoundsException("Index: " + index + ", Size: " + size);
        }
    }

    private void rangeCheckForAdd(int index) {
        if (index < 0 || index > size) {
            throw new IndexOutOfBoundsException("Index: " + index + ", Size: " + size);
        }
    }

    public int size() {
        return size;
    }

    public boolean isEmpty() {
        return size == 0;
    }
}
```

## 3. 手写LinkedList

### 3.1 面试题

```
Q: 请手写一个简易版双向链表LinkedList，要求：
  1. 实现add(E)、get(int)、remove(int)方法
  2. 双向链表结构
  3. 实现addFirst/addLast/getFirst/getLast
```

### 3.2 解答思路

```
核心设计:
  1. 数据结构: Node节点(prev + next + item)
  2. 维护head和tail两个指针
  3. 头插/尾插O(1), 中间插入O(n)
  4. 删除O(1)(已知节点) 或 O(n)(需先查找)

  vs ArrayList:
    ArrayList: 查找O(1), 插入删除O(n)
    LinkedList: 查找O(n), 头尾插入删除O(1)
```

### 3.3 详细实现

```java
public class MyLinkedList<E> {

    private static class Node<E> {
        E item;
        Node<E> prev;
        Node<E> next;

        Node(E item, Node<E> prev, Node<E> next) {
            this.item = item;
            this.prev = prev;
            this.next = next;
        }
    }

    private Node<E> first;
    private Node<E> last;
    private int size;

    // 尾插法
    public boolean add(E e) {
        linkLast(e);
        return true;
    }

    // 指定位置插入
    public void add(int index, E e) {
        rangeCheckForAdd(index);
        if (index == size) {
            linkLast(e);
        } else {
            linkBefore(e, node(index));
        }
    }

    // 头插
    public void addFirst(E e) {
        linkFirst(e);
    }

    // 尾插
    public void addLast(E e) {
        linkLast(e);
    }

    // 获取
    public E get(int index) {
        rangeCheck(index);
        return node(index).item;
    }

    public E getFirst() {
        if (first == null) throw new NoSuchElementException();
        return first.item;
    }

    public E getLast() {
        if (last == null) throw new NoSuchElementException();
        return last.item;
    }

    // 删除
    public E remove(int index) {
        rangeCheck(index);
        return unlink(node(index));
    }

    // --- 核心操作 ---

    private void linkFirst(E e) {
        Node<E> oldFirst = first;
        Node<E> newNode = new Node<>(e, null, oldFirst);
        first = newNode;
        if (oldFirst == null) {
            last = newNode; // 空链表
        } else {
            oldFirst.prev = newNode;
        }
        size++;
    }

    private void linkLast(E e) {
        Node<E> oldLast = last;
        Node<E> newNode = new Node<>(e, oldLast, null);
        last = newNode;
        if (oldLast == null) {
            first = newNode; // 空链表
        } else {
            oldLast.next = newNode;
        }
        size++;
    }

    private void linkBefore(E e, Node<E> succ) {
        Node<E> pred = succ.prev;
        Node<E> newNode = new Node<>(e, pred, succ);
        succ.prev = newNode;
        if (pred == null) {
            first = newNode;
        } else {
            pred.next = newNode;
        }
        size++;
    }

    private E unlink(Node<E> node) {
        E item = node.item;
        Node<E> prev = node.prev;
        Node<E> next = node.next;

        if (prev == null) {
            first = next;
        } else {
            prev.next = next;
            node.prev = null;
        }

        if (next == null) {
            last = prev;
        } else {
            next.prev = prev;
            node.next = null;
        }

        node.item = null;
        size--;
        return item;
    }

    // 查找节点(优化: 前半段从头找, 后半段从尾找)
    private Node<E> node(int index) {
        if (index < (size >> 1)) {
            Node<E> x = first;
            for (int i = 0; i < index; i++) {
                x = x.next;
            }
            return x;
        } else {
            Node<E> x = last;
            for (int i = size - 1; i > index; i--) {
                x = x.prev;
            }
            return x;
        }
    }

    private void rangeCheck(int index) {
        if (index < 0 || index >= size) {
            throw new IndexOutOfBoundsException("Index: " + index + ", Size: " + size);
        }
    }

    private void rangeCheckForAdd(int index) {
        if (index < 0 || index > size) {
            throw new IndexOutOfBoundsException("Index: " + index + ", Size: " + size);
        }
    }

    public int size() {
        return size;
    }
}
```

## 4. 手写线程池

### 4.1 面试题

```
Q: 请手写一个简易版线程池，要求：
  1. 核心线程数 + 最大线程数 + 阻塞队列
  2. 任务超过核心线程数时入队列
  3. 队列满时创建非核心线程(不超过最大线程数)
  4. 拒绝策略(任务太多时)
  5. 说明与JDK ThreadPoolExecutor的区别
```

### 4.2 解答思路

```
核心流程(JDK ThreadPoolExecutor的execute方法):
  1. 当前线程数 < corePoolSize → 创建新线程执行
  2. 当前线程数 >= corePoolSize → 任务入队列
  3. 队列满 + 线程数 < maxPoolSize → 创建非核心线程
  4. 队列满 + 线程数 >= maxPoolSize → 执行拒绝策略

  状态管理:
    - RUNNING: 接受新任务，处理队列任务
    - SHUTDOWN: 不接受新任务，处理队列任务
    - STOP: 不接受新任务，不处理队列，中断进行中任务
    - TERMINATED: 完全终止

  与JDK的区别:
    - JDK用CAS+ReentrantLock保证线程安全
    - JDK的Worker继承了AQS实现独占锁
    - JDK支持空闲线程超时回收(keepAliveTime)
    - JDK支持prestartAllCoreThreads预热核心线程
```

### 4.3 详细实现

```java
public class MyThreadPool {

    // 任务队列
    private final BlockingQueue<Runnable> workQueue;
    // 工作线程集合
    private final HashSet<Worker> workers = new HashSet<>();
    // 核心线程数
    private final int corePoolSize;
    // 最大线程数
    private final int maximumPoolSize;
    // 拒绝策略
    private final RejectedExecutionHandler handler;
    // 线程工厂
    private final ThreadFactory threadFactory;
    // 锁
    private final ReentrantLock mainLock = new ReentrantLock();
    // 线程池状态
    private volatile boolean isShutdown = false;

    // Worker: 包装线程和任务
    private class Worker implements Runnable {
        final Thread thread;
        Runnable firstTask;

        Worker(Runnable firstTask) {
            this.firstTask = firstTask;
            this.thread = threadFactory.newThread(this);
        }

        @Override
        public void run() {
            Runnable task = firstTask;
            firstTask = null;
            try {
                while (task != null || (task = getTask()) != null) {
                    try {
                        task.run();
                    } catch (Exception e) {
                        e.printStackTrace();
                    } finally {
                        task = null;
                    }
                }
            } finally {
                workerExit(this);
            }
        }
    }

    // 从队列获取任务(阻塞)
    private Runnable getTask() {
        while (true) {
            if (isShutdown && workQueue.isEmpty()) {
                return null;
            }
            try {
                Runnable task = workQueue.poll(1, TimeUnit.SECONDS);
                if (task != null) return task;
                // 空闲超时可在此处回收非核心线程(简化版不实现)
            } catch (InterruptedException e) {
                return null;
            }
        }
    }

    public MyThreadPool(int corePoolSize, int maximumPoolSize,
                        int queueCapacity, RejectedExecutionHandler handler) {
        this.corePoolSize = corePoolSize;
        this.maximumPoolSize = maximumPoolSize;
        this.workQueue = new ArrayBlockingQueue<>(queueCapacity);
        this.handler = handler;
        this.threadFactory = Executors.defaultThreadFactory();
    }

    // 执行任务(核心方法)
    public void execute(Runnable command) {
        if (command == null) throw new NullPointerException();

        if (isShutdown) {
            reject(command);
            return;
        }

        // 1. 线程数 < 核心线程数 → 创建核心线程
        if (workers.size() < corePoolSize) {
            if (addWorker(command, true)) return;
        }

        // 2. 入队列
        if (workQueue.offer(command)) {
            return;
        }

        // 3. 队列满 → 创建非核心线程
        if (workers.size() < maximumPoolSize) {
            if (addWorker(command, false)) return;
        }

        // 4. 超过最大线程数 → 拒绝
        reject(command);
    }

    // 添加工作线程
    private boolean addWorker(Runnable firstTask, boolean core) {
        mainLock.lock();
        try {
            if (isShutdown) return false;
            int limit = core ? corePoolSize : maximumPoolSize;
            if (workers.size() >= limit) return false;

            Worker worker = new Worker(firstTask);
            workers.add(worker);
            worker.thread.start();
            return true;
        } finally {
            mainLock.unlock();
        }
    }

    // Worker退出
    private void workerExit(Worker worker) {
        mainLock.lock();
        try {
            workers.remove(worker);
        } finally {
            mainLock.unlock();
        }
    }

    // 拒绝策略
    private void reject(Runnable command) {
        handler.rejectedExecution(command, this);
    }

    // 关闭线程池
    public void shutdown() {
        mainLock.lock();
        try {
            isShutdown = true;
            for (Worker worker : workers) {
                worker.thread.interrupt();
            }
        } finally {
            mainLock.unlock();
        }
    }

    // 拒绝策略接口
    public interface RejectedExecutionHandler {
        void rejectedExecution(Runnable r, MyThreadPool executor);
    }

    // 默认拒绝策略: AbortPolicy(抛异常)
    public static class AbortPolicy implements RejectedExecutionHandler {
        @Override
        public void rejectedExecution(Runnable r, MyThreadPool executor) {
            throw new RejectedExecutionException("Task rejected: pool is full");
        }
    }

    // CallerRunsPolicy: 调用者线程执行
    public static class CallerRunsPolicy implements RejectedExecutionHandler {
        @Override
        public void rejectedExecution(Runnable r, MyThreadPool executor) {
            if (!executor.isShutdown) {
                r.run();
            }
        }
    }
}
```

## 5. 手写BlockingQueue

### 5.1 面试题

```
Q: 请手写一个简易版BlockingQueue，要求：
  1. 基于数组的循环队列
  2. put时队列满则阻塞等待
  3. take时队列空则阻塞等待
  4. 使用wait/notify实现(不用ReentrantLock)
```

### 5.2 解答思路

```
核心设计:
  1. 循环数组: head和tail指针，(tail+1) % capacity == head表示满
  2. synchronized + wait/notifyAll实现阻塞
  3. put: 队列满时wait，放入后notifyAll
  4. take: 队列空时wait，取出后notifyAll

  注意:
    - 用while判断条件(不用if)防止虚假唤醒
    - notifyAll而非notify(防止只有put线程被唤醒导致死锁)
```

### 5.3 详细实现

```java
public class MyBlockingQueue<E> {

    private final Object[] items;
    private int head; // 队头(取元素)
    private int tail; // 队尾(放元素)
    private int count; // 当前元素数
    private final Object lock = new Object();

    public MyBlockingQueue(int capacity) {
        if (capacity <= 0) throw new IllegalArgumentException();
        this.items = new Object[capacity];
        this.head = 0;
        this.tail = 0;
        this.count = 0;
    }

    // put: 放入元素(队列满时阻塞)
    @SuppressWarnings("unchecked")
    public void put(E e) throws InterruptedException {
        synchronized (lock) {
            // while不用if: 防止虚假唤醒
            while (count == items.length) {
                lock.wait();
            }
            items[tail] = e;
            tail = (tail + 1) % items.length;
            count++;
            lock.notifyAll(); // 通知可能等待的take线程
        }
    }

    // take: 取出元素(队列空时阻塞)
    @SuppressWarnings("unchecked")
    public E take() throws InterruptedException {
        synchronized (lock) {
            while (count == 0) {
                lock.wait();
            }
            E item = (E) items[head];
            items[head] = null; // 帮助GC
            head = (head + 1) % items.length;
            count--;
            lock.notifyAll(); // 通知可能等待的put线程
        }
        return item;
    }

    // 非阻塞offer
    public boolean offer(E e) {
        synchronized (lock) {
            if (count == items.length) return false;
            items[tail] = e;
            tail = (tail + 1) % items.length;
            count++;
            lock.notifyAll();
            return true;
        }
    }

    // 非阻塞poll
    @SuppressWarnings("unchecked")
    public E poll() {
        synchronized (lock) {
            if (count == 0) return null;
            E item = (E) items[head];
            items[head] = null;
            head = (head + 1) % items.length;
            count--;
            lock.notifyAll();
            return item;
        }
    }

    public int size() {
        synchronized (lock) {
            return count;
        }
    }
}
```

## 6. 手写LRU缓存

### 6.1 面试题

```
Q: 请手写一个LRU(最近最少使用)缓存，要求：
  1. 实现get(key)和put(key, value)方法
  2. get和put都是O(1)时间复杂度
  3. 超过容量时淘汰最近最少使用的元素
  4. 使用HashMap + 双向链表实现
```

### 6.2 解答思路

```
核心设计:
  1. HashMap: key → Node(快速查找O(1))
  2. 双向链表: 维护访问顺序(最近访问的在头部，最久未访问的在尾部)
  3. get: HashMap查找 → 移动到链表头部
  4. put: 新增/更新 → 移动到头部 → 超容量则删除尾部

  为什么不用LinkedHashMap？
    LinkedHashMap内置accessOrder可以实现LRU，但面试要求手写。
    实际生产中直接用LinkedHashMap或Caffeine。

  O(1)分析:
    HashMap查找: O(1)
    双向链表节点移动: O(1)(已知节点，直接操作指针)
```

### 6.3 详细实现

```java
public class LRUCache<K, V> {

    // 双向链表节点
    private static class Node<K, V> {
        K key;
        V value;
        Node<K, V> prev;
        Node<K, V> next;

        Node(K key, V value) {
            this.key = key;
            this.value = value;
        }
    }

    private final int capacity;
    private final Map<K, Node<K, V>> cache;
    // 哨兵节点: 简化边界处理
    private final Node<K, V> head; // 头部(最近使用)
    private final Node<K, V> tail; // 尾部(最久未使用)

    public LRUCache(int capacity) {
        this.capacity = capacity;
        this.cache = new HashMap<>(capacity);
        // 初始化哨兵节点
        this.head = new Node<>(null, null);
        this.tail = new Node<>(null, null);
        head.next = tail;
        tail.prev = head;
    }

    // get: 获取并移动到头部
    public V get(K key) {
        Node<K, V> node = cache.get(key);
        if (node == null) {
            return null; // 未命中
        }
        moveToHead(node); // 命中后移到头部
        return node.value;
    }

    // put: 插入或更新
    public void put(K key, V value) {
        Node<K, V> node = cache.get(key);
        if (node != null) {
            // 更新已存在的key
            node.value = value;
            moveToHead(node);
        } else {
            // 新增key
            Node<K, V> newNode = new Node<>(key, value);
            cache.put(key, newNode);
            addToHead(newNode);

            // 超容量 → 淘汰尾部
            if (cache.size() > capacity) {
                Node<K, V> lru = removeTail();
                cache.remove(lru.key);
            }
        }
    }

    // --- 双向链表操作 ---

    // 添加到头部(哨兵之后)
    private void addToHead(Node<K, V> node) {
        node.prev = head;
        node.next = head.next;
        head.next.prev = node;
        head.next = node;
    }

    // 删除节点
    private void removeNode(Node<K, V> node) {
        node.prev.next = node.next;
        node.next.prev = node.prev;
    }

    // 移动到头部
    private void moveToHead(Node<K, V> node) {
        removeNode(node);
        addToHead(node);
    }

    // 删除尾部(最久未使用)
    private Node<K, V> removeTail() {
        Node<K, V> lru = tail.prev;
        removeNode(lru);
        return lru;
    }

    // 打印当前缓存(调试用)
    public void printCache() {
        Node<K, V> node = head.next;
        System.out.print("LRU Cache: ");
        while (node != tail) {
            System.out.print("[" + node.key + ":" + node.value + "] ");
            node = node.next;
        }
        System.out.println();
    }
}
```

## 7. 面试题速查

**Q1: 手写HashMap时如何处理哈希冲突？**

```
拉链法: 数组+链表，相同hash的元素用链表连接
扩容: size > capacity * 0.75时数组翻倍，rehash
JDK 8+: 链表>8且数组>=64转红黑树(O(logN)查找)
```

**Q2: ArrayList和LinkedList扩容区别？**

```
ArrayList: 数组扩容1.5倍，Arrays.copyOf复制
LinkedList: 双向链表不需要扩容，链表无限延伸
```

**Q3: 线程池的核心流程？**

```
1. 线程数<core → 新建核心线程
2. 线程数>=core → 入队列
3. 队列满+线程数<max → 新建非核心线程
4. 队列满+线程数>=max → 拒绝策略
```

**Q4: BlockingQueue为什么用while不用if判断条件？**

```
防止虚假唤醒(spurious wakeup)
wait()可能在没有notify的情况下返回
while会重新检查条件，if不会 → 可能导致队列满时仍put
```

**Q5: LRU为什么用双向链表+HashMap？**

```
HashMap: O(1)查找key对应的节点
双向链表: O(1)移动节点到头部(已知节点，直接操作prev/next)
单向链表: 删除节点需要前驱节点(O(N)查找)
哨兵节点: 简化头尾边界处理，避免null检查
```
