### 3.2 短链接生成算法

**方案对比**：

| 方案 | 优点 | 缺点 | 推荐度 |
|------|------|------|--------|
| 自增ID + Base62 | 简单、唯一 | 可预测、可爬取 | ⭐⭐⭐⭐ |
| Hash + 冲突检测 | 不可预测 | 可能冲突 | ⭐⭐⭐ |
| 雪花算法 | 分布式、唯一 | 较长 | ⭐⭐⭐⭐⭐ |
| UUID | 绝对唯一 | 太长(36位) | ⭐⭐ |

**推荐方案：自增ID + Base62编码**

```java
public class ShortUrlGenerator {
    private static final String BASE62 = 
        "0123456789abcdefghijklmnopqrstuvwxyzABCDEFGHIJKLMNOPQRSTUVWXYZ";
    
    /**
     * 将数字ID转为Base62字符串
     * @param id 自增ID
     * @return Base62编码的短链接key
     */
    public String toBase62(long id) {
        if (id == 0) return "0";
        
        StringBuilder sb = new StringBuilder();
        while (id > 0) {
            sb.append(BASE62.charAt((int)(id % 62)));
            id /= 62;
        }
        return sb.reverse().toString();
    }
    
    /**
     * 将Base62字符串转回数字ID
     */
    public long fromBase62(String str) {
        long result = 0;
        for (char c : str.toCharArray()) {
            result = result * 62 + BASE62.indexOf(c);
        }
        return result;
    }
}

// 示例：
// id=1      → Base62 → "1"
// id=62     → Base62 → "10"
// id=12345  → Base62 → "3d7"
// id=1000000 → Base62 → "4c92"  (6位)
```

**为什么选Base62**：
```
Base62 = 数字(0-9) + 小写字母(a-z) + 大写字母(A-Z)
       = 10 + 26 + 26 = 62个字符

容量计算：
  6位Base62 = 62^6 = 568亿条
  7位Base62 = 62^7 = 3.5万亿条  ✅ 足够使用

短URL示例：
  https://short.ly/abc123  (6位)
```

### 3.4 核心代码实现

**创建短链接**：
```java
@Service
public class ShortUrlService {
    @Autowired
    private ShortUrlMapper mapper;
    
    @Autowired
    private RedisTemplate<String, String> redisTemplate;
    
    @Autowired
    private ShortUrlGenerator generator;
    
    @Transactional
    public ShortUrlDTO createShortUrl(String originalUrl, Integer expireDays) {
        // 1. 检查URL是否已存在（可选，避免重复）
        ShortUrl existing = mapper.findByOriginalUrl(originalUrl);
        if (existing != null && existing.getStatus() == 1) {
            return convertToDTO(existing);
        }
        
        // 2. 插入数据库获取自增ID
        ShortUrl shortUrl = new ShortUrl();
        shortUrl.setOriginalUrl(originalUrl);
        shortUrl.setStatus(1);
        if (expireDays != null) {
            shortUrl.setExpireAt(
                LocalDateTime.now().plusDays(expireDays)
            );
        }
        mapper.insert(shortUrl);  // 获得自增ID
        
        // 3. 生成短链接key（Base62编码ID）
        String shortKey = generator.toBase62(shortUrl.getId());
        shortUrl.setShortKey(shortKey);
        mapper.updateShortKey(shortUrl.getId(), shortKey);
        
        // 4. 写入Redis缓存
        cacheShortUrl(shortKey, shortUrl);
        
        return convertToDTO(shortUrl);
    }
    
    private void cacheShortUrl(String shortKey, ShortUrl shortUrl) {
        String cacheKey = "short:" + shortKey;
        Map<String, Object> data = new HashMap<>();
        data.put("original_url", shortUrl.getOriginalUrl());
        data.put("expire_at", shortUrl.getExpireAt());
        data.put("status", shortUrl.getStatus());
        
        redisTemplate.opsForHash().putAll(cacheKey, data);
        redisTemplate.expire(cacheKey, 1, TimeUnit.HOURS);
    }
}
```

**短链接跳转**：
```java
@RestController
public class RedirectController {
    @Autowired
    private ShortUrlService service;
    
    @GetMapping("/{shortKey}")
    public void redirect(@PathVariable String shortKey, 
                         HttpServletResponse response) throws IOException {
        // 1. 查询原始URL
        String originalUrl = service.getOriginalUrl(shortKey);
        
        if (originalUrl == null) {
            response.sendError(404, "短链接不存在或已过期");
            return;
        }
        
        // 2. 异步统计点击次数（不影响跳转速度）
        service.incrementClickCountAsync(shortKey);
        
        // 3. 302重定向
        response.sendRedirect(originalUrl);
    }
}

@Service
public class ShortUrlService {
    public String getOriginalUrl(String shortKey) {
        // 1. 先查Redis缓存
        String cacheKey = "short:" + shortKey;
        String url = (String) redisTemplate.opsForHash()
            .get(cacheKey, "original_url");
        
        if (url != null) {
            return url;  // 缓存命中，直接返回
        }
        
        // 2. 缓存未命中，查数据库
        ShortUrl shortUrl = mapper.findByShortKey(shortKey);
        if (shortUrl == null || shortUrl.getStatus() == 0) {
            return null;
        }
        
        // 检查是否过期
        if (shortUrl.getExpireAt() != null && 
            shortUrl.getExpireAt().isBefore(LocalDateTime.now())) {
            return null;
        }
        
        // 3. 写入缓存
        cacheShortUrl(shortKey, shortUrl);
        
        return shortUrl.getOriginalUrl();
    }
    
    @Async
    public void incrementClickCountAsync(String shortKey) {
        // 1. Redis计数（快速）
        String countKey = "short:count:" + shortKey;
        redisTemplate.opsForValue().increment(countKey);
        
        // 2. 定时批量同步到数据库（减少数据库压力）
        // 由定时任务处理
    }
}
```

### 3.5 性能优化

**缓存策略**：
```java
// 多级缓存
public String getOriginalUrl(String shortKey) {
    // L1: 本地缓存（Caffeine）- 超快
    String url = localCache.getIfPresent(shortKey);
    if (url != null) return url;
    
    // L2: Redis缓存 - 快
    url = redisCache.get(shortKey);
    if (url != null) {
        localCache.put(shortKey, url);
        return url;
    }
    
    // L3: 数据库 - 慢
    url = database.query(shortKey);
    if (url != null) {
        redisCache.set(shortKey, url);
        localCache.put(shortKey, url);
    }
    return url;
}
```

**统计优化**：
```java
// 问题：每次跳转都更新数据库，性能差
// ❌ 同步更新
public void incrementClickCount(String shortKey) {
    mapper.incrementClickCount(shortKey);  // 数据库压力大
}

// ✅ 异步批量更新
@Scheduled(fixedRate = 60000)  // 每分钟执行
public void syncClickCounts() {
    // 1. 从Redis获取所有计数
    Set<String> keys = redisTemplate.keys("short:count:*");
    
    // 2. 批量更新数据库
    for (String key : keys) {
        String shortKey = key.replace("short:count:", "");
        Long count = redisTemplate.opsForValue().get(key);
        if (count != null && count > 0) {
            mapper.incrementClickCount(shortKey, count.intValue());
            redisTemplate.delete(key);  // 清空Redis计数
        }
    }
}
```

## 五、容量估算

### 5.1 存储容量

```
假设：
  - 每天新增 100万条短链接
  - 每条记录 260字节（URL 200B + 字段 60B）
  - 保存 5年

总存储 = 100万/天 × 260B × 365天 × 5年
       = 100万 × 260 × 1825
       ≈ 475GB × 5
       ≈ 2.4TB
       
分库分表：16个分片，每个分片 150GB ✅
```

### 5.2 QPS估算

```
假设：
  - 1亿条短链接
  - 平均每条每天被访问 10次
  - 读写比 100:1

日均请求 = 1亿 × 10 = 10亿次/天
平均QPS = 10亿 / 86400 ≈ 11,574
峰值QPS（×3）≈ 35,000

写入QPS = 35000 / 101 ≈ 350
读取QPS = 35000 - 350 ≈ 34,650

结论：典型的读多写少场景，重点优化读性能 ✅
```

## 六、常见追问

### Q1: 如何保证生成的短链接唯一？

**回答**：
```
方案1：数据库自增ID + Base62
  - MySQL自增ID天然唯一
  - Base62编码保持唯一性
  - 推荐方案 ✅

方案2：分布式ID生成器
  - 雪花算法（Snowflake）
  - 美团Leaf
  - 适合大规模分布式场景

方案3：Hash + 冲突检测
  - MD5/SHA256 hash
  - 冲突则重新生成
  - 性能稍差
```

### Q2: 为什么用302重定向而不是301？

**回答**：
```
301 (Permanent Redirect - 永久重定向):
  ❌ 浏览器会缓存，后续直接访问原URL
  ❌ 无法统计点击次数
  ❌ 无法修改跳转目标

302 (Temporary Redirect - 临时重定向):
  ✅ 每次都经过服务器
  ✅ 可以统计点击次数
  ✅ 可以修改跳转目标
  ✅ 推荐方案

结论：短链接场景用302 ✅
```

### Q3: 如何处理热点短链接？

**回答**：
```
问题：某个短链接被大量访问，单点压力大

解决方案：
1. 多级缓存：
   - 本地缓存（Caffeine）
   - Redis缓存
   - CDN缓存

2. 读写分离：
   - 主库写入
   - 从库读取（多个从库分担压力）

3. 限流保护：
   - 单个短链接访问频率限制
   - 整体系统限流

4. 预热缓存：
   - 热点短链接提前加载到缓存
```

### Q4: 短链接服务挂了怎么办？

**回答**：
```
高可用方案：

1. 应用层：
   - 多实例部署（至少3个）
   - 负载均衡（Nginx/LVS）
   - 健康检查自动摘除故障节点

2. 缓存层：
   - Redis Cluster（主从 + 哨兵）
   - 客户端容错

3. 数据库层：
   - MySQL主从复制
   - 自动故障转移

4. 跨地域：
   - 多机房部署
   - GeoDNS就近访问

目标��99.9%可用性（年故障时间 < 9小时）✅
```

### Q5: 如何防止短链接被恶意爬取？

**回答**：
```
防护措施：

1. 访问频率限制：
   - IP限流：单IP每秒最多10次
   - 用户限流：单用户每分钟最多100次

2. 验证码：
   - 高频访问触发验证码
   - 滑动验证、图形验证

3. 黑名单：
   - 恶意IP加入黑名单
   - 定期清理

4. 随机化：
   - 短链接key随机，不连续
   - 增加爬取难度
```

## 八、关键技术挑战与解决方案

### 8.1 分布式ID生成挑战
**问题**：数据库自增ID在分布式环境下存在单点问题和ID碰撞风险

**解决方案**：基于雪花算法的分布式ID生成器
```java
public class SnowflakeIdGenerator {
    // 41位时间戳 + 10位机器ID + 12位序列号
    private final long workerId;        // 机器ID(0-1023)
    private final long datacenterId;    // 数据中心ID(0-31)
    private long sequence = 0L;         // 序列号(0-4095)
    private long lastTimestamp = -1L;   // 上次生成ID的时间戳

    public synchronized long nextId() {
        long timestamp = System.currentTimeMillis();
        
        // 处理时钟回拨
        if (timestamp < lastTimestamp) {
            throw new RuntimeException("Clock moved backwards");
        }
        
        // 同一毫秒内序列号递增
        if (timestamp == lastTimestamp) {
            sequence = (sequence + 1) & 0xFFF; // 12位序列号掩码
            if (sequence == 0) {
                timestamp = tilNextMillis(lastTimestamp);
            }
        } else {
            sequence = 0L;
        }
        
        lastTimestamp = timestamp;
        
        // 组合ID：时间戳(41) + 数据中心ID(5) + 机器ID(5) + 序列号(12)
        return (
            (timestamp - 1609459200000L) << 22 |
            (datacenterId << 17) |
            (workerId << 12) |
            sequence
        );
    }
    
    private long tilNextMillis(long lastTimestamp) {
        long timestamp = System.currentTimeMillis();
        while (timestamp <= lastTimestamp) {
            timestamp = System.currentTimeMillis();
        }
        return timestamp;
    }
}
```

### 8.2 缓存一致性保障
**问题**：Redis缓存与MySQL数据不一致

**解决方案**：延迟双删策略

```java
    redisTemplate.delete("short:" + shortUrl.getShortKey());
    
    // 2. 更新数据库
    mapper.update(shortUrl);
    
    // 3. 延迟再次删除缓存（处理更新期间的脏读）
    scheduler.schedule(() -> {
        redisTemplate.delete("short:" + shortUrl.getShortKey());
    }, 500, TimeUnit.MILLISECONDS);
}
```

### 8.3 高并发场景下的性能瓶颈
**问题**：热点短链接导致的缓存击穿和数据库压力

**解决方案**：布隆过滤器 + 热点隔离
```java
// 布隆过滤器防止缓存穿透
@Component
public class ShortUrlBloomFilter {
    private BloomFilter<String> filter;
    
    @PostConstruct
    public void init() {
        // 初始化布隆过滤器，预计1亿条数据，误判率0.01
        filter = BloomFilter.create(
            Funnels.stringFunnel(Charset.defaultCharset()),
            100_000_000,
            0.01
        );
        // 加载所有短链接key到过滤器
        List<String> allShortKeys = mapper.findAllShortKeys();
        allShortKeys.forEach(filter::put);
    }
    
    public boolean mightContain(String shortKey) {
        return filter.mightContain(shortKey);
    }
}
```
