1. 按技术栈分类刷题
2. 总结高频考点

### 第四阶段：模拟面试
1. 限时完成系统设计（45分钟）
2. 口头表达设计思路
3. 回答面试官追问
4. 总结改进

## 💡 系统设计通用步骤

### 1. 需求澄清（5分钟）
- 功能需求：核心功能是什么？
- 非功能需求：QPS、延迟、可用性要求？
- 规模估算：DAU、存储、带宽？

### 2. 概要设计（15分钟）
- 核心流程梳理
- 关键组件识别
- 技术选型
- 架构图绘制

### 3. 详细设计（20分钟）
- 数据模型设计
- 接口设计
- 关键算法
- 容错处理

### 4. 优化与扩展（5分钟）
- 性能优化
- 可扩展性
- 监控告警
- 成本优化

## 🎯 面试技巧

### 系统设计面试
1. **先问需求**：不要急于设计，先澄清需求
2. **从简到繁**：先设计最简版本，再逐步优化
3. **主动沟通**：边画边说，让面试官跟上你的思路
4. **权衡取舍**：说明为什么选这个方案，其他方案的优缺点
5. **准备追问**：每个设计点都可能被深挖

### 技术问题面试
1. **先答要点**：简明扼要说结论
2. **再展开**：展开原理、细节
3. **举例说明**：结合实际项目经验
4. **对比分析**：对比不同方案
5. **追问预判**：预判面试官可能追问的点
