## 1. 权限系统概述

```
权限系统四层模型：

  ┌──────────────────────────────────┐
  │  功能权限（菜单/按钮）             │ — 你能看到什么
  ├──────────────────────────────────┤
  │  数据权限（行级/字段级）           │ — 你能看哪些数据
  ├──────────────────────────────────┤
  │  API权限（接口级）                │ — 你能调用什么接口
  ├──────────────────────────────────┤
  │  字段权限（敏感字段脱敏）          │ — 你能看到哪些字段
  └──────────────────────────────────┘

RBAC（Role-Based Access Control）：
  用户 → 角色 → 权限 → 资源
  用户不直接关联权限，通过角色间接关联
```

### 2.1 数据库表设计

```sql
-- 用户表
CREATE TABLE sys_user (
    id BIGINT PRIMARY KEY AUTO_INCREMENT,
    username VARCHAR(50) NOT NULL UNIQUE,
    password VARCHAR(100) NOT NULL,
    dept_id BIGINT COMMENT '部门ID',
    status TINYINT DEFAULT 1 COMMENT '1启用0禁用',
    create_time DATETIME DEFAULT CURRENT_TIMESTAMP
);

-- 角色表
CREATE TABLE sys_role (
    id BIGINT PRIMARY KEY AUTO_INCREMENT,
    role_name VARCHAR(50) NOT NULL,
    role_code VARCHAR(50) NOT NULL UNIQUE,
    data_scope TINYINT DEFAULT 1 COMMENT '1全部 2本部门 3本部门及以下 4仅本人 5自定义',
    status TINYINT DEFAULT 1,
    create_time DATETIME DEFAULT CURRENT_TIMESTAMP
);

-- 权限/菜单表
CREATE TABLE sys_menu (
    id BIGINT PRIMARY KEY AUTO_INCREMENT,
    parent_id BIGINT DEFAULT 0,
    menu_name VARCHAR(50) NOT NULL,
    menu_type CHAR(1) COMMENT 'M目录 C菜单 F按钮',
    path VARCHAR(200) COMMENT '路由地址',
    component VARCHAR(255) COMMENT '组件路径',
    perms VARCHAR(100) COMMENT '权限标识如system:user:list',
    icon VARCHAR(100),
    sort INT DEFAULT 0,
    visible CHAR(1) DEFAULT '1'
);

-- 用户-角色关联表
CREATE TABLE sys_user_role (
    user_id BIGINT NOT NULL,
    role_id BIGINT NOT NULL,
    PRIMARY KEY (user_id, role_id)
);

-- 角色-菜单关联表
CREATE TABLE sys_role_menu (
    role_id BIGINT NOT NULL,
    menu_id BIGINT NOT NULL,
    PRIMARY KEY (role_id, menu_id)
);

-- 角色-部门关联表（数据权限自定义范围）
CREATE TABLE sys_role_dept (
    role_id BIGINT NOT NULL,
    dept_id BIGINT NOT NULL,
    PRIMARY KEY (role_id, dept_id)
);
```

```java
@Entity
@Table(name = "sys_user")
public class SysUser {
    @Id @GeneratedValue(strategy = GenerationType.IDENTITY)
    private Long id;
    private String username;
    private String password;
    private Long deptId;
    private Integer status;
    
    @ManyToMany(fetch = FetchType.EAGER)
    @JoinTable(name = "sys_user_role",
        joinColumns = @JoinColumn(name = "user_id"),
        inverseJoinColumns = @JoinColumn(name = "role_id"))
    private Set<SysRole> roles = new HashSet<>();
}

@Entity
@Table(name = "sys_role")
public class SysRole {
    @Id @GeneratedValue
    private Long id;
    private String roleName;
    private String roleCode;
    private Integer dataScope; // 1全部 2本部门 3本部门及以下 4仅本人 5自定义
    
    @ManyToMany(fetch = FetchType.EAGER)
    @JoinTable(name = "sys_role_menu",
        joinColumns = @JoinColumn(name = "role_id"),
        inverseJoinColumns = @JoinColumn(name = "menu_id"))
    private Set<SysMenu> menus = new HashSet<>();
}
```

## 3. 数据权限

```java
// 数据权限：控制用户能查询哪些数据
// 通过MyBatis拦截器自动拼接SQL条件

@Target(ElementType.METHOD)
@Retention(RetentionPolicy.RUNTIME)
public @interface DataPermission {
    String deptAlias() default "d";      // 部门表别名
    String userAlias() default "u";      // 用户表别名
    String deptField() default "dept_id"; // 部门字段
    String userField() default "create_by"; // 用户字段
}

// MyBatis拦截器实现
@Intercepts(@Signature(type = StatementHandler.class, method = "prepare",
                        args = {Connection.class, Integer.class}))
public class DataPermissionInterceptor implements Interceptor {
    
    @Override
    public Object intercept(Invocation invocation) throws Throwable {
        StatementHandler handler = (StatementHandler) invocation.getTarget();
        BoundSql boundSql = handler.getBoundSql();
        String sql = boundSql.getSql();
        
        // 获取当前用户的数据范围
        LoginUser user = SecurityUtils.getLoginUser();
        String dataScope = buildDataScope(user);
        
        if (dataScope != null) {
            sql = sql.replace("WHERE", "WHERE " + dataScope + " AND ");
            // 或在SQL末尾追加
            if (!sql.contains("WHERE")) {
                sql = sql + " WHERE " + dataScope;
            }
            ReflectUtil.setFieldValue(boundSql, "sql", sql);
        }
        
        return invocation.proceed();
    }
    
    private String buildDataScope(LoginUser user) {
        for (SysRole role : user.getRoles()) {
            switch (role.getDataScope()) {
                case 1: return null; // 全部数据，不限制
                case 2: return "d.id = " + user.getDeptId(); // 本部门
                case 3: return "d.id IN (SELECT id FROM sys_dept WHERE id = " 
                    + user.getDeptId() + " OR FIND_IN_SET(" + user.getDeptId() + ", ancestors))";
                case 4: return "u.id = " + user.getUserId(); // 仅本人
                case 5: return "d.id IN (SELECT dept_id FROM sys_role_dept WHERE role_id = " 
                    + role.getId() + ")"; // 自定义
            }
        }
        return "1=0"; // 默认无权限
    }
}

// 使用
@DataPermission(deptAlias = "d", userAlias = "u")
public List<User> selectUserList(UserQuery query) {
    return userMapper.selectUserList(query);
}
```

## 4. 菜单与按钮权限

```java
// 菜单权限：构建用户的菜单树
@Service
public class MenuService {
    
    public List<MenuTree> buildMenuTree(Long userId) {
        List<SysMenu> menus = menuMapper.selectMenusByUserId(userId);
        
        // 过滤按钮权限
        List<MenuTree> tree = menus.stream()
            .filter(m -> !"F".equals(m.getMenuType())) // F是按钮
            .map(this::toTree)
            .collect(Collectors.toList());
        
        return buildTree(tree, 0L);
    }
    
    public Set<String> getButtonPerms(Long userId) {
        List<SysMenu> menus = menuMapper.selectMenusByUserId(userId);
        return menus.stream()
            .filter(m -> "F".equals(m.getMenuType()))
            .map(SysMenu::getPerms)  // 如 "system:user:add"
            .collect(Collectors.toSet());
    }
}
```

```javascript
// 前端按钮权限指令
// Vue自定义指令
Vue.directive('hasPermi', {
    bind(el, binding) {
        const { value } = binding;
        const allPermissions = store.getters.permissions;
        if (value && !allPermissions.includes(value)) {
            el.parentNode && el.parentNode.removeChild(el);
        }
    }
});
```

## 5. Spring Security集成

```java
// 自定义UserDetailsService
@Service
public class UserDetailsServiceImpl implements UserDetailsService {
    
    @Autowired
    private SysUserMapper userMapper;
    @Autowired
    private SysMenuMapper menuMapper;
    
    @Override
    public UserDetails loadUserByUsername(String username) {
        SysUser user = userMapper.selectByUsername(username);
        if (user == null) {
            throw new UsernameNotFoundException("用户不存在");
        }
        
        // 加载权限
        Set<String> permissions = menuMapper.selectPermsByUserId(user.getId());
        
        return new LoginUser(user, permissions);
    }
}

// JWT过滤器：从Token中解析用户权限
public class JwtAuthenticationFilter extends OncePerRequestFilter {
    
    @Override
    protected void doFilterInternal(HttpServletRequest request, 
                                     HttpServletResponse response, 
                                     FilterChain chain) {
        String token = request.getHeader("Authorization");
        if (token != null && jwtUtil.validate(token)) {
            Long userId = jwtUtil.getUserId(token);
            LoginUser loginUser = redisUtil.get("login:" + userId);
            
            UsernamePasswordAuthenticationToken auth = 
                new UsernamePasswordAuthenticationToken(
                    loginUser, null, loginUser.getAuthorities());
            SecurityContextHolder.getContext().setAuthentication(auth);
        }
        chain.doFilter(request, response);
    }
}

// 接口权限注解
@RestController
@RequestMapping("/system/user")
public class UserController {
    
    @PreAuthorize("@ss.hasPermi('system:user:list')")
    @GetMapping("/list")
    public TableData list(UserQuery query) { ... }
    
    @PreAuthorize("@ss.hasPermi('system:user:add')")
    @PostMapping
    public AjaxResult add(@RequestBody User user) { ... }
    
    @PreAuthorize("@ss.hasPermi('system:user:delete')")
    @DeleteMapping("/{ids}")
    public AjaxResult delete(@PathVariable Long[] ids) { ... }
}

// 权限校验工具
@Component("ss")
public class PermissionService {
    public boolean hasPermi(String permission) {
        LoginUser user = SecurityUtils.getLoginUser();
        return user.getPermissions().contains("*:*:*") // 超级管理员
            || user.getPermissions().contains(permission);
    }
}
```

## 6. 面试题速查

**Q1: RBAC模型的核心思想？**
```
用户→角色→权限→资源，用户不直接关联权限
优点：权限统一管理、角色复用、简化授权
RBAC0基础→RBAC1角色继承→RBAC2责任分离→RBAC3=1+2
```

**Q2: 数据权限怎么实现？**
```
通过MyBatis拦截器，在SQL执行前自动拼接数据范围条件
根据角色的data_scope决定：
  全部/本部门/本部门及以下/仅本人/自定义部门
用@DataPermission注解标记需要数据权限的方法
```

**Q3: 按钮权限怎么实现？**
```
后端：@PreAuthorize注解校验权限标识
前端：v-hasPermi自定义指令，无权限则移除按钮
权限标识格式：模块:实体:操作（如system:user:add）
```

**Q4: 权限缓存方案？**
```
用户登录时加载权限到Redis，Key=login:userId
权限变更时删除对应缓存，下次请求重新加载
避免每次请求都查数据库
```
