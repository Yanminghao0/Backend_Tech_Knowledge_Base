## 1. 秒杀系统核心挑战

### 1.1 场景特征

```
日常QPS: 1,000
秒杀QPS: 100,000~500,000 (100~500倍)
库存: 100~10,000件
持续时间: 几秒到几分钟
售罄速度: < 1秒
```

### 1.2 核心挑战

| 挑战 | 描述 | 应对策略 |
|------|------|----------|
| 瞬时高并发 | 10万+用户同时点击 | 多级流量削峰 |
| 防超卖 | 库存100件不能卖出101件 | Redis Lua原子扣减 |
| 防黄牛 | 机器刷单 | 验证码 + 风控 |
| 系统可用性 | 不能因为秒杀拖垮全站 | 独立部署 + 资源隔离 |
| 用户体验 | 不能让用户看到500错误 | 排队 + 友好提示 |

### 1.3 设计目标

- **不超卖**：库存绝对准确，零容忍
- **不少卖**：有库存就不能被拦截
- **高可用**：秒杀期间系统不宕机
- **不影响主站**：秒杀故障不蔓延到正常交易

## 2. 整体架构设计

### 2.1 架构全景图

```
用户 → CDN(静态资源) → Nginx(限流) → API Gateway(鉴权/限流)
    → 秒杀服务(独立部署)
        ├── Redis(库存预减 + 排队标记)
        ├── RocketMQ(异步下单)
        ├── 限流(Sentinel)
        └── 降级(兜底)
    → 订单服务(异步消费创建订单)
```

### 2.2 分层过滤策略

```
第1层: CDN + 静态页面 — 拦截90%静态请求
第2层: Nginx限流 — 拦截刷子流量
第3层: API Gateway — 鉴权 + IP限流
第4层: 应用层限流 — Sentinel + 验证码
第5层: Redis预减库存 — 库存为0直接拒绝
第6层: MQ削峰 — 异步处理下单请求
```

每一层过滤后到达下一层的流量递减，最终到数据库的流量非常可控：

```
原始请求: 1,000,000
  → CDN过滤后: 100,000 (90%静态资源被CDN拦截)
    → Nginx限流后: 50,000
      → 网关鉴权后: 40,000
        → 验证码过滤后: 20,000
          → Redis库存检查后: 1,000 (库存1000件)
            → MQ消费后: 1,000 (实际下单)
```

### 3.3 网关层限流

```java
/**
 * Spring Cloud Gateway 全局限流过滤器
 * 基于Redis + Lua实现令牌桶
 */
@Component
public class SeckillRateLimiterFilter implements GlobalFilter, Ordered {

    @Autowired
    private StringRedisTemplate redisTemplate;

    private static final String LUA_SCRIPT =
        "local key = KEYS[1] " +
        "local rate = tonumber(ARGV[1]) " +      // 令牌生成速率
        "local capacity = tonumber(ARGV[2]) " +   // 桶容量
        "local now = tonumber(ARGV[3]) " +
        "local requested = tonumber(ARGV[4]) " +
        "local last_time = tonumber(redis.call('hget', key, 'last_time') or now) " +
        "local last_tokens = tonumber(redis.call('hget', key, 'tokens') or capacity) " +
        "local delta = math.max(0, now - last_time) " +
        "local tokens = math.min(capacity, last_tokens + delta * rate) " +
        "if tokens < requested then return 0 end " +
        "tokens = tokens - requested " +
        "redis.call('hmset', key, 'tokens', tokens, 'last_time', now) " +
        "redis.call('expire', key, 60) " +
        "return 1";

    @Override
    public Mono<Void> filter(ServerWebExchange exchange, GatewayFilterChain chain) {
        String path = exchange.getRequest().getPath().value();
        if (!path.startsWith("/api/seckill/")) {
            return chain.filter(exchange);
        }

        String clientIp = getClientIp(exchange);
        String key = "rate_limit:seckill:" + clientIp;

        Long allowed = redisTemplate.execute(
            new DefaultRedisScript<>(LUA_SCRIPT, Long.class),
            Collections.singletonList(key),
            "100",  // 每秒100令牌
            "200",  // 桶容量200
            String.valueOf(System.currentTimeMillis() / 1000),
            "1"
        );

        if (allowed == null || allowed == 0) {
            exchange.getResponse().setStatusCode(HttpStatus.TOO_MANY_REQUESTS);
            exchange.getResponse().getHeaders().setContentType(MediaType.APPLICATION_JSON);
            String body = "{\"code\":429,\"message\":\"请求过于频繁，请稍后重试\"}";
            DataBuffer buffer = exchange.getResponse().bufferFactory().wrap(body.getBytes());
            return exchange.getResponse().writeWith(Mono.just(buffer));
        }

        return chain.filter(exchange);
    }

    @Override
    public int getOrder() {
        return -100;
    }
}
```

## 4. Redis预减库存

### 4.1 库存预热

```java
/**
 * 秒杀活动开始前，将库存加载到Redis

@Service
public class SeckillStockWarmup {

    @Autowired
    private StringRedisTemplate redisTemplate;

    @Autowired
    private SeckillActivityMapper activityMapper;

    /**
     * 活动开始前5分钟预热

    @Scheduled(cron = "0 */5 * * * ?")
    public void warmupStock() {
        List<SeckillActivity> activities = activityMapper.selectUpcoming(LocalDateTime.now().plusMinutes(5));
        for (SeckillActivity activity : activities) {

            redisTemplate.opsForValue().set(stockKey, String.valueOf(activity.getStock()));
            // 初始化售罄标记
            redisTemplate.delete("seckill:soldout:" + activity.getId());
            // 初始化购买记录Set（防止重复购买）
            redisTemplate.delete("seckill:users:" + activity.getId());

            log.info("秒杀活动库存预热, activityId: {}, stock: {}", activity.getId(), activity.getStock());
        }
    }
}
```

### 4.2 预减库存流程

```java
@Service
public class SeckillService {

    @Autowired
    private StringRedisTemplate redisTemplate;

    @Autowired
    private RocketMQTemplate rocketMQTemplate;

    /**
     * 秒杀核心流程
     * 1. 售罄标记检查（快速失败）
     * 2. 重复购买检查
     * 3. Redis预减库存（Lua原子操作）
     * 4. 发送MQ异步下单
     */
    public SeckillResult execute(Long userId, Long activityId) {
        // 1. 售罄标记检查 — 如果已售罄直接返回，不执行后续逻辑
        String soldOutKey = "seckill:soldout:" + activityId;
        if (Boolean.TRUE.equals(redisTemplate.hasKey(soldOutKey))) {
            return SeckillResult.fail("手慢了，商品已抢完");
        }

        // 2. 防重复购买
        String usersKey = "seckill:users:" + activityId;
        Boolean isNew = redisTemplate.opsForValue().setBit(usersKey, userId, true);
        if (Boolean.TRUE.equals(isNew)) {
            // 该bit位之前已经是1，说明已经参与过
            return SeckillResult.fail("您已参与过该秒杀活动");
        }

        // 3. Redis预减库存
        String stockKey = "seckill:stock:" + activityId;
        Long remaining = redisTemplate.opsForValue().decrement(stockKey);
        if (remaining == null) {
            // Redis异常，恢复bit位
            redisTemplate.opsForValue().setBit(usersKey, userId, false);
            return SeckillResult.fail("系统繁忙，请稍后重试");
        }

        if (remaining < 0) {
            // 库存不足，恢复bit位和库存
            redisTemplate.opsForValue().increment(stockKey);
            redisTemplate.opsForValue().setBit(usersKey, userId, false);
            // 设置售罄标记
            redisTemplate.opsForValue().set(soldOutKey, "1", 1, TimeUnit.HOURS);
            return SeckillResult.fail("手慢了，商品已抢完");
        }

        // 4. 发送MQ异步下单
        SeckillOrderMessage message = new SeckillOrderMessage();
        message.setUserId(userId);
        message.setActivityId(activityId);
        message.setOrderNo(generateOrderNo(userId));
        message.setTimestamp(System.currentTimeMillis());

        try {
            rocketMQTemplate.asyncSend("seckill-order-topic", message, new SendCallback() {
                @Override
                public void onSuccess(SendResult sendResult) {
                    log.info("秒杀订单消息发送成功, userId: {}, activityId: {}", userId, activityId);
                }
                @Override
                public void onException(Throwable e) {
                    log.error("秒杀订单消息发送失败, userId: {}, activityId: {}", userId, activityId, e);
                    // 消息发送失败，回滚库存和购买标记
                    redisTemplate.opsForValue().increment(stockKey);
                    redisTemplate.opsForValue().setBit(usersKey, userId, false);
                }
            });
        } catch (Exception e) {
            // 发送异常回滚
            redisTemplate.opsForValue().increment(stockKey);
            redisTemplate.opsForValue().setBit(usersKey, userId, false);
            return SeckillResult.fail("系统繁忙，请稍后重试");
        }

        return SeckillResult.success(message.getOrderNo(), "抢购成功，正在创建订单...");
    }
}
```

## 5. 防超卖Lua脚本

### 5.1 完整Lua脚本

将库存检查、扣减、防重复购买合并在一个Lua脚本中，保证原子性：

```lua
-- seckill.lua
-- KEYS[1]: 库存Key  seckill:stock:{activityId}
-- KEYS[2]: 用户记录Key  seckill:users:{activityId}
-- KEYS[3]: 售罄标记Key  seckill:soldout:{activityId}
-- ARGV[1]: 用户ID
-- ARGV[2]: 售罄标记过期时间(秒)

-- 1. 检查是否售罄
local soldout = redis.call('EXISTS', KEYS[3])
if soldout == 1 then
    return -3  -- 已售罄
end

-- 2. 检查是否已购买
local userBit = redis.call('GETBIT', KEYS[2], ARGV[1])
if userBit == 1 then
    return -2  -- 重复购买
end

-- 3. 检查并扣减库存
local stock = redis.call('GET', KEYS[1])
if stock == nil then
    return -4  -- 活动不存在
end

stock = tonumber(stock)
if stock <= 0 then
    -- 设置售罄标记
    redis.call('SETEX', KEYS[3], ARGV[2], '1')
    return -3  -- 已售罄
end

-- 4. 扣减库存
redis.call('DECR', KEYS[1])

-- 5. 标记已购买
redis.call('SETBIT', KEYS[2], ARGV[1], 1)

return 1  -- 成功
```

### 5.2 Java调用

```java
@Service
public class SeckillLuaService {

    @Autowired
    private StringRedisTemplate redisTemplate;

    private DefaultRedisScript<Long> seckillScript;

    @PostConstruct
    public void init() {
        seckillScript = new DefaultRedisScript<>();
        seckillScript.setLocation(new ClassPathResource("lua/seckill.lua"));
        seckillScript.setResultType(Long.class);
    }

    public SeckillResult executeWithLua(Long userId, Long activityId) {
        String stockKey = "seckill:stock:" + activityId;
        String usersKey = "seckill:users:" + activityId;
        String soldOutKey = "seckill:soldout:" + activityId;

        Long result = redisTemplate.execute(
            seckillScript,
            Arrays.asList(stockKey, usersKey, soldOutKey),
            String.valueOf(userId),
            "3600"  // 售罄标记1小时过期
        );

        if (result == null || result == -4) {
            return SeckillResult.fail("活动不存在");
        }
        if (result == -3) {
            return SeckillResult.fail("手慢了，商品已抢完");
        }
        if (result == -2) {
            return SeckillResult.fail("您已参与过该秒杀活动");
        }

        // Lua脚本执行成功，发送MQ异步下单
        SeckillOrderMessage message = new SeckillOrderMessage();
        message.setUserId(userId);
        message.setActivityId(activityId);
        message.setOrderNo(generateOrderNo(userId));

        SendResult sendResult = rocketMQTemplate.syncSend("seckill-order-topic", message);
        if (sendResult.getSendStatus() != SendStatus.SEND_OK) {
            // MQ发送失败，回滚Redis
            redisTemplate.opsForValue().increment(stockKey);
            redisTemplate.opsForValue().setBit(usersKey, userId, false);
            return SeckillResult.fail("系统繁忙，请稍后重试");
        }

        return SeckillResult.success(message.getOrderNo(), "抢购成功，正在创建订单...");
    }
}
```

### 5.3 数据库最终扣减兜底

```java
/**
 * MQ消费者：异步创建秒杀订单
 * 数据库层面用乐观锁做最终兜底
 */
@RocketMQMessageListener(
    topic = "seckill-order-topic",
    consumerGroup = "seckill-order-consumer",
    messageModel = MessageModel.CLUSTERING
)
@Component
public class SeckillOrderConsumer implements RocketMQListener<SeckillOrderMessage> {

    @Override
    @Transactional(rollbackFor = Exception.class)
    public void onMessage(SeckillOrderMessage message) {
        // 幂等检查
        if (orderMapper.existsByOrderNo(message.getOrderNo())) {
            return;
        }

        // 查询活动信息
        SeckillActivity activity = activityMapper.selectById(message.getActivityId());
        if (activity == null) {
            log.warn("秒杀活动不存在: {}", message.getActivityId());
            return;
        }

        // 数据库乐观锁扣减库存（最终兜底）
        int rows = activityMapper.deductStockWithOptimisticLock(
            message.getActivityId(), activity.getVersion());
        if (rows == 0) {
            // 数据库库存不足，回滚Redis
            redisTemplate.opsForValue().increment("seckill:stock:" + message.getActivityId());
            redisTemplate.opsForValue().setBit("seckill:users:" + message.getActivityId(),
                message.getUserId(), false);

            // 发送失败通知
            notifyService.sendSeckillFail(message.getUserId(), "商品已抢完");
            return;
        }

        // 创建订单
        Order order = new Order();
        order.setOrderNo(message.getOrderNo());
        order.setUserId(message.getUserId());
        order.setSpuId(activity.getSpuId());
        order.setSkuId(activity.getSkuId());
        order.setQuantity(1);
        order.setTotalAmount(activity.getSeckillPrice());
        order.setPayAmount(activity.getSeckillPrice());
        order.setStatus(OrderStatus.PENDING_PAYMENT);
        order.setExpireTime(LocalDateTime.now().plusMinutes(10)); // 秒杀订单10分钟超时
        orderMapper.insert(order);

        // 发送下单成功通知
        notifyService.sendSeckillSuccess(message.getUserId(), message.getOrderNo());

        log.info("秒杀订单创建成功, userId: {}, orderNo: {}", message.getUserId(), message.getOrderNo());
    }
}
```

```java
/**
 * 消费端限流：根据数据库承受能力控制消费速率
 */
@Component
public class SeckillOrderRateLimiter {

    // 基于令牌桶的消费者限流
    private final RateLimiter rateLimiter = RateLimiter.create(500); // 每秒500个订单

    @RocketMQMessageListener(topic = "seckill-order-topic", consumerGroup = "seckill-order-consumer")
    @Component
    public class SeckillOrderConsumer implements RocketMQListener<SeckillOrderMessage> {

        @Override
        public void onMessage(SeckillOrderMessage message) {
            // 限流：等待令牌（最多等待2秒）
            if (!rateLimiter.tryAcquire(2, TimeUnit.SECONDS)) {
                throw new RuntimeException("消费限流，稍后重试"); // 抛异常触发重试
            }

            processOrder(message);
        }
    }
}
```

### 6.3 消费失败重试与死信

```java
/**
 * 消费失败处理策略
 */
@RocketMQMessageListener(
    topic = "seckill-order-topic",
    consumerGroup = "seckill-order-consumer",
    maxReconsumeTimes = 5  // 最多重试5次
)
@Component
public class SeckillOrderConsumerV2 implements RocketMQListener<SeckillOrderMessage> {

    @Override
    public void onMessage(SeckillOrderMessage message) {
        try {
            processOrder(message);
        } catch (OptimisticLockException e) {
            // 乐观锁冲突，不重试直接回滚
            rollbackRedis(message);
        } catch (DuplicateKeyException e) {
            // 重复订单，幂等忽略
            log.info("重复订单忽略: {}", message.getOrderNo());
        } catch (Exception e) {
            log.error("秒杀订单处理异常, orderNo: {}", message.getOrderNo(), e);
            throw e; // 触发重试
        }
    }

    /**
     * 死信队列处理：超过最大重试次数的消息进入死信队列
     */
    @RocketMQMessageListener(
        topic = "%DLQ%seckill-order-consumer",
        consumerGroup = "seckill-dlq-consumer"
    )
    @Component
    public class SeckillDLQConsumer implements RocketMQListener<SeckillOrderMessage> {
        @Override
        public void onMessage(SeckillOrderMessage message) {
            // 死信处理：回滚Redis库存 + 发送补偿通知
            rollbackRedis(message);
            notifyService.sendSeckillFail(message.getUserId(), "系统繁忙，请重新参与");
            // 记录到补偿表，人工介入
            compensationMapper.insert(message);
        }
    }
}
```

```java
@Configuration
public class SentinelConfig {

    @PostConstruct

        // 秒杀接口QPS限流
        FlowRule flowRule1 = new FlowRule();

        flowRule1.setGrade(RuleConstant.FLOW_GRADE_QPS);
        flowRule1.setCount(10000); // 每秒最多10000次
        flowRule1.setLimitApp("default");

        // 线程数限流
        FlowRule flowRule2 = new FlowRule();
        flowRule2.setResource("seckillExecute");
        flowRule2.setGrade(RuleConstant.FLOW_GRADE_THREAD);
        flowRule2.setCount(200); // 最多200个并发线程

        // 熔断规则：异常率超50%熔断
        DegradeRule degradeRule = new DegradeRule();
        degradeRule.setResource("seckillExecute");
        degradeRule.setGrade(RuleConstant.DEGRADE_GRADE_EXCEPTION_RATIO);
        degradeRule.setCount(0.5);          // 异常率阈值50%
        degradeRule.setTimeWindow(10);       // 熔断10秒
        degradeRule.setMinRequestAmount(100); // 最小请求数
        degradeRule.setStatIntervalMs(1000);  // 统计窗口1秒

        // 热点参数限流：单个activityId限流
        ParamFlowRule paramRule = new ParamFlowRule("seckillExecute")
            .setParamIdx(1)  // 第二个参数 activityId
            .setGrade(RuleConstant.FLOW_GRADE_QPS)
            .setCount(5000); // 单个活动每秒5000次

        FlowRuleManager.loadRules(Arrays.asList(flowRule1, flowRule2));
        DegradeRuleManager.loadRules(Collections.singletonList(degradeRule));
        ParamFlowRuleManager.loadRules(Collections.singletonList(paramRule));
    }
}

 * 限流降级处理

@Service
public class SeckillResourceService {

    @SentinelResource(
        value = "seckillExecute",
        blockHandler = "blockHandler",
        fallback = "fallback"
    )
    public SeckillResult execute(Long userId, Long activityId) {
        return seckillService.execute(userId, activityId);
    }

    // 限流处理
    public SeckillResult blockHandler(Long userId, Long activityId, BlockException ex) {
        if (ex instanceof FlowException) {
            return SeckillResult.fail("当前抢购人数过多，请稍后重试");
        }
        if (ex instanceof DegradeException) {
            return SeckillResult.fail("系统繁忙，请稍后再试");
        }
        return SeckillResult.fail("请求被限制，请稍后重试");
    }

    // 降级处理
    public SeckillResult fallback(Long userId, Long activityId, Throwable e) {
        log.error("秒杀执行异常, userId: {}, activityId: {}", userId, activityId, e);
        return SeckillResult.fail("系统异常，请稍后重试");
    }
}
```

### 7.2 多级降级策略

```java
/**
 * 降级开关管理
 */
@Service
public class SeckillDegradeManager {

    @Autowired
    private StringRedisTemplate redisTemplate;

    // 降级开关
    private static final String DEGRADE_VERIFY = "seckill:degrade:verify";       // 关闭验证码
    private static final String DEGRADE_RISK = "seckill:degrade:risk";           // 关闭风控
    private static final String DEGRADE_MQ = "seckill:degrade:mq";               // 同步下单
    private static final String DEGRADE_FULL = "seckill:degrade:full";           // 全面降级

    public boolean isVerifyCodeEnabled() {
        return !Boolean.TRUE.equals(redisTemplate.hasKey(DEGRADE_VERIFY));
    }

    public boolean isRiskCheckEnabled() {
        return !Boolean.TRUE.equals(redisTemplate.hasKey(DEGRADE_RISK));
    }

    public boolean isMQEnabled() {
        return !Boolean.TRUE.equals(redisTemplate.hasKey(DEGRADE_MQ));
    }

    public boolean isFullDegrade() {
        return Boolean.TRUE.equals(redisTemplate.hasKey(DEGRADE_FULL));
    }

    /**
     * 根据系统负载自动降级
     */
    @Scheduled(fixedRate = 5000)
    public void autoDegrade() {
        double cpuUsage = getCpuUsage();
        double memUsage = getMemUsage();
        int activeThreads = getActiveThreads();

        if (cpuUsage > 0.9 || memUsage > 0.9 || activeThreads > 500) {
            // 系统过载，开启全面降级
            redisTemplate.opsForValue().set(DEGRADE_FULL, "1", 5, TimeUnit.MINUTES);
            log.warn("系统过载，开启全面降级: CPU={}, MEM={}, Threads={}", cpuUsage, memUsage, activeThreads);
        } else if (cpuUsage > 0.7 || activeThreads > 300) {
            // 中等负载，关闭非核心功能
            redisTemplate.opsForValue().set(DEGRADE_VERIFY, "1", 5, TimeUnit.MINUTES);
            redisTemplate.opsForValue().set(DEGRADE_RISK, "1", 5, TimeUnit.MINUTES);
        } else {
            // 恢复正常
            redisTemplate.delete(DEGRADE_VERIFY);
            redisTemplate.delete(DEGRADE_RISK);
            redisTemplate.delete(DEGRADE_FULL);
        }
    }
}
```

```java
/**
 * 多维度防刷策略
 */
@Service
public class SeckillSecurityService {

    /**
     * 1. 验证码拦截
     * 秒杀前必须输入验证码，过滤大部分脚本
     */
    public boolean verifyCaptcha(Long userId, String captchaCode) {
        String storedCode = redisTemplate.opsForValue().get("captcha:" + userId);
        if (storedCode == null || !storedCode.equalsIgnoreCase(captchaCode)) {
            return false;
        }
        // 验证后删除，防止复用
        redisTemplate.delete("captcha:" + userId);
        return true;
    }

    /**
     * 2. 频率限制
     * 同一用户3秒内只能请求1次
     */
    public boolean checkFrequency(Long userId) {
        String key = "seckill:freq:" + userId;
        Boolean isNew = redisTemplate.opsForValue().setIfAbsent(key, "1", 3, TimeUnit.SECONDS);
        return Boolean.TRUE.equals(isNew);
    }

    /**
     * 3. 设备指纹
     * 同一设备只能参与一次
     */
    public boolean checkDevice(String deviceFingerprint, Long activityId) {
        String key = "seckill:device:" + activityId + ":" + deviceFingerprint;
        Boolean isNew = redisTemplate.opsForValue().setIfAbsent(key, "1", 1, TimeUnit.HOURS);
        return Boolean.TRUE.equals(isNew);
    }

    /**
     * 4. IP限制
     * 同一IP每秒最多10次请求
     */
    public boolean checkIp(String ip) {
        String key = "seckill:ip:" + ip;
        Long count = redisTemplate.opsForValue().increment(key);
        if (count == 1) {
            redisTemplate.expire(key, 1, TimeUnit.SECONDS);
        }
        return count != null && count <= 10;
    }

    /**
     * 5. 风控规则
     * 综合判断用户是否为黄牛
     */
    public boolean riskCheck(Long userId, String ip, String deviceFingerprint) {
        // 新注册用户限制
        User user = userService.getById(userId);
        if (user.getCreateTime().isAfter(LocalDateTime.now().minusDays(1))) {
            return false;
        }

        // 历史行为分析
        int returnRate = orderService.getReturnRate(userId);
        if (returnRate > 50) { // 退货率超50%
            return false;
        }

        // IP信誉库检查
        if (blacklistService.isBlackIp(ip)) {
            return false;
        }

        return true;
    }
}
```

```java
        return Result.success(dynamicUrl);

     * 动态URL秒杀接口
```

### 9.3 热点数据优化

```java
/**
 * 热点Key分散: 库存Key分片
 * 单个Redis Key会成为热点，将库存分散到多个Key
 */
@Service
public class ShardedStockService {

    private static final int SHARD_COUNT = 10; // 库存分片数

    /**
     * 初始化分片库存
     */
    public void initShardedStock(Long activityId, int totalStock) {
        int perShard = totalStock / SHARD_COUNT;
        int remainder = totalStock % SHARD_COUNT;
        for (int i = 0; i < SHARD_COUNT; i++) {
            int shardStock = perShard + (i < remainder ? 1 : 0);
            redisTemplate.opsForValue().set(
                "seckill:stock:" + activityId + ":" + i,
                String.valueOf(shardStock)
            );
        }
    }

    /**
     * 分片扣减库存
     * 随机选择一个分片扣减，失败则尝试下一个
     */
    public boolean deductShardedStock(Long activityId) {
        // 随机打乱分片顺序
        List<Integer> shards = IntStream.range(0, SHARD_COUNT)
            .boxed().collect(Collectors.toList());
        Collections.shuffle(shards);

        for (int shard : shards) {
            String key = "seckill:stock:" + activityId + ":" + shard;
            Long remaining = redisTemplate.opsForValue().decrement(key);
            if (remaining != null && remaining >= 0) {
                return true; // 扣减成功
            }
            // 该分片库存不足，恢复并尝试下一个
            if (remaining != null) {
                redisTemplate.opsForValue().increment(key);
            }
        }
        return false; // 所有分片都不足
    }
}
```

## 10. 面试题速查

**Q1: 秒杀系统怎么防止超卖？**

核心方案：Redis + Lua脚本原子扣减。Lua脚本中先检查库存是否充足，再扣减，整个过程Redis单线程执行保证原子性。数据库层面用乐观锁CAS做最终兜底。双保险：Redis快速扣减 + 数据库最终一致。

**Q2: 秒杀系统如何做流量削峰？**

多级削峰：① 前端按钮防抖 + 倒计时；② Nginx限流（IP维度）；③ 网关令牌桶限流；④ 验证码过滤（降低2-3倍流量）；⑤ Redis库存预减（库存为0直接拒绝）；⑥ MQ异步下单（将同步请求转化为异步消费）。每一层过滤后流量递减。

**Q3: 为什么用RocketMQ而不是Kafka做秒杀？**

RocketMQ支持事务消息、延迟消息、重试机制和死信队列，更适合业务场景。Kafka吞吐量更高但不支持延迟消息和重试，适合日志场景。秒杀场景消息量在RocketMQ承受范围内，且RocketMQ的消息可靠性保障更适合订单场景。

**Q4: 秒杀库存100件，10万人抢购，怎么设计？**

① 活动前将100件库存预热到Redis；② 10万请求经过多层限流后约5000到达Redis；③ Lua脚本原子扣减，前100个成功，后续返回售罄；④ 成功的100个请求发送MQ；⑤ 消费者按500 TPS限流创建订单；⑥ 用户轮询或WebSocket推送下单结果。

**Q5: 秒杀系统如何做降级？**

分级降级：① 轻度负载→关闭验证码、关闭风控；② 中度负载→降低MQ消费速率；③ 重度负载→返回"活动太火爆"直接拒绝大部分请求，只放少量请求通过；④ 极端情况→关闭秒杀入口，展示活动结束页面。降级开关通过Redis控制，实时生效。

**Q6: 秒杀接口如何防黄牛？**

多重防护：① 验证码拦截脚本流量；② 动态URL（MD5路径）防止提前抓包；③ 设备指纹+IP限制防止多账号刷单；④ 用户行为分析（注册时间、退货率）；⑤ 频率限制（3秒内只能请求1次）；⑥ IP信誉库黑名单。

**Q7: 秒杀消息发送MQ失败怎么办？**

同步发送MQ，失败则回滚Redis库存和购买标记。如果MQ发送成功但消费者处理失败，通过重试机制（最多5次）处理。超过重试次数进入死信队列，人工补偿。消费者端做幂等处理（订单号唯一索引），防止重复消费。

**Q8: Redis挂了秒杀怎么办？**

① Sentinel自动故障转移，从节点提升为主节点（秒级恢复）；② 降级方案：直接走数据库扣减（用乐观锁防超卖），性能下降但功能可用；③ Redis Cluster集群部署避免单点故障；④ 活动前做Redis压测和预热验证。
