### 1.2 核心挑战

| 挑战 | 描述 | 解决方案 |
|------|------|----------|
| 资金安全 | 一分钱都不能错 | 强一致性 + 对账兜底 |
| 幂等性 | 回调可能重复 | 幂等键 + 状态机 |
| 高可用 | 支付不能停 | 多活部署 + 故障转移 |
| 渠道差异 | 各支付渠道接口不同 | 适配层统一抽象 |
| 异步通知 | 支付结果异步回调 | 重试机制 + 补偿 |
| 对账 | 确保系统与渠道一致 | T+1对账 + 差异处理 |

## 2. 支付网关设计

### 2.1 支付网关职责

```java
/**
 * 支付网关是支付系统的统一入口
 * 职责: 路由分发、协议转换、安全校验、流量控制
 */
@Service
public class PaymentGateway {

    @Autowired
    private Map<String, PaymentChannel> channelMap; // 渠道适配器

    @Autowired
    private RiskControlService riskControlService;

    @Autowired
    private IdempotentService idempotentService;

    /**
     * 统一下单入口
     */
    public PaymentResponse createPayment(PaymentRequest request) {
        // 1. 幂等检查
        if (!idempotentService.check(request.getRequestId())) {
            return PaymentResponse.duplicateRequest();
        }

        // 2. 参数校验
        validateRequest(request);

        // 3. 风控检查
        RiskResult riskResult = riskControlService.check(request);
        if (riskResult.isRejected()) {
            return PaymentResponse.riskRejected(riskResult.getReason());
        }

        // 4. 路由选择支付渠道
        PaymentChannel channel = routeChannel(request);
        log.info("支付路由: requestId={}, channel={}", request.getRequestId(), channel.getName());

        // 5. 创建支付单
        PaymentOrder paymentOrder = createPaymentOrder(request, channel);

        // 6. 调用渠道下单
        PaymentResponse response = channel.createPayment(paymentOrder);

        // 7. 记录支付流水
        paymentOrderMapper.insert(paymentOrder);

        return response;
    }

    /**
     * 支付渠道路由策略
     */
    private PaymentChannel routeChannel(PaymentRequest request) {
        // 策略1: 用户指定渠道
        if (request.getPreferredChannel() != null) {
            return channelMap.get(request.getPreferredChannel());
        }

        // 策略2: 根据金额路由
        if (request.getAmount().compareTo(new BigDecimal("0.01")) == 0) {
            return channelMap.get("ALIPAY"); // 小额走支付宝(费率低)
        }

        // 策略3: 根据渠道可用性
        for (String channelName : Arrays.asList("WECHAT", "ALIPAY", "UNIONPAY")) {
            PaymentChannel channel = channelMap.get(channelName);
            if (channel.isAvailable()) {
                return channel;
            }
        }

        throw new PaymentException("无可用支付渠道");
    }
}
```

### 2.2 支付单状态机

```java
/**
 * 支付单状态枚举
 */
public enum PaymentStatus {
    CREATED("已创建"),
    PAYING("支付中"),
    SUCCESS("支付成功"),
    FAILED("支付失败"),
    CLOSED("已关闭"),
    REFUNDING("退款中"),
    REFUNDED("已退款"),
    PARTIAL_REFUNDED("部分退款");

    private final String desc;
}

/**
 * 支付状态流转控制
 */
public class PaymentStateMachine {

    private static final Map<PaymentStatus, Set<PaymentStatus>> TRANSITIONS = new HashMap<>();

    static {
        TRANSITIONS.put(PaymentStatus.CREATED,
            new HashSet<>(Arrays.asList(PaymentStatus.PAYING, PaymentStatus.CLOSED)));
        TRANSITIONS.put(PaymentStatus.PAYING,
            new HashSet<>(Arrays.asList(PaymentStatus.SUCCESS, PaymentStatus.FAILED, PaymentStatus.CLOSED)));
        TRANSITIONS.put(PaymentStatus.SUCCESS,
            new HashSet<>(Arrays.asList(PaymentStatus.REFUNDING)));
        TRANSITIONS.put(PaymentStatus.REFUNDING,
            new HashSet<>(Arrays.asList(PaymentStatus.REFUNDED, PaymentStatus.PARTIAL_REFUNDED, PaymentStatus.SUCCESS)));
    }

    public static void transition(PaymentOrder order, PaymentStatus target) {
        PaymentStatus current = order.getStatus();
        Set<PaymentStatus> allowed = TRANSITIONS.get(current);
        if (allowed == null || !allowed.contains(target)) {
            throw new PaymentStateException(
                String.format("支付状态不能从[%s]流转到[%s]", current.getDesc(), target.getDesc()));
        }
        order.setStatus(target);
        order.setUpdateTime(LocalDateTime.now());
    }
}
```

### 3.1 渠道适配器模式

```java
/**
 * 支付渠道统一接口
 */
public interface PaymentChannel {

    String getName();

    boolean isAvailable();

    /**
     * 创建支付订单(下单到第三方渠道)
     */
    PaymentResponse createPayment(PaymentOrder order);

    /**
     * 查询支付状态
     */
    PaymentQueryResult queryPayment(String paymentNo);

    /**
     * 关闭支付订单
     */
    PaymentResponse closePayment(String paymentNo);

    /**
     * 申请退款
     */
    RefundResponse refund(RefundRequest request);

    /**
     * 验证回调签名
     */
    boolean verifyCallback(Map<String, String> params);

    /**
     * 解析回调数据
     */
    PaymentCallback parseCallback(Map<String, String> params);
}

/**
 * 微信支付渠道实现
 */
@Component
public class WechatPayChannel implements PaymentChannel {

    @Autowired
    private WxPayService wxPayService;

    @Override
    public String getName() { return "WECHAT"; }

    @Override
    public boolean isAvailable() {
        try {
            return wxPayService.ping() == 0;
        } catch (Exception e) {
            return false;
        }
    }

    @Override
    public PaymentResponse createPayment(PaymentOrder order) {
        WxPayUnifiedOrderRequest request = new WxPayUnifiedOrderRequest();
        request.setOutTradeNo(order.getPaymentNo());
        request.setBody(order.getSubject());
        request.setTotalFee(order.getAmount().multiply(BigDecimal.valueOf(100)).intValue());
        request.setSpbillCreateIp(order.getClientIp());
        request.setNotifyUrl(order.getNotifyUrl());
        request.setTradeType(order.getPayType());

        try {
            WxPayUnifiedOrderResult result = wxPayService.unifiedOrder(request);
            PaymentResponse response = new PaymentResponse();
            response.setPrepayId(result.getPrepayId());
            response.setPayUrl(result.getCodeURL());
            response.setStatus(PaymentStatus.PAYING);
            return response;
        } catch (WxPayException e) {
            log.error("微信支付下单失败, paymentNo: {}", order.getPaymentNo(), e);
            PaymentResponse response = new PaymentResponse();
            response.setStatus(PaymentStatus.FAILED);
            response.setErrorCode(e.getErrCode());
            response.setErrorMessage(e.getErrCodeDes());
            return response;
        }
    }

    @Override
    public PaymentQueryResult queryPayment(String paymentNo) {
        try {
            WxPayOrderQueryResult result = wxPayService.queryOrder(null, paymentNo);
            PaymentQueryResult queryResult = new PaymentQueryResult();
            queryResult.setChannelTradeNo(result.getTransactionId());
            queryResult.setPaymentNo(result.getOutTradeNo());

            switch (result.getTradeState()) {
                case "SUCCESS":
                    queryResult.setStatus(PaymentStatus.SUCCESS);
                    break;
                case "REFUND":
                    queryResult.setStatus(PaymentStatus.REFUNDED);
                    break;
                case "NOTPAY":
                case "USERPAYING":
                    queryResult.setStatus(PaymentStatus.PAYING);
                    break;
                case "CLOSED":
                    queryResult.setStatus(PaymentStatus.CLOSED);
                    break;
                default:
                    queryResult.setStatus(PaymentStatus.FAILED);
            }
            queryResult.setPayTime(result.getTimeEnd());
            return queryResult;
        } catch (WxPayException e) {
            throw new PaymentException("微信支付查询失败: " + e.getMessage(), e);
        }
    }

    @Override
    public RefundResponse refund(RefundRequest request) {
        WxPayRefundRequest refundRequest = new WxPayRefundRequest();
        refundRequest.setOutTradeNo(request.getPaymentNo());
        refundRequest.setOutRefundNo(request.getRefundNo());
        refundRequest.setTotalFee(request.getTotalAmount().multiply(BigDecimal.valueOf(100)).intValue());
        refundRequest.setRefundFee(request.getRefundAmount().multiply(BigDecimal.valueOf(100)).intValue());
        refundRequest.setRefundDesc(request.getReason());

        try {
            WxPayRefundResult result = wxPayService.refund(refundRequest);
            RefundResponse response = new RefundResponse();
            response.setRefundNo(request.getRefundNo());
            response.setChannelRefundNo(result.getRefundId());
            response.setStatus("PROCESSING");
            return response;
        } catch (WxPayException e) {
            throw new PaymentException("微信退款失败: " + e.getMessage(), e);
        }
    }

    @Override
    public boolean verifyCallback(Map<String, String> params) {
        try {
            return wxPayService.verifySignature(params, params.get("sign"),
                WXPayConstants.SignType.HMACSHA256);
        } catch (Exception e) {
            return false;
        }
    }

    @Override
    public PaymentCallback parseCallback(Map<String, String> params) {
        PaymentCallback callback = new PaymentCallback();
        callback.setPaymentNo(params.get("out_trade_no"));
        callback.setChannelTradeNo(params.get("transaction_id"));
        callback.setAmount(new BigDecimal(params.get("total_fee")).divide(BigDecimal.valueOf(100)));

        if ("SUCCESS".equals(params.get("result_code"))) {
            callback.setStatus(PaymentStatus.SUCCESS);
        } else {
            callback.setStatus(PaymentStatus.FAILED);
        }
        callback.setPayTime(params.get("time_end"));
        return callback;
    }
}
```

## 4. 异步通知处理

### 4.1 回调处理流程

```java
/**
 * 支付回调处理服务
 * 处理第三方支付渠道的异步通知
 */
@Service
public class PaymentCallbackService {

    @Autowired
    private PaymentOrderMapper paymentOrderMapper;

    @Autowired
    private PaymentFlowMapper flowMapper;

    @Autowired
    private Map<String, PaymentChannel> channelMap;

    @Autowired
    private ApplicationEventPublisher eventPublisher;

    /**
     * 统一回调处理
     */
    @Transactional(rollbackFor = Exception.class)
    public String handleCallback(String channelName, Map<String, String> params) {
        PaymentChannel channel = channelMap.get(channelName);
        if (channel == null) {
            log.error("未知支付渠道: {}", channelName);
            return "FAIL";
        }

        // 1. 验签
        if (!channel.verifyCallback(params)) {
            log.warn("回调验签失败, channel: {}", channelName);
            return "FAIL";
        }

        // 2. 解析回调
        PaymentCallback callback = channel.parseCallback(params);

        // 3. 记录回调流水
        PaymentFlow flow = new PaymentFlow();
        flow.setFlowNo(generateFlowNo());
        flow.setPaymentNo(callback.getPaymentNo());
        flow.setFlowType("NOTIFY");
        flow.setAmount(callback.getAmount());
        flow.setChannel(channelName);
        flow.setChannelTradeNo(callback.getChannelTradeNo());
        flow.setStatus(callback.getStatus().name());
        flow.setRequestContent(JSON.toJSONString(params));
        flowMapper.insert(flow);

        // 4. 幂等检查 — 已处理过的回调直接返回成功
        PaymentOrder order = paymentOrderMapper.selectByPaymentNo(callback.getPaymentNo());
        if (order == null) {
            log.warn("支付单不存在: {}", callback.getPaymentNo());
            return "FAIL";
        }

        if (order.getStatus() == PaymentStatus.SUCCESS) {
            log.info("支付单已成功，重复回调忽略: {}", callback.getPaymentNo());
            return channel.getSuccessResponse();
        }

        // 5. 金额校验
        if (callback.getAmount().compareTo(order.getAmount()) != 0) {
            log.error("回调金额不一致, paymentNo: {}, 应收: {}, 实收: {}",
                callback.getPaymentNo(), order.getAmount(), callback.getAmount());
            // 金额不一致触发告警
            alertService.sendAlert("支付金额不一致告警",
                String.format("支付单: %s, 应收: %s, 实收: %s",
                    callback.getPaymentNo(), order.getAmount(), callback.getAmount()));
            return "FAIL";
        }

        // 6. 状态流转
        if (callback.getStatus() == PaymentStatus.SUCCESS) {
            PaymentStateMachine.transition(order, PaymentStatus.SUCCESS);
            order.setChannelTradeNo(callback.getChannelTradeNo());
            order.setPayTime(LocalDateTime.now());
            paymentOrderMapper.updateById(order);

            // 7. 发布支付成功事件
            eventPublisher.publishEvent(new PaymentSuccessEvent(order));
        } else {
            PaymentStateMachine.transition(order, PaymentStatus.FAILED);
            paymentOrderMapper.updateById(order);
        }

        // 8. 更新流水状态
        flow.setStatus("PROCESSED");
        flowMapper.updateById(flow);

        return channel.getSuccessResponse();
    }
}
```

### 4.2 主动查询补偿

```java
/**
 * 支付状态主动查询补偿
 * 不完全依赖渠道回调，定时主动查询兜底
 */
@Service
public class PaymentQueryCompensator {

    @Autowired
    private PaymentOrderMapper paymentOrderMapper;

    @Autowired
    private Map<String, PaymentChannel> channelMap;

    /**
     * 每5分钟扫描支付中的订单，主动查询状态
     */
    @Scheduled(fixedRate = 300000)
    public void queryPendingPayments() {
        // 查询10分钟前仍处于PAYING状态的订单
        List<PaymentOrder> pendingOrders = paymentOrderMapper.selectByStatusAndBeforeTime(
            PaymentStatus.PAYING,
            LocalDateTime.now().minusMinutes(10)
        );

        for (PaymentOrder order : pendingOrders) {
            try {
                PaymentChannel channel = channelMap.get(order.getChannel());
                PaymentQueryResult result = channel.queryPayment(order.getPaymentNo());

                if (result.getStatus() == PaymentStatus.SUCCESS) {
                    // 渠道已成功但本地未更新，补偿处理
                    handleCompensateSuccess(order, result);
                } else if (result.getStatus() == PaymentStatus.CLOSED ||
                           result.getStatus() == PaymentStatus.FAILED) {
                    // 渠道已关闭/失败
                    handleCompensateFail(order, result);
                }
                // PAYING状态继续等待
            } catch (Exception e) {
                log.error("支付查询补偿失败, paymentNo: {}", order.getPaymentNo(), e);
            }
        }
    }

    /**
     * 超时订单自动关闭
     */
    @Scheduled(fixedRate = 60000)
    public void closeTimeoutOrders() {
        List<PaymentOrder> timeoutOrders = paymentOrderMapper.selectByStatusAndBeforeTime(
            PaymentStatus.PAYING,
            LocalDateTime.now().minusMinutes(30) // 30分钟超时
        );

        for (PaymentOrder order : timeoutOrders) {
            try {
                PaymentChannel channel = channelMap.get(order.getChannel());

                // 先查询渠道状态
                PaymentQueryResult result = channel.queryPayment(order.getPaymentNo());
                if (result.getStatus() == PaymentStatus.SUCCESS) {
                    // 用户实际已支付，不能关闭
                    handleCompensateSuccess(order, result);
                    continue;
                }

                // 调用渠道关闭订单
                channel.closePayment(order.getPaymentNo());
                PaymentStateMachine.transition(order, PaymentStatus.CLOSED);
                paymentOrderMapper.updateById(order);
            } catch (Exception e) {
                log.error("超时订单关闭失败, paymentNo: {}", order.getPaymentNo(), e);
            }
        }
    }
}
```

## 5. 对账系统

### 5.1 对账流程

```
T+1对账流程:
1. 下载渠道账单 (T日所有交易记录)
2. 生成系统账单 (系统T日所有支付记录)
3. 双向比对
   - 系统有、渠道有 → 比对金额是否一致
   - 系统有、渠道无 → 长款(系统多记)
   - 系统无、渠道有 → 短款(系统漏记)
   - 金额不一致 → 差异
4. 差异处理
   - 自动补偿: 可自动处理的直接修复
   - 人工介入: 无法自动处理的转人工
5. 生成对账报告
```

### 5.2 对账实现

```java
/**
 * 对账服务
 */
@Service
public class ReconciliationService {

    @Autowired
    private PaymentOrderMapper paymentOrderMapper;

    @Autowired
    private Map<String, PaymentChannel> channelMap;

    @Autowired
    private ReconciliationDiffMapper diffMapper;

    /**
     * 执行T+1对账
     */
    public ReconciliationReport reconcile(LocalDate date) {
        ReconciliationReport report = new ReconciliationReport();
        report.setDate(date);

        // 对每个支付渠道分别对账
        for (String channelName : Arrays.asList("WECHAT", "ALIPAY", "UNIONPAY")) {
            try {
                ChannelReconResult result = reconcileByChannel(channelName, date);
                report.addChannelResult(channelName, result);
            } catch (Exception e) {
                log.error("对账失败, channel: {}, date: {}", channelName, date, e);
                report.addError(channelName, e.getMessage());
            }
        }

        // 生成对账报告
        report.setTotalDiffCount(diffMapper.countByDateAndStatus(date, "UNRESOLVED"));
        saveReport(report);

        // 发送对账报告
        notifyService.sendReconReport(report);

        return report;
    }

    /**
     * 单渠道对账
     */
    private ChannelReconResult reconcileByChannel(String channelName, LocalDate date) {
        PaymentChannel channel = channelMap.get(channelName);

        // 1. 下载渠道账单
        List<ChannelBillItem> channelBills = channel.downloadBill(date);
        Map<String, ChannelBillItem> channelBillMap = channelBills.stream()
            .collect(Collectors.toMap(ChannelBillItem::getOutTradeNo, item -> item));

        // 2. 查询系统账单
        List<PaymentOrder> systemOrders = paymentOrderMapper.selectByChannelAndDate(
            channelName, date.atStartOfDay(), date.plusDays(1).atStartOfDay());
        Map<String, PaymentOrder> systemOrderMap = systemOrders.stream()
            .collect(Collectors.toMap(PaymentOrder::getPaymentNo, order -> order));

        ChannelReconResult result = new ChannelReconResult();
        result.setChannel(channelName);
        result.setSystemCount(systemOrders.size());
        result.setChannelCount(channelBills.size());

        // 3. 双向比对
        for (PaymentOrder order : systemOrders) {
            ChannelBillItem bill = channelBillMap.get(order.getPaymentNo());
            if (bill == null) {
                // 系统有、渠道无 → 长款
                createDiffRecord(date, channelName, order, null, "SYSTEM_EXTRA");
                result.addLongPayment();
            } else {
                // 双方都有，比对金额
                if (order.getAmount().compareTo(bill.getAmount()) != 0) {
                    createDiffRecord(date, channelName, order, bill, "AMOUNT_MISMATCH");
                    result.addAmountMismatch();
                } else {
                    result.addMatched();
                }
            }
        }

        // 系统无、渠道有 → 短款
        for (ChannelBillItem bill : channelBills) {
            if (!systemOrderMap.containsKey(bill.getOutTradeNo())) {
                createDiffRecord(date, channelName, null, bill, "CHANNEL_EXTRA");
                result.addShortPayment();
            }
        }

        return result;
    }

    /**
     * 创建差异记录
     */
    private void createDiffRecord(LocalDate date, String channel,
            PaymentOrder order, ChannelBillItem bill, String diffType) {
        ReconciliationDiff diff = new ReconciliationDiff();
        diff.setReconDate(date);
        diff.setChannel(channel);
        diff.setDiffType(diffType);
        diff.setStatus("UNRESOLVED");

        if (order != null) {
            diff.setPaymentNo(order.getPaymentNo());
            diff.setSystemAmount(order.getAmount());
            diff.setSystemStatus(order.getStatus().name());
        }
        if (bill != null) {
            diff.setChannelTradeNo(bill.getChannelTradeNo());
            diff.setChannelAmount(bill.getAmount());
            diff.setChannelStatus(bill.getStatus());
        }

        diffMapper.insert(diff);

        // 差异告警
        alertService.sendAlert("对账差异告警",
            String.format("日期: %s, 渠道: %s, 类型: %s, 支付单: %s",
                date, channel, diffType,
                order != null ? order.getPaymentNo() : bill.getOutTradeNo()));
    }
}
```

### 5.3 差异处理

```java
/**
 * 对账差异处理
 */
@Service
public class ReconciliationDiffHandler {

    /**
     * 自动处理可修复的差异
     */
    @Scheduled(fixedRate = 600000)
    public void autoResolveDiffs() {
        List<ReconciliationDiff> diffs = diffMapper.selectUnresolved();
        for (ReconciliationDiff diff : diffs) {
            try {
                switch (diff.getDiffType()) {
                    case "SYSTEM_EXTRA":
                        // 系统有渠道无: 可能是渠道延迟，重新查询确认
                        handleSystemExtra(diff);
                        break;
                    case "CHANNEL_EXTRA":
                        // 渠道有系统无: 可能是回调丢失，主动查询补偿
                        handleChannelExtra(diff);
                        break;
                    case "AMOUNT_MISMATCH":
                        // 金额不一致: 转人工处理
                        diff.setStatus("MANUAL_REQUIRED");
                        diff.setHandleNote("金额不一致，需人工核实");
                        diffMapper.updateById(diff);
                        break;
                }
            } catch (Exception e) {
                log.error("差异处理失败, diffId: {}", diff.getId(), e);
            }
        }
    }

    /**
     * 处理系统有渠道无
     */
    private void handleSystemExtra(ReconciliationDiff diff) {
        // 主动查询渠道
        PaymentChannel channel = channelMap.get(diff.getChannel());
        PaymentQueryResult result = channel.queryPayment(diff.getPaymentNo());

        if (result.getStatus() == PaymentStatus.SUCCESS) {
            // 渠道实际已支付，可能是账单延迟，标记已解决
            diff.setStatus("RESOLVED");
            diff.setHandleNote("渠道确认已支付，账单延迟");
        } else if (result.getStatus() == PaymentStatus.CLOSED) {
            // 渠道已关闭，系统状态需修正
            PaymentOrder order = paymentOrderMapper.selectByPaymentNo(diff.getPaymentNo());
            if (order.getStatus() == PaymentStatus.SUCCESS) {
                // 系统误标记为成功，需要回滚
                log.error("严重: 系统标记成功但渠道已关闭! paymentNo: {}", diff.getPaymentNo());
                diff.setStatus("MANUAL_REQUIRED");
                diff.setHandleNote("系统标记成功但渠道已关闭，需人工紧急处理");
            } else {
                diff.setStatus("RESOLVED");
                diff.setHandleNote("渠道已关闭，系统状态一致");
            }
        }
        diffMapper.updateById(diff);
    }
}
```

## 6. 退款流程

### 6.1 退款设计

```java
/**
 * 退款服务
 */
@Service
public class RefundService {

    @Autowired
    private RefundOrderMapper refundOrderMapper;

    @Autowired
    private PaymentOrderMapper paymentOrderMapper;

    @Autowired
    private Map<String, PaymentChannel> channelMap;

    @Autowired
    private IdempotentService idempotentService;

    /**
     * 申请退款
     */
    @Transactional(rollbackFor = Exception.class)
    public RefundResult applyRefund(RefundRequest request) {
        // 1. 幂等检查
        if (!idempotentService.check(request.getRequestId())) {
            return RefundResult.duplicate();
        }

        // 2. 查询原支付单
        PaymentOrder paymentOrder = paymentOrderMapper.selectByPaymentNo(request.getPaymentNo());
        if (paymentOrder == null) {
            throw new PaymentException("原支付单不存在");
        }
        if (paymentOrder.getStatus() != PaymentStatus.SUCCESS) {
            throw new PaymentException("原支付单状态不允许退款");
        }

        // 3. 校验退款金额
        BigDecimal totalRefunded = refundOrderMapper.sumRefundedAmount(request.getPaymentNo());
        BigDecimal remaining = paymentOrder.getAmount().subtract(totalRefunded);
        if (request.getRefundAmount().compareTo(remaining) > 0) {
            throw new PaymentException("退款金额超过可退金额");
        }

        // 4. 创建退款单
        RefundOrder refundOrder = new RefundOrder();
        refundOrder.setRefundNo(generateRefundNo());
        refundOrder.setPaymentNo(request.getPaymentNo());
        refundOrder.setChannel(paymentOrder.getChannel());
        refundOrder.setTotalAmount(paymentOrder.getAmount());
        refundOrder.setRefundAmount(request.getRefundAmount());
        refundOrder.setReason(request.getReason());
        refundOrder.setStatus("PROCESSING");
        refundOrder.setCreateTime(LocalDateTime.now());
        refundOrderMapper.insert(refundOrder);

        // 5. 调用渠道退款
        PaymentChannel channel = channelMap.get(paymentOrder.getChannel());
        RefundResponse response = channel.refund(request);

        refundOrder.setChannelRefundNo(response.getChannelRefundNo());
        refundOrderMapper.updateById(refundOrder);

        // 6. 更新支付单状态
        if (request.getRefundAmount().compareTo(paymentOrder.getAmount()) == 0) {
            PaymentStateMachine.transition(paymentOrder, PaymentStatus.REFUNDING);
        } else {
            PaymentStateMachine.transition(paymentOrder, PaymentStatus.REFUNDING);
        }
        paymentOrderMapper.updateById(paymentOrder);

        return RefundResult.success(refundOrder.getRefundNo());
    }

    /**
     * 退款回调处理
     */
    @Transactional(rollbackFor = Exception.class)
    public void handleRefundCallback(String channelName, Map<String, String> params) {
        PaymentChannel channel = channelMap.get(channelName);
        if (!channel.verifyCallback(params)) {
            return;
        }

        RefundCallback callback = channel.parseRefundCallback(params);
        RefundOrder refundOrder = refundOrderMapper.selectByRefundNo(callback.getRefundNo());

        if (refundOrder == null || "SUCCESS".equals(refundOrder.getStatus())) {
            return; // 不存在或已处理
        }

        if ("SUCCESS".equals(callback.getStatus())) {
            refundOrder.setStatus("SUCCESS");
            refundOrder.setFinishTime(LocalDateTime.now());

            // 判断是否全额退款
            PaymentOrder paymentOrder = paymentOrderMapper.selectByPaymentNo(refundOrder.getPaymentNo());
            BigDecimal totalRefunded = refundOrderMapper.sumRefundedAmount(refundOrder.getPaymentNo());
            if (totalRefunded.compareTo(paymentOrder.getAmount()) == 0) {
                PaymentStateMachine.transition(paymentOrder, PaymentStatus.REFUNDED);
            } else {
                PaymentStateMachine.transition(paymentOrder, PaymentStatus.PARTIAL_REFUNDED);
            }
            paymentOrderMapper.updateById(paymentOrder);
        } else {
            refundOrder.setStatus("FAILED");
            refundOrder.setFailReason(callback.getFailReason());
        }
        refundOrderMapper.updateById(refundOrder);
    }
}
```

```java
@Service
public class RiskControlService {

    /**
     * 风控检查
     */
    public RiskResult check(PaymentRequest request) {
        // 1. 金额检查
        if (request.getAmount().compareTo(new BigDecimal("50000")) > 0) {
            return RiskResult.reject("单笔金额超过5万限额");
        }

        // 2. 频率检查
        int todayCount = paymentOrderMapper.countTodayByUser(request.getUserId());
        if (todayCount > 20) {
            return RiskResult.reject("今日支付次数超过20次");
        }

        // 3. 日累计限额
        BigDecimal todayTotal = paymentOrderMapper.sumTodayByUser(request.getUserId());
        if (todayTotal.add(request.getAmount()).compareTo(new BigDecimal("100000")) > 0) {
            return RiskResult.reject("日累计支付金额超过10万");
        }

        // 4. IP风险检查
        if (blacklistService.isBlackIp(request.getClientIp())) {
            return RiskResult.reject("IP存在风险");
        }

        // 5. 设备指纹检查
        if (request.getDeviceId() != null &&
            blacklistService.isBlackDevice(request.getDeviceId())) {
            return RiskResult.reject("设备存在风险");
        }

        // 6. 异地支付检测
        String usualCity = userService.getUsualCity(request.getUserId());
        String currentCity = ipService.getCity(request.getClientIp());
        if (usualCity != null && !usualCity.equals(currentCity)) {
            // 异地支付需要二次验证
            return RiskResult.needVerify("异地支付，需要短信验证");
        }

        return RiskResult.pass();
    }
```

```java
/**
 * 幂等服务 — 多重保障

@Service
public class IdempotentService {

    @Autowired
    private StringRedisTemplate redisTemplate;

    /**
     * 方案1: Redis SETNX (推荐)
     * 适用于高频接口
     */
    public boolean check(String requestId) {
        String key = "idempotent:" + requestId;
        Boolean isNew = redisTemplate.opsForValue()
            .setIfAbsent(key, "1", 24, TimeUnit.HOURS);
        return Boolean.TRUE.equals(isNew);
    }

    /**
     * 方案2: 数据库唯一索引 (兜底)
     * 即使Redis失效，数据库唯一索引也能防止重复
     */
    // payment_order表上 out_trade_no 字段有唯一索引
    // 重复插入会抛出 DuplicateKeyException

    /**
     * 方案3: 状态机校验 (业务层)
     * 只允许特定状态下执行操作
     */
    public boolean checkByStatus(String paymentNo, PaymentStatus... allowedStatuses) {
        PaymentOrder order = paymentOrderMapper.selectByPaymentNo(paymentNo);
        if (order == null) return false;
        return Arrays.asList(allowedStatuses).contains(order.getStatus());
    }
}

/**
 * 幂等注解AOP
 */
@Target(ElementType.METHOD)
@Retention(RetentionPolicy.RUNTIME)
public @interface Idempotent {
    String field() default "requestId";  // 幂等字段名
    int expire() default 86400;           // 过期时间(秒)
}

@Aspect
@Component
public class IdempotentAspect {

    @Autowired
    private IdempotentService idempotentService;

    @Around("@annotation(idempotent)")
    public Object check(ProceedingJoinPoint pjp, Idempotent idempotent) throws Throwable {
        // 从方法参数中提取幂等字段
        Object[] args = pjp.getArgs();
        String requestId = extractField(args, idempotent.field());

        if (requestId != null && !idempotentService.check(requestId)) {
            throw new DuplicateRequestException("重复请求");
        }

        try {
            return pjp.proceed();
        } catch (DuplicateKeyException e) {
            // 数据库唯一索引兜底
            throw new DuplicateRequestException("重复请求");
        }
    }
}
```

```java
@Service

    /**
     * 渠道降级: 主渠道不可用时自动切换备用渠道

     * MySQL不可用时，先写Redis，异步补偿到DB
     */
    public PaymentResponse createPaymentDegrade(PaymentRequest request) {
        try {
            return createPaymentNormal(request);
        } catch (DataAccessException e) {
            log.error("数据库故障，启动降级模式", e);
            // 降级: 写Redis暂存，标记为待补偿
            String degradeKey = "payment:degrade:" + request.getRequestId();
            redisTemplate.opsForValue().set(degradeKey, JSON.toJSONString(request), 24, TimeUnit.HOURS);

            PaymentResponse response = new PaymentResponse();
            response.setStatus(PaymentStatus.PAYING);
            response.setMessage("支付处理中，请稍后查询结果");
            return response;
        }
    }
```

## 10. 面试题速查

**Q1: 支付回调如何保证幂等？**

三重保障：① Redis SETNX幂等键检查，重复请求直接返回成功；② 数据库唯一索引(out_trade_no)兜底，重复插入抛DuplicateKeyException；③ 状态机校验，只有PAYING状态才能流转到SUCCESS，已成功的订单重复回调直接忽略。

**Q2: 支付系统怎么保证不丢单？**

① 创建支付单时同步落库，返回给用户支付参数；② 支付回调时更新状态；③ 不完全依赖回调，定时任务主动查询渠道补偿；④ 超时订单自动关闭前先查询渠道确认是否已支付。极端情况通过T+1对账发现差异。

**Q3: 对账系统怎么设计？**

T+1对账：下载渠道账单，查询系统账单，双向比对。差异类型：长款(系统有渠道无)、短款(渠道有系统无)、金额不一致。自动处理可修复差异(如回调丢失导致的短款)，不可自动处理的转人工。每日生成对账报告并发送告警。

**Q4: 退款流程怎么设计？**

创建退款单 → 调用渠道退款接口 → 渠道异步处理 → 退款回调更新状态。支持部分退款：记录累计退款金额，不超过原支付金额。退款后更新原支付单状态(全额退款→REFUNDED，部分退款→PARTIAL_REFUNDED)。

**Q5: 支付系统如何做风控？**

多层风控：① 单笔限额、日累计限额、频率限制；② IP/设备黑名单；③ 异地支付检测需要二次验证；④ 异常行为分析(如短时间内大量支付)；⑤ 大额支付人工审核。风控规则引擎化，可动态配置。

**Q6: 支付渠道接口差异怎么处理？**

适配器模式：定义统一PaymentChannel接口(createPayment/queryPayment/refund/verifyCallback/parseCallback)，每个渠道(微信/支付宝/银联)实现该接口。支付网关通过渠道名称路由到对应实现。新增渠道只需新增适配器实现类。

**Q7: 支付系统高可用怎么保证？**

① 支付网关多实例部署，负载均衡；② Redis Cluster保证幂等键高可用；③ MySQL主从+半同步复制保证数据不丢；④ 渠道降级：主渠道不可用自动切换备用渠道；⑤ 数据库故障降级：先写Redis，异步补偿到DB；⑥ 监控告警全覆盖。

**Q8: 如何防止支付金额被篡改？**

① 创建支付单时在服务端计算金额，不信任前端传入的金额；② 回调时校验回调金额与系统金额是否一致，不一致立即告警；③ API签名验证防止请求被篡改；④ HTTPS传输加密；⑤ 敏感操作(如修改金额)需要二次验证。
