## 1. 系统概述

```
文件存储系统架构：

  ┌──────────┐    上传     ┌──────────────┐    存储    ┌──────────┐
  │  客户端    │ ─────────→ │  文件服务API  │ ─────────→ │ 对象存储  │
  └──────────┘            └──────┬───────┘            │ MinIO/S3 │
                                 │                     └──────────┘
                                 │ 元数据
                          ┌──────▼───────┐
                          │   MySQL      │  文件元数据
                          │  (文件表)     │  (MD5/大小/路径)
                          └──────────────┘
                                 │
                          ┌──────▼───────┐
                          │   Redis      │  分片上传状态
                          │  (分片记录)    │
                          └──────────────┘

核心能力：
  1. 分片上传 — 大文件分块上传，降低单次请求大小
  2. 断点续传 — 上传中断后可恢复，避免重新上传
  3. 秒传     — 相同文件MD5匹配，跳过上传
  4. CDN加速  — 静态文件就近访问
  5. 防盗链   — 签名URL防盗用
```

## 2. 分片上传与断点续传

### 2.1 上传流程

```
分片上传流程：

  客户端                           服务端
    │                                │
    │  1. 请求上传(文件名/大小/MD5)    │
    │ ──────────────────────────────→ │  检查MD5 → 秒传?
    │  ←── 返回uploadId/分片大小      │
    │                                │
    │  2. 计算分片，逐个上传           │
    │ ──→ POST /upload/{uploadId}/part/1
    │ ←── part1 ETag                 │
    │ ──→ POST /upload/{uploadId}/part/2
    │ ←── part2 ETag                 │
    │  ...                           │
    │                                │
    │  3. 合并请求                    │
    │ ──→ POST /upload/{uploadId}/complete
    │ ←── 文件URL                     │  合并分片
    │                                │
```

### 2.2 服务端实现

```java
@RestController
@RequestMapping("/file")
public class FileController {
    
    @Autowired
    private FileService fileService;
    @Autowired
    private RedisTemplate<String, Object> redisTemplate;
    
    private static final int CHUNK_SIZE = 5 * 1024 * 1024; // 5MB
    
    // 1. 初始化上传
    @PostMapping("/upload/init")
    public AjaxResult initUpload(@RequestBody UploadInitRequest req) {
        // 检查秒传
        FileMeta exist = fileService.findByMd5(req.getMd5());
        if (exist != null) {
            return AjaxResult.success("秒传成功", exist.getUrl());
        }
        
        String uploadId = UUID.randomUUID().toString();
        int totalChunks = (int) Math.ceil((double) req.getSize() / CHUNK_SIZE);
        
        // 记录上传状态到Redis
        UploadStatus status = new UploadStatus();
        status.setUploadId(uploadId);
        status.setFileName(req.getFileName());
        status.setFileSize(req.getSize());
        status.setMd5(req.getMd5());
        status.setTotalChunks(totalChunks);
        status.setUploadedChunks(new ArrayList<>());
        
        redisTemplate.opsForValue().set("upload:" + uploadId, status, 24, TimeUnit.HOURS);
        
        return AjaxResult.success()
            .put("uploadId", uploadId)
            .put("chunkSize", CHUNK_SIZE)
            .put("totalChunks", totalChunks);
    }
    
    // 2. 上传分片
    @PostMapping("/upload/{uploadId}/part/{chunkNumber}")
    public AjaxResult uploadPart(@PathVariable String uploadId,
                                  @PathVariable int chunkNumber,
                                  @RequestParam("file") MultipartFile file) {
        UploadStatus status = (UploadStatus) redisTemplate.opsForValue().get("upload:" + uploadId);
        if (status == null) {
            return AjaxResult.error("上传已过期，请重新初始化");
        }
        
        // 存储分片到临时目录
        String chunkPath = "/tmp/upload/" + uploadId + "/" + chunkNumber;
        File chunkFile = new File(chunkPath);
        file.transferTo(chunkFile);
        
        // 记录已上传分片
        status.getUploadedChunks().add(chunkNumber);
        redisTemplate.opsForValue().set("upload:" + uploadId, status, 24, TimeUnit.HOURS);
        
        return AjaxResult.success().put("chunkNumber", chunkNumber);
    }
    
    // 3. 查询已上传分片（断点续传）
    @GetMapping("/upload/{uploadId}/status")
    public AjaxResult getUploadStatus(@PathVariable String uploadId) {
        UploadStatus status = (UploadStatus) redisTemplate.opsForValue().get("upload:" + uploadId);
        if (status == null) {
            return AjaxResult.error("上传记录不存在");
        }
        return AjaxResult.success()
            .put("uploadedChunks", status.getUploadedChunks())
            .put("totalChunks", status.getTotalChunks());
    }
    
    // 4. 合并分片
    @PostMapping("/upload/{uploadId}/complete")
    public AjaxResult completeUpload(@PathVariable String uploadId) {
        UploadStatus status = (UploadStatus) redisTemplate.opsForValue().get("upload:" + uploadId);
        if (status == null || status.getUploadedChunks().size() != status.getTotalChunks()) {
            return AjaxResult.error("分片不完整");
        }
        
        // 合并分片
        String filePath = fileService.mergeChunks(uploadId, status);
        
        // 上传到对象存储
        String url = fileService.uploadToOss(filePath, status.getFileName());
        
        // 保存文件元数据
        FileMeta meta = new FileMeta();
        meta.setFileName(status.getFileName());
        meta.setFileSize(status.getFileSize());
        meta.setMd5(status.getMd5());
        meta.setUrl(url);
        fileService.save(meta);
        
        // 清理临时文件和Redis
        fileService.cleanChunks(uploadId);
        redisTemplate.delete("upload:" + uploadId);
        
        return AjaxResult.success("上传成功", url);
    }
}
```

## 4. 对象存储集成

```java
// MinIO客户端
@Configuration
public class MinioConfig {
    
    @Bean
    public MinioClient minioClient() {
        return MinioClient.builder()
            .endpoint("http://minio:9000")
            .credentials("accessKey", "secretKey")
            .build();
    }
}

@Service
public class OssService {
    
    @Autowired
    private MinioClient minioClient;
    
    private static final String BUCKET = "files";
    
    // 上传文件
    public String upload(String objectName, InputStream stream, long size, String contentType) {
        try {
            minioClient.putObject(
                PutObjectArgs.builder()
                    .bucket(BUCKET)
                    .object(objectName)
                    .stream(stream, size, -1)
                    .contentType(contentType)
                    .build()
            );
            return BUCKET + "/" + objectName;
        } catch (Exception e) {
            throw new RuntimeException("上传失败", e);
        }
    }
    
    // 生成预签名URL（临时访问/下载）
    public String getPresignedUrl(String objectName, int expireSeconds) {
        try {
            return minioClient.getPresignedObjectUrl(
                GetPresignedObjectUrlArgs.builder()
                    .method(Method.GET)
                    .bucket(BUCKET)
                    .object(objectName)
                    .expiry(expireSeconds)
                    .build()
            );
        } catch (Exception e) {
            throw new RuntimeException("生成URL失败", e);
        }
    }
}
```

## 5. CDN加速与防盗链

```
CDN加速：
  用户 → CDN边缘节点 → 源站(对象存储)
  静态文件缓存到边缘节点，就近访问

防盗链方案：
  1. Referer白名单：只允许指定域名访问
  2. 签名URL：URL带时效性签名，过期失效
  3. Token鉴权：需要登录后才能访问
```

```java
// 生成带签名的临时URL（防盗链）
public String generateSignedUrl(String objectName, int expireSeconds) {
    long expireAt = System.currentTimeMillis() / 1000 + expireSeconds;
    String stringToSign = objectName + "\n" + expireAt;
    String signature = hmacSHA1(stringToSign, secretKey);
    
    return cdnDomain + "/" + objectName 
        + "?expires=" + expireAt 
        + "&signature=" + urlEncode(signature);
}
```

## 6. 面试题速查

**Q1: 大文件上传怎么处理？**
```
分片上传：文件切成5MB小块逐个上传
断点续传：记录已上传分片，中断后查询续传
合并：所有分片上传完成后服务端合并
```

**Q2: 秒传原理？**
```
客户端计算文件MD5 → 服务端检查MD5是否存在
存在则直接返回URL，跳过上传
不存在则正常上传，上传后保存MD5
```

**Q3: 对象存储和本地文件系统的区别？**
```
对象存储（MinIO/S3）：分布式、高可用、REST API、无限扩容
本地文件系统：简单但不可扩展、单点故障
生产环境用对象存储 + CDN
```

**Q4: 文件防盗链怎么做？**
```
1. Referer白名单（简单但有局限）
2. 签名URL（推荐）：URL带时效性签名
3. Token鉴明（最安全但复杂）
```
