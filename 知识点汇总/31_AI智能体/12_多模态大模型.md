## 1. 多模态概述

```
多模态(Multimodal) = 文本 + 图像 + 音频 + 视频

  ┌──────────────────────────────────────────────────────┐
  │  输入模式    │  任务          │  模型                  │
  │  ──────   │  ────       │  ────              │
  │  文本→文本  │  对话/翻译     │  GPT/Claude/Qwen      │
  │  图像→文本  │  图像描述/VQA  │  LLaVA/Qwen-VL        │
  │  文本→图像  │  文生图        │  Stable Diffusion/FLUX│

  │  文本→视频  │  文生视频      │  Sora/Kling            │
  └──────────────────────────────────────────────────────┘

## 2. 视觉语言模型架构

### 2.1 LLaVA架构

```
LLaVA = 视觉编码器 + 投影层 + LLM

  ┌──────────────────────────────────────────────────┐
  │  图像                                             │
  │    ↓                                             │
  │  ┌──────────────┐                               │
  │  │ 视觉编码器    │  CLIP ViT-L/14                │
  │  │ (冻结)        │  图像 → 图像特征向量序列        │
  │  └──────┬───────┘                               │
  │         ↓                                       │
  │  ┌──────────────┐                               │
  │  │ 投影层        │  MLP (训练)                   │
  │  │ (训练)        │  图像特征 → LLM嵌入空间         │
  │  └──────┬───────┘                               │
  │         ↓                                       │
  │  [图像token] + [文本token] → LLM → 输出文本      │
  └──────────────────────────────────────────────────┘

  核心思想:
    1. CLIP视觉编码器(预训练，冻结)提取图像特征
    2. 投影层(训练)将图像特征对齐到LLM的文本嵌入空间
    3. 图像token和文本token拼接 → LLM处理
    4. LLM理解图像+文本 → 生成回答

  训练两阶段:
    Stage 1: 只训练投影层(图像特征对齐)
    Stage 2: 微调投影层+LLM(视觉指令跟随)
```

## 4. 语音处理

### 4.1 语音识别(ASR)

```
Whisper(OpenAI开源) — 语音转文本

  特点:
    1. 多语言 — 支持99种语言
    2. 鲁棒性强 — 噪音环境/口音/语速
    3. 翻译 — 可直接翻译成英文
    4. 开源免费 — 可本地部署

  部署:
    # Python
    pip install openai-whisper
    import whisper
    model = whisper.load_model("base")
    result = model.transcribe("audio.mp3")
    print(result["text"])
```

### 4.2 语音合成(TTS)

```
CosyVoice(阿里开源) — 文本转语音

  特点:
    1. 语音克隆 — 3秒样本克隆声音
    2. 情感语音 — 开心/悲伤/愤怒等
    3. 多语言 — 中英日韩
    4. 流式合成 — 实时输出

  实时语音对话:
    用户语音 → Whisper(ASR) → LLM → CosyVoice(TTS) → 语音输出
    流式: 边识别→边生成→边合成(低延迟)
```

## 5. 多模态RAG

```
多模态RAG = 文本+图片+表格混合检索

  ┌──────────────────────────────────────────────────────┐
  │  文档(PDF含图文)                                      │
  │    ├── 文本 → 文本嵌入 → 向量库                        │
  │    ├── 图片 → CLIP嵌入 → 向量库                        │
  │    └── 表格 → 文本描述 → 向量库                        │
  │                                                      │
  │  查询: "这张架构图是什么意思?"                         │
  │    → 检索文本+图片                                     │
  │    → 多模态LLM(如Qwen-VL)理解图片+文本                 │
  │    → 生成回答                                         │
  └──────────────────────────────────────────────────────┘

  实现方案:
    1. 文本+图片统一嵌入(CLIP/Jina-CLIP)
    2. 向量库支持多模态(Milvus/Qdrant)
    3. 检索后用多模态LLM理解图片
    4. 文档解析时保留图片位置信息
```

```
多模态Agent = 视觉理解 + 工具调用

  ┌──────────────────────────────────────────────────────┐

  │  场景2: GUI Agent                                     │
  │  截屏 → GPT-4o理解界面 → 生成操作指令 → 执行          │
  │  (Anthropic Computer Use / OpenAI Operator)          │
  │                                                      │
  │  场景3: 图表分析                                      │
  │  上传图表 → VLM理解数据 → 计算工具分析 → 生成报告     │
  └──────────────────────────────────────────────────────┘

  GUI Agent(Computer Use):
    1. 截屏 → VLM识别界面元素(按钮/输入框/菜单)
    2. VLM生成操作(点击坐标/输入文字)
    3. 执行操作(鼠标/键盘)
    4. 截屏 → 观察结果 → 下一步
    → 实现自动化操作电脑
    代表: Anthropic Computer Use / OpenAI Operator

## 7. 面试题速查

**Q1: LLaVA的架构？**

```
视觉编码器(CLIP ViT，冻结) + 投影层(MLP，训练) + LLM(训练)
图像→CLIP特征→投影层对齐到文本空间→与文本token拼接→LLM处理
训练: Stage1只训投影层, Stage2微调投影层+LLM
```

**Q2: Qwen-VL为什么中文OCR好？**

```
1. 中文训练数据多
2. 多分辨率支持(高清图片)
3. 专门优化了表格/公式/中英混合
4. 开源可微调
```

**Q3: 多模态RAG怎么做？**

```
文档解析: 文本→文本嵌入, 图片→CLIP嵌入, 表格→文本描述
统一向量库: Milvus/Qdrant支持多模态
检索: 文本+图片混合检索
生成: 多模态LLM(Qwen-VL/GPT-4o)理解图片+文本
```

**Q4: GUI Agent怎么工作？**

```
截屏→VLM识别界面元素→生成操作(点击/输入)→执行→截屏观察→循环
代表: Anthropic Computer Use
应用: 自动化操作电脑/填表/测试
```
