## 2. 多模型路由

```java
// 按任务复杂度路由到不同模型(省钱)
@Service
public class ModelRouter {

    @Autowired
    @Qualifier("gptClient")
    private ChatClient gptClient;        // 最强但贵

    @Autowired
    @Qualifier("qwenClient")
    private ChatClient qwenClient;       // 中等

    @Autowired
    @Qualifier("localClient")
    private ChatClient localClient;      // 本地免费

    public ChatClient route(String userInput) {
        // 1. 简单问题 → 本地模型(免费)
        if (isSimpleQuestion(userInput)) {
            return localClient;
        }
        // 2. 中文场景 → 通义千问(中文好且便宜)
        if (isChineseContent(userInput)) {
            return qwenClient;
        }
        // 3. 复杂推理 → GPT-5(最强)
        if (needsComplexReasoning(userInput)) {
            return gptClient;
        }
        // 4. 默认 → 通义千问
        return qwenClient;
    }

    // 用LLM分类或规则判断
    private boolean isSimpleQuestion(String input) {
        return input.length() < 20 ||
               input.contains("翻译") ||
               input.contains("总结");
    }
}
```

## 3. 缓存与成本优化

### 3.1 响应缓存

```java
@Service
public class CachedChatService {

    @Autowired
    private RedisTemplate<String, String> redis;

    @Autowired
    private EmbeddingModel embeddingModel;

    // 精确缓存 — 相同输入直接返回
    public String chatExact(String input) {
        String key = "chat:exact:" + DigestUtils.md5Hex(input);
        String cached = redis.opsForValue().get(key);
        if (cached != null) {
            return cached;  // 命中，省LLM调用
        }
        String response = chatClient.prompt().user(input).call().content();
        redis.opsForValue().set(key, response, Duration.ofHours(24));
        return response;
    }

    // 语义缓存 — 语义相似的输入复用历史回答
    public String chatSemantic(String input) {
        float[] queryVec = embeddingModel.embed(input);

        // 在Redis中搜索相似问题
        String cachedKey = searchSimilarQuestion(queryVec, 0.95f);
        if (cachedKey != null) {
            return redis.opsForValue().get(cachedKey);
        }

        String response = chatClient.prompt().user(input).call().content();

        // 存入缓存(向量+回答)
        storeSemanticCache(input, queryVec, response);
        return response;
    }
}
```

### 3.2 成本优化策略

```
┌──────────────────────────────────────────────────────┐
  │  策略          │  节省比例    │  实现                │
  │  ──────     │  ────    │  ────           │
  │  模型路由      │  40-60%     │  简单→小模型          │
  │  响应缓存      │  20-30%     │  相同/相似问题复用     │
  │  Prompt压缩   │  10-20%     │  减少Token输入        │
  │  批处理API     │  50%        │  非实时用Batch API    │
  │  Prefix Cache │  30-80%     │  共享System Prompt   │
  │  本地模型      │  100%(API)  │  vLLM本地部署        │
  └──────────────────────────────────────────────────────┘

  Token预算管理:
    单次请求限制: max_tokens=2000
    用户日限额: daily_token_budget=100000
    月度总预算: monthly_budget=$500
    超限 → 降级到小模型 / 提示用户

  Prompt压缩:
    1. 去除冗余描述
    2. 用缩写/符号代替长描述
    3. 压缩历史对话(摘要)
    4. 用Prompt Caching(Anthropic/Claude)
```

## 4. 限流与降级

```java
@Service
public class ResilientChatService {

    @Autowired
    private ChatClient primaryClient;   // 主模型
    @Autowired
    private ChatClient fallbackClient;  // 备用模型

    @RateLimiter(name = "llmApi", fallbackMethod = "rateLimitFallback")
    @CircuitBreaker(name = "llmApi", fallbackMethod = "circuitBreakFallback")
    @Retry(name = "llmApi")
    public String chat(String input) {
        try {
            return primaryClient.prompt()
                .user(input)
                .call()
                .content();
        } catch (Exception e) {
            // 主模型失败 → 降级到备用
            return fallbackClient.prompt()
                .user(input)
                .call()
                .content();
        }
    }

    // 限流降级
    public String rateLimitFallback(String input, Exception e) {
        return "当前请求过多，请稍后重试";
    }

    // 熔断降级
    public String circuitBreakFallback(String input, Exception e) {
        // 降级到本地小模型
        return fallbackClient.prompt().user(input).call().content();
    }
}

// application.yml
// resilience4j.ratelimiter:
//   llmApi:
//     limitForPeriod: 50        # 每周期50次
//     limitRefreshPeriod: 1s    # 1秒
//     timeoutDuration: 10s
// resilience4j.circuitbreaker:
//   llmApi:
//     failureRateThreshold: 50  # 失败率50%熔断
//     waitDurationInOpenState: 30s
//     ringBufferSizeInHalfOpenState: 10
```

## 5. 会话与状态管理

```java
@Service
public class SessionManager {

    @Autowired
    private ChatMemory chatMemory;
    @Autowired
    private RedisTemplate<String, String> redis;

    // 多用户会话隔离
    public String chat(String sessionId, String userId, String message) {
        // 会话ID = userId + sessionId
        String conversationId = userId + ":" + sessionId;

        return chatClient.prompt()
            .user(message)
            .advisors(MessageChatMemoryAdvisor.builder()
                .chatMemory(chatMemory)
                .conversationId(conversationId)
                .build())
            .call()
            .content();
    }

    // 会话历史持久化
    public void saveSession(String sessionId, List<Message> messages) {
        redis.opsForValue().set(
            "session:" + sessionId,
            JsonUtils.toJson(messages),
            Duration.ofDays(7)  // 7天过期
        );
    }

    // 恢复会话
    public List<Message> restoreSession(String sessionId) {
        String json = redis.opsForValue().get("session:" + sessionId);
        return json != null ? JsonUtils.parseList(json, Message.class) : new ArrayList<>();
    }
}
```

## 7. 面试题速查

**Q1: 如何降低LLM API成本？**

```
1. 模型路由 — 简单问题用小模型(省40-60%)
2. 响应缓存 — 精确+语义缓存(省20-30%)
3. Prompt压缩 — 减少Token(省10-20%)
4. 批处理API — 非实时用Batch(省50%)
5. Prefix Cache — 共享System Prompt
6. 本地模型 — vLLM部署(省API费)
```

**Q2: 多模型路由怎么做？**

```
按任务复杂度/语言/领域路由:
简单→本地小模型(免费)
中文→通义千问(便宜中文好)
复杂推理→GPT-5/Claude(最强)
用LLM分类或规则判断
```

**Q3: AI应用如何降级？**

```
主模型失败→备用模型→规则兜底
Resilience4j: 限流+熔断+重试
限流: RPM/TPM控制
熔断: 失败率超阈值→暂停→降级
```

**Q4: 多用户会话如何隔离？**

```
conversationId = userId + sessionId
ChatMemory按conversationId隔离
历史持久化到Redis(7天过期)
恢复时从Redis加载
```
