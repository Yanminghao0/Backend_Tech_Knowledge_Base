## 1. 评估概述

```
AI应用评估维度:

  ┌──────────────────────────────────────────────────────┐
  │  维度        │  含义              │  评估方法          │
  │  ──────   │  ────          │  ──────      │
  │  准确性      │  回答是否正确       │  人工/LLM裁判      │
  │  忠实度      │  是否基于检索内容   │  RAGAS自动评估     │
  │  相关性      │  回答是否切题       │  LLM裁判           │
  │  完整性      │  是否回答全面       │  人工/LLM裁判      │
  │  安全性      │  是否有害/不当      │  安全分类器         │
  │  流畅性      │  语言是否通顺       │  LLM裁判           │
  └──────────────────────────────────────────────────────┘

  评估层次:
    离线评估 — 上线前用Golden Set测试
    在线评估 — 上线后持续监控
    A/B测试 — 对比不同模型/Prompt/参数
```

## 2. LLM评估方法

### 2.1 人工评估

```
Likert量表(1-5分):
  1分 — 完全错误
  2分 — 大部分错误
  3分 — 部分正确
  4分 — 大部分正确
  5分 — 完全正确

  成对比较:
  给两个模型的回答，人工选更好的
  适合: 模型对比/AB测试
  问题: 主观差异 → 需要多个标注员

  专家评估:
  领域专家深度评估(如医生评估医学QA)
  成本高但质量最好
```

### 2.2 LLM-as-Judge

```python
# 用GPT-5/Claude评估其他模型输出
def llm_judge(question, answer, reference=None):
    prompt = f"""
    请评估以下AI回答的质量。

    问题: {question}
    回答: {answer}
    {"参考答案: " + reference if reference else ""}

    评分标准(1-5):
    1. 准确性(1-5): 事实是否正确
    2. 完整性(1-5): 是否完全回答了问题
    3. 清晰度(1-5): 表达是否清晰
    4. 安全性(1-5): 是否安全无害

    请输出JSON: {{"accuracy": X, "completeness": X, "clarity": X, "safety": X, "comment": "..."}}
    """
    result = gpt5.invoke(prompt)
    return parse_json(result)

# 批量评估
for sample in test_set:
    score = llm_judge(sample.question, sample.answer, sample.reference)
    print(f"Q: {sample.question[:50]}...")
    print(f"  Scores: {score}")
```

## 3. RAG评估(RAGAS)

### 3.1 RAG评估指标

```
RAG评估四大指标(RAGAS框架):

  ┌──────────────────────────────────────────────────────┐
  │  指标              │  含义                           │
  │  ──────         │  ────                       │
  │  Faithfulness      │  忠实度: 回答是否基于检索内容   │
  │                    │  (不编造检索中没有的信息)       │
  │  Answer Relevancy  │  答案相关性: 回答是否切题       │
  │  Context Precision │  上下文精确率: 检索结果是否相关  │
  │  Context Recall    │  上下文召回率: 是否检索到所有    │
  │                    │  需要的信息                     │
  └──────────────────────────────────────────────────────┘

  其他指标:
    Context Entity Recall — 检索到的实体覆盖率
    Answer Correctness — 答案与参考答案的匹配度
    Answer Similarity — 语义相似度
```

### 3.2 RAGAS实战

```python
from ragas import evaluate
from ragas.metrics import (
    faithfulness, answer_relevancy,
    context_precision, context_recall
)
from datasets import Dataset

# 准备评估数据
eval_data = {
    "question": ["Java的volatile关键字有什么作用?"],
    "answer": ["volatile保证可见性和有序性，但不保证原子性"],
    "contexts": [["volatile是Java的轻量级同步机制，保证可见性、有序性，不保证原子性..."]],
    "ground_truth": ["volatile保证可见性(写入立即刷新主内存)和有序性(禁止指令重排)，不保证原子性"]
}

dataset = Dataset.from_dict(eval_data)

# 评估
results = evaluate(
    dataset,
    metrics=[faithfulness, answer_relevancy, context_precision, context_recall],
)

print(results)
# {'faithfulness': 1.0, 'answer_relevancy': 0.95,
#  'context_precision': 1.0, 'context_recall': 0.9}
```

```
  │  指标            │  含义                             │
  │  ──────       │  ────                         │

    1. 固定测试集 — 100个任务 → 完成率
    2. 对比基线 — Agent A vs Agent B
    3. LLM裁判 — LLM评估Agent的执行轨迹质量
    4. 人工审查 — 抽样查看执行步骤
```

## 6. LangSmith与Langfuse

### 6.1 LangSmith

```
LangSmith — LangChain官方可观测性平台

  功能:
    1. Trace追踪 — 完整请求链路
    2. 评估 — 自动/人工评估
    3. 数据集管理 — Golden Set
    4. A/B测试 — 对比不同配置
    5. 监控 — 实时指标

  Trace链路:
    用户输入 → [Retriever] → [Prompt] → [LLM] → 输出
    每步记录: 输入/输出/耗时/Token/成本
```

### 6.2 Langfuse

```
Langfuse — 开源LLM可观测性(可自部署)

  优势(相比LangSmith):
    1. 开源 — 可自部署(数据不外泄)
    2. 模型无关 — 不绑定LangChain
    3. 多语言 — Python/JS/Java SDK

  部署:
    docker run -p 3000:3000 langfuse/langfuse

  功能:
    Trace追踪 — 完整请求链路
    评估 — LLM-as-Judge自动评估
    成本分析 — Token消耗+费用
    用户反馈 — 点赞/点踩收集
```

```java
// Spring AI + Langfuse集成
// 通过自定义Advisor实现追踪

@Component
public class LangfuseAdvisor implements BaseAdvisor {

    @Override
    public AdvisedResponse aroundCall(AdvisedRequest request, CallAroundAdvisorChain chain) {
        long start = System.currentTimeMillis();
        String traceId = UUID.randomUUID().toString();

        // 记录输入
        langfuseClient.trace(traceId, "input", request.userText());

        AdvisedResponse response = chain.nextAroundCall(request);

        // 记录输出+耗时+Token
        long elapsed = System.currentTimeMillis() - start;
        langfuseClient.event(traceId, "output", response.content(),
            Map.of("latency_ms", elapsed, "tokens", response.metadata().getUsage()));

        return response;
    }
}
```

## 7. 面试题速查

**Q1: RAGAS的四大评估指标？**

```
Faithfulness(忠实度): 回答是否基于检索内容(不幻觉)
Answer Relevancy(答案相关性): 回答是否切题
Context Precision(上下文精确率): 检索结果是否相关
Context Recall(上下文召回率): 是否检索到所有需要信息
```

**Q2: LLM-as-Judge怎么用？**

```
用GPT-5/Claude等强模型评估其他模型输出
按维度打分(准确性/完整性/清晰度/安全性)
优势: 自动化、可批量
注意: 裁判模型要比被评估模型强
```

**Q3: AI应用需要监控哪些指标？**

```
性能: P99延迟/首Token延迟/流式速度
质量: 错误率/用户点踩率/LLM裁判分
成本: Token消耗/日费用/每次调用费用
资源: GPU利用率/显存使用/QPS
```

**Q4: LangSmith和Langfuse的区别？**

```
LangSmith: LangChain官方/SaaS/绑定LangChain
Langfuse: 开源/可自部署/模型无关
选择: 用LangChain → LangSmith; 自部署/多框架 → Langfuse
```
