## 1. 重构原则

### 1.1 重构的两顶帽子

```java
// Martin Fowler的"两顶帽子"比喻：
// 开发软件时，你总是在两顶帽子之间切换：
// 1. 重构帽子：不添加新功能，只改善结构
// 2. 功能帽子：不改善结构，只添加功能
// 永远不要同时戴两顶帽子！

// ❌ 错误做法：一边加功能一边重构
public class OrderService {
    public void processOrder(Order order) {
        // 顺便重构：提取验证方法
        validateOrder(order);  // 这是在重构
        
        // 添加新功能：支持VIP折扣
        if (order.getCustomer().isVip()) {  // 这是加功能
            order.setDiscount(order.getAmount().multiply(new BigDecimal("0.15")));
        }
        
        // 顺便重构：提取保存方法
        saveOrder(order);  // 这是在重构
    }
}

// ✅ 正确做法：分开进行
// 第一步：重构（不加功能）
// - 提取validateOrder方法
// - 提取saveOrder方法
// - 确保测试通过
// - 提交重构代码

// 第二步：加功能（不重构）
// - 在processOrder中添加VIP折扣逻辑
// - 添加测试
// - 提交新功能
```

### 2.3 提取方法实践案例

```java
// 重构前：复杂的订单处理
public OrderResult processOrder(Order order) {
    // 验证订单
    if (order.getItems() == null || order.getItems().isEmpty()) {
        throw new IllegalArgumentException("订单项不能为空");
    }
    if (order.getCustomer() == null) {
        throw new IllegalArgumentException("客户信息不能为空");
    }
    if (order.getAmount() == null || order.getAmount().compareTo(BigDecimal.ZERO) <= 0) {
        throw new IllegalArgumentException("订单金额必须大于0");
    }
    
    // 计算折扣
    BigDecimal discount = BigDecimal.ZERO;
    if (order.getCustomer().isVip()) {
        if (order.getAmount().compareTo(new BigDecimal("1000")) > 0) {
            discount = order.getAmount().multiply(new BigDecimal("0.15"));
        } else {
            discount = order.getAmount().multiply(new BigDecimal("0.10"));
        }
    } else {
        if (order.getAmount().compareTo(new BigDecimal("500")) > 0) {
            discount = order.getAmount().multiply(new BigDecimal("0.05"));
        }
    }
    
    // 扣减库存
    for (OrderItem item : order.getItems()) {
        Product product = productRepository.findById(item.getProductId());
        if (product.getStock() < item.getQuantity()) {
            throw new BusinessException("库存不足: " + product.getName());
        }
        product.setStock(product.getStock() - item.getQuantity());
        productRepository.save(product);
    }
    
    // 保存订单
    order.setDiscount(discount);
    order.setFinalAmount(order.getAmount().subtract(discount));
    order.setStatus(OrderStatus.PROCESSED);
    order.setProcessedAt(LocalDateTime.now());
    orderRepository.save(order);
    
    return OrderResult.success(order);
}

// 重构后：清晰的职责分离
public OrderResult processOrder(Order order) {
    validateOrder(order);
    applyDiscount(order);
    deductStock(order);
    return saveOrder(order);
}

private void validateOrder(Order order) {
    validateOrderItems(order);
    validateCustomer(order);
    validateAmount(order);
}

private void validateOrderItems(Order order) {
    if (CollectionUtils.isEmpty(order.getItems())) {
        throw new IllegalArgumentException("订单项不能为空");
    }
}

private void validateCustomer(Order order) {
    if (order.getCustomer() == null) {
        throw new IllegalArgumentException("客户信息不能为空");
    }
}

private void validateAmount(Order order) {
    if (order.getAmount() == null || 
        order.getAmount().compareTo(BigDecimal.ZERO) <= 0) {
        throw new IllegalArgumentException("订单金额必须大于0");
    }
}

private void applyDiscount(Order order) {
    DiscountPolicy policy = DiscountPolicy.forCustomer(order.getCustomer());
    BigDecimal discount = policy.calculate(order.getAmount());
    order.setDiscount(discount);
    order.setFinalAmount(order.getAmount().subtract(discount));
}

private void deductStock(Order order) {
    for (OrderItem item : order.getItems()) {
        stockService.deduct(item.getProductId(), item.getQuantity());
    }
}

private OrderResult saveOrder(Order order) {
    order.setStatus(OrderStatus.PROCESSED);
    order.setProcessedAt(LocalDateTime.now());
    orderRepository.save(order);
    return OrderResult.success(order);
}

## 4. 搬移方法

### 4.1 搬移到更合适的类

// 重构前：方法在错误的类中（特征依恋）
public class Account {
    private AccountType type;
    private int daysOverdrawn;
    
    double overdraftCharge() {
        // 这个方法主要使用AccountType的数据
        // 应该搬到AccountType中
        if (type.isPremium()) {
            double result = 10;
            if (daysOverdrawn > 7) {
                result += (daysOverdrawn - 7) * 0.85;
            }
            return result;
        } else {
            return daysOverdrawn * 1.75;
        }
    }
    
    double bankCharge() {
        double result = 4.5;
        if (daysOverdrawn > 0) {
            result += overdraftCharge();
        }
        return result;
    }
}

// 重构后：方法搬到AccountType
public class AccountType {
    double overdraftCharge(int daysOverdrawn) {
        if (isPremium()) {
            double result = 10;
            if (daysOverdrawn > 7) {
                result += (daysOverdrawn - 7) * 0.85;
            }
            return result;
        } else {
            return daysOverdrawn * 1.75;
        }
    }
}

public class Account {
    private AccountType type;
    private int daysOverdrawn;
    
    double overdraftCharge() {
        return type.overdraftCharge(daysOverdrawn);  // 委托
    }
    
    double bankCharge() {
        double result = 4.5;
        if (daysOverdrawn > 0) {
            result += type.overdraftCharge(daysOverdrawn);  // 直接调用
        }
        return result;
    }
}

### 6.1 替换为更清晰的算法

// 重构前：复杂难懂的算法
public String findPerson(String[] people) {
    for (int i = 0; i < people.length; i++) {
        for (int j = 0; j < people.length; j++) {
            if (people[i].equals("Don")) {
                return "Don";
            }
            if (people[i].equals("John")) {
                return "John";
            }
            if (people[i].equals("Kent")) {
                return "Kent";
            }
        }
    }
    return "";
}

// 重构后：使用更清晰的算法
public String findPerson(String[] people) {
    List<String> candidates = List.of("Don", "John", "Kent");
    for (String person : people) {
        if (candidates.contains(person)) {
            return person;
        }
    }
    return "";
}

// 进一步重构：使用Stream
public Optional<String> findPerson(List<String> people) {
    Set<String> candidates = Set.of("Don", "John", "Kent");
    return people.stream()
            .filter(candidates::contains)
            .findFirst();
}

// 重构前：O(n²)的查找
public boolean hasDuplicate(List<Order> orders) {
    for (int i = 0; i < orders.size(); i++) {
        for (int j = i + 1; j < orders.size(); j++) {
            if (orders.get(i).getOrderNo().equals(orders.get(j).getOrderNo())) {
                return true;
            }
        }
    }
    return false;
}

// 重构后：O(n)的查找
public boolean hasDuplicate(List<Order> orders) {
    Set<String> seen = new HashSet<>();
    for (Order order : orders) {
        if (!seen.add(order.getOrderNo())) {  // add返回false表示已存在
            return true;
        }
    }
    return false;
}

// 使用Stream
public boolean hasDuplicate(List<Order> orders) {
    long uniqueCount = orders.stream()
            .map(Order::getOrderNo)
            .distinct()
            .count();
    return uniqueCount < orders.size();
}

// 重构前：复杂的条件表达式
public double calculateCharge(Order order) {
    double result;
    if (order.getDate().before(summerStart) || order.getDate().after(summerEnd)) {
        result = order.getQuantity() * order.getItemPrice() * 0.8 + 
                 Math.min(order.getQuantity() * 0.5, 100);
    } else {
        result = order.getQuantity() * order.getItemPrice() * 0.5;
    }
    return result;
}

// 重构后：提取条件和分支
public double calculateCharge(Order order) {
    if (isSummer(order)) {
        return summerCharge(order);
    } else {
        return winterCharge(order);
    }
}

private boolean isSummer(Order order) {
    return !order.getDate().before(summerStart) && 
           !order.getDate().after(summerEnd);
}

private double summerCharge(Order order) {
    return order.getQuantity() * order.getItemPrice() * 0.5;
}

private double winterCharge(Order order) {
    double base = order.getQuantity() * order.getItemPrice() * 0.8;
    double surcharge = Math.min(order.getQuantity() * 0.5, 100);
    return base + surcharge;
}
```

// 重构前：相同的条件被重复检查
public double calculateTotal(Order order) {
    double total = 0;
    
    if (order.isExpress()) {
        total += order.getShippingFee() * 1.5;  // 加急运费
    } else {
        total += order.getShippingFee();
    }
    
    // ... 其他计算
    
    if (order.isExpress()) {  // 条件重复
        total += order.getHandlingFee() * 1.2;  // 加急处理费
    } else {
        total += order.getHandlingFee();
    }
    
    return total;
}

// 重构后：合并条件
public double calculateTotal(Order order) {
    double multiplier = order.isExpress() ? 1.5 : 1.0;
    double handlingMultiplier = order.isExpress() ? 1.2 : 1.0;
    
    return order.getShippingFee() * multiplier 
         + order.getHandlingFee() * handlingMultiplier;
}

// 进一步重构：使用多态
public abstract class ShippingType {
    abstract double shippingMultiplier();
    abstract double handlingMultiplier();
    
    public double calculateTotal(Order order) {
        return order.getShippingFee() * shippingMultiplier()
             + order.getHandlingFee() * handlingMultiplier();
    }
}

public class ExpressShipping extends ShippingType {
    double shippingMultiplier() { return 1.5; }
    double handlingMultiplier() { return 1.2; }
}

public class StandardShipping extends ShippingType {

}

// 重构前：根据类型用switch/if-else
public class Employee {
    private EmployeeType type;
    
    public double monthlySalary() {
        switch (type) {
            case ENGINEER:
                return engineerSalary();
            case SALESMAN:
                return engineerSalary() + commission();
            case MANAGER:
                return engineerSalary() + bonus();
            default:
                throw new IllegalArgumentException("未知员工类型");
        }
    }
    
    public double bonus() {
        switch (type) {
            case ENGINEER:
                return 0;
            case SALESMAN:
                return monthlySalary() * 0.1;
            case MANAGER:
                return monthlySalary() * 0.2;
            default:
                throw new IllegalArgumentException("未知员工类型");
        }
    }
}

                discount = order.getAmount().multiply(new BigDecimal("0.05"));
            }
        } else if ("VIP".equals(customerType)) {
            if (order.getAmount().compareTo(new BigDecimal("1000")) > 0) {
                discount = order.getAmount().multiply(new BigDecimal("0.15"));
            } else {
                discount = order.getAmount().multiply(new BigDecimal("0.10"));
            }
        } else if ("SVIP".equals(customerType)) {
            discount = order.getAmount().multiply(new BigDecimal("0.20"));
            if (order.getItems().size() > 10) {
                discount = discount.add(new BigDecimal("50")); // 额外减免
            }
        } else if ("EMPLOYEE".equals(customerType)) {
            discount = order.getAmount().multiply(new BigDecimal("0.30"));
        } else if ("WHOLESALE".equals(customerType)) {
            BigDecimal volumeDiscount = order.getQuantity() > 100 
                ? new BigDecimal("0.25") : new BigDecimal("0.15");
            discount = order.getAmount().multiply(volumeDiscount);
        }
        
        return discount;
    }

public interface DiscountStrategy {
    BigDecimal calculate(Order order);
    String getType();
}

@Component
public class NormalDiscountStrategy implements DiscountStrategy {
    private static final BigDecimal THRESHOLD = new BigDecimal("1000");
    private static final BigDecimal DISCOUNT_RATE = new BigDecimal("0.05");
    
    @Override
    public BigDecimal calculate(Order order) {
        if (order.getAmount().compareTo(THRESHOLD) > 0) {
            return order.getAmount().multiply(DISCOUNT_RATE);
        }
        return BigDecimal.ZERO;
    }
    
    @Override
    public String getType() { return "NORMAL"; }
}

@Component
public class VipDiscountStrategy implements DiscountStrategy {
    private static final BigDecimal THRESHOLD = new BigDecimal("1000");
    private static final BigDecimal HIGH_RATE = new BigDecimal("0.15");
    private static final BigDecimal LOW_RATE = new BigDecimal("0.10");
    
    @Override
    public BigDecimal calculate(Order order) {
        BigDecimal rate = order.getAmount().compareTo(THRESHOLD) > 0 
            ? HIGH_RATE : LOW_RATE;
        return order.getAmount().multiply(rate);
    }
    
    @Override
    public String getType() { return "VIP"; }
}

@Service
public class OrderService {
    private final Map<String, DiscountStrategy> strategies;
    
    // Spring自动注入所有DiscountStrategy实现
    public OrderService(List<DiscountStrategy> strategyList) {
        this.strategies = strategyList.stream()
                .collect(Collectors.toMap(
                    DiscountStrategy::getType, 
                    Function.identity()
                ));
    }
    
    public BigDecimal calculateDiscount(Order order, String customerType) {
        DiscountStrategy strategy = strategies.get(customerType);
        if (strategy == null) {
            return BigDecimal.ZERO;
        }

// 更复杂的场景：策略选择需要多种条件
public interface PricingStrategy {
    BigDecimal calculate(Order order);
    boolean supports(Order order);  // 判断是否适用
}

// 满减策略
@Component
@Order(1)  // 优先级
public class FullReductionStrategy implements PricingStrategy {
    @Override
    public BigDecimal calculate(Order order) {
        if (order.getAmount().compareTo(new BigDecimal("200")) >= 0) {
            return new BigDecimal("30");  // 满200减30
        }
        return BigDecimal.ZERO;
    }
    
    @Override
    public boolean supports(Order order) {
        return order.getCoupon() != null 
               && "FULL_REDUCTION".equals(order.getCoupon().getType());
    }
}

// 折扣策略
@Component
@Order(2)
public class PercentageDiscountStrategy implements PricingStrategy {
    @Override
    public BigDecimal calculate(Order order) {
        return order.getAmount().multiply(new BigDecimal("0.15"));  // 85折
    }
    
    @Override
    public boolean supports(Order order) {
        return order.getCoupon() != null 
               && "PERCENTAGE".equals(order.getCoupon().getType());
    }
}

// 策略工厂
@Service
public class PricingStrategyFactory {
    private final List<PricingStrategy> strategies;
    
    public PricingStrategyFactory(List<PricingStrategy> strategies) {
        this.strategies = strategies.stream()
                .sorted(Comparator.comparingInt(s -> 
                    s.getClass().getAnnotation(Order.class).value()))
                .collect(Collectors.toList());
    }
    
    public PricingStrategy getStrategy(Order order) {
        return strategies.stream()
                .filter(s -> s.supports(order))
                .findFirst()
                .orElse(new NoDiscountStrategy());
    }
}

## 10. 面试题速查

**Q1: 重构的时机是什么？什么时候不应该重构？**
> 重构时机：1）添加新功能前（准备性重构）。2）修复Bug时（顺手改善）。3）Code Review中发现的改进点。4）技术债偿还Sprint。不应该重构：1）没有测试覆盖的代码（先补测试）。2）临近发布截止日期。3）需要完全重写的代码（重写比重构更划算）。4）不理解代码功能时（先理解再重构）。

**Q2: 提取方法（Extract Method）的标准是什么？**
> 1）方法名应该能表达做什么，而不是怎么做。2）方法长度建议不超过20-30行。3）方法应该做一件事（SRP）。4）方法中的代码在同一抽象层级。5）方法参数不超过3-4个。6）如果方法名需要"and"来描述，可能做了多件事。

**Q3: 卫语句（Guard Clause）相比嵌套if-else有什么优势？**
> 卫语句提前返回，使代码从左到右线性阅读，减少缩进层级。它将"异常情况先处理"的模式显式化——先处理边界条件（空值、非法状态），再处理正常逻辑。嵌套if-else将正常逻辑和异常逻辑混在一起，增加认知负担。卫语句的核心思想是"把正常路径放在最外层"。

**Q4: 何时用多态取代条件表达式？什么时候不适合？**
> 适合：类型不多（3-5种），类型稳定不常变化，多个方法都根据同一类型做switch。不适合：类型经常变化，只有一两个switch语句，类型数量很多但简单。引入多态的代价是增加了类层次和间接调用，如果条件逻辑简单，过度使用多态反而增加复杂度。

**Q5: 策略模式重构的核心步骤是什么？**
> 1）识别变化部分——if-else/switch中的每个分支。2）定义策略接口——声明公共方法。3）为每个分支实现策略类。4）将条件逻辑移到策略选择器（工厂或Map）。5）上下文持有策略引用。6）客户端通过策略接口调用。关键：开闭原则——新增策略不需要修改现有代码。

**Q6: 引入参数对象有什么好处？有什么陷阱？**
> 好处：减少参数数量，参数语义更清晰，参数可以扩展（添加字段不破坏方法签名），促进领域建模。陷阱：可能变成"数据包"（缺乏行为的数据容器），应该让参数对象包含相关行为；过度使用参数对象可能导致过度抽象；需要平衡——2-3个关联紧密的参数可以引入，单个参数不需要。

**Q7: 重构中如何保证安全性？**
> 1）有测试覆盖——重构前确保测试通过。2）小步重构——每次只改一点，立即运行测试。3）使用IDE重构工具——自动重构比手动更安全。4）一次只做一种重构——不要混合多种重构手法。5）先重构后优化——先改善结构，再改善性能。6）持续集成——每次重构后提交并运行CI。
