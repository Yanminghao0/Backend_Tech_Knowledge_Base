## 1. 什么时候需要缓存

### 1.1 缓存使用时机

**明确指标：何时必须使用缓存**

| 维度 | 阈值 | 建议 |
|------|------|------|
| **QPS** | > 1000 | 建议缓存热点数据 |
| **QPS** | > 5000 | 必须使用缓存 |
| **数据量** | > 100万 | 考虑缓存 |
| **数据量** | > 1000万 | 强烈建议缓存 |
| **查询耗时** | > 100ms | 建议缓存 |
| **查询耗时** | > 500ms | 必须缓存 |
| **读写比** | 读:写 > 10:1 | 适合缓存 |
| **热点数据** | 20%数据占80%流量 | 强烈建议缓存 |

### 1.2 典型场景分析

#### 场景1：电商商品详情页

**数据特征**：
```
商品总数：1000万
热门商品：1万个（占总流量的80%）
商品详情QPS：5万
数据库QPS上限：5000
```

**分析**：
```
✅ 必须使用缓存！

理由：
1. QPS: 5万 >> 5000（数据库扛不住）
2. 热点数据：1万个热门商品
3. 读多写少：商品信息变更频率低

缓存策略：
- 本地缓存：缓存Top 1000热门商品（减少Redis压力）
- Redis缓存：缓存Top 10万商品
- 数据库：兜底查询
```

#### 场景2：用户登录信息

**数据特征**：
```
用户总数：1亿
日活用户：1000万
登录校验QPS：10万
Session有效期：30分钟
```

**分析**：
```
✅ 必须使用缓存！

理由：
1. QPS: 10万（极高）
2. 频繁访问：每个请求都要校验登录
3. 数据量大：1亿用户

缓存策略：
- Redis缓存：Session信息（支持分布式部署）
- 过期时间：30分钟（自动清理）
```

#### 场景3：文章详情页

**数据特征**：
```
文章总数：100万
热门文章：1000篇（占总流量的50%）
文章详情QPS：1000
数据库查询耗时：50ms
```

**分析**：
```
🤔 可以使用缓存（非必须）

理由：
1. QPS: 1000（数据库可以承受）
2. 查询耗时：50ms（可接受）
3. 有热点数据：1000篇热门文章

缓存策略：
- Redis缓存：热门1000篇文章
- 数据库：直接查询（查询耗时可接受）
- 优化方向：索引优化、查询优化
```

#### 场景4：后台管理系统

**数据特征**：
```
用户数：100人
操作QPS：10
数据量：10万条
```

**分析**：
```
❌ 不需要缓存

理由：
1. QPS低：10（数据库轻松应对）
2. 数据量小：10万条
3. 实时性要求高：后台数据需要实时

建议：
- 优化SQL查询
- 添加索引
- 不引入缓存复杂度
```

## 2. 缓存选型决策

### 2.1 本地缓存 vs Redis

| 维度 | 本地缓存 | Redis |

| **性能** | ⭐⭐⭐⭐⭐ 极快（纳秒级） | ⭐⭐⭐⭐ 快（毫秒级） |
| **容量** | ⭐⭐ 受JVM内存限制（GB级） | ⭐⭐⭐⭐⭐ 大（TB级） |
| **分布式** | ❌ 不支持 | ✅ 支持 |
| **数据共享** | ❌ 单机独享 | ✅ 多机共享 |
| **持久化** | ❌ 不支持 | ✅ 支持（RDB/AOF） |

| **成本** | ⭐ 低（无额外成本） | ⭐⭐⭐ 中（需要Redis服务器） |

## 3. 本地缓存实战

### 3.1 Caffeine（推荐）

**简介**：Spring Boot官方推荐，性能最强的本地缓存

**性能对比**：
```
Caffeine > Guava Cache > ConcurrentHashMap + 手动过期
```

**基本使用**：

```java
@Configuration
public class CacheConfig {
    
    @Bean
    public Caffeine<Object, Object> caffeineConfig() {
        return Caffeine.newBuilder()
            .maximumSize(10000)  // 最大缓存数量
            .expireAfterWrite(5, TimeUnit.MINUTES)  // 写入后5分钟过期
            .recordStats();  // 开启统计
    }
    
    @Bean
    public CacheManager cacheManager(Caffeine caffeine) {
        CaffeineCacheManager manager = new CaffeineCacheManager();
        manager.setCaffeine(caffeine);
        return manager;
    }
}
```

**注解使用**：

```java
@Service
public class ProductService {
    
    // 查询时自动缓存
    @Cacheable(value = "product", key = "#id")
    public Product getById(Long id) {
        System.out.println("查询数据库: " + id);
        return productMapper.selectById(id);
    }
    
    // 更新时删除缓存
    @CacheEvict(value = "product", key = "#product.id")
    public void update(Product product) {
        productMapper.updateById(product);
    }
    
    // 更新时刷新缓存
    @CachePut(value = "product", key = "#result.id")
    public Product save(Product product) {
        productMapper.insert(product);
        return product;
    }
}
```

**手动使用**：

```java
@Service
public class ProductService {
    
    private final Cache<Long, Product> productCache;
    
    public ProductService() {
        this.productCache = Caffeine.newBuilder()
            .maximumSize(10000)
            .expireAfterWrite(5, TimeUnit.MINUTES)
            .build();
    }
    
    public Product getById(Long id) {
        // 1. 先查缓存
        Product product = productCache.getIfPresent(id);
        if (product != null) {
            return product;
        }
        
        // 2. 查数据库
        product = productMapper.selectById(id);
        
        // 3. 写入缓存
        if (product != null) {
            productCache.put(id, product);
        }
        
        return product;
    }
    
    // 或者使用 get + loader
    public Product getByIdV2(Long id) {
        return productCache.get(id, key -> {
            // 缓存未命中时自动加载
            return productMapper.selectById(key);
        });
    }
}
```

### 3.2 过期策略

**1. 基于时间过期**

```java
// 写入后过期
Caffeine.newBuilder()
    .expireAfterWrite(10, TimeUnit.MINUTES)
    .build();

// 访问后过期
Caffeine.newBuilder()
    .expireAfterAccess(10, TimeUnit.MINUTES)
    .build();

// 自定义过期策略
Caffeine.newBuilder()
    .expireAfter(new Expiry<Long, Product>() {
        @Override
        public long expireAfterCreate(Long key, Product value, long currentTime) {
            // 热门商品：10分钟过期
            if (value.isHot()) {
                return TimeUnit.MINUTES.toNanos(10);
            }
            // 普通商品：5分钟过期
            return TimeUnit.MINUTES.toNanos(5);
        }
        
        @Override
        public long expireAfterUpdate(Long key, Product value, 
                                      long currentTime, long currentDuration) {
            return currentDuration;
        }
        
        @Override
        public long expireAfterRead(Long key, Product value, 
                                    long currentTime, long currentDuration) {
            return currentDuration;
        }
    })
    .build();
```

**2. 基于大小淘汰**

```java
// 最大数量
Caffeine.newBuilder()
    .maximumSize(10000)
    .build();

// 最大权重
Caffeine.newBuilder()
    .maximumWeight(1000000)  // 1MB
    .weigher((Long key, Product value) -> {
        // 计算对象大小
        return value.getSize();
    })
    .build();
```

**3. 淘汰算法**

Caffeine使用 **W-TinyLFU** 算法（性能优于LRU）：

```
W-TinyLFU = Window + TinyLFU

Window（1%）：新数据先进入Window（防止LFU误淘汰）
TinyLFU（99%）：高频数据进入主缓存

优点：
✅ 命中率高于LRU
✅ 适应突发流量
✅ 防止缓存污染
```

### 3.3 监听器

```java
Cache<Long, Product> cache = Caffeine.newBuilder()
    .maximumSize(10000)
    .removalListener((Long key, Product value, RemovalCause cause) -> {
        System.out.println("缓存移除: key=" + key + ", cause=" + cause);
```


### 3.5 刷新机制

```java
LoadingCache<Long, Product> cache = Caffeine.newBuilder()
    .maximumSize(10000)
    .refreshAfterWrite(1, TimeUnit.MINUTES)  // 写入后1分钟异步刷新
    .build(key -> {
        // 加载数据
        return productMapper.selectById(key);
    });

// 查询时自动刷新（异步）
Product product = cache.get(1L);
```

**刷新 vs 过期**：

| 特性 | 刷新（Refresh） | 过期（Expire） |
|------|----------------|---------------|
| **触发时机** | 定时触发 | 访问时触发 |
| **是否阻塞** | 异步刷新（不阻塞） | 同步加载（阻塞） |
| **旧值可用** | ✅ 刷新期间返回旧值 | ❌ 过期后必须重新加载 |
| **适用场景** | 热点数据（保证可用性） | 普通数据 |

## 4. Redis分布式缓存

### 4.1 数据类型选择

**场景1：简单对象（String）**

```java
// 用户信息
@Service
public class UserService {
    
    @Autowired
    private StringRedisTemplate redisTemplate;
    
    public User getById(Long id) {
        String key = "user:" + id;
        
        // 1. 查缓存
        String json = redisTemplate.opsForValue().get(key);
        if (json != null) {
            return JSON.parseObject(json, User.class);
        }
        
        // 2. 查数据库
        User user = userMapper.selectById(id);
        
        // 3. 写缓存
        if (user != null) {
            redisTemplate.opsForValue().set(
                key, 
                JSON.toJSONString(user), 
                30, TimeUnit.MINUTES
            );
        }
        
        return user;
    }
}
```

**场景2：对象属性（Hash）**

```java
// 商品信息（属性较多，部分更新）
@Service
public class ProductService {
    
    public Product getById(Long id) {
        String key = "product:" + id;
        
        // 1. 查缓存
        Map<Object, Object> map = redisTemplate.opsForHash().entries(key);
        if (!map.isEmpty()) {
            return convertToProduct(map);
        }
        
        // 2. 查数据库
        Product product = productMapper.selectById(id);
        
        // 3. 写缓存
        if (product != null) {
            Map<String, String> hash = new HashMap<>();
            hash.put("id", product.getId().toString());
            hash.put("name", product.getName());
            hash.put("price", product.getPrice().toString());
            hash.put("stock", product.getStock().toString());
            
            redisTemplate.opsForHash().putAll(key, hash);
            redisTemplate.expire(key, 1, TimeUnit.HOURS);
        }
        
        return product;
    }
    
    // 更新单个字段（不影响其他字段）
    public void updatePrice(Long id, BigDecimal price) {
        productMapper.updatePrice(id, price);
        
        // 更新缓存中的price字段
        String key = "product:" + id;
        redisTemplate.opsForHash().put(key, "price", price.toString());
    }
}
```

**场景3：列表数据（List）**

```java
// 用户最近浏览记录
@Service
public class BrowseHistoryService {
    
    public void addHistory(Long userId, Long productId) {
        String key = "history:" + userId;
        
        // 添加到列表头部
        redisTemplate.opsForList().leftPush(key, productId.toString());
        
        // 保留最近100条
        redisTemplate.opsForList().trim(key, 0, 99);
        
        // 设置过期时间
        redisTemplate.expire(key, 7, TimeUnit.DAYS);
    }
    
    public List<Long> getHistory(Long userId, int limit) {
        String key = "history:" + userId;
        
        List<String> list = redisTemplate.opsForList().range(key, 0, limit - 1);
        return list.stream()
            .map(Long::parseLong)
            .collect(Collectors.toList());
    }
}
```

**场景4：去重（Set）**

```java
// 文章点赞用户
@Service
public class LikeService {
    
    public boolean like(Long articleId, Long userId) {
        String key = "like:article:" + articleId;
        
        // 添加到Set（自动去重）

    public long getLikeCount(Long articleId) {
        String key = "like:article:" + articleId;
        
        return redisTemplate.opsForSet().size(key);
    }
    
    public boolean isLiked(Long articleId, Long userId) {
        String key = "like:article:" + articleId;
        
        return redisTemplate.opsForSet().isMember(key, userId.toString());
    }
}
```

**场景5：排行榜（Sorted Set）**

```java
// 游戏积分排行榜
@Service
public class RankService {
    
    public void updateScore(Long userId, double score) {
        String key = "rank:score";
        
        // 更新分数
        redisTemplate.opsForZSet().add(key, userId.toString(), score);
    }
    
    public List<UserScore> getTopN(int n) {
        String key = "rank:score";
        
        // 获取Top N（分数从高到低）
        Set<ZSetOperations.TypedTuple<String>> set = 
            redisTemplate.opsForZSet().reverseRangeWithScores(key, 0, n - 1);
        
        return set.stream()
            .map(tuple -> new UserScore(
                Long.parseLong(tuple.getValue()),
                tuple.getScore()
            ))
            .collect(Collectors.toList());
    }
    
    public Long getRank(Long userId) {
        String key = "rank:score";
        
        // 获取排名（从0开始）
        Long rank = redisTemplate.opsForZSet().reverseRank(key, userId.toString());
        
        return rank != null ? rank + 1 : null;
    }
}
```

## 5. 多级缓存架构

### 5.1 架构设计

**三级缓存架构**：

```
请求
  │
  ▼
┌─────────────────────────────┐
│ L1：本地缓存（Caffeine）     │  ← 极快（纳秒级）
│ - Top 100热点数据            │
│ - 过期时间：1分钟            │
└─────────────┬───────────────┘
              │ 未命中
              ▼
┌─────────────────────────────┐
│ L2：分布式缓存（Redis）      │  ← 快（毫秒级）
│ - Top 10000数据             │
│ - 过期时间：10分钟           │
└─────────────┬───────────────┘
              │ 未命中
              ▼
┌─────────────────────────────┐
│ L3：数据库（MySQL）          │  ← 慢（百毫秒级）
│ - 全量数据                  │
└─────────────────────────────┘
```

### 5.2 实现方案

**方案1：手动实现**

```java
@Service
public class ProductService {
    
    // L1：本地缓存
    private final Cache<Long, Product> localCache;
    
    // L2：Redis
    @Autowired
    private StringRedisTemplate redisTemplate;
    
    // L3：数据库
    @Autowired
    private ProductMapper productMapper;
    
    public ProductService() {
        this.localCache = Caffeine.newBuilder()
            .maximumSize(100)  // Top 100
            .expireAfterWrite(1, TimeUnit.MINUTES)  // 1分钟过期
            .build();
    }
    
    public Product getById(Long id) {
        // 1. 查L1本地缓存
        Product product = localCache.getIfPresent(id);
        if (product != null) {
            log.info("L1命中: {}", id);
            return product;
        }
        
        // 2. 查L2 Redis缓存
        String redisKey = "product:" + id;
        String json = redisTemplate.opsForValue().get(redisKey);
        if (json != null) {
            log.info("L2命中: {}", id);
            product = JSON.parseObject(json, Product.class);
            
            // 写入L1
            localCache.put(id, product);
            
            return product;
        }
        
        // 3. 查L3数据库
        log.info("L3查询数据库: {}", id);
        product = productMapper.selectById(id);
        
        if (product != null) {
            // 写入L2
            redisTemplate.opsForValue().set(
                redisKey,
                JSON.toJSONString(product),
                10, TimeUnit.MINUTES
            );
            
            // 写入L1
            localCache.put(id, product);
        }
        
        return product;
    }
}
```

### 5.3 一致性保证

**问题**：更新数据库后，如何保证L1和L2缓存一致？

**方案1：主动删除（推荐）**

```java
@Service
public class ProductService {

        String redisKey = "product:" + product.getId();
        redisTemplate.delete(redisKey);
        
        // 3. 删除L1（本地缓存）
        localCache.invalidate(product.getId());
        
        // 4. 如果是集群，需要通知其他节点删除L1
        notifyOtherNodes(product.getId());

    // 监听消息，删除本地缓存
    @RedisListener(topics = "cache:evict:product")
    public void handleCacheEvict(Long productId) {
        localCache.invalidate(productId);
    }
}
```

## 6. 缓存设计模式

### 6.1 Cache-Aside（旁路缓存）★★★★★

**最常用的模式**

**读流程**：
```
1. 查缓存
2. 缓存命中 → 返回
3. 缓存未命中 → 查数据库
4. 写入缓存
5. 返回
```

**写流程**：
```
1. 更新数据库
2. 删除缓存
```

**代码示例**：
```java
// 读
public Product getById(Long id) {
    // 1. 查缓存
    String json = redisTemplate.opsForValue().get("product:" + id);
    if (json != null) {
        return JSON.parseObject(json, Product.class);
    }
    
    // 2. 查数据库
    Product product = productMapper.selectById(id);
    
    // 3. 写缓存
    if (product != null) {
        redisTemplate.opsForValue().set(
            "product:" + id,
            JSON.toJSONString(product),
            1, TimeUnit.HOURS
        );
    }
    
    return product;
}

// 写
public void update(Product product) {
    // 1. 更新数据库
    productMapper.updateById(product);
    
    // 2. 删除缓存
    redisTemplate.delete("product:" + product.getId());
}
```

**优点**：
- ✅ 简单易用
- ✅ 应用层控制灵活

**缺点**：
- ❌ 需要应用层管理缓存
- ❌ 可能出现短暂不一致

### 6.2 Read-Through（读穿透）

**由缓存层自动加载数据**

**流程**：
```
应用 → 缓存层 → 数据库
       ↑
   缓存层自动加载
```

**代码示例**（Guava Cache）：
```java
LoadingCache<Long, Product> cache = CacheBuilder.newBuilder()
    .maximumSize(10000)
    .expireAfterWrite(10, TimeUnit.MINUTES)
    .build(new CacheLoader<Long, Product>() {
        @Override
        public Product load(Long id) {
            // 缓存层自动加载数据
            return productMapper.selectById(id);
        }
    });

// 应用层只需要调用get
Product product = cache.get(id);  // 缓存未命中时自动加载
```

**优点**：
- ✅ 应用层简单
- ✅ 缓存加载逻辑统一

**缺点**：
- ❌ 缓存层逻辑复杂
- ❌ 不适合Redis（Redis不支持自动加载）

### 6.3 Write-Through（写穿透）

**由缓存层同步写数据库**

**流程**：
```
应用 → 缓存层 → 数据库
       ↓
   缓存层同步写入
```

**代码示例**：
```java
public void update(Product product) {
    // 应用层只写缓存
    cache.put(product.getId(), product);
    
    // 缓存层自动同步到数据库
}

// 缓存层实现
class WriteThroughCache {
    public void put(Long id, Product product) {
        // 1. 写数据库
        productMapper.updateById(product);
        
        // 2. 写缓存
        redisTemplate.opsForValue().set(
            "product:" + id,
            JSON.toJSONString(product),
            1, TimeUnit.HOURS
        );
    }
}
```

**优点**：
- ✅ 缓存和数据库强一致

**缺点**：
- ❌ 写入性能差（同步）
- ❌ 不适合高并发写入

### 6.4 Write-Behind（异步写回）

**缓存层异步批量写数据库**

**流程**：
```
应用 → 缓存（立即返回）
       ↓
   异步批量写数据库
```

**代码示例**：
```java
@Service
public class ProductService {
    
    private final BlockingQueue<Product> writeQueue = new LinkedBlockingQueue<>(10000);
    
    @PostConstruct
    public void startAsyncWriter() {
        // 异步批量写入
        ScheduledExecutorService executor = Executors.newScheduledThreadPool(1);
        executor.scheduleAtFixedRate(() -> {
            List<Product> batch = new ArrayList<>();
            writeQueue.drainTo(batch, 100);  // 最多取100个
            
            if (!batch.isEmpty()) {
                // 批量写数据库
                productMapper.batchUpdate(batch);
            }
        }, 0, 1, TimeUnit.SECONDS);
    }
    
    public void update(Product product) {
        // 1. 写缓存
        redisTemplate.opsForValue().set(
            "product:" + product.getId(),
            JSON.toJSONString(product),
            1, TimeUnit.HOURS
        );
        
        // 2. 加入异步写队列
        writeQueue.offer(product);
    }
}
```

**优点**：
- ✅ 写入性能极高
- ✅ 批量写数据库

**缺点**：
- ❌ 可能丢失数据（宕机）
- ❌ 实现复杂

### 6.5 模式选择

| 模式 | 性能 | 一致性 | 复杂度 | 推荐场景 |
|------|------|--------|--------|---------|
| **Cache-Aside** | ⭐⭐⭐⭐ | ⭐⭐⭐ | ⭐ | **通用场景（推荐）** |
| **Read-Through** | ⭐⭐⭐ | ⭐⭐⭐ | ⭐⭐ | 本地缓存 |
| **Write-Through** | ⭐⭐ | ⭐⭐⭐⭐⭐ | ⭐⭐ | 强一致性场景 |
| **Write-Behind** | ⭐⭐⭐⭐⭐ | ⭐⭐ | ⭐⭐⭐⭐ | 高并发写入 |

## 7. 缓存常见问题

### 7.1 缓存穿透

**问题**：查询不存在的数据，缓存和数据库都没有，每次都打到数据库

**场景**：
```
恶意攻击：查询id=-1的用户（不存在）
→ 缓存没有 → 数据库没有 → 缓存还是没有
→ 每次请求都打到数据库
```

**解决方案1：布隆过滤器（推荐）**

```java
@Service
public class UserService {
    
    @Autowired
    private RedisTemplate<String, Object> redisTemplate;
    
    // 布隆过滤器
    private BloomFilter<Long> bloomFilter;
    
    @PostConstruct
    public void init() {
        // 初始化布隆过滤器
        bloomFilter = BloomFilter.create(
            Funnels.longFunnel(),
            10000000,  // 预计元素数量：1000万
            0.01  // 误判率：1%
        );
        
        // 加载所有用户ID
        List<Long> userIds = userMapper.selectAllIds();
        userIds.forEach(bloomFilter::put);
    }
    
    public User getById(Long id) {
        // 1. 布隆过滤器判断
        if (!bloomFilter.mightContain(id)) {
            return null;  // 一定不存在
        }
        
        // 2. 查缓存
        String json = (String) redisTemplate.opsForValue().get("user:" + id);
        if (json != null) {
            return JSON.parseObject(json, User.class);
        }
        
        // 3. 查数据库
        User user = userMapper.selectById(id);
        
        // 4. 写缓存
        if (user != null) {
            redisTemplate.opsForValue().set(
                "user:" + id,
                JSON.toJSONString(user),
                30, TimeUnit.MINUTES
            );
        }
        
        return user;
    }
}
```

**解决方案2：缓存空值**

### 7.2 缓存击穿

**问题**：热点数据过期，大量请求同时打到数据库

**场景**：
```
热门商品缓存过期
→ 10000个请求同时到达
→ 缓存都未命中
→ 10000个请求同时查数据库
→ 数据库崩溃
```

**解决方案1：互斥锁**

```java
public Product getById(Long id) {
    String key = "product:" + id;
    
    // 1. 查缓存
    String json = redisTemplate.opsForValue().get(key);
    if (json != null) {
        return JSON.parseObject(json, Product.class);
    }
    
    // 2. 缓存未命中，尝试获取锁
    String lockKey = "lock:product:" + id;
    Boolean locked = redisTemplate.opsForValue().setIfAbsent(
        lockKey, 
        "locked", 
        10, TimeUnit.SECONDS
    );
    
    try {
        if (locked) {
            // 获取锁成功，查数据库
            Product product = productMapper.selectById(id);
            
            // 写缓存
            if (product != null) {
                redisTemplate.opsForValue().set(
                    key,
                    JSON.toJSONString(product),
                    1, TimeUnit.HOURS
                );
            }
            
            return product;
        } else {
            // 获取锁失败，等待后重试
            Thread.sleep(50);
            return getById(id);  // 递归重试
        }
    } catch (InterruptedException e) {
        throw new RuntimeException(e);
    } finally {
        // 释放锁
        if (locked) {
            redisTemplate.delete(lockKey);
        }
    }
}
```

**解决方案2：热点数据永不过期**

## 8. 缓存一致性

### 8.1 问题分析

**缓存和数据库不一致的4种情况**：

| 情况 | 操作顺序 | 结果 |
|------|---------|------|
| 1 | 先删缓存 → 更新DB | ✅ 最终一致 |
| 2 | 先更新DB → 删缓存 | ✅ 最终一致（推荐） |
| 3 | 先删缓存 → 更新DB失败 | ❌ 不一致 |
| 4 | 先更新DB → 删缓存失败 | ❌ 不一致 |

### 8.2 策略1：先更新DB，再删缓存（推荐）

```java
public void update(Product product) {
    // 1. 更新数据库
    productMapper.updateById(product);
    
    // 2. 删除缓存
    String key = "product:" + product.getId();
    redisTemplate.delete(key);
}
```

**为什么是"删除"而不是"更新"缓存？**

```
删除缓存：
- 简单
- 下次查询时自动加载最新数据
- 避免缓存更新失败

更新缓存：
- 复杂（需要重新查询或计算）
- 可能更新失败导致不一致
- 浪费（缓存可能不会被访问）
```

### 8.3 策略2：延迟双删

**解决并发问题**：

```
时间线：
T1: 线程A更新DB（price=100）
T2: 线程B查询，缓存未命中，查DB（price=90，旧数据）
T3: 线程A删除缓存
T4: 线程B写入缓存（price=90，旧数据）

结果：缓存是旧数据！
```

**解决方案：延迟双删**

```java
public void update(Product product) {
    String key = "product:" + product.getId();
    
    // 1. 删除缓存
    redisTemplate.delete(key);
    
    // 2. 更新数据库
    productMapper.updateById(product);
    
    // 3. 延迟后再次删除缓存
    CompletableFuture.runAsync(() -> {
        try {
            Thread.sleep(500);  // 延迟500ms
            redisTemplate.delete(key);
        } catch (InterruptedException e) {
            log.error("延迟删除缓存失败", e);
        }
    });
}
```

```
✅ 异步（性能高）
```

```java
            "",
            ""

                }
            }
```

### 10.1 电商商品详情页缓存

**场景**：
```

热门商品：1万个
商品详情QPS：10万
```

**架构设计**：

```
┌─────────────────────────────────┐
│  L1：本地缓存（Caffeine）        │
│  - 缓存Top 100超级热门商品        │
│  - 过期时间：1分钟               │
│  - 命中率：20%                   │
│  - 延迟：<1ms                    │
└─────────────────────────────────┘
              │ 未命中
              ▼
┌─────────────────────────────────┐
│  L2：Redis集群                   │
│  - 缓存Top 10000热门商品         │
│  - 过期时间：10分钟 + 随机值      │
│  - 命中率：60%                   │
│  - 延迟：2ms                     │
└─────────────────────────────────┘
              │ 未命中
              ▼
┌─────────────────────────────────┐
│  L3：MySQL读写分离               │
│  - 全量数据                      │
│  - 命中率：20%                   │
│  - 延迟：50ms                    │
└─────────────────────────────────┘
```

**代码实现**：

```java
@Service
public class ProductService {
    
    // L1：本地缓存
    private final Cache<Long, Product> localCache;
    
    // L2：Redis
    @Autowired
    private StringRedisTemplate redisTemplate;
    
    // L3：数据库
    @Autowired
    private ProductMapper productMapper;
    
    public ProductService() {
        this.localCache = Caffeine.newBuilder()
            .maximumSize(100)
            .expireAfterWrite(1, TimeUnit.MINUTES)
            .recordStats()
            .build();
    }
    
    public Product getById(Long id) {
        // 1. L1本地缓存
        Product product = localCache.getIfPresent(id);
        if (product != null) {
            return product;
        }
        
        // 2. L2 Redis
        String redisKey = "product:" + id;
        String json = redisTemplate.opsForValue().get(redisKey);
        if (json != null) {
            product = JSON.parseObject(json, Product.class);
            localCache.put(id, product);
            return product;
        }
        
        // 3. L3数据库（互斥锁防击穿）
        String lockKey = "lock:product:" + id;
        Boolean locked = redisTemplate.opsForValue().setIfAbsent(
            lockKey, "locked", 10, TimeUnit.SECONDS
        );
        
        try {
            if (locked) {
                // 获取锁成功，查数据库
                product = productMapper.selectById(id);
                
                if (product != null) {
                    // 写Redis（过期时间加随机值）
                    int ttl = 600 + ThreadLocalRandom.current().nextInt(120);
                    redisTemplate.opsForValue().set(
                        redisKey,
                        JSON.toJSONString(product),
                        ttl, TimeUnit.SECONDS
                    );
                    
                    // 写本地缓存
                    localCache.put(id, product);
                }
                
                return product;
            } else {
                // 获取锁失败，等待后重试
                Thread.sleep(50);
                return getById(id);
            }
        } catch (InterruptedException e) {
            throw new RuntimeException(e);
        } finally {
            if (locked) {
                redisTemplate.delete(lockKey);
            }
        }
    }
    
    @Scheduled(fixedRate = 60000)  // 每分钟统计
    public void printStats() {
        CacheStats stats = localCache.stats();
        log.info("本地缓存命中率: {}", stats.hitRate());
    }
}
```

```
P99延迟：10ms

L2命中率：60%
L3命中率：20%
```

**场景**：

```java
@Service
public class RankService {
    
    @Autowired
    private StringRedisTemplate redisTemplate;
    
    /**
     * 更新分数
     */
    public void updateScore(Long userId, double score) {
        String key = "rank:score";
        redisTemplate.opsForZSet().add(key, userId.toString(), score);
    }
    
    /**
     * 批量更新分数（Pipeline优化）
     */
    public void batchUpdateScore(Map<Long, Double> scoreMap) {
        String key = "rank:score";
        
        redisTemplate.executePipelined((RedisCallback<Object>) connection -> {
            scoreMap.forEach((userId, score) -> {
                connection.zAdd(
                    key.getBytes(),
                    score,
                    userId.toString().getBytes()
                );
            });
            return null;
        });
    }
    
    /**
     * 获取Top N
     */
    public List<UserScore> getTopN(int n) {
        String key = "rank:score";
        
        // 查询Top N（分数从高到低）
        Set<ZSetOperations.TypedTuple<String>> set = 
            redisTemplate.opsForZSet().reverseRangeWithScores(key, 0, n - 1);
        
        if (set == null) {
            return Collections.emptyList();
        }
        
        return set.stream()
            .map(tuple -> new UserScore(
                Long.parseLong(tuple.getValue()),
                tuple.getScore()
            ))
            .collect(Collectors.toList());
    }
    
    /**
     * 获取用户排名
     */
    public Long getUserRank(Long userId) {
        String key = "rank:score";
        
        Long rank = redisTemplate.opsForZSet().reverseRank(key, userId.toString());
        
        return rank != null ? rank + 1 : null;
    }
    
    /**
     * 获取用户周边排名（前后各5名）
     */
    public List<UserScore> getNearbyRank(Long userId) {
        String key = "rank:score";
        
        Long rank = redisTemplate.opsForZSet().reverseRank(key, userId.toString());
        if (rank == null) {
            return Collections.emptyList();
        }
        
        // 获取前后各5名
        long start = Math.max(0, rank - 5);
        long end = rank + 5;
        
        Set<ZSetOperations.TypedTuple<String>> set = 
            redisTemplate.opsForZSet().reverseRangeWithScores(key, start, end);
        
        return set.stream()
            .map(tuple -> new UserScore(
                Long.parseLong(tuple.getValue()),
                tuple.getScore()
            ))
            .collect(Collectors.toList());
    }
}
```

```
3. 检查是否有大量冷数据查询
4. 检查缓存Key是否设计合理
```

**解决方案**：
```
1. 延长缓存过期时间
2. 增加缓存容量
3. 预热热点数据
4. 优化缓存Key设计
```

### 问题2：缓存雪崩

**症状**：
```
凌晨3点，数据库QPS突然飙升到10万
数据库响应时间从10ms暴增到5秒
部分请求超时
```

**排查步骤**：
```
1. 检查Redis监控，是否大量Key同时过期
2. 检查Redis是否宕机
3. 检查缓存过期时间设置
```

**解决方案**：
```
1. 过期时间加随机值
2. 多级缓存兜底
3. 限流降级
4. Redis集群高可用
```

### 问题3：缓存穿透

**症状**：
```
请求ID=-1的数据（不存在）
每次都打到数据库
数据库慢查询增加
```

**排查步骤**：
```
1. 查看慢查询日志，发现大量查询不存在的数据
2. 查看应用日志，发现异常参数
```

**解决方案**：
```
1. 布隆过滤器
2. 缓存空值
3. 参数校验
```

### 问题4：内存溢出

**症状**：
```
Redis内存使用率：95%
部分Key被淘汰
缓存命中率下降
```

**排查步骤**：
```
1. redis-cli INFO memory  # 查看内存使用
2. redis-cli --bigkeys     # 查找大Key
3. 分析Key过期策略
```

**解决方案**：
```
1. 增加Redis内存
2. 删除无用Key
3. 设置合理的过期时间
4. 优化数据结构（Hash代替String）
```
