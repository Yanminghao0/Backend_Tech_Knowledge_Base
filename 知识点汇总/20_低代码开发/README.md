## 🎯 学习目标

1. **理解低代码平台架构**：元数据驱动、模型驱动、声明式开发的核心理念
2. **掌握表单引擎**：可视化设计器、Schema协议、动态渲染引擎
3. **了解流程引擎**：BPMN 2.0规范、流程设计器、执行引擎
4. **数据集成能力**：数据源管理、API编排、ETL
5. **性能与扩展**：渲染优化、插件机制、二次开发

---

## 📊 知识图谱

```
低代码开发
├── 平台架构
│   ├── 元数据驱动引擎
│   ├── 核心组件（表单/流程/报表/规则）
│   ├── 技术选型（前端Vue/React + 后端Java/Node）
│   ├── 扩展性设计（插件机制、自定义组件）
│   └── 安全性设计（权限模型、数据隔离）
├── 表单引擎
│   ├── 可视化设计器（拖拽编排、属性面板）
│   ├── Schema协议（JSON Schema、Form Schema）
│   ├── 渲染引擎（动态渲染、条件显隐、联动）
│   ├── 数据模型（字段类型、校验规则、默认值）
│   └── 业务规则引擎（公式计算、流程触发）
├── 流程引擎
│   ├── BPMN 2.0规范
│   ├── 流程设计器（节点编排、网关、事件）
│   ├── 流程执行引擎（任务分配、会签、回退）
│   └── 任务调度与超时处理
├── 数据集成
│   ├── 数据源管理（多数据库支持）
│   ├── API编排与聚合
│   ├── ETL工具与数据同步
│   └── 外部系统集成
└── 性能优化
    ├── 表单渲染优化（虚拟DOM、懒加载）
    ├── 流程执行优化（异步、缓存）
    ├── 数据查询优化（分页、索引）
    └── 系统扩展策略（微服务化、CDN）
```

## 🔧 主流低代码平台对比

| 平台 | 类型 | 前端技术 | 后端技术 | 适用场景 |
|------|------|----------|----------|----------|
| 钉钉宜搭 | SaaS | React | Java | 企业内部应用 |

| 阿里低代码引擎 | 开源 | React | Node | 中后台应用 |
| Amis(百度) | 开源 | React | - | 中后台表单页面 |
