## 一、可视化搭建概述

### 1.1 什么是低代码可视化搭建

低代码可视化搭建是一种通过图形化界面（拖拽、配置）而非手写代码来构建页面的技术方案。其核心思想是将页面抽象为**数据（Schema）**，由**渲染引擎**负责将数据还原为真实页面。

```
┌─────────────────────────────────────────────────────────────────┐
│                    可视化搭建核心流程                               │
├─────────────────────────────────────────────────────────────────┤
│                                                                 │
│   ┌──────────┐     拖拽      ┌──────────┐    序列化   ┌───────┐ │
│   │ 物料面板  │ ──────────→  │  画布区   │ ────────→  │Schema │ │
│   │ (组件库)  │              │ (编辑器)  │            │ (JSON)│ │
│   └──────────┘              └──────────┘            └───┬───┘ │
│                                                        │     │
│   ┌──────────┐     配置      ┌──────────┐    反序列化  │     │
│   │ 属性面板  │ ←──────────  │  选中组件  │ ←─────────  │     │
│   │ (表单)    │              │  实例     │             │     │
│   └──────────┘              └──────────┘             │     │
│                                                       │     │
│                                          ┌────────────▼───┐ │
│                                          │   渲染引擎      │ │
│                                          │ (页面还原输出)  │ │
│                                          └────────────────┘ │
└─────────────────────────────────────────────────────────────────┘
```

### 1.2 适用场景

| 场景 | 说明 | 典型代表 |
|------|------|----------|
| 表单页面 | 问卷调查、数据录入、审批流程 | 钉钉智能表单、飞书多维表格 |
| 报表大屏 | 数据可视化看板、监控大屏 | DataV、Grafana |
| 营销活动 | 落地页、H5活动页、促销页 | 阿里 Bach、百度 H5 |
| 后台管理 | CRUD列表、详情页、配置页 | Amis、Form-Generator |
| 小程序/APP | 混合渲染的移动端页面 | 京东 Taro、美团 mtvue |

### 1.3 核心设计原则

```
1. 数据驱动 —— 页面即数据，渲染即解析
   - 一切操作最终都转化为对 Schema 的修改
   - 渲染引擎只负责"读取 Schema → 生成页面"

2. 协议先行 —— Schema 是编辑器和渲染器的唯一桥梁
   - 编辑器产出 Schema，渲染器消费 Schema
   - Schema 版本化，保证向前兼容

3. 物料闭环 —— 组件注册即可用
   - 物料遵循统一规范开发
   - 自动注册到物料面板，自动生成属性配置表单

4. 扩展性 —— 插件化架构
   - 自定义组件、自定义属性配置器、自定义校验规则
   - 一切皆可扩展
```

### 3.3 Schema 校验

```typescript
import Ajv from 'ajv';

// 定义 Schema 校验规则
const pageSchemaValidator = new Ajv().compile({
  type: 'object',
  required: ['version', 'type', 'id', 'props'],
  properties: {
    version: { type: 'string', pattern: '^\\d+\\.\\d+\\.\\d+$' },
    type: { type: 'string', const: 'Page' },
    id: { type: 'string', minLength: 1 },
    props: { type: 'object' },
    children: {
      type: 'array',
      items: { $ref: '#/definitions/componentNode' }
    }
  },
  definitions: {
    componentNode: {
      type: 'object',
      required: ['type', 'id', 'props'],
      properties: {
        type: { type: 'string', minLength: 1 },
        id: { type: 'string', minLength: 1 },
        props: { type: 'object' },
        children: {
          type: 'array',
          items: { $ref: '#/definitions/componentNode' }
        }
      }
    }
  }
});

// 校验函数
function validateSchema(schema: unknown): { valid: boolean; errors?: string[] } {
  const valid = pageSchemaValidator(schema);
  if (!valid) {
    const errors = pageSchemaValidator.errors?.map(
      e => `${e.instancePath}: ${e.message}`
    ) || [];
    return { valid: false, errors };
  }
  return { valid: true };
}
```

### 3.4 Schema 版本迁移

```typescript
// 版本迁移器：处理 Schema 协议升级
const migrations = [
  {
    from: '1.0.0',
    to: '1.1.0',
    migrate(schema: any): any {
      // v1.1.0: 新增 dataSource 字段
      return traverse(schema, (node) => ({
        ...node,
        dataSource: node.dataSource || null
      }));
    }
  },
  {
    from: '1.1.0',
    to: '1.2.0',
    migrate(schema: any): any {
      // v1.2.0: events 结构调整
      return traverse(schema, (node) => {
        if (node.events) {
          node.events = node.events.map((evt: any) => ({
            type: evt.event || evt.type,
            action: evt.action || { type: 'script', config: { code: evt.handler } }
          }));
        }
        return node;
      });
    }
  }
];

function migrateSchema(schema: any, targetVersion: string): any {
  let current = schema;
  while (current.version !== targetVersion) {
    const migration = migrations.find(m => m.from === current.version);
    if (!migration) break;
    current = migration.migrate(current);
    current.version = migration.to;
  }
  return current;
}
```

## 四、可视化编辑器实现

### 4.1 编辑器状态管理

```typescript
import { create } from 'zustand';

interface EditorState {
  // 页面 Schema
  schema: PageSchema;

  // 当前选中的组件ID
  selectedId: string | null;

  // 当前悬停的组件ID
  hoverId: string | null;

  // 是否处于拖拽中
  isDragging: boolean;

  // 编辑模式：edit | preview
  mode: 'edit' | 'preview';

  // 操作方法
  selectNode: (id: string | null) => void;
  hoverNode: (id: string | null) => void;
  setMode: (mode: 'edit' | 'preview') => void;
  addNode: (node: ComponentNode, parentId: string, index?: number) => void;
  removeNode: (id: string) => void;
  moveNode: (id: string, targetParentId: string, index: number) => void;
  updateNodeProps: (id: string, props: Record<string, any>) => void;
}

const useEditorStore = create<EditorState>((set, get) => ({
  schema: createEmptyPage(),
  selectedId: null,
  hoverId: null,
  isDragging: false,
  mode: 'edit',

  selectNode: (id) => set({ selectedId: id }),
  hoverNode: (id) => set({ hoverId: id }),
  setMode: (mode) => set({ mode }),

  addNode: (node, parentId, index) =>
    set((state) => {
      const newSchema = cloneDeep(state.schema);
      const parent = findNodeById(newSchema, parentId);
      if (!parent) return state;
      if (index === undefined) {
        parent.children = parent.children || [];
        parent.children.push(node);
      } else {
        parent.children = parent.children || [];
        parent.children.splice(index, 0, node);
      }
      return { schema: newSchema, selectedId: node.id };
    }),

  removeNode: (id) =>
    set((state) => {
      const newSchema = cloneDeep(state.schema);
      removeNodeById(newSchema, id);
      return { schema: newSchema, selectedId: null };
    }),

  moveNode: (id, targetParentId, index) =>
    set((state) => {
      const newSchema = cloneDeep(state.schema);
      const node = findNodeById(newSchema, id);
      if (!node) return state;
      removeNodeById(newSchema, id);
      const targetParent = findNodeById(newSchema, targetParentId);
      if (!targetParent) return state;
      targetParent.children = targetParent.children || [];
      targetParent.children.splice(index, 0, node);
      return { schema: newSchema };
    }),

  updateNodeProps: (id, props) =>
    set((state) => {
      const newSchema = cloneDeep(state.schema);
      const node = findNodeById(newSchema, id);
      if (!node) return state;
      node.props = { ...node.props, ...props };
      return { schema: newSchema };
    }),
}));
```

### 4.2 选中态与高亮

// 编辑器组件 - 带选中态的渲染包裹器
function EditableWrapper({ node, children }: { node: ComponentNode; children: React.ReactNode }) {
  const { selectedId, hoverId, selectNode, hoverNode } = useEditorStore();

  const isSelected = selectedId === node.id;
  const isHovered = hoverId === node.id;

  return (
    <div
      data-node-id={node.id}
      onClick={(e) => {
        e.stopPropagation();
        selectNode(node.id);
      }}
      onMouseEnter={(e) => {
        e.stopPropagation();
        hoverNode(node.id);
      }}
      onMouseLeave={() => hoverNode(null)}
      style={{
        outline: isSelected ? '2px solid #1890ff' : isHovered ? '1px dashed #1890ff' : 'none',
        outlineOffset: '-2px',
        cursor: 'pointer',
        position: 'relative',

      {isSelected && (
        <div style={{
          position: 'absolute',
          top: '-20px',

          height: '20px',

          background: '#1890ff',

          borderRadius: '2px 2px 0 0',
          zIndex: 100,
        }}>

  );

## 五、渲染引擎设计

### 5.1 渲染引擎核心

渲染引擎是低代码平台的运行时核心，负责将 Schema 转换为真实页面。

```typescript
import React, { ReactNode } from 'react';

// 组件注册表
const componentMap = new Map<string, React.ComponentType<any>>();

// 注册组件
function registerComponent(name: string, component: React.ComponentType<any>) {
  componentMap.set(name, component);
}

// 渲染单个节点
function renderNode(node: ComponentNode, context: RenderContext): ReactNode {
  const Component = componentMap.get(node.type);
  if (!Component) {
    console.warn(`未注册的组件类型: ${node.type}`);
    return null;
  }

  // 条件渲染
  if (node.condition && !evalCondition(node.condition, context)) {
    return null;
  }

  // 处理循环
  if (node.loop) {
    const list = getDataSource(node.loop.dataField, context) || [];
    return list.map((item: any, index: number) => {
      const childContext = {
        ...context,
        [node.loop!.itemName]: item,
        index,
      };
      return (
        <Component key={`${node.id}_${index}`} {...node.props}>
          {node.children?.map(child => renderNode(child, childContext))}
        </Component>
      );
    });
  }

  // 处理事件绑定
  const eventHandlers: Record<string, Function> = {};
  if (node.events) {
    node.events.forEach(evt => {
      eventHandlers[`on${capitalize(evt.type)}`] = (e: any) => {
        handleEvent(evt.action, e, context);
      };
    });
  }

  return (
    <Component key={node.id} {...node.props} {...eventHandlers} style={node.style}>
      {node.children?.map(child => renderNode(child, context))}
    </Component>
  );
}

// 页面渲染入口
function PageRenderer({ schema, dataSource }: { schema: PageSchema; dataSource?: Record<string, any> }) {
  const context: RenderContext = {
    dataSource: dataSource || {},
    router: null, // 由外部注入
    message: null,
  };

  return (
    <div className="lowcode-page" style={schema.props.style}>
      {schema.children?.map(child => renderNode(child, context))}
    </div>
  );
}
```

### 5.2 运行时与设计时的分离

// 设计时渲染器：在编辑器中使用，带选中态、辅助线等
function DesignRenderer({ schema }: { schema: PageSchema }) {
  return (
    <div className="lowcode-design-canvas" style={schema.props.style}>
      {schema.children?.map(child => renderDesignNode(child))}
    </div>
  );

function renderDesignNode(node: ComponentNode): ReactNode {
  const Component = componentMap.get(node.type);
  if (!Component) return null;

  return (
    <EditableWrapper key={node.id} node={node}>
      <Component {...node.props} style={node.style}>
        {node.children?.map(child => renderDesignNode(child))}
      </Component>
    </EditableWrapper>
  );
}

// 运行时渲染器：在生产环境使用，纯净无编辑态
function RuntimeRenderer({ schema, dataSource }: { schema: PageSchema; dataSource?: Record<string, any> }) {
  const context: RenderContext = { dataSource: dataSource || {} };
  return (
    <div className="lowcode-runtime-page" style={schema.props.style}>
      {schema.children?.map(child => renderNode(child, context))}
    </div>
  );
}

// 物料注册中心
class MaterialRegistry {
  private metas = new Map<string, ComponentMeta>();
  private components = new Map<string, React.ComponentType<any>>();

  // 注册物料
  register(meta: ComponentMeta, component: React.ComponentType<any>) {
    this.metas.set(meta.componentName, meta);
    this.components.set(meta.componentName, component);
  }

  // 批量注册
  registerAll(materials: { meta: ComponentMeta; component: React.ComponentType<any> }[]) {
    materials.forEach(({ meta, component }) => this.register(meta, component));
  }

  // 获取物料元数据
  getMeta(name: string): ComponentMeta | undefined {
    return this.metas.get(name);
  }

  // 获取所有物料（按分类分组）
  getAllByCategory(): Record<string, ComponentMeta[]> {
    const grouped: Record<string, ComponentMeta[]> = {};
    this.metas.forEach(meta => {
      if (!grouped[meta.category]) grouped[meta.category] = [];
      grouped[meta.category].push(meta);
    });
    return grouped;
  }

  // 根据元数据创建默认节点
  createNode(metaName: string): ComponentNode | null {
    const meta = this.metas.get(metaName);
    if (!meta) return null;
    return {
      id: generateId(metaName),
      type: meta.componentName,
      props: { ...meta.defaultProps },
      children: meta.defaultChildren?.map(child => ({ ...child })) || [],
    };
  }
}

const materialRegistry = new MaterialRegistry();

## 七、拖拽交互系统

### 7.1 拖拽核心逻辑

```typescript
type DropPosition = 'before' | 'after' | 'inside';

interface DragData {
  nodeId?: string;         // 已有节点拖拽
  materialName?: string;   // 从物料面板拖入新组件
}

class DragSystem {
  private dragData: DragData | null = null;
  private dropCallbacks: Function[] = [];

  // 开始拖拽
  startDrag(data: DragData) {
    this.dragData = data;
  }

  // 判断是否可以放置
  canDrop(targetNode: ComponentNode, position: DropPosition): boolean {
    if (!this.dragData) return false;

    // 物料面板拖入：检查目标是否是容器
    if (this.dragData.materialName) {
      const meta = materialRegistry.getMeta(this.dragData.materialName);
      if (position === 'inside') {
        return targetNode.type === 'Container' || targetNode.type === 'Page';
      }
      return true;
    }

    // 已有节点移动：不能拖入自身或子节点
    if (this.dragData.nodeId === targetNode.id) return false;
    if (isDescendant(targetNode.id, this.dragData.nodeId)) return false;

    return true;
  }

  // 计算放置位置
  calculateDropPosition(
    e: DragEvent,
    targetRect: DOMRect
  ): DropPosition {
    const y = e.clientY - targetRect.top;
    const height = targetRect.height;

    if (y < height * 0.25) return 'before';
    if (y > height * 0.75) return 'after';
    return 'inside';
  }

  // 触发放置回调
  notifyDrop(
    targetId: string,
    position: DropPosition,
    dragData: DragData
  ) {
    this.dropCallbacks.forEach(cb => cb(targetId, position, dragData));
  }

  onDrop(callback: Function) {
    this.dropCallbacks.push(callback);
  }

  endDrag() {
    this.dragData = null;
  }
}
```

## 九、撤销重做机制

### 9.1 命令模式实现

```typescript
interface HistoryAction {
  type: string;
  description: string;
  // 执行
  execute(state: PageSchema): PageSchema;
  // 撤销
  undo(state: PageSchema): PageSchema;
}

// 具体命令：添加节点
class AddNodeAction implements HistoryAction {
  type = 'addNode';
  description: string;

  constructor(
    private node: ComponentNode,
    private parentId: string,
    private index?: number
  ) {
    this.description = `添加组件 ${node.type}`;
  }

  execute(state: PageSchema): PageSchema {
    const newSchema = cloneDeep(state);
    const parent = findNodeById(newSchema, this.parentId);
    if (parent) {
      parent.children = parent.children || [];
      if (this.index !== undefined) {
        parent.children.splice(this.index, 0, this.node);
      } else {
        parent.children.push(this.node);
      }
    }
    return newSchema;
  }

  undo(state: PageSchema): PageSchema {
    const newSchema = cloneDeep(state);
    removeNodeById(newSchema, this.node.id);
    return newSchema;
  }
}

// 历史管理器
class HistoryManager {
  private undoStack: { action: HistoryAction; stateBefore: PageSchema }[] = [];
  private redoStack: { action: HistoryAction; stateBefore: PageSchema }[] = [];
  private maxStack = 50;

  record(action: HistoryAction, stateBefore: PageSchema) {
    this.undoStack.push({ action, stateBefore });
    if (this.undoStack.length > this.maxStack) {
      this.undoStack.shift();
    }
    this.redoStack = []; // 新操作清空 redo
  }

  undo(currentState: PageSchema): PageSchema | null {
    const entry = this.undoStack.pop();
    if (!entry) return null;
    this.redoStack.push({ ...entry, stateBefore: currentState });
    return entry.action.undo(currentState);
  }

  redo(currentState: PageSchema): PageSchema | null {
    const entry = this.redoStack.pop();
    if (!entry) return null;
    this.undoStack.push({ ...entry, stateBefore: currentState });
    return entry.action.execute(entry.stateBefore);
  }

  canUndo(): boolean {
    return this.undoStack.length > 0;
  }

  canRedo(): boolean {
    return this.redoStack.length > 0;
  }
}
```

## 十一、面试题速查

### Q1：低代码可视化搭建的核心是什么？

> 核心是**数据驱动渲染**。页面被抽象为 JSON Schema，编辑器产出 Schema，渲染引擎消费 Schema 还原页面。所有操作（拖拽、配置、删除）本质上都是对 Schema 树的增删改查。

### Q2：Schema 协议设计有哪些关键点？

> 1. **版本化**：version 字段 + 迁移机制，保证向前兼容
> 2. **树形结构**：children 递归嵌套，天然映射 DOM 树
> 3. **节点唯一标识**：每个节点有 id，用于选中、操作、历史记录
> 4. **可扩展**：通过 dataSource、events、condition、loop 等字段支持动态能力
> 5. **校验机制**：用 JSON Schema 对协议做校验，防止非法数据

### Q3：渲染引擎如何实现运行时和设计时的分离？

> 设计时渲染在组件外包裹 `EditableWrapper`，注入选中态、悬停态、拖拽目标等编辑功能。运行时渲染直接渲染组件本身，不带任何编辑态。两者共享同一个 Schema 和组件注册表，但渲染逻辑完全独立。

### Q4：拖拽系统如何判断放置位置？

> 通过计算鼠标在目标组件矩形中的相对位置。通常将矩形分为三个区域：上 25% 为 `before`（插入到前方），下 25% 为 `after`（插入到后方），中间 50% 为 `inside`（作为子组件插入）。同时需要校验拖拽合法性，如不能拖入自身子节点。

### Q5：物料体系如何实现"注册即可用"？

> 每个物料包含三部分：1）React/Vue 组件实现；2）`ComponentMeta` 元数据描述（属性、事件、插槽）；3）默认属性和默认子节点。注册时组件进入 `componentMap` 供渲染引擎使用，元数据进入物料面板供用户拖拽，`propsConfig` 自动生成属性配置面板。用户不需要手写任何配置代码。

### Q6：撤销重做用什么模式实现？

> **命令模式**。每个操作封装为一个 `HistoryAction` 对象，包含 `execute` 和 `undo` 两个方法。执行时入 undo 栈，撤销时弹出到 redo 栈。需要限制栈大小（如 50）防止内存溢出。新操作执行时清空 redo 栈。

### Q7：低代码平台如何保证组件的扩展性？

> 1. **自定义组件**：遵循 `ComponentMeta` 规范开发，注册即可用
> 2. **自定义属性设置器**：通过 setter 字段指定自定义渲染器
> 3. **自定义事件动作**：扩展 EventAction 类型，注册自定义处理器
> 4. **插件机制**：编辑器核心提供生命周期钩子（onInit、onNodeAdd、onRender 等），插件可拦截和修改行为

### Q8：Schema 嵌套很深时，性能如何优化？

> 1. **节点查找用 Map 索引**：构建 `id -> node` 的扁平索引，O(1) 查找
> 2. **不可变更新 + 引用复用**：使用 immer 或 structuredClone 做局部更新，未变化的子树保持引用不变
> 3. **渲染分片**：大页面采用虚拟滚动或分块渲染
> 4. **防抖更新**：属性面板频繁修改时做防抖，避免高频 re-render
