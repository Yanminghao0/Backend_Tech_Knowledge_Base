## 1. 构建工具

### 1.1 Webpack vs Vite

```
┌──────────────────────────────────────────────────┐
│  Webpack              │  Vite                    │
│  ───────             │  ────                    │
│  打包所有模块→bundle   │  Dev: ESM原生加载          │
│  Dev启动慢(全量构建)   │  Dev启动极快(按需加载)     │
│  HMR全量更新           │  HMR精确更新              │
│  生态成熟              │  生态快速成长              │
│  配置复杂              │  配置简洁                 │
│  生产打包成熟           │  生产用Rollup打包         │
│  适合大型项目           │  适合新项目               │
└──────────────────────────────────────────────────┘

Vite Dev快的原因:
  浏览器原生ESM → 请求哪个模块就编译哪个 → 不打包
  依赖预构建(esbuild) → node_modules依赖一次性编译缓存
```

### 1.2 Webpack核心概念

```javascript
// webpack.config.js
const path = require('path');
const HtmlWebpackPlugin = require('html-webpack-plugin');

module.exports = {
    mode: 'development',  // development | production
    entry: './src/index.js',  // 入口

    output: {
        path: path.resolve(__dirname, 'dist'),
        filename: '[name].[contenthash].js',  // 内容hash缓存
        clean: true,  // 构建前清理dist
    },

    // 模块处理规则
    module: {
        rules: [
            // Babel转译
            {
                test: /\.js$/,
                exclude: /node_modules/,
                use: {
                    loader: 'babel-loader',
                    options: {
                        presets: ['@babel/preset-env', '@babel/preset-react']
                    }
                }
            },
            // CSS
            { test: /\.css$/, use: ['style-loader', 'css-loader'] },
            // SCSS
            { test: /\.scss$/, use: ['style-loader', 'css-loader', 'sass-loader'] },
            // 图片
            { test: /\.(png|jpg|gif)$/, type: 'asset/resource' },
            // TypeScript
            { test: /\.tsx?$/, use: 'ts-loader', exclude: /node_modules/ },
        ]
    },

    // 插件
    plugins: [
        new HtmlWebpackPlugin({ template: './public/index.html' }),
    ],

    // 开发服务器
    devServer: {
        hot: true,         // HMR热更新
        port: 3000,
        proxy: {
            '/api': {
                target: 'http://localhost:8080',
                changeOrigin: true,
            }
        }
    },

    // 路径别名
    resolve: {
        alias: {
            '@': path.resolve(__dirname, 'src'),
        },
        extensions: ['.js', '.jsx', '.ts', '.tsx'],
    },

    // 优化
    optimization: {
        splitChunks: {
            chunks: 'all',  // 代码分割
            cacheGroups: {
                vendor: {
                    test: /[\\/]node_modules[\\/]/,
                    name: 'vendors',
                    chunks: 'all',
                }
            }
        }
    }
};
```

## 2. Vite配置

```javascript
// vite.config.js
import { defineConfig } from 'vite';
import vue from '@vitejs/plugin-vue';
import { resolve } from 'path';

export default defineConfig({
    plugins: [
        vue(),
        // React: import react from '@vitejs/plugin-react'
        // react(),
    ],

    resolve: {
        alias: {
            '@': resolve(__dirname, 'src'),
        }
    },

    server: {
        port: 3000,
        open: true,  // 自动打开浏览器
        proxy: {
            '/api': {
                target: 'http://localhost:8080',
                changeOrigin: true,
                rewrite: (path) => path.replace(/^\/api/, ''),
            }
        }
    },

    build: {
        outDir: 'dist',
        sourcemap: false,
        rollupOptions: {
            output: {
                // 代码分割
                manualChunks: {
                    vendor: ['vue', 'vue-router', 'pinia'],
                    utils: ['lodash', 'axios', 'dayjs'],
                },
                // 文件名hash
                chunkFileNames: 'js/[name]-[hash].js',
                entryFileNames: 'js/[name]-[hash].js',
                assetFileNames: 'assets/[name]-[hash].[ext]',
            }
        },
        // 压缩
        minify: 'terser',
        terserOptions: {
            compress: {
                drop_console: true,  // 去除console
                drop_debugger: true,
            }
        },
        // 分包大小警告阈值
        chunkSizeWarningLimit: 1000,
    },

    // CSS预处理器
    css: {
        preprocessorOptions: {
            scss: {
                additionalData: `@import "@/styles/variables.scss";`
            }
        }
    },

    // 环境变量
    define: {
        __APP_VERSION__: JSON.stringify('1.0.0'),
    }
});
```

## 3. 包管理

### 3.1 npm vs yarn vs pnpm

```
┌──────────────────────────────────────────────────┐
│  npm           │  yarn          │  pnpm          │
│  ───          │  ────         │  ────          │
│  Node自带       │  Facebook     │  最新推荐       │
│  package-lock  │  yarn.lock    │  pnpm-lock     │
│  串行安装       │  并行安装      │  并行+硬链接    │
│  node_modules大 │  node_modules大│  node_modules小 │
│  扁平结构       │  扁平结构      │  符号链接结构   │
│                │  workspace    │  workspace     │
│                │               │  严格依赖隔离   │
└──────────────────────────────────────────────────┘

pnpm优势:
  1. 磁盘节省 — 硬链接共享全局store, 多项目共用同一版本包
  2. 严格隔离 — 符号链接结构, 防止幽灵依赖(未声明的包被引用)
  3. 安装快 — 并行+硬链接
  4. Monorepo — workspace原生支持
```

```bash
# pnpm常用命令
pnpm install              # 安装所有依赖
pnpm add axios            # 安装生产依赖
pnpm add -D vite          # 安装开发依赖
pnpm add -w lodash        # 安装到workspace根
pnpm remove axios         # 移除
pnpm update               # 更新所有
pnpm run dev              # 运行脚本
pnpm dlx create-vite      # 临时执行(类似npx)
```

### 3.2 Monorepo

```json
// pnpm-workspace.yaml
packages:
  - 'packages/*'
  - 'apps/*'
```

```
monorepo结构:
  my-monorepo/
  ├── pnpm-workspace.yaml
  ├── package.json
  ├── apps/
  │   ├── web/          # 前端应用
  │   └── admin/        # 管理后台
  ├── packages/
  │   ├── ui/           # 共享UI组件库
  │   ├── utils/        # 共享工具
  │   └── types/        # 共享类型定义
  └── tsconfig.base.json

  优势:
  - 代码共享 — 多项目复用组件/工具
  - 统一管理 — 统一依赖版本/构建/发布
  - 原子提交 — 跨项目修改一次提交
  - 依赖一致 — pnpm workspace保证版本一致
```

### 4.1 类型系统

```typescript
// 基本类型
let name: string = 'Alice';

let isActive: boolean = true;

enum Status {
    Pending = 'PENDING',

let status: Status = Status.Active;

interface User {
    id: number;
    name: string;

    readonly email: string; // 只读

    code: number;
    message: string;
    data: T;
}

// 类型别名
type ID = string | number;
```

### 4.2 高级类型

```typescript
// 泛型
function identity<T>(value: T): T {
    return value;
}

function getProperty<T, K extends keyof T>(obj: T, key: K): T[K] {
    return obj[key];
}

const user: User = { id: 1, name: 'Alice', email: 'a@b.com' };
getProperty(user, 'name');  // string

// 内置工具类型
type PartialUser = Partial<User>;           // 所有属性可选
type RequiredUser = Required<User>;         // 所有属性必填
type ReadonlyUser = Readonly<User>;         // 所有属性只读
type UserPreview = Pick<User, 'id' | 'name'>;  // 选取部分属性
type UserWithoutEmail = Omit<User, 'email'>;    // 排除属性
type UserRecord = Record<string, User>;         // 键值对
type ReturnType<T> = T extends (...args: any[]) => infer R ? R : never;

// 映射类型
type Stringify<T> = {
    [K in keyof T]: string;
};

// 条件类型
type IsString<T> = T extends string ? true : false;
type A = IsString<string>;  // true
type B = IsString<number>;  // false
```

### 4.3 React + TypeScript

```tsx
import { useState, useEffect } from 'react';

// 组件Props类型
interface ButtonProps {
    variant?: 'primary' | 'secondary' | 'danger';
    size?: 'small' | 'medium' | 'large';
    disabled?: boolean;
    onClick?: (event: React.MouseEvent<HTMLButtonElement>) => void;
    children: React.ReactNode;
}

function Button({
    variant = 'primary',
    size = 'medium',
    disabled = false,
    onClick,
    children
}: ButtonProps) {
    return (
        <button
            className={`btn btn-${variant} btn-${size}`}
            disabled={disabled}
            onClick={onClick}
        >
            {children}
        </button>
    );
}

// 泛型组件
function List<T>({ items, render }: {
    items: T[];
    render: (item: T, index: number) => React.ReactNode;
}) {
    return <ul>{items.map((item, i) => <li key={i}>{render(item, i)}</li>)}</ul>;
}

// API类型定义
interface UserDTO {
    id: number;
    name: string;
    email: string;
}

async function fetchUsers(): Promise<UserDTO[]> {
    const response = await fetch('/api/users');
    return response.json();
}

function UserList() {
    const [users, setUsers] = useState<UserDTO[]>([]);
    const [loading, setLoading] = useState(true);

    useEffect(() => {
        fetchUsers().then(data => {
            setUsers(data);
            setLoading(false);
        });
    }, []);

    if (loading) return <div>Loading...</div>;

    return (
        <List
            items={users}
            render={(user) => <span>{user.name} ({user.email})</span>}
        />
    );
}
```

## 6. 面试题速查

**Q1: Vite为什么比Webpack快？**

```
Dev: Vite利用浏览器原生ESM, 按需编译, 不打包
     Webpack需要全量构建打包
依赖预构建: esbuild(Go编写)比Babel快10-100倍
HMR: Vite精确模块热更新, Webpack需重新构建受影响模块
生产: Vite用Rollup打包, Tree-shaking更好
```

**Q2: pnpm比npm好在哪里？**

```
1. 磁盘节省 — 硬链接共享全局store
2. 安装快 — 并行+硬链接
3. 严格隔离 — 符号链接防止幽灵依赖
4. Monorepo — workspace原生支持
```

**Q3: TypeScript的泛型？**

```
泛型 = 类型参数化
function identity<T>(value: T): T
好处: 类型安全 + 复用性
约束: <T extends SomeType> 限制T必须满足条件
推断: 调用时自动推断T类型
```

**Q4: ESLint和Prettier的区别？**

```
ESLint: 代码质量检查(未使用变量/隐式类型转换)
Prettier: 代码格式化(缩进/引号/换行)
配合: ESLint管质量, Prettier管格式, eslint-config-prettier解决冲突
```
