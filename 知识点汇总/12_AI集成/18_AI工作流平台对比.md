## 1. AI工作流平台概述

### 什么是AI工作流平台

```
低代码/无代码平台，通过可视化拖拽构建AI应用：

  用户输入 → [LLM节点] → [条件判断] → [工具调用] → [输出]
                ↑                         ↓
            [知识库检索] ← ← ← ← ← ←

价值：
  - 降低AI应用开发门槛
  - 快速原型验证
  - 可视化调试
  - 非技术人员可参与构建
```

### 应用场景

```
1. 智能客服：多轮对话+知识库+人工转接
2. 文档处理：解析→提取→分类→存储
3. 内容生成：选题→大纲→写作→审核
4. 数据分析：查询→分析→可视化→报告
5. 流程自动化：触发→处理→通知
```

## 2. Dify

### 核心特性

```
Dify = 开源LLM应用开发平台

功能：
  - 可视化Prompt编排
  - RAG知识库管理
  - Agent工具链
  - 工作流编排
  - 模型管理（多模型切换）
  - API发布
  - 日志和标注

部署：Docker自部署 / 云服务
```

### 工作流示例

```
Dify工作流：智能客服

  [开始] → [意图识别(LLM)] → [条件判断]
                                    ├─ 知识库问答 → [RAG检索] → [LLM生成] → [输出]
                                    ├─ 查询订单 → [API调用] → [LLM总结] → [输出]
                                    └─ 人工转接 → [通知客服] → [输出]
```

### Dify优势

```
1. 一站式：Prompt编排+RAG+Agent+发布
2. 可视化：拖拽式工作流
3. 多模型：OpenAI/Claude/Qwen/Ollama统一管理
4. 企业级：多租户、权限、审计
5. 开源：Apache 2.0，可私有部署
```

## 3. Flowise

### 核心特性

```
Flowise = LangChain的可视化版本

特点：
  - 拖拽式构建LangChain流程
  - 节点即LangChain组件（LLM/Tool/Memory/Retriever）
  - 实时预览和调试
  - API导出
  - 开源
```

### vs Dify

```
Flowise：
  - 更灵活（任意LangChain组件组合）
  - 适合开发者快速原型
  - 无知识库管理（需自己搭建）
  - 无模型管理

Dify：
  - 更完整（含RAG/知识库/模型管理/发布）
  - 适合企业级应用
  - 工作流更强大
  - 更重
```

## 4. n8n

### 核心特性

```
n8n = 开源工作流自动化平台 + AI能力

特点：
  - 400+集成节点（数据库/API/SaaS/消息队列）
  - AI节点（LLM调用/Agent/RAG）
  - 代码节点（JavaScript/Python）
  - 定时触发/Webhook触发
  - 自部署
```

### AI工作流示例

```
n8n工作流：邮件自动分类回复

  [邮件接收Trigger] → [LLM分类]
    ├─ 紧急 → [Slack通知] + [LLM草拟回复] → [人工审核] → [发送]
    ├─ 普通 → [LLM自动回复] → [发送]
    └─ 垃圾 → [标记删除]
```

### n8n优势

```
1. 集成丰富：400+预置节点
2. AI原生：内置LangChain节点
3. 灵活触发：定时/Webhook/事件
4. 自部署：数据不外传
5. 代码节点：复杂逻辑可写代码
```

## 5. LangFlow

### 核心特性

```
LangFlow = LangChain的GUI设计器

特点：
  - 与LangChain完全对应
  - 拖拽构建Chain/Agent/RAG
  - 实时调试
  - 导出Python代码
  - 开源
```

## 6. 平台对比与选型

| 平台 | 定位 | 优势 | 适用 | 部署 |
|------|------|------|------|------|
| Dify | LLM应用平台 | 全功能(编排+RAG+发布) | 企业AI应用 | Docker/云 |
| Flowise | LangChain可视化 | 灵活组合 | 快速原型 | Docker |
| n8n | 自动化+AI | 400+集成 | 流程自动化 | Docker/云 |
| LangFlow | LangChain GUI | 代码导出 | 开发者 | Docker |

### 选型建议

```
企业AI应用（客服/知识库）→ Dify（全功能，企业级）
快速原型验证 → Flowise（灵活，快速）
业务流程自动化 → n8n（集成丰富）
开发者可视化调试 → LangFlow（导出代码）
Java团队 → Spring AI + LangChain4j（代码级控制）
```

## 7. 面试要点

### Q1: 什么时候用工作流平台，什么时候写代码？

```
用工作流平台：
  - 快速原型验证
  - 非技术人员参与
  - 流程频繁变更
  - 简单AI应用

写代码：
  - 复杂业务逻辑
  - 性能要求高
  - 需要细粒度控制
  - 企业级生产环境
```

### Q2: Dify的核心能力？

```
1. Prompt编排：可视化构建多轮对话
2. RAG知识库：文档管理+向量化+检索
3. Agent：工具调用+多步推理
4. 工作流：条件分支+循环+并行
5. 模型管理：多模型统一管理+路由
6. API发布：一键暴露为REST API
7. 运维：日志/监控/标注/评估
```
