## 1. 知识库概述

### 企业知识库的挑战

```
1. 文档量大：万级~百万级文档
2. 格式多样：PDF/Word/Markdown/HTML/图片
3. 权限复杂：部门/项目/密级隔离
4. 更新频繁：文档增删改需要实时同步
5. 多租户：SaaS场景需租户隔离
6. 准确性：不能返回过时或错误信息
```

### 架构

```
┌──────────────────────────────────────────┐
│              用户提问                      │
├──────────────────────────────────────────┤
│          权限过滤 + 租户隔离               │
├──────────────────────────────────────────┤
│          RAG检索（向量+关键词）            │
│  ┌──────┐  ┌──────┐  ┌──────┐          │
│  │向量DB │  │ES全文 │  │知识图谱│          │
│  └──────┘  └──────┘  └──────┘          │
├──────────────────────────────────────────┤
│          重排序 + 引用来源                 │
├──────────────────────────────────────────┤
│          LLM生成回答                      │
└──────────────────────────────────────────┘
```

## 2. 文档管理体系

### 文档生命周期

```java
@Service
public class DocumentLifecycleService {
    
    // 文档状态机
    // UPLOADING → PROCESSING → READY → UPDATED → DELETED
    
    public Document upload(MultipartFile file, String tenantId, String userId) {
        Document doc = Document.builder()
            .id(IdUtil.nextId())
            .tenantId(tenantId)
            .fileName(file.getOriginalFilename())
            .status(DocumentStatus.PROCESSING)
            .uploadedBy(userId)
            .build();
        
        documentMapper.insert(doc);
        
        // 异步处理：解析→切分→向量化→存储
        asyncProcess(doc, file);
        
        return doc;
    }
    
    @Async
    public void asyncProcess(Document doc, MultipartFile file) {
        try {
            // 1. 解析文档
            String text = documentParser.parse(file);
            
            // 2. 切分
            List<TextChunk> chunks = documentSplitter.split(text, 800, 100);
            
            // 3. 向量化 + 存储带权限标记
            for (TextChunk chunk : chunks) {
                Document vectorDoc = new Document(chunk.getText(), Map.of(
                    "docId", doc.getId(),
                    "tenantId", doc.getTenantId(),
                    "department", doc.getDepartment(),
                    "classification", doc.getClassification(),
                    "chunkIndex", chunk.getIndex()
                ));
                vectorStore.add(List.of(vectorDoc));
            }
            
            // 4. 全文索引（ES）
            esClient.index(doc.getId(), text, doc.getMetadata());
            
            // 5. 更新状态
            doc.setStatus(DocumentStatus.READY);
            documentMapper.updateById(doc);
            
        } catch (Exception e) {
            doc.setStatus(DocumentStatus.FAILED);
            documentMapper.updateById(doc);
        }
    }
}
```

## 3. 权限与多租户

### 租户隔离

```java
@Service
public class TenantAwareRagService {
    
    @Autowired private VectorStore vectorStore;
    
    // 检索时强制加租户过滤
    public List<Document> search(String tenantId, String query, int topK) {
        // 方案1: 元数据过滤（推荐）
        SearchRequest request = SearchRequest.builder()
            .query(query)
            .topK(topK)
            .similarityThreshold(0.7)
            .filterExpression("tenantId == '" + tenantId + "'")  // 租户隔离
            .build();
        
        return vectorStore.similaritySearch(request);
    }
    
    // 带权限的检索
    public List<Document> searchWithPermission(String tenantId, String userId, 
            Set<String> userDepartments, String query, int topK) {
        
        // 1. 先检索（带租户过滤）
        List<Document> candidates = search(tenantId, query, topK * 3);
        
        // 2. 权限过滤
        return candidates.stream()
            .filter(doc -> hasPermission(doc, userId, userDepartments))
            .limit(topK)
            .collect(Collectors.toList());
    }
    
    private boolean hasPermission(Document doc, String userId, Set<String> departments) {
        String classification = doc.getMetadata().get("classification");
        String docDepartment = doc.getMetadata().get("department");
        
        // 公开文档：所有人可读
        if ("public".equals(classification)) return true;
        
        // 内部文档：同部门可读
        if ("internal".equals(classification)) {
            return departments.contains(docDepartment);
        }
        
        // 机密文档：只有上传者和管理员
        if ("confidential".equals(classification)) {
            return userId.equals(doc.getMetadata().get("uploadedBy"))
                || userRoleChecker.isAdmin(userId);
        }
        
        return false;
    }
}
```

## 4. 增量更新机制

### 文档更新同步

```java
@Service
public class DocumentUpdateService {
    
    // 文档更新时同步向量库
    public void updateDocument(Long docId, MultipartFile newFile) {
        // 1. 删除旧向量
        vectorStore.delete(filterExpression("docId == " + docId));
        
        // 2. 删除旧ES索引
        esClient.delete(docId);
        
        // 3. 重新处理新文档
        Document doc = documentMapper.selectById(docId);
        doc.setVersion(doc.getVersion() + 1);
        documentMapper.updateById(doc);
        
        asyncProcess(doc, newFile);
    }
    
    // 文档删除时清理向量
    public void deleteDocument(Long docId) {
        // 1. 标记删除
        documentMapper.softDelete(docId);
        
        // 2. 删除向量
        vectorStore.delete(filterExpression("docId == " + docId));
        
        // 3. 删除ES索引
        esClient.delete(docId);
    }
    
    // 定时同步：检测源文档变更
    @Scheduled(fixedDelay = 3600000)  // 每小时
    public void syncChangedDocuments() {
        List<Document> changed = documentMapper.findUpdatedSince(lastSyncTime);
        for (Document doc : changed) {
            updateDocument(doc.getId(), loadFile(doc));
        }
    }
}
```

## 5. 知识图谱增强

### GraphRAG

```java
@Service
public class GraphRagService {
    
    @Autowired private VectorStore vectorStore;
    @Autowired private Neo4jClient neo4jClient;
    
    // 构建知识图谱
    public void buildKnowledgeGraph(List<Document> docs) {
        for (Document doc : docs) {
            // LLM提取实体和关系
            String prompt = """
                从以下文档中提取实体和关系，输出JSON：
                文档：%s
                
                格式：
                {
                  "entities": [{"name": "Spring Boot", "type": "框架"}],
                  "relations": [{"from": "Spring Boot", "to": "Spring", "type": "基于"}]
                }
                """.formatted(doc.getText());
            
            String response = chatClient.prompt().user(prompt).call().content();
            KnowledgeGraphData kg = parseJson(response);
            
            // 存入Neo4j
            for (Entity e : kg.getEntities()) {
                neo4jClient.query("MERGE (n:Entity {name: $name, type: $type})")
                    .bind(e.getName()).to("name")
                    .bind(e.getType()).to("type").run();
            }
            for (Relation r : kg.getRelations()) {
                neo4jClient.query(
                    "MATCH (a:Entity {name: $from}), (b:Entity {name: $to}) " +
                    "MERGE (a)-[:RELATION {type: $type}]->(b)")
                    .bind(r.getFrom()).to("from")
                    .bind(r.getTo()).to("to")
                    .bind(r.getType()).to("type").run();
            }
        }
    }
    
    // GraphRAG检索：向量检索 + 图谱扩展
    public String ask(String query) {
        // 1. 向量检索获取初始文档
        List<Document> docs = vectorStore.similaritySearch(
            SearchRequest.builder().query(query).topK(5).build());
        
        // 2. 提取实体，图谱扩展
        Set<String> entities = extractEntities(query);
        List<String> relatedEntities = neo4jClient.query(
            "MATCH (n:Entity)-[:RELATION*1..2]-(m:Entity) " +
            "WHERE n.name IN $names RETURN DISTINCT m.name")
            .bind(entities).to("names")
            .fetchAs(String.class).all();
        
        // 3. 用扩展的实体补充检索
        if (!relatedEntities.isEmpty()) {
            String expandedQuery = query + " " + String.join(" ", relatedEntities);
            List<Document> extra = vectorStore.similaritySearch(
                SearchRequest.builder().query(expandedQuery).topK(3).build());
            docs.addAll(extra);
        }
        
        // 4. LLM生成
        return generate(query, docs);
    }
}
```

## 6. 面试要点

### Q1: 企业知识库的权限怎么做？

```
三层权限控制：
1. 租户隔离：向量检索时强制加tenantId过滤
2. 部门权限：internal级别文档仅同部门可见
3. 密级控制：confidential级别仅上传者和管理员可见

实现：向量DB元数据过滤 + 应用层权限校验
```

### Q2: 文档更新后向量库怎么同步？

```
1. 删除旧向量（按docId过滤删除）
2. 重新解析→切分→向量化→存储
3. 版本号递增（防止旧版本干扰）
4. 增量同步：定时检测源文档变更

注意：删除用软删除（标记），避免向量DB物理删除的性能问题
```

### Q3: GraphRAG和普通RAG的区别？

```
普通RAG：向量检索 → 相关文档片段
GraphRAG：向量检索 + 知识图谱实体扩展

GraphRAG优势：
1. 实体关系推理（"Spring Boot基于什么框架"→追溯关系链）
2. 多跳推理（A→B→C的关系链）
3. 全局视角（图谱可视化知识结构）

代价：需要构建和维护知识图谱（Neo4j）
```

### Q4: 知识库的文档格式怎么处理？

```
1. Apache Tika：统一解析PDF/Word/HTML/Excel等
2. OCR：图片文档用Tesseract识别
3. 表格处理：Markdown表格或结构化提取
4. 代码文档：按函数/类切分
5. 图片：多模态Embedding（CLIP）或图片描述
```
