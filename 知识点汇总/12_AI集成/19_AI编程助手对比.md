## 2. Cursor

### 核心特性

```
Cursor = VS Code Fork + 深度AI集成

优势：
  1. 全项目理解：索引整个代码库，跨文件理解
  2. Agent模式：自主完成多步骤编码任务
  3. 多模型支持：GPT-5/Claude 4/Claude 4 Opus
  4. 代码库聊天：@Codebase 引用整个项目
  5. Composer：多文件同时编辑
  6. 终端AI：命令行辅助
```

### 核心功能

```
Tab补全：基于上下文的智能补全（比Copilot更准）
Cmd+K：选中代码→AI修改/重构/解释
Cmd+L：侧边栏聊天，@文件引用
Composer：多文件编辑模式（Ctrl+I）
Agent：自主完成"添加一个用户注册功能"等复杂任务
@Codebase：全项目搜索+理解
@Web：联网搜索最新文档
```

### 适用场景

```
✅ 从零搭建项目（Agent模式）
✅ 跨文件重构
✅ 理解大型代码库
✅ 快速原型开发
```

## 3. GitHub Copilot

### 核心特性

```
GitHub Copilot = GitHub原生AI编程助手

产品矩阵：
  Copilot Individual：个人版（$10/月）
  Copilot Business：企业版（$19/月）
  Copilot Enterprise：企业版（$39/月，含知识库）

优势：
  1. IDE集成广：VS Code/IntelliJ/Visual Studio/Neovim
  2. GitHub生态：PR审查/Issue处理/Actions
  3. Ghost Text：行内补全体验好
  4. Copilot Chat：支持@workspace/@terminal
  5. Copilot Edits：多文件编辑（2024新增）
```

### vs Cursor

```
Copilot优势：
  - IDE选择多（IntelliJ支持好）
  - GitHub深度集成
  - 行内补全体验流畅
  - 企业级管理

Cursor优势：
  - 全项目理解更深
  - Agent模式更强
  - 多模型可选
  - Composer多文件编辑更成熟
```

## 5. Continue

### 核心特性

```
Continue = 开源AI编程助手（VS Code/JetBrains插件）

特点：
  1. 开源：Apache 2.0
  2. 本地模型：支持Ollama本地模型
  3. 自定义：可配置任意LLM API
  4. 隐私：代码不外传（用本地模型时）
  5. 免费：无订阅费用
```

### 配置示例

```json
// ~/.continue/config.json
{
  "models": [
    {
      "title": "GPT-5",
      "provider": "openai",
      "model": "gpt-5",
      "apiKey": "sk-xxx"
    },
    {
      "title": "Local Ollama",
      "provider": "ollama",
      "model": "qwen3:72b"
    }
  ],
  "embeddingsProvider": {
    "provider": "ollama",
    "model": "nomic-embed-text"
  }
}
```

## 7. 面试要点

### Q1: AI编程助手的原理？

```
代码补全：
  1. 收集上下文：当前文件+光标位置+导入语句+相关文件
  2. 构建Prompt：上下文 + "请补全以下代码"
  3. LLM生成：流式输出补全建议
  4. Ghost Text：灰色文本展示，Tab确认

全项目理解：
  1. 代码库索引：向量化所有文件
  2. 检索相关文件：根据当前任务检索相关代码
  3. 构建上下文：相关代码 + 当前文件
  4. LLM生成：基于完整上下文的回答
```

### Q2: Cursor和Copilot的区别？

```
Cursor：
  - 独立IDE（VS Code Fork）
  - Agent模式（自主完成多步任务）
  - Composer（多文件同时编辑）
  - 全项目理解更深
  - 多模型可选

Copilot：
  - 插件形式（多IDE支持）
  - GitHub生态集成
  - 行内补全体验好
  - 企业管理功能
```

### Q3: 如何安全使用AI编程助手？

```
1. 敏感代码：不发送到云端（用Continue+本地模型）
2. 代码审查：AI生成的代码必须人工审查
3. 安全扫描：AI代码可能引入漏洞
4. 版权合规：AI可能生成与开源项目相似的代码
5. 企业策略：制定AI编程工具使用规范
6. 敏感信息：不在Prompt中包含密码/密钥
```
