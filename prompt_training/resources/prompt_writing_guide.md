# 提示词写作指南

## 基本原则

### 1. 明确性原则
- **角色定位要具体**：不要写"你是一位专家"，要写"你是一位有10年经验的Python开发专家"
- **任务描述要清晰**：不要写"帮我写代码"，要写"帮我写一个用户登录验证的Python函数"
- **要求要具体**：不要写"写得好一点"，要写"代码要有注释，包含错误处理，符合PEP8规范"

### 2. 结构化原则
- **背景信息**：提供必要的上下文
- **具体要求**：列出详细的任务要求
- **输出格式**：指定期望的输出格式
- **注意事项**：补充重要的约束条件

### 3. 可操作原则
- **要求要可执行**：避免模糊不清的要求
- **格式要明确**：便于后续处理和使用
- **约束要合理**：不要设置过于严格的限制

## 常见问题与解决方案

### 问题1：提示词太模糊
**错误示例：**
```
帮我写一个程序
```

**正确示例：**
```
你是一位Python开发专家。请帮我写一个用户管理系统，要求如下：
- 功能：用户注册、登录、信息修改
- 数据库：使用SQLite
- 界面：命令行界面
- 安全：密码加密存储
```

### 问题2：缺少输出格式
**错误示例：**
```
帮我分析这个数据
```

**正确示例：**
```
请按以下格式输出分析结果：
## 数据概览
[数据的基本信息]

## 关键发现
[重要的发现]

## 建议
[基于分析的建议]
```

### 3. 问题3：要求过于复杂
**错误示例：**
```
写一个完美的程序，要快、要美、要安全、要易用、要便宜
```

**正确示例：**
```
写一个程序，优先考虑以下要求：
1. 性能：处理1000条记录不超过1秒
2. 安全：输入验证和SQL注入防护
3. 易用：清晰的用户界面和错误提示
```

## 提示词优化技巧

### 1. 使用具体数字
- 不要写"写一个长一点的故事"
- 要写"写一个2000字的故事"

### 2. 提供示例
- 不要写"写一个邮件"
- 要写"写一个邮件，参考以下格式：主题：[主题]，正文：[正文内容]"

### 3. 分步骤说明
- 不要写"帮我做这个项目"
- 要写"请按以下步骤完成：1. 需求分析 2. 设计方案 3. 实现代码 4. 测试验证"

### 4. 设置约束条件
- 不要写"写一个程序"
- 要写"写一个程序，要求：使用Python，不超过100行代码，包含错误处理"

## 不同类型提示词的特点

### 1. 分析类提示词
- 重点：数据收集、分析方法、结论推导
- 格式：结构化输出，包含概述、分析、结论
- 注意：要求基于数据，结论要客观

### 2. 创作类提示词
- 重点：创意表达、情感色彩、风格一致
- 格式：包含大纲、作品、说明
- 注意：目标受众要明确，创新点要突出

### 3. 技术类提示词
- 重点：功能实现、代码质量、性能要求
- 格式：包含代码、注释、测试
- 注意：要求要具体，约束要合理

### 4. 商务类提示词
- 重点：专业表达、逻辑清晰、目标明确
- 格式：包含背景、要求、输出
- 注意：语气要恰当，内容要实用

## 实践建议

### 1. 从简单开始
- 先写简单的提示词
- 逐步增加复杂度
- 不断优化改进

### 2. 多练习多总结
- 记录好的提示词
- 分析失败的原因
- 总结成功经验

### 3. 参考优秀范例
- 学习别人的提示词
- 分析其优点
- 应用到自己的写作中

### 4. 持续改进
- 根据使用效果调整
- 收集反馈意见
- 不断优化完善
