# 提示词最佳实践

## 核心原则

### 1. 明确性原则
- **角色定位要具体**：不要写"你是一位专家"，要写"你是一位有10年经验的Python开发专家"
- **任务描述要清晰**：不要写"帮我写代码"，要写"帮我写一个用户登录验证的Python函数"
- **要求要具体**：不要写"写得好一点"，要写"代码要有注释，包含错误处理，符合PEP8规范"

### 2. 结构化原则
- **背景信息**：提供必要的上下文
- **具体要求**：列出详细的任务要求
- **输出格式**：指定期望的输出格式
- **注意事项**：补充重要的约束条件

### 3. 可操作原则
- **要求要可执行**：避免模糊不清的要求
- **格式要明确**：便于后续处理和使用
- **约束要合理**：不要设置过于严格的限制

## 提示词结构模板

### 基础模板
```
你是一位[具体角色]。请帮我[具体任务]，要求如下：

**基本信息：**
- [关键信息1]：[具体内容]
- [关键信息2]：[具体内容]
- [关键信息3]：[具体内容]

**具体要求：**
1. [要求1]
2. [要求2]
3. [要求3]
4. [要求4]
5. [要求5]

**输出格式：**
请按以下格式输出：
[格式要求]

**注意事项：**
- [注意事项1]
- [注意事项2]
- [注意事项3]
```

## 优化技巧

### 1. 使用具体数字
- 不要写"写一个长一点的故事"
- 要写"写一个2000字的故事"

### 2. 提供示例
- 不要写"写一个邮件"
- 要写"写一个邮件，参考以下格式：主题：[主题]，正文：[正文内容]"

### 3. 分步骤说明
- 不要写"帮我做这个项目"
- 要写"请按以下步骤完成：1. 需求分析 2. 设计方案 3. 实现代码 4. 测试验证"

### 4. 设置约束条件
- 不要写"写一个程序"
- 要写"写一个程序，要求：使用Python，不超过100行代码，包含错误处理"

## 不同类型提示词的特点

### 1. 分析类提示词
- **重点**：数据收集、分析方法、结论推导
- **格式**：结构化输出，包含概述、分析、结论
- **注意**：要求基于数据，结论要客观

### 2. 创作类提示词
- **重点**：创意表达、情感色彩、风格一致
- **格式**：包含大纲、作品、说明
- **注意**：目标受众要明确，创新点要突出

### 3. 技术类提示词
- **重点**：功能实现、代码质量、性能要求
- **格式**：包含代码、注释、测试
- **注意**：要求要具体，约束要合理

### 4. 商务类提示词
- **重点**：专业表达、逻辑清晰、目标明确
- **格式**：包含背景、要求、输出
- **注意**：语气要恰当，内容要实用

## 实践建议

### 1. 从简单开始
- 先写简单的提示词
- 逐步增加复杂度
- 不断优化改进

### 2. 多练习多总结
- 记录好的提示词
- 分析失败的原因
- 总结成功经验

### 3. 参考优秀范例
- 学习别人的提示词
- 分析其优点
- 应用到自己的写作中

### 4. 持续改进
- 根据使用效果调整
- 收集反馈意见
- 不断优化完善

## 常见问题与解决方案

### 问题1：提示词太模糊
**解决方案**：使用具体的数字、示例和约束条件

### 问题2：缺少输出格式
**解决方案**：明确指定输出格式和结构

### 问题3：要求过于复杂
**解决方案**：设置优先级，分步骤说明

### 问题4：没有约束条件
**解决方案**：设置合理的约束和限制

### 问题5：缺少背景信息
**解决方案**：提供必要的上下文和背景信息
