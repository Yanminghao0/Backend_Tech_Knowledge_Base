# 提示词常见错误与解决方案

## 错误类型1：过于模糊

### 错误示例
```
帮我写一个程序
```

### 问题分析
- 没有明确要写什么程序
- 没有说明程序的功能
- 没有指定使用的技术栈
- 没有说明输出格式

### 正确示例
```
你是一位Python开发专家。请帮我写一个用户管理系统，要求如下：

**功能要求：**
- 用户注册：用户名、邮箱、密码
- 用户登录：验证用户名和密码
- 用户信息修改：修改个人资料
- 用户列表：显示所有用户

**技术要求：**
- 使用Python和SQLite数据库
- 密码使用bcrypt加密
- 包含输入验证和错误处理
- 代码要有详细注释

**输出格式：**
请按以下结构输出：
## 完整代码
[完整的Python代码]

## 使用说明
[如何运行和使用程序]

## 测试建议
[如何测试程序功能]
```

## 错误类型2：缺少角色定位

### 错误示例
```
帮我分析这个数据
```

### 问题分析
- 没有明确AI的身份
- 没有说明分析的专业领域
- 没有指定分析方法
- 没有说明输出格式

### 正确示例
```
你是一位数据分析专家，有5年Python和机器学习经验。请帮我分析这个销售数据，要求如下：

**分析背景：**
- 数据来源：公司销售系统
- 分析目的：找出销售趋势和问题
- 数据时间：2023年全年
- 数据量：10万条记录

**分析要求：**
1. 数据概览：基本统计信息
2. 趋势分析：月度销售趋势
3. 产品分析：各产品销量对比
4. 客户分析：客户购买行为
5. 问题识别：销售中的问题点

**输出格式：**
请按以下结构输出：
## 数据概览
[数据的基本信息]

## 趋势分析
[销售趋势分析]

## 产品分析
[产品销量分析]

## 客户分析
[客户行为分析]

## 问题与建议
[发现的问题和改进建议]
```

## 错误类型3：要求过于复杂

### 错误示例
```
写一个完美的网站，要快、要美、要安全、要易用、要便宜、要SEO友好
```

### 问题分析
- 要求太多，没有优先级
- 没有具体的技术要求
- 没有明确的功能范围
- 没有说明预算和时间限制

### 正确示例
```
你是一位全栈开发专家。请帮我设计一个企业官网，要求如下：

**项目背景：**
- 公司类型：科技公司
- 目标用户：企业客户
- 主要功能：产品展示、公司介绍、联系方式
- 预算：5万元以内
- 时间：2个月

**技术要求：**
- 前端：React + TypeScript
- 后端：Node.js + Express
- 数据库：MongoDB
- 部署：阿里云服务器

**功能要求：**
1. 首页：公司介绍和产品概览
2. 产品页：产品详细介绍
3. 关于我们：公司历史和文化
4. 联系我们：联系方式和表单

**设计原则：**
- 简洁专业，符合科技公司形象
- 响应式设计，支持移动端
- 加载速度快，SEO友好
- 安全可靠，包含基础防护

**输出格式：**
请按以下结构输出：
## 技术方案
[技术架构和选型说明]

## 功能设计
[详细的功能设计]

## 开发计划
[开发时间安排]

## 预算估算
[各项费用估算]
```

## 错误类型4：缺少输出格式

### 错误示例
```
帮我写一个学习计划
```

### 问题分析
- 没有说明学习目标
- 没有指定学习时间
- 没有说明输出格式
- 没有提供背景信息

### 正确示例
```
你是一位学习规划专家。请帮我制定一个Python学习计划，要求如下：

**学习背景：**
- 学习目标：掌握Python数据分析
- 当前水平：编程零基础
- 可用时间：每天2小时，周末4小时
- 学习偏好：视频教程+实践项目

**计划要求：**
1. 制定3个月的学习路径
2. 每周安排具体学习任务
3. 包含阶段性检测点
4. 提供学习资源推荐
5. 考虑实际执行难度

**输出格式：**
请按以下结构输出：
## 学习目标
[明确的学习目标]

## 学习路径
### 第1个月：基础阶段
- 周1-2：[具体任务]
- 周3-4：[具体任务]

### 第2个月：进阶阶段
- 周5-6：[具体任务]
- 周7-8：[具体任务]

### 第3个月：应用阶段
- 周9-10：[具体任务]
- 周11-12：[具体任务]

## 检测点
- 第1个月末：[检测内容]
- 第2个月末：[检测内容]
- 第3个月末：[检测内容]

## 推荐资源
- 书籍：[推荐书籍]
- 课程：[推荐课程]
- 实践：[实践建议]
```

## 错误类型5：没有约束条件

### 错误示例
```
帮我写一个故事
```

### 问题分析
- 没有说明故事类型
- 没有指定故事长度
- 没有说明目标受众
- 没有设置创作约束

### 正确示例
```
你是一位创意写作专家。请帮我创作一个悬疑推理故事，要求如下：

**创作要求：**
- 类型：悬疑推理
- 长度：3000字左右
- 主题：真相与正义
- 风格：紧张刺激，逻辑严密
- 目标受众：成年读者

**角色设定：**
- 主角：经验丰富的侦探
- 配角：案件相关人员
- 反派：隐藏的凶手

**情节要求：**
1. 开头要有吸引力，设置悬念
2. 中间要有冲突和转折
3. 结尾要揭示真相，发人深省
4. 对话要生动自然
5. 描写要生动具体

**输出格式：**
请按以下结构输出：
## 故事大纲
[简要大纲]

## 完整故事
[完整故事内容]

## 创作说明
- 主题表达：[如何表达主题]
- 写作技巧：[使用的技巧]
- 读者感受：[希望读者感受到什么]
```

## 总结

### 避免常见错误的方法
1. **明确角色定位**：给AI一个具体的身份
2. **详细描述任务**：说明要完成什么
3. **提供背景信息**：给出必要的上下文
4. **列出具体要求**：详细说明任务标准
5. **指定输出格式**：便于后续处理
6. **设置合理约束**：避免过于复杂的要求

### 优化提示词的步骤
1. **确定目标**：明确要达成什么目标
2. **分析需求**：分析具体的需求
3. **设计结构**：设计提示词的结构
4. **编写内容**：编写具体的提示词
5. **测试验证**：测试提示词的效果
6. **优化改进**：根据效果进行优化
